## 3. 数据结构基础

> 数据结构是安全工具的底层骨架。理解数据结构不是学术练习——它直接影响你能否读懂安全工具的源码、理解攻击的本质机制、以及在面对复杂安全问题时选择正确的解题思路。本节从安全从业者的视角，系统讲解哈希表、树、图三大核心数据结构在攻防场景中的应用原理。

### 3.1 为什么安全工程师需要理解数据结构

很多安全从业者会问：我又不写安全工具，为什么要懂数据结构？答案是：安全工具的每一个关键决策背后都是数据结构的选择，而这个选择直接决定了工具的性能边界和安全特性。

| 安全场景 | 底层数据结构 | 为什么必须理解 |
|---------|------------|--------------|
| WAF拦截恶意请求 | 前缀树/AC自动机 | 不理解就无法理解WAF为什么能被绕过（模式匹配的盲区） |
| 密码存储与验证 | 哈希表+哈希函数 | 不理解就无法理解彩虹表攻击的原理和盐值防御的机制 |
| 布隆过滤器判断恶意URL | 位数组+多个哈希函数 | 不理解就无法理解误报率的计算和为什么它不能被用作唯一判断依据 |
| 数据库索引优化 | B+树 | 不理解就无法理解SQL注入时为什么某些查询慢、某些快，以及如何利用索引盲区 |
| 攻击路径分析 | 有向图+最短路径 | 不理解就无法自动化渗透测试的路径规划 |
| 二进制逆向分析 | 控制流图（CFG） | 不理解就无法进行漏洞模式识别和自动化代码审计 |
| 蜜罐诱捕系统 | 状态机+哈希表 | 不理解就无法设计高效的攻击者行为追踪系统 |

**核心认知：** 安全攻防的本质是信息战——谁能在海量数据中更快、更准地找到目标信息，谁就占据优势。数据结构决定了"找信息"的效率和正确性。

### 3.2 哈希表在安全中的应用

哈希表（Hash Table）是安全领域使用频率最高的数据结构，没有之一。从密码存储到会话管理，从WAF规则匹配到速率限制，哈希表无处不在。

#### 3.2.1 哈希表的基本原理

哈希表通过哈希函数将键（Key）映射到数组的特定位置（桶，Bucket），实现近乎 O(1) 的查找、插入和删除。

```text
哈希表内部结构：

键(Key) → 哈希函数 H(key) → 桶索引(index) → 存储位置

示例：
  H("admin")  = 3  →  buckets[3]  →  {username: "admin", role: "admin"}
  H("guest")  = 7  →  buckets[7]  →  {username: "guest", role: "user"}
  H("test")   = 3  →  buckets[3]  →  碰撞！需要链表法/开放寻址法处理
```

**时间复杂度对比：**

| 操作 | 哈希表 | 数组 | 链表 | 二叉搜索树 |
|------|-------|------|------|----------|
| 查找 | O(1) 平均 | O(n) | O(n) | O(log n) |
| 插入 | O(1) 平均 | O(n) | O(1) | O(log n) |
| 删除 | O(1) 平均 | O(n) | O(n) | O(log n) |
| 最坏情况 | O(n) 碰撞时 | O(n) | O(n) | O(n) 退化时 |

> **关键警示：** 哈希表的 O(1) 是"平均情况"。当碰撞（Collision）严重时，性能会退化到 O(n)。这个特性被攻击者利用，就是著名的 HashDoS 攻击。

#### 3.2.2 安全应用一：密码存储

哈希表最核心的安全应用是密码存储。现代系统不再存储明文密码，而是存储密码的哈希值。

```python
import hashlib
import os

# ======== 正确的密码存储方案 ========

def hash_password(password: str) -> str:
    """
    使用 PBKDF2 算法对密码进行哈希
    - salt: 随机盐值，防止彩虹表攻击
    - iterations: 迭代次数，增加暴力破解成本
    """
    salt = os.urandom(32)  # 32字节随机盐
    key = hashlib.pbkdf2_hmac(
        'sha256',
        password.encode('utf-8'),
        salt,
        iterations=600000  # OWASP 2023 推荐迭代次数
    )
    # 存储格式：算法$迭代次数$盐$哈希值
    return f"pbkdf2:sha256:600000${salt.hex()}${key.hex()}"

def verify_password(stored_hash: str, password: str) -> bool:
    """验证密码是否匹配存储的哈希"""
    parts = stored_hash.split('$')
    algorithm_info = parts[0]
    salt = bytes.fromhex(parts[1])
    stored_key = parts[2]

    iterations = int(algorithm_info.split(':')[2])
    key = hashlib.pbkdf2_hmac(
        'sha256',
        password.encode('utf-8'),
        salt,
        iterations=iterations
    )
    return key.hex() == stored_key


# ======== 密码存储中的常见错误 ========

# 错误1：明文存储（绝对禁止）
# database.save(password="mypassword123")  # 一旦数据库泄露，全部暴露

# 错误2：使用MD5/SHA1等快速哈希（容易被暴力破解）
# md5_hash = hashlib.md5(password.encode()).hexdigest()
# GPU每秒可计算数十亿次MD5，8位纯数字密码几秒破解

# 错误3：不加盐（容易被彩虹表攻击）
# sha256_hash = hashlib.sha256(password.encode()).hexdigest()
# 攻击者可以预先计算常见密码的哈希值进行查表

# 错误4：盐值太短或可预测
# salt = "fixed_salt"  # 攻击者可以针对此盐值构建彩虹表
```

**密码哈希算法选择对比：**

| 算法 | 安全性 | 速度 | 适用场景 | 备注 |
|------|-------|------|---------|------|
| MD5 | ✗ 已破解 | 极快 | 绝不用于密码 | GPU每秒数十亿次计算 |
| SHA-1 | ✗ 已破解 | 很快 | 绝不用于密码 | 2017年Google实现了实际碰撞 |
| SHA-256 | △ 不推荐 | 快 | 校验文件完整性 | 速度太快，不适合密码 |
| bcrypt | ✓ 推荐 | 慢（设计如此） | 通用密码存储 | 自带盐值，可调节cost |
| scrypt | ✓ 推荐 | 慢 | 需要内存硬化的场景 | 抗GPU/ASIC |
| Argon2 | ✓ 最佳 | 慢 | 新项目首选 | 2015年密码哈希竞赛冠军 |
| PBKDF2 | ✓ 可用 | 可调 | 合规性要求的场景 | NIST推荐，迭代次数决定安全强度 |

#### 3.2.3 安全应用二：会话管理

Web应用使用哈希表存储会话（Session）数据，将Session ID映射到用户信息。

```python
import secrets
import time

class SessionStore:
    """
    基于哈希表的会话存储
    实际生产中通常使用Redis作为后端
    """
    def __init__(self, max_age=3600):
        self._store = {}           # 核心哈希表：session_id → session_data
        self._max_age = max_age    # 会话过期时间（秒）

    def create_session(self, user_id: str, role: str = "user") -> str:
        """
        创建新会话
        使用secrets模块生成密码学安全的随机Session ID
        """
        # 安全要点：Session ID 必须是密码学安全的随机数
        # 错误做法：session_id = str(random.randint(100000, 999999))
        # 上述做法的熵只有 ~20 bits，可以被暴力枚举
        session_id = secrets.token_urlsafe(32)  # 256 bits 熵

        self._store[session_id] = {
            'user_id': user_id,
            'role': role,
            'created_at': time.time(),
            'last_accessed': time.time(),
            'ip_address': None,     # 可选：绑定IP防劫持
            'user_agent': None      # 可选：绑定UA防劫持
        }
        return session_id

    def get_session(self, session_id: str) -> dict | None:
        """获取会话数据，同时检查是否过期"""
        session = self._store.get(session_id)
        if session is None:
            return None

        # 安全检查：会话是否过期
        if time.time() - session['last_accessed'] > self._max_age:
            del self._store[session_id]  # 清除过期会话
            return None

        session['last_accessed'] = time.time()  # 更新最后访问时间
        return session

    def destroy_session(self, session_id: str):
        """销毁会话（用户登出时调用）"""
        self._store.pop(session_id, None)

    def cleanup_expired(self):
        """清理所有过期会话（定期执行）"""
        now = time.time()
        expired = [
            sid for sid, data in self._store.items()
            if now - data['last_accessed'] > self._max_age
        ]
        for sid in expired:
            del self._store[sid]
        return len(expired)


# ======== 会话管理中的安全问题 ========

# 问题1：Session ID 可预测
# 如果Session ID使用线性递增或弱随机数生成器，攻击者可以预测其他用户的Session ID

# 问题2：Session 固定攻击（Session Fixation）
# 攻击者诱导用户使用攻击者已知的Session ID登录
# 防御：登录后必须重新生成Session ID

# 问题3：Session 未及时过期
# 用户登出后Session仍然有效，攻击者可以复用
# 防御：登出时立即销毁Session，设置合理的过期时间

# 问题4：会话存储哈希表未做定期清理
# 大量过期会话堆积导致内存耗尽（DoS攻击向量）
# 防御：定期执行cleanup_expired()，设置最大会话数限制
```

**会话管理安全检查清单：**

| 检查项 | 安全要求 | 常见漏洞 |
|--------|---------|---------|
| Session ID 生成 | 使用 `secrets.token_urlsafe(32)` 或更高熵值 | 使用 `rand()` 或时间戳 |
| Session ID 长度 | 至少128 bits 熵 | 6位数字（仅百万级） |
| 登录后处理 | 重新生成 Session ID | 复用原有 Session ID |
| 过期机制 | 设置合理的 max_age | 永不过期 |
| 存储位置 | HttpOnly + Secure Cookie | 存储在 localStorage（可被 XSS 窃取） |
| 绑定信息 | 绑定 IP + User-Agent | 无绑定（可被劫持） |
| 登出处理 | 服务端销毁 Session | 仅清除客户端 Cookie |

#### 3.2.4 安全应用三：速率限制

使用哈希表实现基于 IP 地址的请求速率限制，这是防御暴力破解和 DDoS 的基础手段。

```python
import time
from collections import defaultdict

class RateLimiter:
    """
    基于滑动窗口的速率限制器
    使用哈希表存储每个客户端的请求记录
    """
    def __init__(self, max_requests=100, window_seconds=60):
        self._requests = defaultdict(list)  # IP → [timestamp1, timestamp2, ...]
        self._max_requests = max_requests
        self._window_seconds = window_seconds

    def is_allowed(self, client_ip: str) -> tuple[bool, dict]:
        """
        检查客户端请求是否在速率限制内
        返回 (是否允许, 响应头信息)
        """
        now = time.time()
        window_start = now - self._window_seconds

        # 清理窗口外的旧记录
        self._requests[client_ip] = [
            ts for ts in self._requests[client_ip]
            if ts > window_start
        ]

        remaining = self._max_requests - len(self._requests[client_ip])
        headers = {
            'X-RateLimit-Limit': str(self._max_requests),
            'X-RateLimit-Remaining': str(max(0, remaining)),
            'X-RateLimit-Reset': str(int(now + self._window_seconds))
        }

        if remaining <= 0:
            return False, headers

        self._requests[client_ip].append(now)
        return True, headers

    def block_ip(self, client_ip: str, duration=3600):
        """
        主动封锁IP（用于检测到恶意行为时）
        """
        # 实际实现中会写入Redis或防火墙规则
        pass


# ======== 速率限制的绕过与对抗 ========

# 攻击者绕过速率限制的常见方法：
# 1. IP轮换：使用代理池、Tor网络切换IP
# 2. 分布式攻击：从多个IP同时发起请求
# 3. X-Forwarded-For伪造：如果服务端信任代理头
# 4. 慢速攻击：控制请求速率刚好在阈值之下

# 防御者的应对策略：
# 1. 多维度限速：IP + 账号 + API路径 组合限制
# 2. 不信任客户端IP头：使用TCP连接的真实IP
# 3. 行为分析：检测请求模式而非仅看频率
# 4. 验证码挑战：对可疑请求弹出验证
```

#### 3.2.5 安全应用四：布隆过滤器

布隆过滤器（Bloom Filter）是基于多个哈希函数的概率数据结构，用于快速判断一个元素是否存在于集合中。安全领域中大量用于恶意 URL/域名/IP 的快速筛选。

```python
import math
import mmh3  # MurmurHash3，布隆过滤器常用的非加密哈希函数
from bitarray import bitarray

class BloomFilter:
    """
    布隆过滤器实现
    核心特性：
    - 如果判断"不存在"，则一定不存在（零假阴性）
    - 如果判断"存在"，则可能存在（有假阳性）
    - 不支持删除操作（除非使用 Counting Bloom Filter）
    """
    def __init__(self, expected_items: int, fp_rate: float = 0.01):
        """
        参数：
            expected_items: 预期存储的元素数量
            fp_rate: 期望的假阳性率（默认1%）
        """
        self.expected_items = expected_items
        self.fp_rate = fp_rate

        # 根据预期元素数和假阳性率计算最优的位数组大小和哈希函数数量
        self.size = self._optimal_size(expected_items, fp_rate)
        self.hash_count = self._optimal_hash_count(self.size, expected_items)
        self.bit_array = bitarray(self.size)
        self.bit_array.setall(0)
        self.count = 0  # 已添加的元素计数

    def _optimal_size(self, n: int, p: float) -> int:
        """计算最优位数组大小：m = -(n * ln(p)) / (ln(2))^2"""
        m = -(n * math.log(p)) / (math.log(2) ** 2)
        return int(m)

    def _optimal_hash_count(self, m: int, n: int) -> int:
        """计算最优哈希函数数量：k = (m/n) * ln(2)"""
        k = (m / n) * math.log(2)
        return int(k)

    def add(self, item: str):
        """添加元素到布隆过滤器"""
        for i in range(self.hash_count):
            # 使用不同的seed产生不同的哈希值
            index = mmh3.hash(item, i) % self.size
            self.bit_array[index] = 1
        self.count += 1

    def contains(self, item: str) -> bool:
        """
        检查元素是否可能存在
        返回False = 一定不存在
        返回True = 可能存在（需要进一步确认）
        """
        for i in range(self.hash_count):
            index = mmh3.hash(item, i) % self.size
            if self.bit_array[index] == 0:
                return False  # 至少有一位为0，元素一定不存在
        return True  # 所有位都为1，元素可能存在

    def current_fp_rate(self) -> float:
        """计算当前的实际假阳性率"""
        # fp_rate ≈ (1 - e^(-k*n/m))^k
        exponent = -self.hash_count * self.count / self.size
        return (1 - math.exp(exponent)) ** self.hash_count


# ======== 安全应用示例：恶意URL快速筛选 ========

def create_malicious_url_filter():
    """
    构建恶意URL布隆过滤器
    实际场景：Chrome Safe Browsing、uBlock Origin 等使用类似技术
    """
    # 假设有100万个已知恶意URL
    bf = BloomFilter(expected_items=1_000_000, fp_rate=0.001)

    # 模拟添加已知恶意URL
    malicious_urls = [
        "http://evil-phishing-site.com/login",
        "http://malware-download.net/payload.exe",
        "http://xss-exploit.com/steal?cookie=...",
        # ... 实际场景中会有百万级数据
    ]
    for url in malicious_urls:
        bf.add(url)

    # 查询时的使用模式
    test_url = "http://suspicious-site.com/page"
    if bf.contains(test_url):
        # 布隆过滤器说"可能存在"
        # 必须进一步确认（查询完整数据库或云端API）
        # 因为这可能是假阳性
        print(f"[!] {test_url} 可能是恶意URL，需要进一步确认")
    else:
        # 布隆过滤器说"一定不存在"
        # 可以安全放行，无需查询完整数据库
        print(f"[OK] {test_url} 确认不是恶意URL")

    # 显示资源使用情况
    size_kb = len(bf.bit_array) / 8 / 1024
    print(f"位数组大小: {size_kb:.1f} KB")
    print(f"哈希函数数量: {bf.hash_count}")
    print(f"当前假阳性率: {bf.current_fp_rate():.6f}")
    return bf

# 布隆过滤器在安全产品中的应用：
# 1. Chrome Safe Browsing：本地布隆过滤器快速判断URL是否安全
# 2. 邮件网关：快速判断发件域名是否在黑名单中
# 3. DNS过滤：快速判断域名是否属于恶意域名
# 4. 入侵检测系统：快速匹配恶意特征码
# 5. 密码强度检查：判断密码是否在已知泄露密码库中（如Have I Been Pwned）
```

**布隆过滤器参数与资源消耗对照表：**

| 预期元素数 | 假阳性率 | 位数组大小 | 哈希函数数 | 内存消耗 |
|-----------|---------|----------|----------|---------|
| 10,000 | 1% | 95,850 bits | 7 | 12 KB |
| 100,000 | 1% | 958,505 bits | 7 | 117 KB |
| 1,000,000 | 1% | 9,585,058 bits | 7 | 1.14 MB |
| 1,000,000 | 0.1% | 14,377,587 bits | 10 | 1.71 MB |
| 10,000,000 | 0.01% | 191,701,166 bits | 13 | 22.8 MB |

> **安全设计启示：** 一个包含 100 万个恶意 URL 的布隆过滤器仅需约 1.2 MB 内存，且查询时间恒定。这就是为什么浏览器能在不联网的情况下快速判断 URL 安全性。

#### 3.2.6 哈希碰撞攻击（HashDoS）

哈希表的 O(1) 性能依赖于良好的哈希函数分布。如果攻击者能构造大量产生相同哈希值的键，哈希表就会退化为链表，性能从 O(1) 退化到 O(n)，导致拒绝服务。

```python
# ======== HashDoS 攻击原理演示 ========

# 以下演示说明哈希碰撞如何影响性能

class NaiveHashTable:
    """
    简化的哈希表实现（使用链表法处理碰撞）
    用于演示 HashDoS 的原理
    """
    def __init__(self, size=16):
        self.size = size
        self.buckets = [[] for _ in range(size)]
        self.collision_count = 0

    def _hash(self, key: str) -> int:
        """一个故意设计糟糕的哈希函数（仅用于演示）"""
        return sum(ord(c) for c in key) % self.size

    def put(self, key, value):
        index = self._hash(key)
        bucket = self.buckets[index]

        # 检查是否已存在
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return

        if len(bucket) > 0:
            self.collision_count += 1
        bucket.append((key, value))

    def get(self, key):
        index = self._hash(key)
        for k, v in self.buckets[index]:
            if k == key:
                return v
        return None

    def max_chain_length(self) -> int:
        """最长链表长度——衡量碰撞严重程度"""
        return max(len(b) for b in self.buckets)


# 正常情况：哈希分布均匀
normal_ht = NaiveHashTable(size=16)
normal_keys = [f"user_{i}" for i in range(100)]
for key in normal_keys:
    normal_ht.put(key, "data")
print(f"正常情况 - 碰撞次数: {normal_ht.collision_count}, 最长链: {normal_ht.max_chain_length()}")

# 攻击情况：构造碰撞键
# 攻击者通过分析哈希函数，构造大量碰撞的键
attack_ht = NaiveHashTable(size=16)
# 这些键的字符ASCII码之和都是 16 的倍数 + 相同余数
attack_keys = [f"a{'a' * i}" for i in range(100)]
# 在真实攻击中，攻击者会精确计算出能产生碰撞的输入
for key in attack_keys:
    attack_ht.put(key, "data")
print(f"攻击情况 - 碰撞次数: {attack_ht.collision_count}, 最长链: {attack_ht.max_chain_length()}")
```

### 3.3 树结构在安全中的应用

树（Tree）是一种层次化的数据结构，在安全领域中用于组织和检索大量分层数据。理解树结构有助于理解数据库索引、域名过滤、文件系统安全等核心问题。

#### 3.3.1 二叉搜索树（BST）与安全

二叉搜索树是最基础的树结构：左子节点值小于父节点，右子节点值大于父节点。平均查找时间 O(log n)，但最坏情况（数据有序插入时）退化为 O(n)。

```python
class BSTNode:
    """二叉搜索树节点"""
    def __init__(self, key, value=None):
        self.key = key
        self.value = value or key
        self.left = None
        self.right = None

class BST:
    """二叉搜索树——用于安全场景中的高效查找"""
    def __init__(self):
        self.root = None

    def insert(self, key, value=None):
        if self.root is None:
            self.root = BSTNode(key, value)
            return
        current = self.root
        while True:
            if key < current.key:
                if current.left is None:
                    current.left = BSTNode(key, value)
                    return
                current = current.left
            elif key > current.key:
                if current.right is None:
                    current.right = BSTNode(key, value)
                    return
                current = current.right
            else:
                current.value = value  # 更新已存在的键
                return

    def search(self, key) -> tuple[bool, int]:
        """
        查找键值，返回 (是否找到, 比较次数)
        比较次数体现了安全工具的性能指标
        """
        current = self.root
        comparisons = 0
        while current:
            comparisons += 1
            if key == current.key:
                return True, comparisons
            elif key < current.key:
                current = current.left
            else:
                current = current.right
        return False, comparisons


# ======== 安全应用：IP地理位置查找 ========

def ip_to_int(ip: str) -> int:
    """将IPv4地址转换为整数，便于BST比较"""
    parts = [int(p) for p in ip.split('.')]
    return (parts[0] << 24) + (parts[1] << 16) + (parts[2] << 8) + parts[3]

# 构建IP地址到地理位置的BST索引
ip_geo_bst = BST()
ip_ranges = [
    ("192.168.0.0", "内网-北京机房"),
    ("10.0.0.0", "内网-上海机房"),
    ("172.16.0.0", "内网-广州机房"),
    ("203.0.113.0", "已知恶意IP段-东欧"),
    ("198.51.100.0", "已知代理IP段-美国"),
]
for ip, location in ip_ranges:
    ip_geo_bst.insert(ip_to_int(ip), location)

# 查找某个IP的地理归属
found, comparisons = ip_geo_bst.search(ip_to_int("192.168.0.1"))
print(f"查找结果: {'找到' if found else '未找到'}, 比较次数: {comparisons}")
```

> **安全启示：** BST 最坏情况下会退化为链表（O(n)），这就是为什么实际系统使用自平衡树（红黑树、AVL树）或跳表来保证 O(log n) 的查找性能。

#### 3.3.2 前缀树（Trie）与恶意域名检测

前缀树（Trie，发音同"try"）是专门为字符串前缀匹配设计的树结构。每个节点代表一个字符，从根到叶的路径构成完整的字符串。在安全领域中，前缀树被广泛用于恶意域名检测、敏感词过滤和WAF规则匹配。

```python
class TrieNode:
    """前缀树节点"""
    def __init__(self):
        self.children = {}      # 字符 → 子节点
        self.is_end = False     # 是否是某个域名的结尾
        self.metadata = None    # 附加信息（如威胁类型、严重等级）

class DomainTrie:
    """
    域名前缀树——用于恶意域名快速检测
    支持精确匹配和后缀匹配（域名从右向左插入）
    """
    def __init__(self):
        self.root = TrieNode()

    def add_domain(self, domain: str, threat_type: str = "malicious"):
        """
        添加恶意域名
        注意：域名从右向左插入（TLD → SLD → 子域名）
        这样可以高效匹配 *.evil.com 等通配符规则
        """
        # 将域名反转后逐字符插入
        # "malware.evil.com" → "moc.live.erawlam"
        reversed_domain = domain[::-1]
        node = self.root
        for char in reversed_domain:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True
        node.metadata = {
            'domain': domain,
            'threat_type': threat_type,
            'added_at': __import__('time').time()
        }

    def is_malicious(self, domain: str) -> dict | None:
        """
        检查域名是否为恶意域名
        支持精确匹配和上级域名匹配
        例如：如果 evil.com 是恶意域名，则 sub.evil.com 也会被匹配
        """
        reversed_domain = domain[::-1]
        node = self.root
        for i, char in enumerate(reversed_domain):
            if char not in node.children:
                return None
            node = node.children[char]
            # 检查是否匹配到一个完整的恶意域名
            if node.is_end:
                # 确保匹配的是完整的域名标签边界
                # 例如 "not-evil.com" 不应该匹配 "evil.com"
                remaining = reversed_domain[i+1:]
                if not remaining or remaining[0] == '.':
                    return node.metadata
        return None

    def count_domains(self) -> int:
        """统计存储的域名数量"""
        count = 0
        stack = [self.root]
        while stack:
            node = stack.pop()
            if node.is_end:
                count += 1
            stack.extend(node.children.values())
        return count

    ("cryptominer.xyz", "挖矿脚本"),

    "download.malware-host.net",  # 应该匹配

    "evil-phishing.com",          # 应该匹配（精确）
    "not-evil-phishing.com",      # 不应该匹配（只是包含子串）
```

#### 3.3.3 Merkle树与数据完整性验证

Merkle树（哈希树）是一种二叉树结构，其中每个叶子节点是数据块的哈希值，每个非叶子节点是其子节点哈希值的组合哈希。它是现代数据完整性验证的基石。

```text
Merkle树结构示意：

                    Root Hash
                   /         \
            Hash(AB)         Hash(CD)
           /       \        /       \
      Hash(A)   Hash(B)  Hash(C)  Hash(D)
         |         |        |        |
       Data A    Data B   Data C   Data D

验证原理：
- 如果 Data A 被篡改，Hash(A) 会改变
- Hash(A) 改变导致 Hash(AB) 改变
- Hash(AB) 改变导致 Root Hash 改变
- 只需对比 Root Hash 就能检测任意数据篡改
```

```python
import hashlib

class MerkleTree:
    """
    Merkle树实现
    安全应用：数据完整性验证、分布式系统同步、区块链
    """
    def __init__(self, data_blocks: list[bytes]):
        self.data_blocks = data_blocks
        self.leaves = [self._hash(block) for block in data_blocks]
        self.tree = self._build_tree(self.leaves)
        self.root = self.tree[-1][0] if self.tree else None

    def _hash(self, data: bytes) -> str:
        """计算SHA-256哈希"""
        return hashlib.sha256(data).hexdigest()

    def _build_tree(self, leaves: list[str]) -> list[list[str]]:
        """自底向上构建Merkle树"""
        tree = [leaves]
        current_level = leaves

        while len(current_level) > 1:
            next_level = []
            for i in range(0, len(current_level), 2):
                left = current_level[i]
                # 如果节点数为奇数，复制最后一个节点
                right = current_level[i + 1] if i + 1 < len(current_level) else left
                parent_hash = self._hash((left + right).encode())
                next_level.append(parent_hash)
            tree.append(next_level)
            current_level = next_level

        return tree

    def get_root_hash(self) -> str:
        """获取根哈希——任何数据变化都会改变这个值"""
        return self.root

    def get_proof(self, index: int) -> list[tuple[str, str]]:
        """
        获取某个数据块的存在性证明（Merkle Proof）
        返回路径上每层的兄弟节点哈希和方向（left/right）
        用于分布式系统中验证某个数据块是否属于这棵树
        """
        if index >= len(self.leaves):
            return []

        proof = []
        current_index = index

        for level in self.tree[:-1]:  # 不包含根节点
            if current_index % 2 == 0:
                # 当前节点是左子节点，兄弟在右边
                sibling_index = current_index + 1
                direction = 'right'
            else:
                # 当前节点是右子节点，兄弟在左边
                sibling_index = current_index - 1
                direction = 'left'

            # 获取兄弟节点的哈希
            if sibling_index < len(level):
                sibling_hash = level[sibling_index]
            else:
                sibling_hash = level[current_index]  # 奇数节点时复制自身

            proof.append((sibling_hash, direction))
            current_index //= 2

        return proof

    def verify_proof(self, data: bytes, proof: list[tuple[str, str]], root_hash: str) -> bool:
        """
        验证数据块的Merkle Proof
        即使没有完整的数据集，也能验证某个数据块是否属于这棵树
        """
        current_hash = self._hash(data)

        for sibling_hash, direction in proof:
            if direction == 'right':
                combined = current_hash + sibling_hash
            else:
                combined = sibling_hash + current_hash
            current_hash = self._hash(combined.encode())

        return current_hash == root_hash


# ======== 安全应用示例 ========

# 场景3：区块链

# 轻节点只需下载区块头（包含Merkle Root），无需下载全部交易

# 实际演示：文件完整性校验

    b"file_2_content: users.db",
    b"file_3_content: secrets.env",
    b"file_4_content: app.log",
```

#### 3.3.4 B树/B+树与数据库索引安全

B树和B+树是关系型数据库索引的核心数据结构。理解它们有助于理解SQL注入的性能影响、数据库查询优化，以及为什么某些注入技术比其他技术更高效。

```text
B+树结构（MySQL InnoDB 聚簇索引）：

                    [30 | 60]
                   /    |    \
           [10|20]  [40|50]  [70|80]
          /  |  \   /  |  \   /  |  \
        [叶子节点通过链表相连]
        10→20→30→40→50→60→70→80

B+树的关键特性：
1. 所有数据都存储在叶子节点（非叶子节点只存索引键）
2. 叶子节点通过双向链表连接（支持范围查询）
3. 树的高度通常为2-4层（即使存储上亿条记录）
4. 每次查找需要 log_m(n) 次磁盘IO（m为阶数，n为记录数）
```

**B+树对SQL注入的影响：**

| 注入场景 | 是否利用索引 | 性能影响 | 安全影响 |
|---------|------------|---------|---------|
| `WHERE id=1 AND 1=1` | 使用主键索引 | O(log n) | 快速响应，难以通过延迟判断 |
| `WHERE username='admin' AND SUBSTRING(password,1,1)='a'` | 可能使用索引 | O(log n) | 布尔盲注可行但较慢 |
| `WHERE id=1 AND SLEEP(5)` | 不影响索引查找 | 额外延迟5秒 | 时间盲注的标准技术 |
| `WHERE username LIKE '%admin%'` | 无法使用索引 | O(n) 全表扫描 | 慢查询可被利用作为DoS |
| `WHERE 1=1 ORDER BY (SELECT COUNT(*) FROM information_schema.tables)` | 无索引 | O(n) 复杂子查询 | 报错注入可能触发慢查询 |

```sql
-- 理解B+树索引对安全的实际意义

-- 1. 覆盖索引与注入信息泄露
-- 如果查询只需要索引列，MySQL不会读取数据页（Covering Index）
-- 攻击者可以利用这一点判断目标列是否被索引

-- 判断id列是否有索引（通过响应时间差异）：
SELECT * FROM users WHERE id = 1;              -- 使用聚簇索引，极快
SELECT * FROM users WHERE id = 1 AND 1=1;      -- 仍然使用索引
SELECT * FROM users WHERE email = 'test@test.com'; -- 如果email无索引，较慢

-- 2. 索引与ORDER BY注入
-- B+树叶子节点的有序性使得ORDER BY查询高效
-- 攻击者可以利用ORDER BY进行基于错误的注入：
-- ' ORDER BY (SELECT CASE WHEN (1=1) THEN 1 ELSE (SELECT 1 UNION SELECT 2) END)--
-- 通过观察排序是否报错来提取信息

-- 3. 索引碎片与安全
-- 频繁的INSERT/DELETE会导致B+树页分裂和碎片化
-- 碎片化的索引会降低查询性能，放大注入攻击的影响
-- 定期执行 OPTIMIZE TABLE 是数据库维护的一部分
```

#### 3.3.5 自平衡树：红黑树与安全工具性能

红黑树是实际系统中使用最广泛的自平衡二叉搜索树。Linux内核的进程调度器、Java的 TreeMap、C++ 的 std::map 都使用红黑树。

```text
红黑树的五个性质（保证树的高度为 O(log n)）：

1. 每个节点是红色或黑色
2. 根节点是黑色
3. 所有叶子节点（NIL）是黑色
4. 红色节点的子节点必须是黑色（不能有两个连续的红色节点）
5. 从任意节点到其叶子节点的所有路径包含相同数量的黑色节点

性质4和5共同保证：最长路径不超过最短路径的两倍
→ 树高 ≤ 2 * log₂(n+1)
→ 查找、插入、删除均为 O(log n)
```

**红黑树在安全工具中的应用：**

| 工具/系统 | 使用位置 | 作用 |
|----------|---------|------|
| Linux内核 | CFS进程调度器（`rb_root`） | O(log n) 插入/查找进程 |
| iptables/nftables | 规则匹配 | 高效匹配网络包规则 |
| Java TreeMap | IDS规则存储 | 有序存储IP范围规则 |
| Nginx | 定时器管理 | 高效管理连接超时 |
| IDS/IPS | 规则排序 | Snort规则按优先级排序 |

### 3.4 图结构在安全中的应用

图（Graph）是描述实体间关系最灵活的数据结构。在安全领域中，网络拓扑、权限关系、代码执行路径都可以用图来建模，而图算法（最短路径、连通分量、拓扑排序等）为自动化安全分析提供了数学基础。

#### 3.4.1 图的基本概念

```text
图的分类：

1. 无向图（Undirected Graph）
   - 边没有方向
   - 示例：主机间的网络连接（A↔B）

2. 有向图（Directed Graph）
   - 边有方向
   - 示例：权限关系（A→B 表示A对B有权限）

3. 加权图（Weighted Graph）
   - 边有权重
   - 示例：攻击成本图（边权重=攻击难度）

4. 有向无环图（DAG）
   - 没有环的有向图
   - 示例：依赖关系、攻击步骤序列

图的存储方式：
1. 邻接矩阵：O(V²)空间，O(1)判断边是否存在
2. 邻接表：O(V+E)空间，适合稀疏图
3. 邻接哈希表：O(V+E)空间，适合动态增删
```

#### 3.4.2 网络拓扑图与攻击路径分析

网络拓扑图是渗透测试中最重要的图结构应用。节点代表主机、路由器、防火墙等网络设备，边代表网络连接关系。

```python
from collections import deque, defaultdict
import heapq

class NetworkGraph:
    """
    网络拓扑图——用于攻击路径分析和渗透规划
    支持：
    - BFS/DFS遍历网络节点
    - Dijkstra最短路径（最小攻击成本路径）
    - 连通分量分析（隔离网络区域识别）
    """
    def __init__(self):
        self.graph = defaultdict(dict)    # node → {neighbor: weight}
        self.node_info = {}               # node → 属性信息

    def add_node(self, node_id: str, **properties):
        """添加网络节点"""
        self.node_info[node_id] = properties

    def add_edge(self, from_node: str, to_node: str, cost: float = 1.0, bidirectional=True):
        """
        添加网络连接
        cost: 攻击成本/难度（值越小越容易攻击）
        """
        self.graph[from_node][to_node] = cost
        if bidirectional:
            self.graph[to_node][from_node] = cost

    def find_attack_path(self, source: str, target: str) -> tuple[list[str], float]:
        """
        使用Dijkstra算法找到从源节点到目标节点的最小成本路径
        成本可以是：攻击难度、跳数、时间等
        """
        distances = {source: 0}
        predecessors = {source: None}
        priority_queue = [(0, source)]
        visited = set()

        while priority_queue:
            current_dist, current_node = heapq.heappop(priority_queue)

            if current_node in visited:
                continue
            visited.add(current_node)

            if current_node == target:
                # 回溯构建路径
                path = []
                node = target
                while node is not None:
                    path.append(node)
                    node = predecessors[node]
                return path[::-1], current_dist

            for neighbor, cost in self.graph[current_node].items():
                if neighbor in visited:
                    continue
                new_dist = current_dist + cost
                if neighbor not in distances or new_dist < distances[neighbor]:
                    distances[neighbor] = new_dist
                    predecessors[neighbor] = current_node
                    heapq.heappush(priority_queue, (new_dist, neighbor))

        return [], float('inf')  # 无法到达

    def find_all_reachable(self, source: str) -> set[str]:
        """
        BFS找出从源节点可达的所有节点
        用于评估攻击者入侵一个节点后的影响范围
        """
        reachable = set()
        queue = deque([source])
        reachable.add(source)

        while queue:
            node = queue.popleft()
            for neighbor in self.graph[node]:
                if neighbor not in reachable:
                    reachable.add(neighbor)
                    queue.append(neighbor)
        return reachable

    def find_connected_components(self) -> list[set[str]]:
        """
        找出所有的连通分量
        每个连通分量是一个独立的网络区域
        """
        visited = set()
        components = []

        all_nodes = set(self.graph.keys())
        for node_list in self.graph.values():
            all_nodes.update(node_list.keys())

        for node in all_nodes:
            if node not in visited:
                component = set()
                queue = deque([node])
                while queue:
                    current = queue.popleft()
                    if current in visited:
                        continue
                    visited.add(current)
                    component.add(current)
                    for neighbor in self.graph[current]:
                        if neighbor not in visited:
                            queue.append(neighbor)
                components.append(component)
        return components

    def find_critical_nodes(self) -> list[tuple[str, int]]:
        """
        找出关键节点（割点）——移除后会增加连通分量数的节点
        这些节点是网络架构的单点故障，也是防御重点
        """
        # 简化实现：找出度数最高的节点
        degrees = []
        all_nodes = set(self.graph.keys())
        for node_list in self.graph.values():
            all_nodes.update(node_list.keys())

        for node in all_nodes:
            degree = len(self.graph.get(node, {}))
            degrees.append((node, degree))
        return sorted(degrees, key=lambda x: -x[1])
```

**攻击路径分析的决策矩阵：**

| 分析维度 | 图算法 | 安全决策支持 |
|---------|--------|------------|
| 最短攻击路径 | Dijkstra / A* | 确定攻击者最可能的入侵路径，优先加固 |
| 所有可达节点 | BFS / DFS | 评估某个节点被攻陷后的影响范围 |
| 连通分量 | BFS / Union-Find | 识别网络隔离是否有效 |
| 关键节点（割点） | Tarjan算法 | 找出单点故障，部署冗余 |
| 最大流/最小割 | Ford-Fulkerson | 确定网络带宽瓶颈和DDoS防护重点 |
| 环路检测 | DFS / 拓扑排序 | 检测依赖循环导致的安全策略矛盾 |

#### 3.4.3 权限关系图与提权分析

权限关系图是有向图的经典安全应用：节点是用户、角色、资源，边是权限关系。通过图的传递闭包分析，可以发现权限提升路径和违反最小权限原则的问题。

```python
class PermissionGraph:
    """
    权限关系图——用于分析权限提升路径和最小权限审计
    """
    def __init__(self):
        self.graph = defaultdict(set)    # entity → set of entities it can access/control
        self.reverse_graph = defaultdict(set)  # 反向图：entity → set of entities that can access it
        self.node_types = {}             # entity → type (user/role/resource/service)

    def add_entity(self, entity_id: str, entity_type: str):
        """添加实体（用户/角色/资源/服务）"""
        self.node_types[entity_id] = entity_type

    def add_permission(self, from_entity: str, to_entity: str, permission_type: str = "access"):
        """
        添加权限关系
        permission_type: access/own/admin/execute/read/write
        """
        self.graph[from_entity].add((to_entity, permission_type))
        self.reverse_graph[to_entity].add((from_entity, permission_type))

    def find_privilege_escalation_paths(self, start_entity: str, target_entity: str) -> list[list[str]]:
        """
        找出所有可能的权限提升路径
        从 start_entity 到 target_entity 的所有路径
        """
        all_paths = []

        def dfs(current, target, path, visited):
            if current == target:
                all_paths.append(list(path))
                return
            for neighbor, perm_type in self.graph.get(current, set()):
                if neighbor not in visited:
                    visited.add(neighbor)
                    path.append((neighbor, perm_type))
                    dfs(neighbor, target, path, visited)
                    path.pop()
                    visited.discard(neighbor)

        visited = {start_entity}
        path = [(start_entity, "start")]
        dfs(start_entity, target_entity, path, visited)
        return all_paths

    def find_excessive_permissions(self) -> list[dict]:
        """
        检测过度权限——违反最小权限原则的情况
        """
        violations = []

        # 规则1：普通用户不应有admin权限
        for user, perms in self.graph.items():
            if self.node_types.get(user) == "user":
                for target, perm_type in perms:
                    if perm_type == "admin":
                        violations.append({
                            "type": "excessive_admin",
                            "entity": user,
                            "target": target,
                            "severity": "HIGH"
                        })

        # 规则2：服务账号不应能访问其他服务
        for service, perms in self.graph.items():
            if self.node_types.get(service) == "service":
                for target, perm_type in perms:
                    if self.node_types.get(target) == "service" and service != target:
                        violations.append({
                            "type": "service_to_service_access",
                            "entity": service,
                            "target": target,
                            "severity": "MEDIUM"
                        })

        # 规则3：检查是否存在权限传递链
        # 例如：user→role1→role2→admin，如果user不应该有admin权限
        for user, perms in self.graph.items():
            if self.node_types.get(user) == "user":
                reachable = self._get_reachable_with_transitive(user)
                for entity, path in reachable:
                    if self.node_types.get(entity) == "resource" and len(path) > 2:
                        violations.append({
                            "type": "transitive_permission",
                            "entity": user,
                            "target": entity,
                            "path": " → ".join(path),
                            "severity": "MEDIUM"
                        })

        return violations

    def _get_reachable_with_transitive(self, start: str) -> list[tuple[str, list[str]]]:
        """计算传递闭包——找出通过权限链可达的所有实体"""
        reachable = []
        visited = set()
        queue = deque([(start, [start])])

        while queue:
            current, path = queue.popleft()
            for neighbor, _ in self.graph.get(current, set()):
                if neighbor not in visited:
                    visited.add(neighbor)
                    new_path = path + [neighbor]
                    reachable.append((neighbor, new_path))
                    queue.append((neighbor, new_path))

        return reachable
```

#### 3.4.4 控制流图（CFG）与漏洞分析

控制流图是二进制逆向工程和代码审计的核心数据结构。节点代表基本块（Basic Block，即一段顺序执行的指令序列），边代表跳转关系（条件跳转、无条件跳转、函数调用等）。

```text
控制流图示例（一个简单的登录验证函数）：

         ┌─────────────────────┐
         │   Block 1: 入口     │
         │   read(username)    │
         │   read(password)    │
         └──────────┬──────────┘
                    │
         ┌──────────▼──────────┐
         │   Block 2: 判断     │
         │   if user == NULL   │
         └────┬─────────┬──────┘
              │true     │false
         ┌────▼─────┐   │
         │ Block 3: │   │
         │ return   │   │
         │ "error"  │   │
         └──────────┘   │
               ┌────────▼──────────┐
               │   Block 4: 判断   │
               │   if pass == NULL │
               └──┬────────────┬───┘
                  │true        │false
             ┌────▼────┐  ┌───▼────────────────┐
             │ Block 5:│  │  Block 6: 核心逻辑  │
             │ return  │  │  hash = md5(pass)   │
             │ "error" │  │  query = "SELECT *  │
             └─────────┘  │  WHERE u='" + user  │
                          │  + "' AND p='" +    │
                          │  hash + "'"         │
                          └────────┬───────────┘
                                   │
                          ┌────────▼───────────┐
                          │   Block 7: 返回     │
                          │   return result     │
                          └────────────────────┘

安全分析要点：
- Block 6 中存在SQL注入：用户输入直接拼接到SQL查询中
- 通过CFG可以追踪"用户输入"数据流到"SQL执行"的完整路径
- 这种分析称为"污点分析"（Taint Analysis）
```

```python
class CFGNode:
    """控制流图节点"""
    def __init__(self, block_id: str, instructions: list[str]):
        self.block_id = block_id
        self.instructions = instructions
        self.successors = []      # 后继节点
        self.predecessors = []    # 前驱节点
        self.tainted_vars = set() # 被污染的变量（污点分析用）

class ControlFlowGraph:
    """
    控制流图——用于漏洞模式识别和污点分析
    """
    def __init__(self):
        self.nodes = {}          # block_id → CFGNode
        self.entry_node = None

    def add_node(self, block_id: str, instructions: list[str]) -> CFGNode:
        node = CFGNode(block_id, instructions)
        self.nodes[block_id] = node
        if self.entry_node is None:
            self.entry_node = node
        return node

    def add_edge(self, from_id: str, to_id: str):
        self.nodes[from_id].successors.append(self.nodes[to_id])
        self.nodes[to_id].predecessors.append(self.nodes[from_id])

    def taint_analysis(self, source_blocks: set[str], sink_patterns: list[str]) -> list[dict]:
        """
        污点分析：追踪从源（source）到汇（sink）的数据流
        source: 用户输入点（HTTP参数、文件读取等）
        sink_pattern: 危险操作模式（SQL执行、命令执行等）
        返回所有从source可达的sink点，以及路径
        """
        findings = []

        for source_id in source_blocks:
            if source_id not in self.nodes:
                continue

            # 从source开始BFS，追踪被污染的数据流
            visited = set()
            queue = deque([(self.nodes[source_id], {source_id})])

            while queue:
                current_node, path = queue.popleft()
                if current_node.block_id in visited:
                    continue
                visited.add(current_node.block_id)

                # 检查当前节点是否包含危险操作
                for instruction in current_node.instructions:
                    for pattern in sink_patterns:
                        if pattern in instruction:
                            findings.append({
                                "source": source_id,
                                "sink": current_node.block_id,
                                "instruction": instruction,
                                "pattern": pattern,
                                "path": list(path),
                                "severity": "CRITICAL"
                            })

                # 继续追踪到后继节点
                for successor in current_node.successors:
                    if successor.block_id not in visited:
                        new_path = path | {successor.block_id}
                        queue.append((successor, new_path))

        return findings

    def find_dead_code(self) -> list[str]:
        """
        检测不可达代码（可能隐藏后门）
        从入口节点出发，所有不可达的基本块都是可疑的
        """
        reachable = set()
        queue = deque([self.entry_node])

        while queue:
            node = queue.popleft()
            if node.block_id in reachable:
                continue
            reachable.add(node.block_id)
            for successor in node.successors:
                queue.append(successor)

        unreachable = set(self.nodes.keys()) - reachable
        return list(unreachable)

# 添加基本块

cfg.add_node("check_user", ["if username == NULL"])
cfg.add_node("err_no_user", ["return '请输入用户名'"])
cfg.add_node("check_pass", ["if password == NULL"])
cfg.add_node("err_no_pass", ["return '请输入密码'"])
cfg.add_node("auth_logic", [
    "hash = md5(password)",

    "result = db.execute(query)"

# 添加控制流边
cfg.add_edge("entry", "check_user")
cfg.add_edge("check_user", "err_no_user")

cfg.add_edge("check_pass", "err_no_pass")

cfg.add_edge("auth_logic", "return_result")
```

#### 3.4.5 调用图（Call Graph）与数据流追踪

调用图描述程序中函数间的调用关系。在安全分析中，调用图用于追踪敏感数据的流转路径（数据从哪里来、经过哪些函数、最终到哪里去）。

```python
class CallGraph:
    """
    调用图——用于数据流追踪和污点传播分析
    """
    def __init__(self):
        self.graph = defaultdict(set)          # caller → set of callees
        self.reverse_graph = defaultdict(set)  # callee → set of callers
        self.functions = {}                     # func_name → function_info

    def add_function(self, name: str, params: list[str] = None, returns: bool = False,
                     is_source: bool = False, is_sink: bool = False, is_sanitizer: bool = False):
        """添加函数节点，标记其安全属性"""
        self.functions[name] = {
            "params": params or [],
            "returns": returns,
            "is_source": is_source,        # 用户输入源（如 request.get()）
            "is_sink": is_sink,            # 危险操作（如 db.execute()）
            "is_sanitizer": is_sanitizer   # 清洗函数（如 escape()）
        }

    def add_call(self, caller: str, callee: str):
        """添加调用关系"""
        self.graph[caller].add(callee)
        self.reverse_graph[callee].add(caller)

    def find_source_to_sink_paths(self) -> list[dict]:
        """
        找出所有从source函数到sink函数的调用路径
        这是安全审计自动化的核心功能
        """
        sources = [name for name, info in self.functions.items() if info.get("is_source")]
        sinks = [name for name, info in self.functions.items() if info.get("is_sink")]

        paths = []
        for source in sources:
            for sink in sinks:
                found_paths = self._find_all_paths(source, sink)
                for path in found_paths:
                    # 检查路径中是否有清洗函数
                    has_sanitizer = any(
                        self.functions.get(node, {}).get("is_sanitizer")
                        for node in path
                    )
                    paths.append({
                        "source": source,
                        "sink": sink,
                        "path": path,
                        "sanitized": has_sanitizer,
                        "risk": "LOW" if has_sanitizer else "HIGH"
                    })

        return paths

    def _find_all_paths(self, start: str, end: str, max_depth: int = 10) -> list[list[str]]:
        """找出两个函数之间的所有调用路径（限深度防止无限递归）"""
        paths = []

        def dfs(current, target, path, depth):
            if depth > max_depth:
                return
            if current == target:
                paths.append(list(path))
                return
            for callee in self.graph.get(current, set()):
                if callee not in path:  # 防止循环
                    path.append(callee)
                    dfs(callee, target, path, depth + 1)
                    path.pop()

        dfs(start, end, [start], 0)
        return paths

    def get_callees(self, func_name: str) -> set[str]:
        """获取某个函数调用的所有函数"""
        return self.graph.get(func_name, set())

    def get_callers(self, func_name: str) -> set[str]:
        """获取调用某个函数的所有函数"""
        return self.reverse_graph.get(func_name, set())
```

#### 3.5.2 入侵检测系统（IDS）的规则匹配

Snort/Suricata等IDS使用多模式匹配算法在高速网络流量中检测恶意特征。其核心是Aho-Corasick自动机（基于前缀树的多模式匹配）：

```python
class AhoCorasickNode:
    """Aho-Corasick自动机节点"""
    def __init__(self):
        self.children = {}
        self.failure = None       # 失败指针（类似KMP的next数组）
        self.output = []          # 匹配到的模式列表
        self.depth = 0            # 节点深度

class AhoCorasick:
    """
    Aho-Corasick多模式匹配算法
    时间复杂度：O(n + m + z)
    - n: 文本长度
    - m: 所有模式串总长度
    - z: 匹配次数

    安全应用：IDS/IPS规则匹配、WAF特征检测、恶意软件签名扫描
    """
    def __init__(self):
        self.root = AhoCorasickNode()

    def add_pattern(self, pattern: str, rule_id: str):
        """添加匹配模式"""
        node = self.root
        for char in pattern:
            if char not in node.children:
                child = AhoCorasickNode()
                child.depth = node.depth + 1
                node.children[char] = child
            node = node.children[char]
        node.output.append(rule_id)

    def build(self):
        """构建失败指针（BFS）"""
        queue = deque()
        # 第一层节点的失败指针指向根节点
        for child in self.root.children.values():
            child.failure = self.root
            queue.append(child)

        while queue:
            current = queue.popleft()
            for char, child in current.children.items():
                queue.append(child)

                # 查找失败指针
                failure = current.failure
                while failure is not None and char not in failure.children:
                    failure = failure.failure
                child.failure = failure.children.get(char, self.root) if failure else self.root

                # 合并输出
                child.output = child.output + child.failure.output

    def search(self, text: str) -> list[tuple[int, str]]:
        """
        在文本中搜索所有匹配的模式
        返回 (位置, 规则ID) 列表
        """
        matches = []
        node = self.root

        for i, char in enumerate(text):
            while node is not self.root and char not in node.children:
                node = node.failure
            node = node.children.get(char, self.root)

            for rule_id in node.output:
                matches.append((i, rule_id))

        return matches


# ======== 实际使用：IDS规则匹配 ========

    ("base64_decode", "EVD-001: 混淆代码"),
    ("SELECT * FROM information_schema", "SQLI-003: Schema探测"),
```

#### 3.5.3 密码强度检查器中的数据结构组合

一个完善的密码强度检查器需要组合多种数据结构：

```python
class PasswordStrengthChecker:
    """
    密码强度检查器——组合多种数据结构
    1. 布隆过滤器：快速检查是否在已知泄露密码库中
    2. 前缀树：检查是否包含常见密码模式
    3. 哈希表：存储检查规则和统计信息
    """
    def __init__(self):
        # 布隆过滤器：存储已知泄露密码（如Have I Been Pwned数据集）
        self.leaked_passwords = BloomFilter(expected_items=10_000_000, fp_rate=0.001)

        # 前缀树：存储常见密码模式
        self.common_patterns = TrieNode()

        # 哈希表：存储字符类别规则
        self.rules = {
            'min_length': 12,
            'max_length': 128,
            'require_uppercase': True,
            'require_lowercase': True,
            'require_digit': True,
            'require_special': True,
            'max_consecutive': 3,      # 最大连续相同字符
            'max_sequential': 3,       # 最大键盘序列（如abc, 123）
        }

        # 键盘序列模式（存储在哈希表中用于快速查找）
        self.keyboard_sequences = {
            'qwerty', 'qwertz', 'azerty',   # 键盘行
            'abcdef', 'abcdefgh',            # 字母序列
            '123456', '12345678',            # 数字序列
            'password', 'admin', 'letmein',  # 常见密码
        }

    def check(self, password: str) -> dict:
        """
        综合评估密码强度
        返回：得分、级别、详细分析
        """
        score = 0
        issues = []
        details = {}

        # 1. 长度检查
        length = len(password)
        details['length'] = length
        if length < self.rules['min_length']:
            issues.append(f"密码长度不足（{length} < {self.rules['min_length']}）")
        else:
            score += min(length * 2, 30)  # 长度最多贡献30分

        # 2. 字符类别分析
        has_upper = any(c.isupper() for c in password)
        has_lower = any(c.islower() for c in password)
        has_digit = any(c.isdigit() for c in password)
        has_special = any(not c.isalnum() for c in password)
        char_types = sum([has_upper, has_lower, has_digit, has_special])
        details['char_types'] = char_types

        if char_types < 3:
            issues.append(f"字符类别不足（仅{char_types}类，建议至少3类）")
        score += char_types * 5

        # 3. 信息熵估算
        charset_size = 0
        if has_lower: charset_size += 26
        if has_upper: charset_size += 26
        if has_digit: charset_size += 10
        if has_special: charset_size += 33

        import math
        entropy = length * math.log2(charset_size) if charset_size > 0 else 0
        details['entropy_bits'] = round(entropy, 1)

        if entropy < 40:
            issues.append(f"信息熵过低（{entropy:.0f} bits < 40 bits）")
        elif entropy < 60:
            score += 10
        elif entropy < 80:
            score += 20
        else:
            score += 30

        # 4. 常见泄露密码检查（布隆过滤器）
        if self.leaked_passwords.contains(password.lower()):
            issues.append("密码出现在已知泄露数据库中（高风险）")
            score -= 30

        # 5. 键盘序列和常见模式检查
        password_lower = password.lower()
        for seq in self.keyboard_sequences:
            if seq in password_lower:
                issues.append(f"包含常见模式: '{seq}'")
                score -= 10
                break

        # 6. 连续重复字符检查
        max_consecutive = 1
        current_consecutive = 1
        for i in range(1, len(password)):
            if password[i] == password[i-1]:
                current_consecutive += 1
                max_consecutive = max(max_consecutive, current_consecutive)
            else:
                current_consecutive = 1

        if max_consecutive > self.rules['max_consecutive']:
            issues.append(f"连续重复字符过多（{max_consecutive} > {self.rules['max_consecutive']}）")
            score -= 10

        # 计算最终得分和级别
        score = max(0, min(100, score))
        if score >= 80:
            level = "强"
        elif score >= 60:
            level = "中"
        elif score >= 40:
            level = "弱"
        else:
            level = "极弱"

        return {
            "score": score,
            "level": level,
            "issues": issues,
            "details": details,
            "recommendation": self._get_recommendation(score, issues)
        }

    def _get_recommendation(self, score: int, issues: list[str]) -> str:
        if score >= 80:
            return "密码强度良好"
        recommendations = []
        if score < 60:
            recommendations.append("建议使用密码管理器生成随机密码")
        recommendations.append("建议长度至少12位，包含大小写字母、数字和特殊字符")
        recommendations.append("避免使用个人信息、常见词汇或键盘序列")
        return " | ".join(recommendations)


# 使用示例

test_passwords = [
    "123456",
    "your_password!",
    "correcthorsebatterystaple",
    "aB3$kL9!mN2&pQ7@",
    "aaaaaaaa",
]

for pwd in test_passwords:
    result = checker.check(pwd)
    masked = pwd[:3] + "*" * (len(pwd) - 3)
    print(f"密码: {masked:30s} 得分: {result['score']:3d} 级别: {result['level']}")
    if result['issues']:
        for issue in result['issues']:
            print(f"  ⚠ {issue}")
```

### 3.6 进阶话题：概率数据结构与大规模安全系统

当安全系统需要处理海量数据时（如每天数十亿次DNS查询、TB级网络流量），精确数据结构的内存和计算开销变得不可接受。概率数据结构通过接受一定的误差换取数量级的性能提升。

| 数据结构 | 用途 | 时间复杂度 | 空间复杂度 | 特性 |
|---------|------|----------|----------|------|
| 布隆过滤器 | 集合成员判断 | O(k) | ~1.44 log₂(1/ε) × n bits | 无假阴性，有假阳性 |
| Count-Min Sketch | 频率估计 | O(d) | O(w × d) | 可能高估，不会低估 |
| HyperLogLog | 基数估算 | O(m) | O(m) | 误差 ~1.04/√m |
| 滑动窗口计数器 | 实时速率统计 | O(1) | O(window_size) | 精确的时间窗口计数 |

```python
class CountMinSketch:
    """
    Count-Min Sketch：用于估计元素出现频率
    安全应用：DDoS流量分析、异常请求频率检测、僵尸网络行为识别
    """
    def __init__(self, width: int = 1000, depth: int = 5):
        self.width = width
        self.depth = depth
        self.table = [[0] * width for _ in range(depth)]
        self.hash_seeds = list(range(depth))

    def _hash(self, item: str, seed: int) -> int:
        """使用不同种子的哈希函数"""
        h = 0
        for i, c in enumerate(item):
            h = (h * 31 + ord(c) + seed * 17) % self.width
        return h

    def add(self, item: str, count: int = 1):
        """添加元素（增加计数）"""
        for i in range(self.depth):
            index = self._hash(item, self.hash_seeds[i])
            self.table[i][index] += count

    def estimate(self, item: str) -> int:
        """估计元素出现频率（返回所有行的最小值）"""
        return min(
            self.table[i][self._hash(item, self.hash_seeds[i])]
            for i in range(self.depth)
        )


# ======== 使用场景：DDoS攻击检测 ========

# 创建Count-Min Sketch来追踪IP请求频率

# 模拟正常流量
import random
for _ in range(100000):
    ip = f"192.168.{random.randint(1, 254)}.{random.randint(1, 254)}"
    cms.add(ip)

# 模拟DDoS攻击流量（某个IP发送大量请求）
attack_ip = "10.0.0.66"
for _ in range(50000):
    cms.add(attack_ip)

# 检测异常频率
test_ips = ["192.168.1.1", "192.168.50.50", attack_ip]
threshold = 1000  # 频率阈值

for ip in test_ips:
    freq = cms.estimate(ip)
    status = "[ALERT: DDoS嫌疑]" if freq > threshold else "[正常]"
    print(f"IP: {ip:20s} 估计频率: {freq:6d} {status}")
```

### 3.7 常见误区与纠正

| 误区 | 纠正 |
|------|------|
| "哈希表查找永远是O(1)" | 平均O(1)，碰撞严重时退化为O(n)。HashDoS攻击就是利用这一点 |
| "布隆过滤器说存在就一定存在" | 布隆过滤器有假阳性，判断"存在"需要进一步确认。只有"不存在"是确定的 |
| "数据结构只是程序员的事" | 安全工具的每一个关键决策（WAF规则匹配、IDS特征扫描、密码存储）都基于数据结构选择 |
| "B+树和B树是一样的" | B+树所有数据在叶子节点，叶子节点通过链表相连，支持范围查询。B树每个节点都存数据 |
| "前缀树只用于自动补全" | 前缀树在安全中大量用于域名黑名单匹配、WAF规则引擎、敏感词过滤 |
| "图算法太慢不适合实时安全" | BFS/DFS是O(V+E)，对于百万节点的图在毫秒级完成。BloodHound分析AD域环境就是实时的 |
| "理解算法实现就够了" | 还需要理解数据结构的空间/时间权衡——这决定了在有限资源下安全工具的设计选择 |

**关键要点回顾：**

1. **哈希表**是安全工具的基础设施，但必须理解其碰撞退化特性和HashDoS攻击面
2. **前缀树**是大规模模式匹配的最优选择，WAF和IDS的核心引擎都依赖它
3. **Merkle树**是数据完整性验证的数学基石，从Git到区块链都依赖它
4. **B+树**决定了数据库索引的行为，理解它有助于理解SQL注入的性能影响
5. **图结构**是权限分析、攻击路径规划和代码审计的数学基础
6. **概率数据结构**（布隆过滤器、Count-Min Sketch）在大规模安全系统中以可接受的误差换取数量级的性能提升
7. 安全工具的实际架构是多种数据结构的**组合使用**，而非单一数据结构
