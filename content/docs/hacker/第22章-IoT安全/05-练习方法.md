# 第22章 IoT安全 - 练习方法

### 22.1.3 网络隔离环境

IoT测试必须在隔离网络中进行，避免测试流量影响生产环境或触犯法律：

```bash
# 创建独立的测试网络（使用VLAN或独立路由器）
# 推荐使用物理隔离：独立路由器 + 独立WAN口

# 使用Docker创建隔离的IoT模拟网络
docker network create --driver bridge --subnet 172.20.0.0/24 iot-lab

# 搭建MQTT Broker（模拟不安全配置用于练习）
docker run -d --name mosquitto-unsafe \
  --network iot-lab \
  -p 1883:1883 -p 9001:9001 \
  -v /path/to/unsafe-mosquitto.conf:/mosquitto/config/mosquitto.conf \
  eclipse-mosquitto

# 不安全的Mosquitto配置示例
cat > /path/to/unsafe-mosquitto.conf << 'EOF'
listener 1883
allow_anonymous true
per_listener_settings false
log_type all
connection_messages true
EOF

# 搭建安全配置的MQTT Broker（对比练习用）
docker run -d --name mosquitto-safe \
  --network iot-lab \
  -p 1884:1884 \
  -v /path/to/safe-mosquitto.conf:/mosquitto/config/mosquitto.conf \
  -v /path/to/certs:/mosquitto/certs \
  eclipse-mosquitto

# 安全的Mosquitto配置
cat > /path/to/safe-mosquitto.conf << 'EOF'
listener 1884
allow_anonymous false
password_file /mosquitto/config/passwd
require_certificate true
cafile /mosquitto/certs/ca.crt
certfile /mosquitto/certs/server.crt
keyfile /mosquitto/certs/server.key
tls_version tlsv1.2
EOF
```

**完整实操流程**：

```bash
# 步骤1：获取目标固件
# 来源1：厂商官网下载（推荐新手使用）
# 来源2：从物理设备提取（进阶）
# 来源3：固件下载站（如 https://firmware.meraki.io/）

# 示例：下载OpenWrt固件用于练习
wget https://downloads.openwrt.org/releases/23.05.3/targets/ramips/mt7621/openwrt-23.05.3-ramips-mt7621-generic-squashfs-sysupgrade.bin

# 步骤2：使用binwalk分析固件结构
binwalk openwrt-*.bin
# 输出示例：
# DECIMAL       HEXADECIMAL     DESCRIPTION
# 0             0x0             TP-Link firmware header
# 512           0x200           LZMA compressed data
# 1048576       0x100000        Squashfs filesystem

# 步骤3：提取文件系统
binwalk -e openwrt-*.bin
cd _openwrt-*.bin.extracted/

# 步骤4：分析文件系统结构
ls -la squashfs-root/
# 常见目录：etc/ bin/ sbin/ usr/ lib/ var/

# 步骤5：搜索硬编码凭证
grep -rn "password\|passwd\|secret\|key\|token" \
  squashfs-root/etc/ --include="*.conf" --include="*.ini" --include="*.json"

# 搜索可能的后门账号
cat squashfs-root/etc/passwd
cat squashfs-root/etc/shadow

# 步骤6：分析启动脚本
ls squashfs-root/etc/init.d/
cat squashfs-root/etc/init.d/*
# 关注：启动了哪些服务、是否有不安全的启动参数

# 步骤7：查找敏感文件
find squashfs-root/ -name "*.pem" -o -name "*.key" -o -name "*.crt" \
  -o -name "*.p12" -o -name "id_rsa" -o -name "id_dsa"

# 步骤8：使用strings分析二进制文件
strings squashfs-root/usr/bin/* | grep -i "http\|ftp\|admin\|root"
```

```bash
# 步骤1：识别UART接口
# 拆开目标设备（如旧路由器），在PCB上寻找4针接口
# 通常标记为J1、CON1、UART等
# 用万用表测量引脚电压：
#   VCC: 3.3V或5V
#   GND: 0V（与外壳/屏蔽层导通）
#   TXD: 待机时高电平（3.3V），有数据时跳变
#   RXD: 待机时高电平或低电平

# 连接关系：
#   设备TXD → USB转TTL RXD
#   设备RXD → USB转TTL TXD
#   设备GND → USB转TTL GND
#   VCC不连接（由设备自身供电）

# 步骤3：检测波特率
# 常见波特率：9600, 38400, 57600, 115200
```

```bash
import asyncio
from bleak import BleakClient

TARGET = "AA:BB:CC:DD:EE:FF"  # 替换为目标MAC

async def enumerate(address):
    async with BleakClient(address) as client:
        for service in client.services:
            print(f"\n[Service] {service.uuid} - {service.description}")
            for char in service.characteristics:
                props = ",".join(char.properties)
                print(f"  [Char] {char.uuid} ({props})")
                if "read" in char.properties:
                    try:
                        val = await client.read_gatt_char(char.uuid)

asyncio.run(enumerate(TARGET))
```

| 检查项 | 预期安全行为 | 常见问题 |
|--------|-------------|----------|
| 配对模式 | 使用带OOB或Passkey的Secure Connections | 使用Just Works且无MITM保护 |
| GATT权限 | 敏感特征值需要认证后读写 | 允许未认证读写敏感数据 |
| 广播信息 | 不泄露设备型号/固件版本 | 广播包含过多设备信息 |
| 重放攻击 | 使用随机数/计数器防止重放 | 固定命令可重放 |
| 地址隐私 | 使用随机地址 | 使用固定公共地址 |

```python
# 步骤1：探测MQTT Broker
import socket

def scan_mqtt(host, port=1883):
    try:
        sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        sock.settimeout(3)
        result = sock.connect_ex((host, port))
        if result == 0:
            print(f"[+] MQTT Broker found at {host}:{port}")
        sock.close()
    except Exception as e:
        print(f"[-] Error: {e}")

# 扫描常见端口
for port in [1883, 8883, 1884]:
    scan_mqtt("192.168.1.1", port)
```

## 22.3 常见问题排错指南

### 硬件连接问题

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| UART无输出 | 波特率错误 | 尝试9600/38400/57600/115200 |
| UART无输出 | TX/RX接反 | 交换TX和RX连接 |
| UART乱码 | 电压不匹配 | 确认设备是3.3V还是5V，使用对应电平的转换器 |
| Flash读取失败 | 芯片未正确连接 | 检查焊接点，确认CS引脚连接 |
| Flash读取失败 | 芯片被锁定 | 尝试擦除后重新读取 |
| JTAG连接失败 | SWD/JTAG被禁用 | 尝试NRST脉冲或特殊解锁序列 |
| BLE扫描不到 | 驱动问题 | 检查蓝牙适配器是否被系统识别 |

### 软件工具问题

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| binwalk提取失败 | 压缩算法不支持 | 安装sasquatch/jefferson等扩展工具 |
| QEMU仿真崩溃 | 内核版本不匹配 | 使用FirmAE的自动配置功能 |
| Ghidra无法分析 | 架构选择错误 | 手动指定正确的处理器类型 |
| KillerBee无法识别设备 | 固件不匹配 | 重新刷入正确的Packet Sniffer固件 |
| Suricata规则不生效 | 规则语法错误 | 使用suricata --list-keywords检查 |

## 22.7 法律与伦理提醒

**合法测试的边界**：

1. **仅测试自己拥有的设备**或已获得书面授权的设备
2. **在隔离网络中进行测试**，避免影响他人设备
3. **不要对生产环境中的IoT设备进行攻击测试**
4. **发现漏洞后遵循负责任披露流程**
5. **记录所有测试活动**，保留授权文件

**练习设备来源建议**：

- 二手市场购买旧路由器/摄像头（成本低，合法拥有）

- 开源硬件项目（如Arduino、ESP32开发板）
