## 25.6 移动设备取证实战

### 25.6.1 移动取证的核心挑战

移动设备取证（Mobile Device Forensics）是数字取证中最具挑战性的分支之一，其复杂度远超传统计算机取证。原因在于移动设备同时整合了多种数据来源：通话记录、短信、即时通讯应用、GPS位置、Wi-Fi连接历史、传感器数据、生物识别信息等，而这些数据分散在加密存储、安全芯片、云端同步等多个层面。

与传统PC取证的核心区别：

| 维度 | PC取证 | 移动取证 |
|------|--------|----------|
| 存储访问 | 硬盘直接访问（拆盘/启动盘） | 通常需要通过设备接口或云服务间接获取 |
| 加密程度 | BitLocker/FileVault可选 | 全盘加密已成默认（iOS/Android 10+） |
| 数据更新 | 静态镜像即可 | OTA更新、远程擦除随时可能发生 |
| 云同步 | 本地数据为主 | 核心数据（照片、通讯录、聊天记录）可能仅在云端 |
| 生物识别 | 通常无 | 指纹/面部识别解锁、支付验证 |
| 碎片化 | x86架构统一 | 芯片组、安全元件、引导加载程序差异巨大 |

### 25.6.2 取证提取的四个层级

移动取证按照数据提取深度分为四个层级，每一级都涉及更复杂的技术手段：

```text
Level 1: Manual Extraction（手动提取）
  └── 直接操作设备界面，截屏/录屏记录
  └── 数据完整性低，仅适合快速筛查

Level 2: Logical Extraction（逻辑提取）
  └── 通过ADB/iTunes备份接口提取用户数据
  └── 覆盖：通讯录、短信、应用数据、媒体文件
  └── 受操作系统权限限制，无法获取已删除数据

Level 3: File System Extraction（文件系统提取）
  └── 获取完整文件系统结构
  └── 可分析：数据库、配置文件、缓存、日志
  └── 部分已删除数据可恢复

Level 4: Physical Extraction（物理提取）
  └── 直接读取闪存芯片的原始二进制数据
  └── 包含文件系统未索引的扇区
  └── 已删除数据恢复成功率最高
```

物理提取是取证的"金标准"，但在加密设备上实现难度极高。现代设备（Android 10+强制文件级加密、iOS A7芯片后全盘加密）使得Level 4提取在没有设备解锁密钥的情况下几乎不可能完成。

### 25.6.3 Android设备取证

#### 25.6.3.1 提取方法详解

**ADB逻辑提取**

ADB（Android Debug Bridge）是最基础的提取通道，但前提是设备已启用USB调试模式：

```bash
# 检查设备连接状态
adb devices

# 获取设备基础信息
adb shell getprop ro.build.version.release     # Android版本
adb shell getprop ro.product.model             # 设备型号
adb shell getprop ro.serialno                  # 序列号

# 创建完整备份（不需要root权限）
adb backup -all -shared -f full_backup.ab

# 提取短信数据库（需要root或ADB授权）
adb pull /data/data/com.android.providers.telephony/databases/mmssms.db

# 提取通话记录
adb pull /data/data/com.android.providers.contacts/databases/contacts2.db

# 提取WhatsApp消息
adb pull /data/data/com.whatsapp/databases/msgstore.db
```

> **关键点**：`adb backup`在Android 12+中已被Google正式废弃，现代设备必须依赖其他提取方式。

**ADB Pull的文件系统结构**

Android关键取证路径：

| 路径 | 内容 | 取证价值 |
|------|------|----------|
| `/data/data/<package>/databases/` | 应用SQLite数据库 | 核心数据：聊天、账户、缓存 |
| `/data/data/<package>/shared_prefs/` | 应用配置XML | 用户设置、token、状态标志 |
| `/data/data/<package>/files/` | 应用私有文件 | 下载文件、临时数据 |
| `/data/media/0/` | 用户存储（内部） | 照片、视频、下载、文档 |
| `/data/system/users/0/` | 系统用户数据 | 锁屏、WiFi密码、账户信息 |
| `/data/misc/wifi/` | Wi-Fi配置 | 已连接网络列表、密码 |
| `/data/system/packages.xml` | 已安装应用列表 | 安装时间、权限、UID |

**Bootloader模式提取**

对于无法正常启动或ADB未开启的设备，可通过Bootloader/Fastboot模式操作：

```bash
# 进入Fastboot模式（通常：关机状态下按住电源+音量下键）
fastboot devices

# 检查bootloader状态
fastboot oem device-info

# 注意：解锁bootloader会触发恢复出厂设置
# 这会破坏证据完整性，取证场景中尽量避免
fastboot oem unlock

# 读取分区（需先解锁bootloader）
fastboot dump userdata userdata_dump.img
fastboot dump system system_dump.img
```

> **取证风险**：解锁bootloader会触发wipedata/factory reset，导致原始数据丢失。取证人员必须在取证报告中明确记录是否触发了此操作，并评估对证据效力的影响。

**Recovery模式提取**

```bash
# 进入Recovery模式（通常：关机状态下按住电源+音量上键）
# 然后通过ADB sideload推送工具

# 使用TWRP Recovery（如果已安装）
# TWRP允许挂载/data分区并直接访问
adb pull /data/ ./extracted_data/
```

#### 25.6.3.2 Android加密机制与取证影响

Android的加密策略直接影响取证难度：

**文件级加密（FBE, File-Based Encryption）**
- Android 10+默认启用
- 每个文件独立加密，使用设备唯一密钥 + 用户凭据
- 设备首次启动（Before First Unlock, BFU）状态下，仅BOOT区数据可访问
- 用户解锁后（After First Unlock, AFU），DE（Device Encrypted）区数据可访问
- 取证窗口：AFU状态下提取效率最高

**全盘加密（FDE, Full Disk Encryption）**
- Android 5.0-9.0的默认模式
- 整个用户数据分区加密，密钥由用户锁屏密码派生
- 已基本被FBE取代，但部分旧设备仍在使用

```python
# 判断Android设备加密状态的取证分析方法
import json

def analyze_encryption_state(device_info):
    """分析设备加密状态，评估取证可行性"""
    
    android_version = device_info.get('android_version', 0)
    security_patch = device_info.get('security_patch', '')
    boot_state = device_info.get('boot_state', 'unknown')
    lock_screen = device_info.get('lock_screen', 'none')
    
    analysis = {
        'encryption_type': 'FBE' if android_version >= 10 else 'FDE',
        'boot_status': 'BFU' if boot_state == 'locked' else 'AFU',
        'extraction_feasibility': 'low',
        'recommended_method': 'none'
    }
    
    # 评估提取可行性
    if boot_state == 'afu' and lock_screen == 'none':
        analysis['extraction_feasibility'] = 'high'
        analysis['recommended_method'] = 'file_system_extraction'
    elif boot_state == 'afu':
        analysis['extraction_feasibility'] = 'medium'
        analysis['recommended_method'] = 'logical_extraction_with_unlock'
    else:  # BFU
        analysis['extraction_feasibility'] = 'low'
        analysis['recommended_method'] = 'limited_manual_only'
    
    # 检查安全补丁级别
    if security_patch >= '2024-01':
        analysis['notes'] = '近期安全补丁，已知漏洞可能性低'
    
    return analysis
```

### 25.6.4 iOS设备取证

#### 25.6.4.1 iOS安全架构与取证影响

iOS的安全架构是移动取证中最坚固的，理解其安全模型是取证工作的前提：

```text
┌─────────────────────────────────────────────┐
│               iOS安全架构层次                 │
├─────────────────────────────────────────────┤
│  Secure Enclave Processor (SEP)             │
│  ├── 设备唯一UID密钥（硬件绑定）             │
│  ├── 生物识别数据（Touch ID/Face ID）        │
│  └── Apple Pay支付密钥                       │
├─────────────────────────────────────────────┤
│  全盘加密（Data Protection）                 │
│  ├── Class A: 完全锁定保护（首次解锁前）      │
│  ├── Class B: 后台创建文件保护               │
│  ├── Class C: 首次解锁后可访问               │
│  └── Class D: 始终可访问                     │
├─────────────────────────────────────────────┤
│  沙盒机制                                    │
│  ├── 每个应用独立容器                        │
│  ├── 进程间通信受限                          │
│  └── 系统文件与用户数据隔离                  │
├─────────────────────────────────────────────┤
│  代码签名 + 沙盒 + 地址空间随机化（ASLR）     │
└─────────────────────────────────────────────┘
```

**iOS取证的核心挑战：**
- **iTunes备份加密**：iOS 11+用户可设置加密备份密码，不知道密码则无法解密
- **密码复杂度**：6位PIN暴力破解耗时合理，但字母数字密码几乎不可行
- **远程擦除**：设备联网状态下，用户可通过"Find My iPhone"远程擦除
- **自动更新**：iOS自动安装安全更新，可能修复取证依赖的漏洞

**备份目录结构分析**

iOS备份文件名是SHA256(文件相对路径)的前两位，完整路径为`<Manifest.db>`中的映射关系：

```python
import sqlite3
import hashlib

def parse_ios_manifest(manifest_path):
    """解析iOS备份Manifest.db，获取文件路径映射"""
    conn = sqlite3.connect(manifest_path)
    cursor = conn.cursor()
    
    # Manifest.db存储了备份中所有文件的元数据
    cursor.execute("""
        SELECT fileID, domain, relativePath, 
               flags, fileLength
        FROM Files
        WHERE relativePath IS NOT NULL
        ORDER BY domain, relativePath
    """)
    
    critical_paths = {}
    for row in cursor.fetchall():
        file_id, domain, rel_path, flags, size = row
        
        # 取证关键路径
        if any(kw in rel_path for kw in [
            'sms.db', 'call_history.db', 'AddressBook',
            'Safari/History', 'MapsData', 'Health',
            'KnowledgeC', 'netusage', 'PowerLog',
            'Photos.sqlite', 'HomeDomain/Library/Preferences'
        ]):
            critical_paths[rel_path] = {
                'file_id': file_id,
                'domain': domain,
                'size': size,
                'category': categorize_ios_artifact(rel_path)
            }
    
    conn.close()
    return critical_paths

def categorize_ios_artifact(path):
    """分类iOS取证关键文件"""
    categories = {
        '通讯记录': ['sms.db', 'call_history.db', 'AddressBook'],
        '位置数据': ['MapsData', 'location', 'consolidated'],
        '浏览历史': ['Safari/History', 'WebClips'],
        '健康数据': ['Health', 'healthdb'],
        '系统活动': ['KnowledgeC', 'DuetExpert'],
        '网络活动': ['netusage', 'NetworkInterfaces'],
        '电源状态': ['PowerLog', 'CurrentPowerlog'],
        '照片元数据': ['Photos.sqlite', 'MediaAnalysis'],
        '应用数据': ['Containers/Data', 'Group Containers'],
    }
    
    for category, keywords in categories.items():
        if any(kw in path for kw in keywords):
            return category
    return '其他'
```

### 25.6.5 应用数据深度解析

#### 25.6.5.1 SQLite数据库取证

移动设备上90%以上的应用数据存储在SQLite数据库中。掌握SQLite取证是移动取证的核心技能。

```python
import sqlite3
import json
import os
from datetime import datetime, timezone

class MobileForensicsDB:
    """移动设备SQLite数据库取证分析器"""
    
    def __init__(self, db_path):
        self.db_path = db_path
        self.conn = sqlite3.connect(db_path)
        self.conn.row_factory = sqlite3.Row
    
    def analyze_database_structure(self):
        """分析数据库结构，识别所有表和索引"""
        cursor = self.conn.cursor()
        
        # 获取所有表
        cursor.execute("""
            SELECT name, sql FROM sqlite_master 
            WHERE type='table' AND name NOT LIKE 'sqlite_%'
            ORDER BY name
        """)
        tables = {}
        for row in cursor.fetchall():
            table_name = row['name']
            cursor.execute(f"PRAGMA table_info({table_name})")
            columns = [col['name'] for col in cursor.fetchall()]
            cursor.execute(f"SELECT COUNT(*) FROM {table_name}")
            count = cursor.fetchone()[0]
            tables[table_name] = {
                'schema': row['sql'],
                'columns': columns,
                'row_count': count
            }
        
        return tables
    
    def parse_android_sms(self):
        """解析Android短信数据库"""
        cursor = self.conn.cursor()
        cursor.execute("""
            SELECT _id, address, body, date, date_sent, type, read, 
                   seen, subject, thread_id
            FROM sms
            ORDER BY date DESC
        """)
        
        messages = []
        for row in cursor.fetchall():
            msg = {
                'id': row['_id'],
                'address': row['address'],
                'body': row['body'],
                'date': datetime.fromtimestamp(
                    row['date'] / 1000, tz=timezone.utc
                ).isoformat() if row['date'] else None,
                'date_sent': datetime.fromtimestamp(
                    row['date_sent'] / 1000, tz=timezone.utc
                ).isoformat() if row['date_sent'] else None,
                'direction': 'received' if row['type'] == 1 else 'sent',
                'read': bool(row['read']),
                'seen': bool(row['seen']),
                'thread_id': row['thread_id']
            }
            messages.append(msg)
        
        return messages
    
    def parse_whatsapp_messages(self, key=None):
        """解析WhatsApp消息数据库（支持SQLCipher解密）"""
        if key:
            self.conn.execute(f"PRAGMA key = '{key}'")
        
        cursor = self.conn.cursor()
        
        # WhatsApp数据库结构会随版本更新变化
        # 以下是通用查询方式
        try:
            # 较新版本WhatsApp
            cursor.execute("""
                SELECT _id, key_remote_jid, data, timestamp, 
                       status, from_me, media_url, media_mime_type,
                       media_size, media_duration, latitude, longitude
                FROM messages
                WHERE key_remote_jid IS NOT NULL
                ORDER BY timestamp DESC
            """)
        except sqlite3.OperationalError:
            # 旧版本WhatsApp
            cursor.execute("""
                SELECT _id, key_remote_jid, data, timestamp, 
                       status, from_me
                FROM messages
                ORDER BY timestamp DESC
            """)
        
        messages = []
        for row in cursor.fetchall():
            msg = {
                'id': row['_id'],
                'chat_jid': row['key_remote_jid'],
                'content': row['data'],
                'timestamp': row['timestamp'],
                'status': row['status'],
                'from_me': bool(row['from_me']),
            }
            # 添加可选字段
            for field in ['media_url', 'media_mime_type', 
                          'media_size', 'media_duration',
                          'latitude', 'longitude']:
                if field in row.keys():
                    msg[field] = row[field]
            
            messages.append(msg)
        
        return messages
    
    def parse_wechat_messages(self):
        """解析微信EnMicroMsg.db数据库"""
        # 微信使用自定义加密（基于IMEI+UIN的MD5派生）
        # 解密密钥 = MD5(IMEI + UIN)[0:7]
        cursor = self.conn.cursor()
        
        cursor.execute("""
            SELECT msgId, talker, content, type, createTime, 
                   imgPath, isSend, status
            FROM message
            ORDER BY createTime DESC
        """)
        
        # 微信消息类型映射
        wechat_msg_types = {
            1: 'text', 3: 'image', 34: 'voice',
            42: 'contact_card', 43: 'video', 47: 'emoji',
            48: 'location', 49: 'link/file', 10000: 'system',
            10002: 'system_notification'
        }
        
        messages = []
        for row in cursor.fetchall():
            msg = {
                'id': row['msgId'],
                'talker': row['talker'],
                'content': row['content'],
                'type': wechat_msg_types.get(row['type'], f'unknown_{row["type"]}'),
                'timestamp': datetime.fromtimestamp(
                    row['createTime'] / 1000, tz=timezone.utc
                ).isoformat() if row['createTime'] else None,
                'is_sent': bool(row['isSend']),
                'media_path': row['imgPath'],
                'status': row['status']
            }
            messages.append(msg)
        
        return messages
    
    def recover_deleted_records(self):
        """尝试恢复已删除的SQLite记录"""
        cursor = self.conn.cursor()
        
        # 读取数据库原始页面
        with open(self.db_path, 'rb') as f:
            raw_data = f.read()
        
        recovered = []
        
        # 获取所有表结构
        cursor.execute("""
            SELECT name FROM sqlite_master 
            WHERE type='table' AND name NOT LIKE 'sqlite_%'
        """)
        tables = [row['name'] for row in cursor.fetchall()]
        
        # 在原始数据中搜索可能的已删除记录
        # SQLite删除记录时不会立即擦除数据，只是标记为可用空间
        for table in tables:
            cursor.execute(f"PRAGMA table_info({table})")
            columns = [col['name'] for col in cursor.fetchall()]
            
            # 搜索已删除记录（简化方法）
            for col in columns:
                pattern = col.encode('utf-8')
                if pattern in raw_data:
                    # 找到可能的列数据
                    offset = raw_data.find(pattern)
                    context = raw_data[max(0, offset-200):offset+200]
                    recovered.append({
                        'table': table,
                        'column': col,
                        'offset': offset,
                        'context': context
                    })
        
        return recovered
    
    def close(self):
        self.conn.close()
```

```python
def derive_wechat_key(imei, uin):
    """
    微信数据库解密密钥推导
    密钥 = MD5(IMEI + UIN)[0:7]
    
    注意：不同微信版本可能使用不同密钥算法
    - 较新版本：使用设备UUID代替IMEI
    - 部分版本使用6位或7位密钥

    key_7 = md5_hash[:7]
    
    # 部分版本使用6位
    key_6 = md5_hash[:6]
    
    return {
        'key_7': key_7,
        'key_6': key_6,
        'full_md5': md5_hash,
        'note': 'IMEI可通过拨号盘输入*#06#获取，UIN存储在shared_prefs/auth_info_key_prefs.xml中'
    }
```

```python
def build_location_timeline(locations):
    """构建位置时间线，识别常去地点和移动模式"""
    from collections import defaultdict
    
    # 按小时分组
    hourly_pattern = defaultdict(list)
    for loc in locations:
        if loc.get('timestamp'):
            dt = datetime.fromtimestamp(loc['timestamp'] / 1000)
            hour = dt.hour
            hourly_pattern[hour].append(loc)
    
    # 识别常去地点（聚类分析简化版）
    frequent_locations = []
    location_cluster = defaultdict(int)
    
    for loc in locations:
        if loc.get('lat') and loc.get('lng'):
            # 简化聚类：四舍五入到小数点后3位
            cluster_key = f"{round(loc['lat'], 3)},{round(loc['lng'], 3)}"
            location_cluster[cluster_key] += 1
    
    # 按访问频率排序
    sorted_locations = sorted(
        location_cluster.items(), 
        key=lambda x: x[1], reverse=True
    )
    
    return {
        'hourly_pattern': {h: len(v) for h, v in hourly_pattern.items()},
        'frequent_locations': [
            {'coordinates': loc, 'visit_count': count}
            for loc, count in sorted_locations[:10]
        ],
        'insights': generate_location_insights(hourly_pattern)
    }

    evening_active = sum(len(hourly_pattern.get(h, [])) for h in range(18, 22))
    night_active = sum(len(hourly_pattern.get(h, [])) for h in range(0, 5))

    if morning_active > 0 and evening_active > 0:
        insights.append("呈现规律的早晚活动模式，可能反映工作-居住地点往返")
```

### 25.6.8 证据链与法律合规

移动取证的证据效力取决于完整的证据链（Chain of Custody）记录：

**证据链记录要素：**

```python
import hashlib
from datetime import datetime

class ChainOfCustody:
    """取证证据链管理器"""
    
    def __init__(self, case_id, examiner):
        self.case_id = case_id
        self.examiner = examiner
        self.entries = []
    
    def add_entry(self, action, item_desc, details=None):
        """添加证据链记录"""
        entry = {
            'timestamp': datetime.now().isoformat(),
            'case_id': self.case_id,
            'examiner': self.examiner,
            'action': action,
            'item_description': item_desc,
            'details': details or {},
            'witness': None
        }
        self.entries.append(entry)
        return entry
    
    def record_device_seizure(self, device_info):
        """记录设备扣押"""
        return self.add_entry('SEIZURE', '移动设备', {
            'device_model': device_info.get('model'),
            'serial_number': device_info.get('serial'),
            'imei': device_info.get('imei'),
            'condition': device_info.get('condition'),
            'battery_level': device_info.get('battery'),
            'network_state': device_info.get('network_state'),  # 飞行模式/联网
            'seizure_location': device_info.get('location'),
            'seizure_time': device_info.get('time')
        })
    
    def record_data_acquisition(self, acquisition_info):
        """记录数据提取"""
        # 计算镜像哈希确保完整性
        image_path = acquisition_info.get('image_path')
        sha256 = self.calculate_hash(image_path) if image_path else None
        
        return self.add_entry('ACQUISITION', '数据提取', {
            'method': acquisition_info.get('method'),  # logical/file_system/physical
            'tool': acquisition_info.get('tool'),
            'tool_version': acquisition_info.get('tool_version'),
            'image_path': image_path,
            'image_size': acquisition_info.get('size'),
            'sha256': sha256,
            'start_time': acquisition_info.get('start_time'),
            'end_time': acquisition_info.get('end_time'),
            'errors': acquisition_info.get('errors', [])
        })
    
    def calculate_hash(self, file_path):
        """计算文件SHA256哈希"""
        sha256 = hashlib.sha256()
        with open(file_path, 'rb') as f:
            for chunk in iter(lambda: f.read(8192), b''):
                sha256.update(chunk)
        return sha256.hexdigest()
    
    def generate_report(self):
        """生成证据链报告"""
        report = {
            'case_id': self.case_id,
            'total_entries': len(self.entries),
            'examiner': self.examiner,
            'entries': self.entries,
            'integrity': self.verify_chain_integrity()
        }
        return report
    
    def verify_chain_integrity(self):
        """验证证据链完整性"""
        issues = []
        
        for i, entry in enumerate(self.entries):
            # 检查时间顺序
            if i > 0:
                prev_time = datetime.fromisoformat(
                    self.entries[i-1]['timestamp']
                )
                curr_time = datetime.fromisoformat(entry['timestamp'])
                if curr_time < prev_time:
                    issues.append(f"时间顺序异常: 第{i}条记录时间早于前一条")
            
            # 检查必要字段
            required_fields = ['timestamp', 'examiner', 'action']
            for field in required_fields:
                if not entry.get(field):
                    issues.append(f"第{i}条记录缺少必填字段: {field}")
        
        return {
            'valid': len(issues) == 0,
            'issues': issues
        }
```

### 25.6.9 常见误区与纠正

**误区1：删除的数据无法恢复**

事实：SQLite数据库删除记录后，数据仍在文件的空闲空间中。直到SQLite执行VACUUM操作或操作系统覆盖了相应扇区，已删除数据通常可以恢复。Android的`journal_mode=WAL`模式下，WAL文件中也可能包含已删除数据。

**误区2：加密设备无法取证**

事实：加密确实增加了取证难度，但并非无法取证。AFU（After First Unlock）状态下，加密密钥在内存中，可通过内存dump获取。此外，许多应用数据已同步到云端，可通过云服务提取。商业工具也在持续开发针对加密设备的新方法。

**误区3：恢复出厂设置=数据彻底清除**

事实：Android的恢复出厂设置通常只是标记数据分区为空闲空间，并不执行全盘覆写。使用专业数据恢复工具（如Autopsy、X-Ways）仍可恢复大量数据。只有执行了"安全擦除"（多次覆写）才使数据基本不可恢复。

**误区4：云备份数据不可信**

事实：云备份数据在法律上具有证据效力，但需要额外的完整性验证。取证人员应记录云数据提取的时间、方式、账号，并与设备本地数据交叉验证。多个数据源相互印证的证据链比单一来源更有说服力。

**误区5：取证工具的输出可以直接作为证据**

事实：任何工具都可能有bug或解析错误。取证报告应包含工具版本、配置参数、处理日志，并对关键发现进行人工验证。美国NIST的CFTT（Computer Forensic Tool Testing）项目提供了标准测试用例。

```python
def analyze_device_logs(log_path):
    """分析移动设备日志中的取证关键事件"""
    
    key_events = {
        'device_boot': [],
        'screen_lock': [],
        'app_install': [],
        'app_uninstall': [],
        'network_connect': [],
        'location_request': [],
        'permission_grant': [],
        'data_access': [],
        'file_operation': []
    }
    
    with open(log_path, 'r', encoding='utf-8', errors='ignore') as f:
        for line in f:
            # 设备启动
            if 'Sys_boot_completed' in line:
                key_events['device_boot'].append(line.strip())
            
            # 屏幕解锁/锁定
            elif 'screen_toggled' in line:
                key_events['screen_lock'].append(line.strip())
            
            # 应用安装
            elif 'PackageInstaller' in line or 'install' in line.lower():
                if 'INSTALL' in line or 'package_added' in line:
                    key_events['app_install'].append(line.strip())
            
            # 网络连接
            elif 'WifiStateMachine' in line or 'NetworkAgent' in line:
                key_events['network_connect'].append(line.strip())
            
            # 位置请求
            elif 'LocationManager' in line:
                key_events['location_request'].append(line.strip())
    
    # 生成统计摘要
    summary = {
        'total_lines': sum(len(v) for v in key_events.values()),
        'event_counts': {k: len(v) for k, v in key_events.items() if v},
        'events': key_events
    }
    
    return summary
```

### 25.6.11 自动化取证流水线

将上述技术整合为可复用的自动化取证流程：

```python
import os
import json
import sqlite3
from datetime import datetime
from pathlib import Path

class MobileForensicsPipeline:
    """移动设备自动化取证流水线"""
    
    def __init__(self, case_id, output_dir):
        self.case_id = case_id
        self.output_dir = Path(output_dir) / case_id
        self.output_dir.mkdir(parents=True, exist_ok=True)
        self.results = {}
        self.errors = []
    
    def run_full_analysis(self, data_path, platform='android'):
        """执行完整取证分析"""
        
        print(f"[*] 开始取证分析 - 案件: {self.case_id}")
        print(f"[*] 平台: {platform}, 数据路径: {data_path}")
        
        # 1. 基础信息提取
        self.extract_device_info(data_path, platform)
        
        # 2. 通讯记录分析
        self.analyze_communications(data_path, platform)
        
        # 3. 应用数据分析
        self.analyze_applications(data_path, platform)
        
        # 4. 位置信息分析
        self.analyze_locations(data_path, platform)
        
        # 5. 浏览历史分析
        self.analyze_browser_history(data_path, platform)
        
        # 6. 多媒体文件分析
        self.analyze_media_files(data_path, platform)
        
        # 7. 生成报告
        report = self.generate_report()
        
        print(f"[+] 分析完成，报告已生成: {self.output_dir}/report.json")
        return report
    
    def extract_device_info(self, data_path, platform):
        """提取设备基础信息"""
        info = {'platform': platform}
        
        if platform == 'android':
            # 尝试读取build.prop
            build_prop = os.path.join(data_path, 'system', 'build.prop')
            if os.path.exists(build_prop):
                with open(build_prop, 'r') as f:
                    for line in f:
                        if '=' in line:
                            key, value = line.strip().split('=', 1)
                            info[key] = value
        elif platform == 'ios':
            # 尝试读取Manifest.plist
            manifest = os.path.join(data_path, 'Manifest.plist')
            if os.path.exists(manifest):
                info['manifest'] = 'found'
        
        self.results['device_info'] = info
    
    def analyze_communications(self, data_path, platform):
        """分析通讯记录"""
        comms = {'messages': [], 'calls': []}
        
        # 搜索短信数据库
        sms_patterns = ['mmssms.db', 'sms.db', 'telephony.db']
        for pattern in sms_patterns:
            for db_file in Path(data_path).rglob(pattern):
                try:
                    conn = sqlite3.connect(str(db_file))
                    cursor = conn.cursor()
                    cursor.execute("SELECT * FROM sms LIMIT 1000")
                    columns = [desc[0] for desc in cursor.description]
                    rows = cursor.fetchall()
                    comms['messages'].append({
                        'source': str(db_file),
                        'count': len(rows),
                        'columns': columns,
                        'sample': [dict(zip(columns, row)) for row in rows[:10]]
                    })
                    conn.close()
                except Exception as e:
                    self.errors.append(f"解析短信失败 {db_file}: {e}")
        
        self.results['communications'] = comms
    
    def analyze_applications(self, data_path, platform):
        """分析已安装应用"""
        apps = []
        
        if platform == 'android':
            packages_xml = os.path.join(data_path, 'system', 'packages.xml')
            if os.path.exists(packages_xml):
                import xml.etree.ElementTree as ET
                tree = ET.parse(packages_xml)
                for package in tree.iter('package'):
                    apps.append({
                        'name': package.get('name'),
                        'code_path': package.get('codePath'),
                        'install_time': package.get('ft'),
                    })
        
        self.results['applications'] = apps
    
    def analyze_locations(self, data_path, platform):
        """分析位置数据"""
        locations = []
        
        # 搜索位置相关数据库
        location_patterns = [
            'location_history', 'locations.db',
            'consolidated.db', 'cache_encryptedB.db'
        ]
        for pattern in location_patterns:
            for db_file in Path(data_path).rglob(f'*{pattern}*'):
                try:
                    conn = sqlite3.connect(str(db_file))
                    cursor = conn.cursor()
                    cursor.execute(
                        "SELECT name FROM sqlite_master WHERE type='table'"
                    )
                    tables = [row[0] for row in cursor.fetchall()]
                    locations.append({
                        'source': str(db_file),
                        'tables': tables
                    })
                    conn.close()
                except Exception as e:
                    self.errors.append(f"解析位置数据失败 {db_file}: {e}")
        
        self.results['locations'] = locations
    
    def analyze_browser_history(self, data_path, platform):
        """分析浏览器历史"""
        history = []
        
        for db_file in Path(data_path).rglob('*History*'):
            if db_file.suffix == '.db':
                try:
                    conn = sqlite3.connect(str(db_file))
                    cursor = conn.cursor()
                    # 通用的Chrome/WebKit历史查询
                    try:
                        cursor.execute(
                            "SELECT url, title, visit_count, last_visit_time "
                            "FROM urls ORDER BY last_visit_time DESC LIMIT 500"
                        )
                        rows = cursor.fetchall()
                        history.append({
                            'source': str(db_file),
                            'count': len(rows),
                            'entries': [
                                {'url': r[0], 'title': r[1], 'visits': r[2]}
                                for r in rows[:50]
                            ]
                        })
                    except sqlite3.OperationalError:
                        pass
                    conn.close()
                except Exception as e:
                    self.errors.append(f"解析浏览器历史失败 {db_file}: {e}")
        
        self.results['browser_history'] = history
    
    def analyze_media_files(self, data_path, platform):
        """分析多媒体文件"""
        media = {'photos': 0, 'videos': 0, 'audio': 0}
        
        extensions = {
            'photos': ['.jpg', '.jpeg', '.png', '.heic', '.webp'],
            'videos': ['.mp4', '.mov', '.3gp', '.mkv'],
            'audio': ['.mp3', '.aac', '.ogg', '.wav', '.m4a']
        }
        
        for category, exts in extensions.items():
            for ext in exts:
                files = list(Path(data_path).rglob(f'*{ext}'))
                media[category] += len(files)
        
        self.results['media'] = media
    
    def generate_report(self):
        """生成取证报告"""
        report = {
            'case_id': self.case_id,
            'analysis_time': datetime.now().isoformat(),
            'platform': self.results.get('device_info', {}).get('platform'),
            'summary': {
                'device_info': bool(self.results.get('device_info')),
                'communication_records': len(
                    self.results.get('communications', {}).get('messages', [])
                ),
                'installed_apps': len(
                    self.results.get('applications', [])
                ),
                'location_sources': len(
                    self.results.get('locations', [])
                ),
                'browser_entries': sum(
                    entry['count'] 
                    for entry in self.results.get('browser_history', [])
                ),
                'media_files': self.results.get('media', {})
            },
            'detailed_results': self.results,
            'errors': self.errors,
            'integrity_hash': None  # 实际应用中应计算报告哈希
        }
        
        # 保存报告
        report_path = self.output_dir / 'report.json'
        with open(report_path, 'w', encoding='utf-8') as f:
            json.dump(report, f, ensure_ascii=False, indent=2)
        
        return report


# 使用示例
if __name__ == '__main__':
    pipeline = MobileForensicsPipeline(
        case_id='CASE-2024-001',
        output_dir='./forensics_output'
    )
    
    report = pipeline.run_full_analysis(
        data_path='/path/to/extracted_data/',
        platform='android'
    )
    
    print(json.dumps(report['summary'], indent=2, ensure_ascii=False))
```

### 25.6.12 本节小结

移动设备取证是一个技术密集、法律敏感的领域。核心要点：

1. **取证层级**：手动提取→逻辑提取→文件系统提取→物理提取，层级越高数据越完整但技术难度越大
2. **加密对抗**：现代设备默认全盘/文件级加密，取证窗口主要在AFU状态下
3. **平台差异**：Android开放性带来多种提取途径但碎片化严重；iOS封闭架构安全但提取方法有限
4. **应用层数据**：SQLite是移动应用数据的通用存储，掌握SQLite取证是核心技能
5. **云数据**：设备不可获取时，云备份是重要替代来源，但需法律授权
6. **证据链**：完整的证据链记录是证据效力的基础，从设备扣押到数据提取每一步都必须记录
7. **工具选择**：根据预算、场景、技术能力选择合适工具，开源工具链在成本敏感场景下是可靠选择

移动取证的技术发展速度极快，取证人员需要持续关注设备安全更新、新漏洞披露和工具版本迭代。建立自动化取证流水线可以显著提高效率和一致性。
