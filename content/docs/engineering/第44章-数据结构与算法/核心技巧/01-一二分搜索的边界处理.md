## 二分搜索的边界处理

二分搜索是算法面试与工程实践中出现频率最高的基础算法之一。LeetCode 上有超过 100 道题目直接或间接使用二分搜索，然而其边界条件的处理之微妙，使得即便是资深程序员也时常踩坑——死循环、越界、漏解、多解，错误形态五花八门。本节从理论根基出发，系统梳理二分搜索的两种区间定义、六大经典变体，以及最实用的"二分答案"模式，帮助读者建立完整的二分搜索知识体系。

---

### 核心原理：为什么二分搜索有效

#### 基本思想

二分搜索的本质是**利用有序性进行排除**：每次比较将搜索范围缩小一半，时间复杂度为 O(log n)。对一个 n = 10⁹ 的有序数组，最多只需 30 次比较即可定位目标。

#### 循环不变式（Loop Invariant）

二分搜索正确性的核心保障是**循环不变式**——在每次迭代前后都成立的条件：

- **左闭右闭 [left, right]**：若 target 存在，必在区间 [left, right] 内。当 left > right 时，区间为空，target 不存在。
- **左闭右开 [left, right)**：若 target 存在，必在区间 [left, right) 内。当 left == right 时，区间为空，target 不存在。

#### 复杂度分析

设数组长度为 n：

| 指标 | 值 | 推导 |
|------|----|------|
| 时间复杂度 | O(log₂ n) | 每次减半：n → n/2 → n/4 → ... → 1 |
| 空间复杂度（迭代） | O(1) | 仅使用 left、right、mid 三个变量 |
| 空间复杂度（递归） | O(log n) | 递归深度为 log₂ n |
| 比较次数上限 | ⌊log₂ n⌋ + 1 | 包含成功与失败两种情况 |

### 两种区间定义：全面对比

#### 左闭右闭 [left, right]

区间 [left, right] 表示 left 和 right 都是有效索引。

**关键决策点：**
- 初始值：left = 0, right = len(nums) - 1（right 是数组最后一个有效索引）
- 循环条件：left <= right（区间非空时继续搜索）
- 右边界更新：right = mid - 1（mid 已检查，排除它）

```python
def binary_search(nums: list[int], target: int) -> int:
    """左闭右闭区间二分搜索"""
    left, right = 0, len(nums) - 1  # right 指向最后一个有效元素

    while left <= right:  # 区间非空
        mid = left + (right - left) // 2  # 防溢出写法
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # [mid+1, right]
        else:
            right = mid - 1  # [left, mid-1]

    return -1  # 区间为空，未找到
```

**执行轨迹示例：** 在 `[1, 3, 5, 7, 9, 11]` 中搜索 target = 7

第1轮: left=0, right=5, mid=2, nums[2]=5 < 7  → left=3
第2轮: left=3, right=5, mid=4, nums[4]=9 > 7  → right=3
第3轮: left=3, right=3, mid=3, nums[3]=7 == 7 → 返回 3

#### 左闭右开 [left, right)

区间 [left, right) 表示 left 是有效索引，right 不是。

**关键决策点：**
- 初始值：left = 0, right = len(nums)（right 指向数组尾部之后）
- 循环条件：left < right（区间非空时继续搜索）
- 右边界更新：right = mid（mid 已检查，排除它，但 mid 本身可能是右边界）

```python
def binary_search_v2(nums: list[int], target: int) -> int:
    """左闭右开区间二分搜索"""
    left, right = 0, len(nums)  # right 指向数组尾部之后

    while left < right:  # 区间非空
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1  # [mid+1, right)
        else:
            right = mid  # [left, mid)

    return -1  # 区间为空，未找到
```

#### 区间定义对比表

| 特性 | 左闭右闭 [left, right] | 左闭右开 [left, right) |
|------|------------------------|------------------------|
| 初始 right | `len(nums) - 1` | `len(nums)` |
| 循环条件 | `left <= right` | `left < right` |
| 左边界更新 | `left = mid + 1` | `left = mid + 1` |
| 右边界更新 | `right = mid - 1` | `right = mid` |
| 空区间表示 | `left > right` | `left == right` |
| 与标准库对齐 | C++ STL（部分） | Python bisect / STL lower_bound |
| 推荐场景 | 直觉清晰，面试首选 | 需要精确边界控制时 |

> **选择建议**：面试时使用左闭右闭更直观；工程实践中，左闭右开与 Python `bisect` 模块、C++ STL 的 `lower_bound` / `upper_bound` 保持一致，更易与标准库配合。

### 常见 Bug 模式与根因分析

#### Bug 1：死循环

**症状**：程序永远不结束，CPU 占用飙升。

**根因**：区间没有严格缩小。最典型的错误是 `right = mid`（左闭右闭场景）或 `left = mid`（左闭右开场景），导致区间大小不变。

```python
# ❌ 错误：左闭右闭中 right = mid，当 left = mid 时死循环
while left <= right:
    mid = (left + right) // 2
    if nums[mid] < target:
        left = mid  # 死循环！left == mid 时不缩小
    else:
        right = mid - 1

# ✅ 正确：必须 left = mid + 1
while left <= right:
    mid = (left + right) // 2
    if nums[mid] < target:
        left = mid + 1  # 严格缩小区间
    else:
        right = mid - 1
```

#### Bug 2：整数溢出

**症状**：mid 计算结果为负数，导致数组越界。

**根因**：`(left + right) // 2` 在 left 和 right 都很大时可能溢出。

```python
# ❌ 危险：当 left + right > 2³¹-1 时溢出
mid = (left + right) // 2

# ✅ 安全：差值计算不会溢出
mid = left + (right - left) // 2

# ✅ 更优雅（Python 3.10+）
mid = (left + right) // 2  # Python 自动处理大整数
```

> **注意**：Python 的整数自动提升机制使其不受此问题影响。但在 C++、Java、Go 中必须使用防溢出写法。

#### Bug 3：Off-by-One（差一错误）

**症状**：目标存在时返回 -1，或返回位置偏离一格。

**根因**：循环条件与边界更新不匹配。`<` 与 `<=`、`mid - 1` 与 `mid` 的组合必须与区间定义严格对应。

#### 速查对照表

| 区间类型 | 循环条件 | 右边界更新 | 组合正确 |
|---------|---------|-----------|---------|
| [left, right] | `<=` | `mid - 1` | ✅ |
| [left, right] | `<` | `mid - 1` | ❌ 会漏掉 left == right 的情况 |
| [left, right) | `<` | `mid` | ✅ |
| [left, right) | `<=` | `mid` | ❌ 可能死循环 |

### 六大经典变体

#### 变体一：查找左边界（第一个等于 target 的位置）

当数组中有重复元素时，普通二分搜索可能返回任意一个匹配位置。左边界搜索确保返回最靠前的匹配。

```python
def find_left_boundary(nums: list[int], target: int) -> int:
    """查找第一个等于 target 的位置（左闭右闭）"""
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1  # 即使 nums[mid] == target，也继续向左搜索

    # left 是第一个 >= target 的位置
    if left < len(nums) and nums[left] == target:
        return left
    return -1
```

**核心思路**：找到 target 时，不立即返回，而是将 right = mid - 1 继续向左收缩，最终 left 指向第一个匹配位置。

#### 变体二：查找右边界（最后一个等于 target 的位置）

```python
def find_right_boundary(nums: list[int], target: int) -> int:
    """查找最后一个等于 target 的位置（左闭右闭）"""
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] <= target:
            left = mid + 1  # 即使 nums[mid] == target，也继续向右搜索
        else:
            right = mid - 1

    # right 是最后一个 <= target 的位置
    if right >= 0 and nums[right] == target:
        return right
    return -1
```

**与左边界变体的对称性**：左边界搜索向左收缩（`right = mid - 1`），右边界搜索向右收缩（`left = mid + 1`）。

#### 变体三：lower_bound（第一个大于等于 target 的位置）

```python
def lower_bound(nums: list[int], target: int) -> int:
    """查找第一个 >= target 的位置（左闭右开）"""
    left, right = 0, len(nums)

    while left < right:
        mid = left + (right - left) // 2
        if nums[mid] < target:
            left = mid + 1
        else:
            right = mid  # right 可能就是答案，不能跳过

    return left  # left == right 时返回插入位置
```

**等价于**：C++ 的 `std::lower_bound`、Python 的 `bisect.bisect_left`。

#### 变体四：upper_bound（第一个大于 target 的位置）

```python
def upper_bound(nums: list[int], target: int) -> int:
    """查找第一个 > target 的位置（左闭右开）"""
    left, right = 0, len(nums)

    while left < right:
        mid = left + (right - left) // 2
        if nums[mid] <= target:
            left = mid + 1
        else:
            right = mid

    return left
```

**等价于**：C++ 的 `std::upper_bound`、Python 的 `bisect.bisect_right`。

#### 变体五：查找旋转排序数组中的最小值

旋转排序数组（如 `[4, 5, 6, 7, 0, 1, 2]`）的特点是：最小值是唯一的"断点"，将数组分为两个有序部分。

```python
def find_min(nums: list[int]) -> int:
    """在旋转排序数组中查找最小值"""
    left, right = 0, len(nums) - 1

    while left < right:
        mid = left + (right - left) // 2
        if nums[mid] > nums[right]:
            # 最小值在 mid 右侧（mid 不可能是最小值）
            left = mid + 1
        else:
            # 最小值在 [left, mid] 内（mid 可能是最小值）
            right = mid

    return nums[left]  # left == right，指向最小值
```

**判断逻辑**：比较 `nums[mid]` 与 `nums[right]`——若 `nums[mid] > nums[right]`，说明最小值在右半部分；否则最小值在左半部分或就是 mid。

#### 变体六：在旋转排序数组中搜索

```python
def search_rotated(nums: list[int], target: int) -> int:
    """在旋转排序数组中搜索目标值"""
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2

        if nums[mid] == target:
            return mid

        # 判断哪半部分是有序的
        if nums[left] <= nums[mid]:
            # 左半部分 [left, mid] 有序
            if nums[left] <= target < nums[mid]:
                right = mid - 1  # target 在左半部分
            else:
                left = mid + 1   # target 在右半部分
        else:
            # 右半部分 [mid, right] 有序
            if nums[mid] < target <= nums[right]:
                left = mid + 1   # target 在右半部分
            else:
                right = mid - 1  # target 在左半部分

    return -1
```

**关键判断**：`nums[left] <= nums[mid]` 时左半部分有序，再判断 target 是否落在有序区间内。

### 二分答案：最重要的实战模式

"二分答案"是二分搜索最高频的工程应用模式——问题本身没有有序数组，但**答案空间具有单调性**，可以用二分搜索在答案空间上查找。

#### 核心思想

如果能回答"答案 ≤ x 是否可行"，那么答案空间就具有单调性：
  - 可行 → 答案可能更大
  - 不可行 → 答案可能更小
因此可以在 [答案下界, 答案上界] 上二分搜索

#### 经典问题：最小化最大值（LC 410. 分割数组的最大值）

给定正整数数组 nums 和整数 m，将数组分成 m 个非空连续子数组，使得最大子数组和最小。

```python
def split_array(nums: list[int], m: int) -> int:
    """二分答案：将数组分成 m 段，最小化最大子数组和"""

    def can_split(max_sum: int) -> bool:
        """贪心检查：每段和不超过 max_sum 时，能否分成 ≤ m 段"""
        count = 1
        current_sum = 0
        for num in nums:
            if current_sum + num > max_sum:
                count += 1
                current_sum = num
                if count > m:
                    return False
            else:
                current_sum += num
        return True

    # 二分搜索答案空间
    left = max(nums)       # 答案下界：每段至少包含最大元素
    right = sum(nums)      # 答案上界：整个数组作为一段

    while left < right:
        mid = left + (right - left) // 2
        if can_split(mid):
            right = mid      # 可行，尝试更小的答案
        else:
            left = mid + 1   # 不可行，需要更大的答案

    return left
```

#### 二分答案的通用模板

```python
def binary_search_answer():
    """二分答案通用框架"""
    # 1. 确定答案的上下界
    left, right = lower_bound, upper_bound

    # 2. 二分搜索
    while left < right:
        mid = left + (right - left) // 2
        if check(mid):       # 检查 mid 是否可行
            right = mid      # 求最小可行值时：缩小上界
        else:
            left = mid + 1   # 扩大下界

    return left  # 最终 left == right，即为答案
```

**使用场景举例**：

| 问题类型 | 答案空间 | check 函数 | 求最大/最小 |
|---------|---------|-----------|------------|
| 分割数组最小化最大和 | [max(nums), sum(nums)] | 贪心分段计数 | 最小 |
| 能否在 D 天内运完包裹 | [max(weights), sum(weights)] | 贪心模拟 | 最小 |
| 吃香蕉的最小速度 | [1, max(piles)] | 计算总耗时 | 最小 |
| 在排序数组中找位置 | [0, n] | 比较目标值 | 精确定位 |

### 语言标准库中的二分搜索

#### Python —— bisect 模块

```python
import bisect

nums = [1, 3, 5, 7, 7, 7, 9, 11]

# lower_bound：第一个 >= 7 的位置
bisect.bisect_left(nums, 7)   # 返回 3

# upper_bound：第一个 > 7 的位置
bisect.bisect_right(nums, 7)  # 返回 6

# 插入并保持有序
bisect.insort(nums, 6)        # nums = [1, 3, 5, 6, 7, 7, 7, 9, 11]
```

#### C++ —— STL 算法

```cpp
#include <algorithm>
#include <vector>

std::vector<int> nums = {1, 3, 5, 7, 7, 7, 9, 11};

// lower_bound：第一个 >= 7 的迭代器
auto it1 = std::lower_bound(nums.begin(), nums.end(), 7);

// upper_bound：第一个 > 7 的迭代器
auto it2 = std::upper_bound(nums.begin(), nums.end(), 7);

// 自定义比较器
auto it3 = std::lower_bound(nums.begin(), nums.end(), 7,
    [](int a, int b) { return a > b; });  // 降序数组
```

#### Go —— sort.Search

```go
import "sort"

nums := []int{1, 3, 5, 7, 7, 7, 9, 11}

// 等价于 lower_bound：第一个 >= 7 的位置
idx := sort.Search(len(nums), func(i int) bool {
    return nums[i] >= 7
})  // 返回 3
```

### 浮点数二分搜索

浮点数二分不依赖整数的离散性，用**精度（epsilon）** 替代精确相等判断。

```python
def sqrt_binary(x: float) -> float:
    """浮点二分求平方根"""
    if x < 0:
        raise ValueError("Cannot compute square root of negative number")

    left, right = 0.0, x
    if x < 1:
        right = 1.0  # x < 1 时平方根 > x，需要修正右边界

    epsilon = 1e-9
    while right - left > epsilon:
        mid = left + (right - left) / 2
        if mid * mid < x:
            left = mid
        else:
            right = mid

    return left

# 测试
# sqrt_binary(4.0)   → 2.0
# sqrt_binary(8.0)   → 2.828427124...
# sqrt_binary(0.25)  → 0.5
```

**浮点二分的注意事项：**
- 循环条件使用 `right - left > epsilon` 而非精确比较
- epsilon 的选择取决于精度需求（1e-9 适用于大多数场景）
- 需要特别处理 x < 1 的情况（此时平方根可能大于 x 本身）
- 浮点运算的累积误差可能导致结果略微偏离，必要时进行边界修正

### 最佳实践清单

1. **统一区间定义**：整个代码库（或项目团队）使用同一种区间约定，降低认知负担
2. **防溢出**：在 C++/Java/Go 中始终使用 `left + (right - left) / 2`
3. **画图验证**：面试或编码时，用 3-5 个元素的小数组手动追踪 left、right、mid 的变化
4. **覆盖边界用例**：空数组、单元素数组、target 不存在、target 在首尾位置、所有元素相同
5. **区分 mid 能否被排除**：找到 target 时，是立即返回还是继续搜索，取决于要找第几个
6. **优先使用标准库**：Python 的 `bisect`、C++ 的 `lower_bound/upper_bound` 已经过充分测试，优先使用

### 常见问题

**Q: 为什么有些场景推荐左闭右开？**

A: 三个原因：(1) 与 Python 切片、C++ STL 迭代器语义一致；(2) 空区间表示更自然（left == right 而非 left > right）；(3) 在 lower_bound/upper_bound 等需要精确控制边界时，左闭右开的 mid 更新（`right = mid` 而非 `right = mid - 1`）更不容易出错。

**Q: 二分搜索只能用于排序数组吗？**

A: 不是。二分搜索的本质是**在具有单调性的空间上排除一半**。排序数组是最直观的场景，但任何满足单调性的搜索空间都可以使用二分搜索——例如"二分答案"模式，搜索空间是答案的取值范围而非数组索引。

**Q: 如何快速判断用哪种区间定义？**

A: 看两个关键点：(1) right 的初始值是否包含有效元素——`len(nums)-1` 对应左闭右闭，`len(nums)` 对应左闭右开；(2) 找到 target 后是否需要继续搜索——如果是，用左闭右开配合 lower_bound 模式更方便。

**Q: 死循环了怎么排查？**

A: 检查三点：(1) 区间是否真的在缩小（对比循环条件与边界更新）；(2) 边界更新是 `mid + 1` / `mid - 1` 还是 `mid`；(3) 用最小输入（2个元素）手动模拟一遍，观察是否出现 left == right 且无法退出的情况。

**Q: 二分答案的 check 函数怎么设计？**

A: check 函数的核心问题是"给定一个候选答案 x，验证它是否满足约束"。通常使用贪心或模拟：遍历数据，按条件累加/分组，最后判断是否满足限制条件。关键是 check 的时间复杂度不能超过 O(n)，否则整体复杂度退化。

### 总结

二分搜索看似简单，实则暗藏玄机。掌握其核心在于三个要点：

1. **选好区间定义**：左闭右闭或左闭右开，选定后保持一致
2. **匹配好循环条件与边界更新**：`<=` 配 `mid - 1`，`<` 配 `mid`，不可混搭
3. **识别单调性**：不限于排序数组，任何具有单调性的搜索空间都可以二分

"二分答案"是工程实践中最高频的模式——当问题本身没有有序结构，但答案空间具有单调性时，就能用二分搜索 + check 函数将暴力搜索的 O(n × answer_range) 优化到 O(n × log(answer_range))。这是从"会写二分"到"会用二分"的关键跨越。
