## 17.2 汇编语言基础

汇编语言是逆向工程的"母语"。无论你使用多么高级的反编译工具（IDA Pro、Ghidra、Binary Ninja），最终面对的仍然是汇编指令。本节从CPU执行模型出发，系统讲解x86/x64和ARM汇编的核心知识，帮助你建立完整的汇编阅读能力。

### 17.2.1 汇编语言的本质

#### 什么是汇编语言

汇编语言（Assembly Language）是一种**低级编程语言**，它用人类可读的助记符（mnemonic）直接对应CPU的机器指令。每一条汇编指令几乎都一一对应一条机器码指令，是人类能直接理解的最底层编程语言。

```text
高级语言 → 编译器 → 汇编语言 → 汇编器 → 机器码 → CPU执行
```

- **高级语言**（C/C++/Rust）：关注"做什么"
- **汇编语言**：关注"怎么做"——精确控制每一步CPU操作
- **机器码**：CPU直接执行的二进制编码

汇编语言与CPU架构**强绑定**。x86汇编与ARM汇编完全不同，就像英语和日语的差异。逆向工程师必须至少掌握x86/x64（桌面/服务器领域）和ARM/AArch64（移动/嵌入式领域）两种架构。

#### 为什么逆向工程必须懂汇编

反编译器（Decompiler）生成的伪代码并不总是准确的。以下场景必须回到汇编层面才能做出正确判断：

| 场景 | 为什么伪代码不够 |
|------|----------------|
| 编译器优化 | `-O2`以上优化会彻底重排代码结构，伪代码可能遗漏分支或合并逻辑 |
| 反混淆 | 混淆器（OLLVM、Tigress）生成的代码只有在汇编层面才能识别模式 |
| Shellcode分析 | Shellcode没有函数表、没有导入表，只有原始机器码 |
| 漏洞利用 | 精确的栈布局、寄存器状态、ROP链构造都需要逐条指令分析 |
| 性能优化 | 指令级优化（SIMD、分支预测、缓存对齐）必须看汇编 |
| 加壳/脱壳 | 壳代码的自解密逻辑通常用汇编直接编写 |

### 17.2.2 CPU执行模型

在深入指令之前，必须理解CPU是如何执行代码的。

#### 寄存器（Registers）

寄存器是CPU内部的高速存储单元，访问速度比内存快100倍以上。理解寄存器是读懂汇编的第一步。

**x86通用寄存器：**

| 寄存器 | 全称 | 传统用途 | 说明 |
|--------|------|----------|------|
| EAX | Accumulator | 函数返回值、算术运算 | `idiv`指令隐式使用 |
| EBX | Base | 基址寄存器 | 通用，较少特殊用途 |
| ECX | Counter | 循环计数器、`rep`前缀计数 | `thiscall`约定中传this指针 |
| EDX | Data | I/O操作、`idiv`的高位 | 乘除法隐式使用 |
| ESI | Source Index | 字符串操作源地址 | `movsb`等隐式使用 |
| EDI | Destination Index | 字符串操作目标地址 | `stosb`等隐式使用 |
| ESP | Stack Pointer | 栈顶指针 | **不可随意修改** |
| EBP | Base Pointer | 栈帧基址 | 帧指针，用于访问局部变量和参数 |

**x64扩展：**

x64架构将所有通用寄存器扩展到64位，并新增了8个通用寄存器：

```text
; x64寄存器命名规则
RAX    ; 64位版本（R前缀）
EAX    ; 低32位
AX     ; 低16位
AL     ; 低8位
AH     ; 第8-15位

; 新增寄存器
R8-R15       ; 64位
R8D-R15D     ; 低32位
R8W-R15W     ; 低16位
R8B-R15B     ; 低8位

; 特殊寄存器
RIP    ; 64位指令指针（替代EIP）
RSP    ; 64位栈指针
RBP    ; 64位帧指针
```

**x86与x64的关键差异：**

| 特性 | x86 (32位) | x64 (64位) |
|------|-----------|-----------|
| 通用寄存器数量 | 8个 | 16个 |
| 寄存器宽度 | 32位 | 64位 |
| 默认地址宽度 | 32位 | 64位 |
| 函数参数传递 | 主要通过栈 | 前4/6个参数通过寄存器 |
| 指令指针 | EIP（不可直接读写） | RIP（可用于PC相对寻址） |
| 虚拟地址空间 | 4GB | 128TB（48位实现） |
| RED ZONE | 无 | 128字节（System V ABI） |

#### 标志寄存器（EFLAGS/RFLAGS）

标志寄存器是CPU的"状态灯"，每条算术或比较指令都会修改其中的标志位，条件跳转指令则根据标志位决定是否跳转。

**核心标志位：**

| 标志 | 位 | 名称 | 含义 | 典型触发 |
|------|---|------|------|----------|
| CF | 0 | 进位标志 | 无符号运算溢出或借位 | `add`超过最大值、`sub`借位 |
| ZF | 6 | 零标志 | 结果为零 | `cmp`两数相等、`test`结果为零 |
| SF | 7 | 符号标志 | 结果为负数（最高位为1） | 运算结果为负 |
| OF | 11 | 溢出标志 | 有符号运算溢出 | 正+正=负，负+负=正 |
| PF | 2 | 奇偶标志 | 低8位中1的个数为偶数 | 校验用 |
| DF | 10 | 方向标志 | 控制字符串操作方向 | `cld`=正向，`std`=反向 |

**标志位的联动关系——以`cmp eax, ebx`为例：**

`cmp`指令执行`eax - ebx`但不保存结果，只更新标志位：

```text
cmp eax, ebx    ; 内部执行 eax - ebx，更新EFLAGS

; 判断逻辑：
; 如果 eax == ebx → ZF=1（结果为零）
; 如果 eax != ebx → ZF=0
; 如果 eax < ebx（有符号）→ SF ≠ OF
; 如果 eax > ebx（有符号）→ SF == OF 且 ZF=0
; 如果 eax < ebx（无符号）→ CF=1
; 如果 eax > ebx（无符号）→ CF=0 且 ZF=0
```

**有符号vs无符号跳转的区别——逆向中最常见的困惑之一：**

```asm
; 有符号比较（把操作数看作补码有符号数）
jg  greater     ; Jump if Greater (有符号) → SF==OF 且 ZF==0
jl  less        ; Jump if Less (有符号) → SF≠OF
jge greater_eq  ; Jump if Greater or Equal (有符号)
jle less_eq     ; Jump if Less or Equal (有符号)

; 无符号比较（把操作数看作纯二进制无符号数）
ja  above       ; Jump if Above (无符号) → CF==0 且 ZF==0
jb  below       ; Jump if Below (无符号) → CF==1
jae above_eq    ; Jump if Above or Equal (无符号)
jbe below_eq    ; Jump if Below or Equal (无符号)
```

记忆技巧：**g/l**用于有符号（Greater/Less），**a/b**用于无符号（Above/Below）。在逆向中，看到`jg`就想想"这个变量可能是int"，看到`ja`就想想"这个变量可能是unsigned int"。

#### 内存模型

CPU访问内存时，需要理解以下概念：

**字节序（Endianness）：**

x86/x64使用**小端序**（Little-Endian），即低位字节存储在低地址：

```text
; 假设内存地址0x1000处存储值 0x12345678
; 地址:  0x1000  0x1001  0x1002  0x1003
; 内容:  0x78    0x56    0x34    0x12
;         低位 --------------------> 高位

; 在调试器中查看内存（以字节显示）会看到: 78 56 34 12
; 而不是直觉上的: 12 34 56 78
```

这是逆向分析中**最常见的错误来源**之一。当你在十六进制编辑器中看到`78 56 34 12`，实际值是`0x12345678`，不是`0x78563412`。

**对齐（Alignment）：**

现代CPU要求数据按其大小对齐以获得最佳性能：

```text
; 4字节整数应位于4的倍数地址
; 地址 0x1000 → 合法（4的倍数）
; 地址 0x1001 → 未对齐（性能损失，某些架构会触发异常）
```

### 17.2.3 x86/x64指令集详解

#### 寻址模式（Addressing Modes）

寻址模式决定了指令如何定位操作数。掌握寻址模式是读懂汇编的关键。

```asm
; 1. 立即数寻址 — 操作数直接编码在指令中
mov eax, 42              ; eax = 42
mov eax, 0xFF00FF00      ; eax = 0xFF00FF00

; 2. 寄存器寻址 — 操作数在寄存器中
mov eax, ebx             ; eax = ebx

; 3. 直接寻址 — 操作数在指定内存地址
mov eax, [0x401000]      ; eax = *(0x401000)  读取绝对地址处的值

; 4. 寄存器间接寻址 — 内存地址在寄存器中
mov eax, [ebx]           ; eax = *ebx

; 5. 基址+偏移寻址 — 最常见，用于结构体成员和栈变量
mov eax, [ebp + 8]       ; eax = 第一个函数参数（32位cdecl）
mov eax, [ebp - 4]       ; eax = 第一个局部变量
mov eax, [ebx + 0x10]    ; eax = 结构体成员（偏移0x10处）

; 6. 基址+索引*比例+偏移寻址 — 数组访问
mov eax, [ebx + ecx*4]       ; eax = arr[ecx]（int数组，每元素4字节）
mov eax, [ebx + ecx*8 + 16]  ; eax = arr[ecx].field（结构体数组，偏移16）

; 7. RIP相对寻址（x64独有）
mov eax, [rip + 0x1234]  ; 相对于当前指令地址的偏移，用于PIC代码
```

**寻址模式的汇编语法对照（Intel vs AT&T）：**

| 特性 | Intel语法（IDA/Ghidra默认） | AT&T语法（GCC默认） |
|------|---------------------------|---------------------|
| 操作数顺序 | `mov dst, src` | `mov src, dst` |
| 寄存器前缀 | 无 | `%` |
| 立即数前缀 | 无 | `$` |
| 内存引用 | `[eax]` | `(%eax)` |
| 偏移 | `[eax + 4]` | `4(%eax)` |
| 比例索引 | `[eax + ecx*4]` | `(%eax, %ecx, 4)` |
| 大小后缀 | `byte ptr [eax]` | `movb`（b/w/l/q后缀） |

本书统一使用**Intel语法**，这也是IDA Pro、Ghidra、Binary Ninja等主流逆向工具的默认语法。

#### 数据传输指令

```asm
; mov — 最基本的数据传输指令
mov eax, ebx              ; 寄存器 ← 寄存器
mov eax, [ebx]            ; 寄存器 ← 内存
mov [ebx], eax            ; 内存 ← 寄存器
mov eax, 42               ; 寄存器 ← 立即数
; 注意：mov不能直接 内存←内存，必须通过寄存器中转

; lea — 加载有效地址（不访问内存，纯地址计算）
lea eax, [ebx + 4]        ; eax = ebx + 4（加法，不读内存）
lea eax, [ebx + ecx*4]    ; eax = ebx + ecx*4（乘加运算）
; lea常被编译器用于算术运算优化，不只是"取地址"

; push/pop — 栈操作
push eax                  ; esp -= 4; [esp] = eax
pop eax                   ; eax = [esp]; esp += 4
; x64下：rsp -= 8 / rsp += 8（64位栈操作）

; xchg — 交换（注意：隐含LOCK语义，多线程安全但较慢）
xchg eax, ebx             ; eax ↔ ebx

; movsx / movzx — 符号扩展/零扩展
movzx eax, byte ptr [ebx]  ; 将1字节零扩展到32位（无符号扩展）
movsx eax, byte ptr [ebx]  ; 将1字节符号扩展到32位（有符号扩展）
; 这两个在逆向中非常重要，能帮你判断变量的实际类型
```

**movzx与movsx的判断意义：**

当你看到`movzx`，说明源操作数是**无符号**类型（如`unsigned char`）。当你看到`movsx`，说明源操作数是**有符号**类型（如`char`或`signed char`）。这是逆向推断变量类型的直接线索。

#### 算术运算指令

```asm
; 基本算术
add eax, ebx              ; eax += ebx，更新标志位
sub eax, ebx              ; eax -= ebx，更新标志位
imul eax, ebx             ; eax *= ebx（有符号乘法，结果截断为32位）
imul eax, ebx, 5          ; eax = ebx * 5（三操作数形式，编译器常用于常量乘法优化）
idiv ecx                  ; edx:eax / ecx → 商在eax，余数在edx
                          ; 注意：idiv之前必须用cdq/cwd将eax符号扩展到edx:eax

; 常量乘法优化——编译器的常见技巧
; 编译器会用移位和加法替代乘法，识别这些模式非常重要：
; x * 2  →  shl eax, 1       或  lea eax, [eax+eax]
; x * 3  →  lea eax, [eax+eax*2]
; x * 4  →  shl eax, 2
; x * 5  →  lea eax, [eax+eax*4]
; x * 7  →  lea eax, [eax*8-sub] 或 mov+sub组合
; x * 10 →  lea eax, [eax+eax*4] 再 shl eax, 1
; x * 12 →  lea eax, [eax+eax*2] 再 shl eax, 2

; 常量除法优化——识别这些能帮你推断原始代码中的除法：
; x / 2  →  shr eax, 1（无符号）或 sar eax, 1（有符号）
; x / 3  →  乘以0xAAAAAAAB再右移33位（魔法数除法）
; x / 10 →  乘以0xCCCCCCCD再右移35位
; 识别方法：如果看到 imul reg, 0xNNNNNNNN 后面跟着 shr/sar，基本就是除法优化
```

**idiv的陷阱：**

```asm
; idiv 使用隐式操作数 edx:eax
; 除法前必须将eax符号扩展到edx:eax，否则会得到错误结果甚至崩溃
mov eax, [ebp-4]          ; 加载被除数
cdq                        ; 将eax符号扩展到edx:eax（重要！）
idiv dword ptr [ebp-8]    ; 除以除数
; 商在eax，余数在edx
```

#### 位运算指令

位运算在逆向中用途极广——加密算法、哈希函数、权限检查、位域操作都大量使用。

```asm
; 基本位运算
and eax, ebx              ; eax &= ebx（位与，常用于掩码/清除指定位）
or  eax, ebx              ; eax |= ebx（位或，常用于设置标志位）
xor eax, ebx              ; eax ^= ebx（位异或，最常用的逆向指令之一）
not eax                   ; eax = ~eax（按位取反）

; xor eax, eax — 最常见的清零方式
xor eax, eax              ; eax = 0（比 mov eax, 0 少1字节机器码）
; 在逆向中看到 xor reg, reg 基本就是"赋值为0"

; 移位操作
shl eax, cl               ; 逻辑左移（低位补0，用于乘以2^n）
shr eax, cl               ; 逻辑右移（高位补0，用于无符号除以2^n）
sar eax, cl               ; 算术右移（高位补符号位，用于有符号除以2^n）
shl eax, 3                ; eax *= 8（常量左移）

; 循环移位
rol eax, cl               ; 循环左移（溢出位回到低位）
ror eax, cl               ; 循环右移
; 循环移位在加密算法（RC5、MD5、SHA系列）中广泛使用

; 位测试
bt  eax, 5                ; 测试第5位是否为1（结果存入CF）
bts eax, 5                ; 测试并设置第5位
btr eax, 5                ; 测试并清除第5位

; 位扫描
bsf eax, ebx              ; 从低位到高位扫描，找到第一个为1的位
bsr eax, ebx              ; 从高位到低位扫描，找到第一个为1的位
```

**位运算的实际应用场景：**

```asm
; 权限检查（位掩码）
test eax, 0x04            ; 检查第2位是否为1（例如：写权限）
jnz  has_write_perm       ; 不为零说明有权限

; 提取特定位域（例如提取bit 8-15）
shr eax, 8                ; 右移8位
and eax, 0xFF             ; 保留低8位

; 判断奇偶
test eax, 1               ; 测试最低位
jz   is_even              ; 最低位为0则是偶数

; 快速取模（2的幂）
and eax, 0x0F             ; eax % 16（仅对2的幂有效）
```

#### 比较与控制流指令

```asm
; 比较指令
cmp eax, ebx              ; 执行 eax - ebx，只更新标志位，不保存结果
test eax, eax             ; 执行 eax & eax，测试eax是否为零
test eax, 0x01            ; 测试特定位

; 条件设置（不跳转，而是设置字节值——编译器用于三元表达式）
sete  al                  ; al = (ZF==1) ? 1 : 0  → 等于时al=1
setne al                  ; al = (ZF==0) ? 1 : 0  → 不等于时al=1
setg  al                  ; al = (大于,有符号) ? 1 : 0
setl  al                  ; al = (小于,有符号) ? 1 : 0
seta  al                  ; al = (大于,无符号) ? 1 : 0
setb  al                  ; al = (小于,无符号) ? 1 : 0

; 条件移动（CMOV指令——避免分支预测失败的性能损失）
cmove eax, ebx            ; 如果ZF==1（相等），则 eax = ebx
cmovne eax, ebx           ; 如果ZF==0（不等），则 eax = ebx
; 编译器在 -O2 以上经常用 CMOV 替代短的 if-else 分支

; 无条件跳转
jmp  target               ; 跳转到目标地址
call target               ; push(返回地址); jmp target
ret                       ; pop → jmp（返回到调用者）
```

#### 字符串操作指令

```asm
; 方向标志控制
cld                        ; DF=0，地址递增（正向处理）——最常见
std                        ; DF=1，地址递减（反向处理）——罕见

; 字符串移动
movsb                      ; [edi] = [esi]; esi±1; edi±1  （移动1字节）
movsw                      ; [edi] = [esi]; esi±2; edi±2  （移动2字节）
movsd                      ; [edi] = [esi]; esi±4; edi±4  （移动4字节）
rep movsb                  ; 重复ecx次：memcpy(dst=edi, src=esi, len=ecx)

; 字符串比较
cmpsb                      ; 比较 [esi] 和 [edi]，更新标志位
repe cmpsb                 ; 相等时重复比较（类似memcmp的前缀匹配）
repne cmpsb                ; 不相等时重复比较（类似strchr查找字符）

; 字符串存储
stosb                      ; [edi] = al; edi±1
rep stosb                  ; 重复ecx次：memset(edi, al, ecx)

; 字符串扫描
scasb                      ; 比较 al 和 [edi]
repne scasb                ; 在内存中查找al的值（类似strchr）
```

在逆向中，看到`rep movsb`基本就是`memcpy`，看到`rep stosb`基本就是`memset`，看到`repne scasb`基本就是`strlen`或`strchr`。

#### 系统指令与特殊指令

```asm
; nop — 空操作（用于对齐、占位、热补丁）
nop                        ; 不做任何事，占1字节
; 有时用于使函数地址对齐到16字节边界

; int3 — 调试断点（0xCC）
int3                       ; 触发调试中断，调试器会暂停
; 反调试技术中会扫描代码中的0xCC来检测断点

; cpuid — 获取CPU信息
mov eax, 0
cpuid                     ; 返回CPU厂商字符串到ebx:edx:ecx

; 系统调用（x86 Linux）
mov eax, 1                ; syscall number: sys_exit
mov ebx, 0                ; exit code
int 0x80                  ; 触发系统调用

; 系统调用（x64 Linux）
mov eax, 60               ; syscall number: sys_exit
xor edi, edi              ; exit code = 0
syscall                   ; 触发系统调用

; 系统调用（Windows）
; Windows不直接使用syscall，而是通过ntdll.dll中的包装函数
; syscall号随Windows版本变化，不能硬编码
```

### 17.2.4 调用约定（Calling Conventions）

调用约定决定了函数如何传递参数、如何返回值、谁负责清理栈。这是逆向分析函数调用时**必须首先识别**的信息。

#### cdecl（32位C默认）

```asm
; 调用者清理栈，参数从右到左压栈
push arg3                 ; 第3个参数
push arg2                 ; 第2个参数
push arg1                 ; 第1个参数
call function
add esp, 12               ; 调用者清理3个参数（3 * 4字节）
; 返回值在eax
```

#### stdcall（32位Windows API默认）

```asm
; 被调用者清理栈，参数从右到左压栈
push arg3
push arg2
push arg1
call function
; 不需要add esp，函数内部用 ret N 清理
; 函数内部：ret 12（返回并清理12字节）
```

#### fastcall（32位优化）

```asm
; 前两个整型参数通过ecx和edx传递，其余通过栈
mov ecx, arg1             ; 第1个参数 → ecx
mov edx, arg2             ; 第2个参数 → edx
push arg3                 ; 第3个参数及之后通过栈
call function
add esp, 4                ; 清理栈参数
```

#### System V AMD64 ABI（64位Linux/macOS默认）

```asm
; 前6个整型/指针参数通过寄存器传递
mov rdi, arg1             ; 第1个参数
mov rsi, arg2             ; 第2个参数
mov rdx, arg3             ; 第3个参数
mov rcx, arg4             ; 第4个参数
mov r8,  arg5             ; 第5个参数
mov r9,  arg6             ; 第6个参数
; 浮点参数通过xmm0-xmm7传递
; 调用者清理栈，返回值在rax（整型）或xmm0（浮点）
call function

; RED ZONE：rsp下方128字节是"红色区域"
; 叶子函数（不调用其他函数的函数）可以不调整rsp就使用这128字节
; 这是一个重要的优化，但也增加了逆向分析的复杂度
```

#### Windows x64调用约定

```asm
; 与System V的主要差异：
; 1. 只有4个寄存器参数：rcx, rdx, r8, r9
; 2. 没有RED ZONE
; 3. 必须分配"影子空间"（Shadow Space）至少32字节
; 4. 栈必须16字节对齐

sub rsp, 32               ; 分配影子空间（即使参数少于4个也要分配）
mov rcx, arg1
mov rdx, arg2
mov r8,  arg3
mov r9,  arg4
call function
add rsp, 32               ; 释放影子空间
```

#### 调用约定速查表

| 约定 | 参数传递 | 栈清理 | 返回值 | 使用场景 |
|------|---------|--------|--------|---------|
| cdecl | 栈（右到左） | 调用者 | eax | 32位C/Linux |
| stdcall | 栈（右到左） | 被调用者 | eax | 32位Win API |
| fastcall | ecx,edx + 栈 | 被调用者 | eax | 32位优化 |
| thiscall | ecx(this) + 栈 | 被调用者 | eax | 32位C++ MSVC |
| System V | rdi,rsi,rdx,rcx,r8,r9 | 调用者 | rax | 64位Linux/macOS |
| Win64 | rcx,rdx,r8,r9 + 影子空间 | 调用者 | rax | 64位Windows |

### 17.2.5 栈帧深入解析

栈是函数调用的核心数据结构。理解栈帧布局是分析缓冲区溢出、ROP链构造、局部变量布局的基础。

#### 32位典型栈帧布局

```text
高地址
┌──────────────────────┐
│    函数参数 arg3      │ [ebp + 16]
├──────────────────────┤
│    函数参数 arg2      │ [ebp + 12]
├──────────────────────┤
│    函数参数 arg1      │ [ebp + 8]
├──────────────────────┤
│    返回地址 (EIP)     │ [ebp + 4]    ← call指令压入
├──────────────────────┤
│    旧的EBP           │ [ebp]        ← push ebp
├──────────────────────┤
│    局部变量 var1      │ [ebp - 4]
├──────────────────────┤
│    局部变量 var2      │ [ebp - 8]
├──────────────────────┤
│    局部变量 var3      │ [ebp - 12]
├──────────────────────┤
│    对齐填充（如有）    │
├──────────────────────┤
│    callee保存的寄存器  │
├──────────────────────┤
│    ...               │ ← ESP（栈顶）
低地址
```

```asm
; 对应的汇编代码
function:
    push ebp              ; 保存旧的帧指针
    mov ebp, esp          ; 建立新帧指针
    sub esp, 0x20         ; 为局部变量分配空间（32字节）
    push ebx              ; 保存callee-saved寄存器
    push esi

    ; 此时：
    ; [ebp + 8]  = arg1
    ; [ebp + 12] = arg2
    ; [ebp + 16] = arg3
    ; [ebp - 4]  = var1
    ; [ebp - 8]  = var2
    ; ...

    ; 函数体...

    pop esi               ; 恢复callee-saved寄存器
    pop ebx
    mov esp, ebp          ; 释放局部变量
    pop ebp               ; 恢复旧的帧指针
    ret                   ; 返回
```

#### 64位栈帧（System V ABI）

```text
高地址
┌──────────────────────┐
│    更多参数（如有）    │ [rbp + 24+]
├──────────────────────┤
│    返回地址 (RIP)     │ [rbp + 8]
├──────────────────────┤
│    旧的RBP           │ [rbp]
├──────────────────────┤
│    局部变量...        │ [rbp - 8], [rbp - 16]...
├──────────────────────┤
│                      │
│    RED ZONE (128B)   │ [rsp] 到 [rsp - 127]
│    叶子函数可直接使用  │
│                      │
低地址
```

#### 寄存器保存约定

在函数调用中，某些寄存器的值必须被保留：

| 类别 | x86/x64 | ARM | 谁负责保存 |
|------|---------|-----|-----------|
| Caller-saved（易失性） | eax, ecx, edx (x86); rax, rcx, rdx, r8-r11 (x64) | r0-r3, r12 | 调用者保存 |
| Callee-saved（非易失性） | ebx, esi, edi, ebp (x86); rbx, rbp, r12-r15 (x64) | r4-r11 | 被调用者保存 |

逆向技巧：如果一个函数开头就`push ebx; push esi; push edi`，结尾`pop edi; pop esi; pop ebx`，说明它使用了这些callee-saved寄存器来存储重要值。这些被保存的值通常就是函数中的关键局部变量。

### 17.2.6 编译器生成的典型模式

理解编译器如何将高级语言翻译成汇编，是高效逆向分析的关键。以下覆盖最常见的编译模式。

#### 函数序言与尾声

**标准帧指针模式：**

```asm
; 函数序言（Prologue）
push ebp                  ; 保存旧的栈帧基址
mov ebp, esp              ; 建立新的栈帧
sub esp, 0x40             ; 分配局部变量空间（0x40 = 64字节）

; 函数尾声（Epilogue）
mov esp, ebp              ; 释放局部变量
pop ebp                   ; 恢复旧的栈帧基址
ret                       ; 返回
; 或者等价的：leave; ret（leave = mov esp, ebp + pop ebp）
```

**帧指针省略模式（-fomit-frame-pointer）：**

```asm
; GCC/Clang在x64下默认开启此优化
function:
    sub rsp, 0x28         ; 直接分配栈空间，不使用rbp
    ; ... 函数体 ...
    add rsp, 0x28         ; 释放栈空间
    ret
; 注意：此时所有局部变量通过 [rsp + offset] 访问
; 逆向时rsp值会变化，需要跟踪计算偏移
```

#### if-else结构

```c
// C源码
if (x > 10) {
    y = 1;
} else {
    y = 2;
}
```

```asm
; 编译结果（未优化 -O0）
cmp dword ptr [ebp-4], 10  ; 比较 x 和 10
jle else_branch             ; 如果 x <= 10，跳到else
mov dword ptr [ebp-8], 1   ; y = 1
jmp end_if                  ; 跳过else
else_branch:
mov dword ptr [ebp-8], 2   ; y = 2
end_if:

; 编译结果（优化 -O2，可能使用CMOV）
cmp dword ptr [rbp-4], 10
setg al                     ; al = (x > 10) ? 1 : 0
movzx eax, al
; 或者使用CMOV：
mov eax, 2                  ; 默认y = 2
cmp dword ptr [rbp-4], 10
mov edx, 1                  ; 如果x > 10
cmovg eax, edx              ; 条件赋值：如果大于，eax = 1
```

#### 三元表达式

```c
// C源码
int y = (x > 0) ? x : -x;  // 取绝对值
```

```asm
; 典型编译结果
mov eax, [ebp-4]            ; eax = x
test eax, eax               ; 测试x是否为0
jns positive                ; 如果x >= 0（SF=0），跳到positive
neg eax                     ; eax = -x
positive:
mov [ebp-8], eax            ; y = eax
```

#### switch语句

**跳转表实现（密集case值）：**

```c
switch (x) {
    case 0: y = 10; break;
    case 1: y = 20; break;
    case 2: y = 30; break;
    case 3: y = 40; break;
    default: y = 0; break;
}
```

```asm
cmp eax, 3                  ; 检查范围
ja  default_case            ; 如果x > 3（无符号），跳到default
jmp [jump_table + eax*4]    ; 通过跳转表跳转

jump_table:
    dd case_0    ; 地址：&case_0
    dd case_1    ; 地址：&case_1
    dd case_2    ; 地址：&case_2
    dd case_3    ; 地址：&case_3
```

**if-else链实现（稀疏case值）：**

```asm
cmp eax, 100
je case_100
cmp eax, 200
je case_200
cmp eax, 500
je case_500
jmp default_case
```

**二分查找实现（大量稀疏case值）：**

编译器可能对大量case值使用二分查找结构，形成嵌套的cmp-je/ja/jb树。

#### 循环结构

```asm
; === for循环 ===
; for (int i = 0; i < 10; i++) { body; }
xor ecx, ecx              ; i = 0
loop_start:
cmp ecx, 10               ; i < 10?
jge loop_end               ; 如果i >= 10，退出
; ... 循环体 ...
inc ecx                   ; i++
jmp loop_start
loop_end:

; === while循环 ===
; while (x != 0) { x = x / 2; }
loop_start:
test eax, eax             ; x == 0?
jz loop_end               ; 为零则退出
shr eax, 1                ; x /= 2
jmp loop_start
loop_end:

; === do-while循环 ===
; do { body; } while (x < 10);
loop_start:
; ... 循环体 ...
cmp eax, 10               ; x < 10?
jl loop_start              ; 如果小于，继续循环

; === 编译器循环优化 ===
; -O2下编译器可能：
; 1. 循环展开（Loop Unrolling）：一次处理4个元素
; 2. 强度削减（Strength Reduction）：用加法替代乘法
; 3. 循环不变量外提：把循环外能计算的提前计算
; 4. 使用rep指令：对memset/memcpy类操作直接使用rep
```

#### 数组与结构体访问

```asm
; === 一维数组访问 ===
; int arr[10]; 访问 arr[i]
mov eax, [ebp-0x40]            ; eax = i
mov eax, [ebp + eax*4 - 0x30] ; eax = arr[i]
; 计算公式：基址 + 索引 * 元素大小
; int(4字节), short(2字节), char(1字节), double(8字节)

; === 二维数组访问 ===
; int matrix[3][4]; 访问 matrix[i][j]
; 行优先存储：地址 = base + (i * cols + j) * sizeof(int)
mov eax, [ebp-0x40]            ; eax = i
imul eax, eax, 16              ; eax = i * 16（i * 4列 * 4字节）
add eax, [ebp-0x44]            ; eax += j（注意j也需要乘以4）
mov eax, [ebp + eax*4 - 0x30]  ; 读取值

; === 结构体访问 ===
; struct Point { int x; int y; int z; };
; p.y 的偏移为 4（sizeof(int) = 4）
mov eax, [ebp-0x20]            ; eax = p（结构体起始地址）
mov eax, [eax + 4]             ; eax = p.y（偏移4字节）

; === 结构体指针访问 ===
; struct Point *pp;
; pp->z 的偏移为 8
mov eax, [ebp-0x24]            ; eax = pp（指针值）
mov eax, [eax + 8]             ; eax = pp->z

; === 结构体数组访问 ===
; struct { int x, y, z; } points[10];
; points[i].y = 基址 + i * 12 + 4
mov eax, [ebp-0x40]            ; eax = i
imul eax, eax, 12              ; eax = i * 12（每个Point占12字节）
mov dword ptr [ebp + eax - 0x30 + 4], 42  ; points[i].y = 42
```

#### 虚函数调用（C++多态）

```asm
; C++虚函数调用通过vtable（虚函数表）实现
; 假设 this 指针在 ecx（thiscall约定）

; 1. 从对象获取vtable指针
mov eax, [ecx]              ; eax = this->vptr（对象第一个字段是vtable指针）

; 2. 从vtable获取函数地址
mov edx, [eax + 0x8]        ; edx = vtable[2]（第3个虚函数，偏移0x8）

; 3. 调用
call edx                    ; 调用虚函数

; vtable布局：
; [0x00] 第1个虚函数指针
; [0x04] 第2个虚函数指针（32位）
; [0x08] 第3个虚函数指针
; ...
; x64下每个指针占8字节：[0x00], [0x08], [0x10], ...
```

**C++ RTTI（运行时类型信息）：**

vtable指针前面通常还有RTTI信息，可以通过负偏移找到：

```asm
mov eax, [ecx]              ; eax = vptr
mov eax, [eax - 4]          ; eax = RTTI指针（vtable前面）
; RTTI中包含类名字符串，可用于识别对象的实际类型
```

#### 异常处理（SEH/SEH64）

```asm
; Windows结构化异常处理（SEH）
; 在栈上有一个SEH链表，fs:[0]指向链表头
; 逆向中看到fs:[0]的读写基本就是异常处理代码

; 32位SEH注册
push handler_addr           ; 异常处理函数地址
push fs:[0]                 ; 保存旧的SEH链表头
mov fs:[0], esp             ; 注册新的SEH节点

; 64位Windows使用基于表的异常处理（不再修改栈）
; 异常处理信息存储在.pdata段中
; 每个函数有一个RUNTIME_FUNCTION条目
```

#### 编译器内联函数

-O2以上优化经常内联小函数，逆向时需要注意：

```asm
; 内联前：两个独立函数
; int add(int a, int b) { return a + b; }
; int result = add(x, 3);

; 内联后：直接嵌入调用点
lea eax, [ecx + 3]          ; result = x + 3（直接内联）

; 识别内联的线索：
; 1. 函数体很短且没有call指令
; 2. 代码片段出现在不合理的上下文中
; 3. 局部变量布局不连续
; 4. 同一段代码在多处重复出现
```

### 17.2.7 ARM汇编基础

ARM（Advanced RISC Machine）是RISC架构，采用Load/Store设计理念——只有Load/Store指令才能访问内存，所有运算都在寄存器之间进行。这与x86的CISC设计（大多数指令可直接操作内存）形成鲜明对比。

#### ARM vs x86的设计哲学差异

| 特性 | x86/x64 (CISC) | ARM (RISC) |
|------|---------------|------------|
| 指令长度 | 可变长（1-15字节） | 固定长（ARM: 32位, Thumb: 16位） |
| 内存访问 | 几乎所有指令可访问内存 | 只有LDR/STR访问内存 |
| 指令数量 | 极多（数百种变体） | 较少，但每条功能明确 |
| 条件执行 | 仅跳转指令可条件执行 | **几乎任何指令都可条件执行** |
| 寄存器数量 | 8-16个 | 16个（R0-R15） |
| 代码密度 | 高（复杂指令） | 低（简单指令多），Thumb模式改善 |
| 功耗 | 较高 | 较低（设计目标） |

#### ARM模式与寄存器

ARM处理器支持多种执行模式，每种模式有不同的寄存器集合：

**用户模式（User Mode）寄存器：**

```text
通用寄存器：
  R0-R7    : 低寄存器，所有指令可访问
  R8-R12   : 高寄存器，部分指令受限
  R13 (SP) : 栈指针（Stack Pointer）
  R14 (LR) : 链接寄存器（Link Register）——保存函数返回地址
  R15 (PC) : 程序计数器（Program Counter）

特殊寄存器：
  CPSR     : 当前程序状态寄存器
  SPSR     : 保存的程序状态寄存器（异常时保存CPSR）
```

**ARM状态寄存器（CPSR）的标志位：**

```text
| 31 | 30 | 29 | 28 | 27 | ... | 5 | 4 | 3:0 |
| N  | Z  | C  | V  | Q  |     | T | M | M   |
 N = Negative（负数标志）
 Z = Zero（零标志）
 C = Carry（进位标志）
 V = Overflow（溢出标志）
 T = Thumb状态（1=Thumb模式，0=ARM模式）
 M = 处理器模式
```

#### ARM条件执行——ARM最独特的特性

几乎每条ARM指令都可以通过条件码后缀来条件执行，这是ARM架构区别于x86的核心特征：

```asm
; 条件码后缀示例
MOVEQ R0, #1       ; 如果Z标志为1（相等），则 R0 = 1
MOVNE R0, #2       ; 如果Z标志为0（不相等），则 R0 = 2
ADDGT R0, R1, R2   ; 如果大于（有符号），则 R0 = R1 + R2
SUBLE R0, R1, R2   ; 如果小于等于（有符号），则 R0 = R1 - R2

; 条件码列表
EQ  ; Equal                    Z=1
NE  ; Not Equal                Z=0
CS/HS ; Carry Set/Unsigned >=  C=1
CC/LO ; Carry Clear/Unsigned < C=0
MI  ; Minus/Negative           N=1
PL  ; Plus/Positive            N=0
VS  ; Overflow Set             V=1
VC  ; Overflow Clear           V=0
HI  ; Unsigned Higher          C=1 and Z=0
LS  ; Unsigned Lower or Same   C=0 or Z=1
GE  ; Signed Greater or Equal  N=V
LT  ; Signed Less Than         N≠V
GT  ; Signed Greater Than      Z=0 and N=V
LE  ; Signed Less or Equal     Z=1 or N≠V
AL  ; Always（默认）           无条件执行
```

**条件执行在逆向中的意义：**

```asm
; if (a > b) c = 1; else c = 2;
; ARM编译结果（不使用跳转）：
CMP R0, R1
MOVGT R2, #1       ; 如果a > b，c = 1
MOVLE R2, #2       ; 如果a <= b，c = 2
; 没有任何跳转指令！代码是线性的
; 这对逆向分析和混淆识别都有重要影响
```

#### ARM Load/Store指令

```asm
; === Load指令（读内存到寄存器）===
LDR  R0, [R1]            ; R0 = *R1（从R1指向的地址读4字节）
LDR  R0, [R1, #4]        ; R0 = *(R1 + 4)（基址 + 立即数偏移）
LDR  R0, [R1, R2]        ; R0 = *(R1 + R2)（基址 + 寄存器偏移）
LDR  R0, [R1, R2, LSL #2]; R0 = *(R1 + R2*4)（基址 + 寄存器*比例）
LDRB R0, [R1]            ; R0 = *(uint8_t*)R1（加载1字节，零扩展）
LDRSB R0, [R1]           ; R0 = *(int8_t*)R1（加载1字节，符号扩展）
LDRH R0, [R1]            ; R0 = *(uint16_t*)R1（加载2字节，零扩展）
LDRSH R0, [R1]           ; R0 = *(int16_t*)R1（加载2字节，符号扩展）

; === Store指令（写寄存器到内存）===
STR  R0, [R1]            ; *R1 = R0
STRB R0, [R1]            ; *(uint8_t*)R1 = R0（写1字节）
STRH R0, [R1]            ; *(uint16_t*)R1 = R0（写2字节）

; === 前索引（Pre-indexed）：先更新基址再访问 ===
LDR  R0, [R1, #4]!       ; R1 += 4; R0 = *R1（!后缀更新R1）

; === 后索引（Post-indexed）：先访问再更新基址 ===
LDR  R0, [R1], #4        ; R0 = *R1; R1 += 4（先读后更新）
; 后索引常用于数组遍历：每次访问后自动推进指针
```

#### ARM算术与逻辑指令

```asm
; 算术
ADD R0, R1, R2            ; R0 = R1 + R2
ADD R0, R1, #5            ; R0 = R1 + 5（立即数）
SUB R0, R1, R2            ; R0 = R1 - R2
RSB R0, R1, R2            ; R0 = R2 - R1（反向减法）
MUL R0, R1, R2            ; R0 = R1 * R2
MLA R0, R1, R2, R3        ; R0 = R1 * R2 + R3（乘加）

; 逻辑
AND R0, R1, R2            ; R0 = R1 & R2
ORR R0, R1, R2            ; R0 = R1 | R2
EOR R0, R1, R2            ; R0 = R1 ^ R2（异或，ARM用EOR不是XOR）
BIC R0, R1, R2            ; R0 = R1 & ~R2（位清除，ARM独有）

; 移位（移位可以作为任何数据处理指令的一部分）
LSL R0, R1, #3            ; R0 = R1 << 3（逻辑左移）
LSR R0, R1, #3            ; R0 = R1 >> 3（逻辑右移）
ASR R0, R1, #3            ; R0 = R1 >> 3（算术右移，保留符号位）
ROR R0, R1, #3            ; R0 = R1 循环右移3位

; 带移位的操作——ARM的优雅设计
ADD R0, R1, R2, LSL #3    ; R0 = R1 + (R2 << 3) = R1 + R2*8
SUB R0, R1, R2, LSR #1    ; R0 = R1 - (R2 >> 1) = R1 - R2/2
; 一条指令完成 乘加/乘减 运算，x86需要两条
```

#### ARM分支与函数调用

```asm
; 条件分支
CMP R0, R1
BEQ label            ; 相等则跳转
BNE label            ; 不相等则跳转
BGT label            ; 大于（有符号）则跳转
BLT label            ; 小于（有符号）则跳转
B label              ; 无条件跳转

; 函数调用
BL function          ; 将返回地址存入LR（R14），跳转到function
; 函数中可以使用 STMFD SP!, {R4-R11, LR} 保存寄存器和返回地址
; 返回时使用 LDMFD SP!, {R4-R11, PC} 恢复寄存器并跳回

; 函数返回
BX LR                ; 跳转到LR中的地址（可能切换ARM/Thumb模式）
; 或
MOV PC, LR           ; 等价于BX LR（但不切换模式）
```

#### ARM函数序言与尾声

```asm
; 标准ARM函数序言
STMFD SP!, {R4-R11, LR}    ; 保存寄存器到栈（STMFD = STMDB = 压栈）
SUB SP, SP, #0x20           ; 分配局部变量空间

; 函数体...

; 标准ARM函数尾声
ADD SP, SP, #0x20           ; 释放局部变量空间
LDMFD SP!, {R4-R11, PC}    ; 恢复寄存器并返回（LDMFD = LDMIA = 出栈）
; 注意：恢复PC就等于返回，LR被弹出到PC
```

#### AArch64（ARM64）

AArch64是ARM的64位架构，用于现代手机（iPhone 5s以后、Android旗舰机）和Apple Silicon Mac。

```asm
; === AArch64寄存器 ===
; X0-X28  : 64位通用寄存器
; X29 (FP): 帧指针（Frame Pointer）
; X30 (LR): 链接寄存器（Link Register）
; SP      : 栈指针（不再是X31，有专用名）
; XZR     : 零寄存器（读取永远为0，写入忽略）

; W0-W28  : X0-X28的低32位

; === AArch64调用约定 ===
; X0-X7  : 前8个参数
; X0-X1  : 返回值（最大128位用X0+X1）
; X8     : 间接返回地址（用于返回大型结构体）
; X9-X15 : 临时寄存器（caller-saved）
; X16-X17: PLT/Linker暂存寄存器（不要在代码中使用）
; X18    : 平台保留（macOS用作TLS）
; X19-X28: callee-saved寄存器
; X29    : 帧指针（callee-saved）
; X30    : 链接寄存器

; === AArch64 Load/Store ===
LDR X0, [X1]               ; X0 = *X1
LDR X0, [X1, #8]           ; X0 = *(X1 + 8)
STR X0, [X1, #8]!          ; X1 += 8; *X1 = X0（前索引写回）
LDR X0, [X1], #8           ; X0 = *X1; X1 += 8（后索引）

; === AArch64算术 ===
ADD X0, X1, X2             ; X0 = X1 + X2
ADD X0, X1, X2, LSL #3    ; X0 = X1 + X2*8
SUB X0, X1, X2             ; X0 = X1 - X2
MUL X0, X1, X2             ; X0 = X1 * X2
SDIV X0, X1, X2            ; X0 = X1 / X2（有符号除法）
UDIV X0, X1, X2            ; X0 = X1 / X2（无符号除法）

; === AArch64分支 ===
CBZ X0, label              ; if (X0 == 0) goto label
CBNZ X0, label             ; if (X0 != 0) goto label
TBZ X0, #5, label          ; if (bit 5 of X0 == 0) goto label（位测试分支）
TBNZ X0, #5, label         ; if (bit 5 of X0 != 0) goto label
; CBZ/CBNZ/TBZ/TBNZ 是AArch64新增的高效分支指令

BL function                ; 调用函数
BLR X0                     ; 间接调用（X0中是函数地址）
RET                        ; 返回（默认返回X30/LR）
```

#### Thumb模式

Thumb是ARM的压缩指令模式，使用16位指令（Thumb-2扩展为16/32位混合），用于提高代码密度：

```asm
; Thumb模式特点：
; 1. 16位指令宽度（Thumb-2支持16/32位混合）
; 2. 只能访问R0-R7（低寄存器），高寄存器访问受限
; 3. 大多数指令无条件执行
; 4. 代码密度更高（约减少30%代码大小）

; 切换模式：
BX R0               ; R0的最低位决定模式
                     ; 最低位=0 → ARM模式
                     ; 最低位=1 → Thumb模式
; 编译器通常自动管理模式切换
; 反汇编器（IDA/Ghidra）会自动检测并切换显示模式

; Thumb-2示例（与ARM语法相同，但支持16位短编码）
MOVS R0, #5         ; Thumb中的MOVS
ADD R0, R0, R1      ; Thumb中的ADD
IT EQ               ; If-Then块（Thumb-2独有）
MOVEQ R2, #1        ; 仅在前面IT条件满足时执行
```

### 17.2.8 实战：完整函数逆向分析

下面通过一个完整示例演示如何将汇编代码还原为C源码。

#### 原始汇编代码（32位x86，cdecl约定）

```asm
; int __cdecl calculate(int a, int b)
calculate:
    push ebp
    mov ebp, esp
    sub esp, 8                  ; 分配8字节局部变量空间
    mov eax, [ebp+8]           ; eax = a
    imul eax, [ebp+0xC]        ; eax = a * b
    mov [ebp-4], eax           ; local_1 = a * b
    cmp dword ptr [ebp+8], 0   ; 比较a和0
    jle .L1                    ; 如果a <= 0，跳到L1
    mov eax, [ebp+8]           ; eax = a
    sub eax, [ebp+0xC]         ; eax = a - b
    mov [ebp-8], eax           ; local_2 = a - b
    jmp .L2
.L1:
    mov eax, [ebp+0xC]         ; eax = b
    add eax, [ebp+8]           ; eax = b + a
    mov [ebp-8], eax           ; local_2 = b + a
.L2:
    mov eax, [ebp-4]           ; eax = local_1
    add eax, [ebp-8]           ; eax = local_1 + local_2
    mov esp, ebp
    pop ebp
    ret
```

#### 还原过程

**第一步：识别函数签名**
- `push ebp; mov ebp, esp` → 标准序言，有栈帧
- `[ebp+8]` → 第一个参数（32位cdecl，第一个参数在[ebp+8]）
- `[ebp+0xC]` → 第二个参数（偏移12 = 8 + 4）
- `sub esp, 8` → 两个局部变量（各4字节）

**第二步：逐步翻译**

```c
int calculate(int a, int b) {
    int local_1, local_2;

    // mov eax, [ebp+8] / imul eax, [ebp+0xC]
    // mov [ebp-4], eax
    local_1 = a * b;

    // cmp dword ptr [ebp+8], 0 / jle .L1
    if (a > 0) {
        // mov eax, [ebp+8] / sub eax, [ebp+0xC]
        // mov [ebp-8], eax
        local_2 = a - b;
    } else {
        // mov eax, [ebp+0xC] / add eax, [ebp+8]
        // mov [ebp-8], eax
        local_2 = b + a;
    }

    // mov eax, [ebp-4] / add eax, [ebp-8]
    return local_1 + local_2;
}
```

**第三步：简化（编译器优化后的代码可能更简洁）**

```c
int calculate(int a, int b) {
    int product = a * b;
    int diff = (a > 0) ? (a - b) : (a + b);
    return product + diff;
}
```

#### 同一函数的x64版本（System V ABI）

```asm
calculate:
    ; 无序言（叶子函数，使用RED ZONE）
    mov edx, edi              ; edx = a（第1个参数在edi）
    imul edx, esi             ; edx = a * b（第2个参数在esi）
    test edi, edi             ; 测试a是否为0
    jle .L1
    sub edi, esi              ; edi = a - b
    jmp .L2
.L1:
    add esi, edi              ; esi = b + a（= a + b）
    mov edi, esi
.L2:
    lea eax, [rdx + rdi]      ; eax = product + diff
    ret
; 注意：叶子函数没有调整rsp，直接使用RED ZONE
; 代码更紧凑，只用3个寄存器，没有栈操作
```

### 17.2.9 常见反汇编陷阱与误区

#### 陷阱1：数据与代码混淆

```text
; 反汇编器是线性扫描的，可能把数据误识别为代码
; 例如跳转表中的地址被反汇编器当作指令解析
; 解决方法：在IDA中按'C'转代码，按'D'转数据，按'U'取消定义
```

#### 陷阱2：编译器优化打乱代码结构

```asm
; 原始：x * 7
; 编译器优化：lea eax, [eax+eax*2] → add eax, eax → ... 
; 或者：lea eax, [eax*8] → sub eax, original
; 看起来不像乘法，但本质是乘法优化

; 识别方法：
; 1. lea reg, [reg+reg*N] 通常是乘以(N+1)
; 2. imul reg, reg, constant 通常是乘以常量
; 3. 魔法数除法：imul reg, 0xNNNNNNNN + shr 基本是除法
```

#### 陷阱3：有符号vs无符号的误判

```asm
; 如果你把有符号比较当作无符号比较来理解，分析结果完全错误
; 看到 jg/jl/jge/jle → 有符号比较 → 变量可能是int
; 看到 ja/jb/jae/jbe → 无符号比较 → 变量可能是unsigned
; 看到 movsx → 有符号扩展 → 源操作数是有符号类型
; 看到 movzx → 无符号扩展 → 源操作数是无符号类型
```

#### 陷阱4：大端序vs小端序

```text
; x86/x64是小端序（Little-Endian）
; 在内存中看到 78 56 34 12 → 实际值是 0x12345678
; ARM默认小端，但某些嵌入式配置为大端
; 大端序：0x12345678 在内存中存储为 12 34 56 78
```

#### 陷阱5：对齐导致的"垃圾数据"

```asm
; 编译器为了对齐会插入nop或填充字节
nop                          ; 用于函数对齐到16字节边界
; 有时用 lea esi, [esi] 等"假指令"代替nop
; 这些不是有效逻辑，不要分析其"功能"
```

#### 陷阱6：异常处理代码的干扰

```asm
; SEH/SEH64的注册和清理代码会出现在函数体中
; 不要将这些代码当作正常的业务逻辑
; 识别特征：
; 32位：fs:[0]的读写
; 64位：.pdata段的RUNTIME_FUNCTION条目
; C++：__try/__except/__finally 的编译结果
```

#### 陷阱7：Tail Call优化

```asm
; 编译器可能将函数末尾的调用优化为跳转
; 原始代码：
; int foo(int x) { ... return bar(x); }
; 
; 优化后（Tail Call）：
; jmp bar          ← 用jmp代替call，节省一次ret
; 
; 这会导致反编译器可能将两个函数合并为一个
; 识别方法：函数末尾是jmp而不是call+ret组合
```

### 17.2.11 本节小结

本节系统讲解了逆向工程所需的汇编语言基础：
