# 核心技巧

理论是地基，技巧是高楼。在掌握了进程与线程的核心概念（PCB、状态机、上下文切换、线程模型、CFS调度器）之后，本节聚焦于**"怎么用好"**——将理论知识转化为工程实践中的关键能力。

## 本节定位

"核心技巧"是第04章从理论走向实战的桥梁。它不重复理论基础中的概念定义，而是回答三个关键问题：

1. **怎么写** —— 进程创建、线程同步、协程调度的标准代码模式和最佳实践
2. **怎么快** —— 上下文切换优化、锁策略选择、并发性能调优的系统性方法
3. **怎么排** —— 死锁检测、竞态条件排查、性能瓶颈定位的实战工具链

## 阅读建议

### 初学者路径（由浅入深）

1. **先读技巧一的第1-2节**：掌握进程创建（fork/spawn）和线程同步（Lock/Condition/Semaphore）的基本操作，这是日常开发的高频场景
2. **跳到技巧一的第3节**：理解协程的本质（用户态调度）和 asyncio 的基本用法
3. **再读技巧一的第4节**：掌握并发模型的选择逻辑——什么时候用多进程、多线程还是协程
4. **然后读技巧二的第1-2节**：学会量化上下文切换开销，理解不同锁策略的适用场景
5. **最后读技巧二的第5-7节**：通过实战案例巩固知识，了解 Amdahl 定律等方法论

### 进阶读者路径

如果你已经熟悉基本操作，可以直接进入技巧二的深度内容：

- **技巧二第3.4节（伪共享）**：这是一个经常被忽视但影响巨大的性能杀手，缓存行填充可以带来 5-20 倍的性能提升
- **技巧二第2.3节（锁策略决策树）**：面对具体场景时，用这个决策树快速选对锁
- **技巧二第7节（Amdahl定律）**：理解并行优化的理论上限，避免无谓的核心数堆砌

## 两大技巧的关系

技巧一和技巧二不是孤立的两个文件，而是一个渐进式的学习路径：

技巧一（基础操作）              技巧二（性能优化）
┌──────────────────┐          ┌──────────────────┐
│ 进程创建          │    →     │ 上下文切换开销    │
│ 线程同步          │    →     │ 锁策略选型        │
│ 协程调度          │    →     │ 异步IO调优        │
│ 调试工具          │    →     │ 性能监控工具箱    │
│ 误区避坑          │    →     │ 优化方法论        │
└──────────────────┘          └──────────────────┘
    "会用"                          "用好"
    How to use                     How to optimize

技巧一确保你能**正确地**创建进程、同步线程、调度协程；技巧二确保你能**高效地**使用这些能力——量化开销、选对策略、定位瓶颈、验证效果。
