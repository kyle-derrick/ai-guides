## 17.5 类型系统与数据结构

高级语言中的类型信息在编译后几乎完全丢失——反汇编器看到的只是一条条指令和连续的内存地址。逆向工程的核心技能之一，就是从这些无类型的字节流中还原出原始的数据结构和类型信息。本节系统讲解各类数据类型在内存中的表示方式，以及在逆向分析中识别和重建它们的方法。

### 17.5.1 基本类型在内存中的表示

#### 整数类型

整数类型根据是否带符号，采用不同的编码方式：

| 类型 | 大小（x86） | 大小（x64） | 编码方式 | 值域 |
|------|------------|------------|---------|------|
| `char` | 1字节 | 1字节 | 无符号或有符号（实现定义） | 0~255 或 -128~127 |
| `unsigned char` | 1字节 | 1字节 | 原码 | 0~255 |
| `short` | 2字节 | 2字节 | 补码（有符号） | -32768~32767 |
| `unsigned short` | 2字节 | 2字节 | 原码 | 0~65535 |
| `int` | 4字节 | 4字节 | 补码（有符号） | -2³¹~2³¹-1 |
| `unsigned int` | 4字节 | 4字节 | 原码 | 0~2³²-1 |
| `long` | 4字节 | 8字节 | 补码 | 平台相关 |
| `long long` | 8字节 | 8字节 | 补码 | -2⁶³~2⁶³-1 |
| `pointer` | 4字节 | 8字节 | 无符号整数 | 虚拟地址空间 |

**字节序（Endianness）** 决定了多字节值在内存中的存储顺序。x86/x64 采用小端序（Little-Endian），即低位字节存放在低地址：

```c
uint32_t value = 0x12345678;
// 内存布局（低地址 → 高地址）：78 56 34 12
```

在反汇编中识别整数的方法：
- **立即数加载**：`mov eax, 42` 直接暴露常量值
- **内存加载宽度**：`movzx`（零扩展）暗示无符号，`movsx`（符号扩展）暗示有符号
- **比较指令**：`ja/jb` 用于无符号比较，`jg/jl` 用于有符号比较——这是区分有符号与无符号类型的关键线索
- **除法指令**：`div` 是无符号除法，`idiv` 是有符号除法

#### 浮点类型

浮点数采用 IEEE 754 标准：

| 类型 | 大小 | 符号位 | 指数位 | 尾数位 | 精度 |
|------|------|--------|--------|--------|------|
| `float` | 4字节 | 1位 | 8位 | 23位 | ~7位十进制 |
| `double` | 8字节 | 1位 | 11位 | 52位 | ~15位十进制 |
| `long double` | 10/16字节 | 1位 | 15/15位 | 64/112位 | ~18/33位十进制 |

浮点数的编码公式：`(-1)^sign × 1.mantissa × 2^(exponent - bias)`

在反汇编中识别浮点数的线索：
- 使用 XMM 寄存器（`movss`/`movsd`/`addss`/`mulsd` 等 SSE 指令）
- 使用 x87 FPU 栈（`fld`/`fstp`/`fadd` 等以 `f` 开头的指令）
- `cvtss2sd`/`cvtsd2ss` 是 float↔double 转换
- `cvtsi2ss`/`cvtsd2si` 是整数↔浮点转换

#### 常量的逆向识别模式

```nasm
; 整数常量：直接出现在立即数中
mov     dword ptr [rbp-4], 100      ; int x = 100

; 浮点常量：通常从 .rodata 殀加载，或通过 __xmm@ 前缀的全局变量
movss   xmm0, dword ptr [rip + .LC0] ; float pi = 3.14f
; .LC0: .long 0x4048f5c3   （IEEE 754 编码的 3.14）

; 字符串常量：lea 加载 .rodata 地址
lea     rdi, [rip + .LC1]           ; "Hello, World!\n"
```

### 17.5.2 结构体的内存布局

#### 对齐与填充（Padding）

结构体成员按声明顺序存放，但每个成员的起始地址必须是其自身大小的整数倍（对齐要求）。编译器在成员之间插入填充字节以满足对齐：

```c
struct Example {
    char a;       // 偏移 0，1字节
    // 3字节填充（使 b 对齐到4字节边界）
    int b;        // 偏移 4，4字节
    char c;       // 偏移 8，1字节
    // 7字节填充（使 d 对齐到8字节边界）
    long long d;  // 偏移 16，8字节
};
// sizeof = 24字节
```

对比调整声明顺序后的效果：

```c
struct Optimized {
    long long d;  // 偏移 0，8字节
    int b;        // 偏移 8，4字节
    char a;       // 偏移 12，1字节
    char c;       // 偏移 13，1字节
    // 2字节尾部填充（使结构体大小为8的倍数）
};
// sizeof = 16字节（节省了8字节）
```

**结构体大小** 必须是最大成员对齐值的整数倍。`Example` 中最大对齐值为 8（来自 `long long`），所以总大小 24 正好是 8 的倍数。

#### 紧凑结构体（Packed Struct）

使用 `#pragma pack(1)` 或 `__attribute__((packed))` 可以取消填充，使结构体紧凑排列：

```c
#pragma pack(push, 1)
struct Packed {
    char a;       // 偏移 0，1字节
    int b;        // 偏移 1，4字节（无对齐要求）
    char c;       // 偏移 5，1字节
    long long d;  // 偏移 6，8字节
};
#pragma pack(pop)
// sizeof = 14字节
```

Packed 结构体在协议解析、文件格式解析中很常见。逆向识别特征：成员访问使用非对齐指令（如 `mov eax, dword ptr [rdx+1]`），或者编译器插入额外的字节拷贝来处理非对齐访问。

#### 位域（Bitfield）

位域允许在单个字节或字中定义多个小范围的字段：

```c
struct Flags {
    unsigned int enabled  : 1;   // bit 0
    unsigned int mode     : 3;   // bits 1-3
    unsigned int priority : 4;   // bits 4-7
    unsigned int reserved : 24;  // bits 8-31
};
// sizeof = 4字节
```

位域在逆向中的识别方法：
- `and` 指令配合掩码提取位域值：`and eax, 0x0F` 提取低4位
- `shl`/`shr` 指令移位后配合 `or`/`and` 设置或读取位域
- 常见掩码值（0x1, 0x3, 0x7, 0xF, 0xFF, 0xFFFF 等）暗示位域操作

#### 在逆向中重建结构体

在 IDA Pro/Ghidra 中重建结构体的步骤：

1. **确定大小**：观察 `malloc`/`new` 的大小参数，或栈帧中 `sub rsp, N` 的 N
2. **识别成员偏移**：记录所有 `[base + offset]` 形式的访问，偏移即为成员在结构体中的位置
3. **推断成员类型**：
   - 8字节偏移对齐 + 被用作 `call [rax+offset]` → 函数指针或 vtable 指针
   - 4字节值参与算术运算 → `int`
   - 4字节值与浮点指令交互 → `float`
   - 值被传递给字符串函数（`strlen`/`strcpy`/`printf %s`）→ `char*`
   - 值被用作 `malloc` 的参数 → `size_t`
4. **在 IDA 中创建结构体**：`Structures` 窗口 → `Create struct`，逐步添加成员，然后将局部变量的类型设为该结构体指针

```python
# IDAPython 自动分析结构体访问模式
import idautils, idc

def find_struct_accesses(func_addr):
    """扫描函数中所有基址+偏移形式的内存访问"""
    accesses = {}
    for head in idautils.Heads(func_addr, idc.find_func_end(func_addr)):
        mnem = idc.print_insn_mnem(head)
        for i in range(2):
            op = idc.get_operand_value(head, i)
            if 0 < op < 0x1000:  # 合理的结构体偏移范围
                if op not in accesses:
                    accesses[op] = []
                accesses[op].append(hex(head))
    return accesses
```

### 17.5.3 C++ 类的内存布局

#### 虚函数表（vtable）机制

C++ 虚函数通过虚函数表实现多态。每个含虚函数的类有一个 vtable（存储在 `.rodata` 段），每个含虚函数的对象在头部有一个 vptr（虚表指针）：

```cpp
class Animal {
public:
    virtual void speak() = 0;
    virtual ~Animal() {}
    int id;
};
// sizeof(Animal) = 16（8字节vptr + 4字节id + 4字节填充）

class Dog : public Animal {
public:
    void speak() override { /* bark */ }
    virtual void fetch();
    int tricks;
};
// sizeof(Dog) = 24（8字节vptr + 4字节id + 4字节tricks偏移对齐 + 8字节tricks对齐）
```

内存布局（Dog 对象）：

```text
偏移 0:  vptr → Dog::vtable
偏移 8:  id (int)
偏移 12: (4字节填充)
偏移 16: tricks (int)
偏移 20: (4字节填充)
```

Dog 的 vtable 内容：

```text
vtable[0]: Dog::~Dog()      // 虚析构函数（complete）
vtable[1]: Dog::~Dog()      // 虚析构函数（deleting）
vtable[2]: Dog::speak()     // 覆盖的虚函数
vtable[3]: Dog::fetch()     // 新增的虚函数
```

#### 多重继承的内存布局

```cpp
class Flyable {
public:
    virtual void fly();
    int altitude;
};

class Swimmable {
public:
    virtual void swim();
    int depth;
};

class Duck : public Flyable, public Swimmable {
public:
    void fly() override;
    void swim() override;
    int color;
};
```

Duck 的内存布局：

```text
偏移 0:  vptr1 → Duck的Flyable子对象vtable
偏移 8:  altitude (int)
偏移 12: (4字节填充)
偏移 16: vptr2 → Duck的Swimmable子对象vtable
偏移 24: depth (int)
偏移 28: color (int)
偏移 32: (尾部填充)
// sizeof(Duck) = 32
```

关键特征：多重继承时，派生类对象中有**多个 vptr**。第二个及后续基类子对象的 vtable 中，虚函数指针会经过 thunk 调整 `this` 指针：

```nasm
; thunk 示例：调整 this 指针以指向正确的基类子对象
_Duck::swim thunk:
    sub     rdi, 16          ; this -= 16，从 Flyable 子对象调整到 Swimmable 子对象
    jmp     Duck::swim()     ; 跳转到实际实现
```

#### 虚继承的内存布局

虚继承用于解决菱形继承问题，引入了 `vbase offset` 机制：

```cpp
class A { public: int x; virtual ~A(); };
class B : virtual public A { public: int y; virtual ~B(); };
class C : virtual public A { public: int z; virtual ~C(); };
class D : public B, public C { public: int w; virtual ~D(); };
```

虚继承的对象布局更加复杂：
- vtable 中增加了 `vbase offset` 条目，用于运行时定位虚基类子对象
- 每个虚继承的基类子对象不按声明顺序排列，而是放在对象的末尾
- 构造函数中需要额外的代码来设置 vbase offset

#### RTTI（运行时类型信息）

GCC/Clang 的 RTTI 结构体：

```text
typeinfo 对象（在 .rodata 中）：
  偏移 0:  vptr → __cxxabiv1::__si_class_type_info 的 vtable（单继承）
           或 → __cxxabiv1::__vmi_class_type_info 的 vtable（多继承/虚继承）
  偏移 8:  指向类名字符串的指针（如 "4Duck" 或 "3Dog"）

si_class_type_info 额外字段：
  偏移 16: 指向基类 typeinfo 的指针

vmi_class_type_info 额外字段：
  偏移 16: flags（__vmi_class_type_info 的标志位）
  偏移 20: base_count（基类数量）
  偏移 24: base_info 数组（每个包含 offset_flags + base_typeinfo_ptr）
```

在逆向中识别 RTTI 的方法：
- IDA 中查看 vtable 上方的 `off_XXXXXXX` 引用，通常指向 typeinfo 结构
- 类名字符串（带长度前缀，如 `"4Duck"`）出现在 `.rodata` 段
- `typeid()` 和 `dynamic_cast` 的实现依赖 RTTI

#### 在逆向中重建 C++ 类层次

```python
# IDAPython：扫描 vtable 结构
def dump_vtable(vtable_addr, count=10):
    """打印 vtable 中的函数指针"""
    print(f"vtable @ {hex(vtable_addr)}:")
    for i in range(count):
        func_ptr = idc.get_qword(vtable_addr + i * 8)
        if func_ptr == 0:
            break
        name = idc.get_func_name(func_ptr)
        print(f"  [{i}] {hex(func_ptr)} : {name}")
```

重建步骤：
1. **定位 vtable**：搜索 `.rodata` 段中连续的函数指针数组（每个元素都是有效函数地址）
2. **找到 vptr 初始化**：构造函数中的 `mov qword ptr [rax], offset vtable_X` 语句
3. **确定继承关系**：比较不同类的 vtable——如果 vtable B 的前 N 项与 vtable A 完全相同，说明 B 继承自 A
4. **在工具中创建类结构**：在 IDA 的 Structures 窗口创建结构体，在 Local Types 中定义类

### 17.5.4 数组与指针

#### 数组的内存布局

数组在内存中是连续存储的元素序列，没有额外的元数据（不像 Java/C# 的数组有长度信息）：

```c
int arr[5] = {10, 20, 30, 40, 50};
// 内存布局（小端序）：
// 0A 00 00 00  14 00 00 00  1E 00 00 00  28 00 00 00  32 00 00 00
//   arr[0]        arr[1]        arr[2]        arr[3]        arr[4]
```

数组访问在汇编中的模式：

```nasm
; arr[i]（int数组，基址在rbx，索引在rcx）
mov     eax, [rbx + rcx*4]    ; eax = arr[i]，4是sizeof(int)

; 二维数组 arr[row][col]（int[3][4]）
; arr[row][col] 的地址 = base + (row * 4 + col) * 4
mov     eax, [rbx + rdx*4]    ; rdx = row * 4 + col
```

**识别数组的线索**：
- 循环中出现 `base + index * scale` 形式的地址计算
- `scale` 值为 1/2/4/8，对应 char/short/int/long long 或指针
- `rep movsb`/`rep stosb` 可能是 `memcpy`/`memset` 操作数组
- `malloc(n * sizeof(T))` 或 `calloc(n, sizeof(T))` 明确暗示数组分配

#### 指针与指针运算

指针的本质是一个存储地址的整数。在逆向中，指针和数组的边界往往模糊，但有一些判断方法：

```nasm
; 单个指针解引用：通常没有乘法缩放
mov     rax, [rbp-8]          ; 加载指针值
mov     eax, [rax]            ; 解引用

; 数组遍历：指针递增或索引递增
; 方式1：指针递增
add     qword ptr [rbp-8], 4  ; ptr++

; 方式2：索引递增
inc     dword ptr [rbp-4]     ; i++
mov     eax, [rbx + rcx*4]   ; arr[i]
```

#### 多级指针

多级指针（如链表、树、哈希表桶）在逆向中很常见：

```nasm
; 三级指针访问：node->next->data->value
mov     rax, [rbp-8]          ; node
mov     rax, [rax+8]          ; node->next（偏移8暗示next在结构体偏移8处）
mov     rax, [rax+16]         ; node->next->data（偏移16）
mov     eax, [rax]            ; node->next->data->value
```

识别链式数据结构的特征：循环中出现 `[reg + const]` 形式的指针跟随，且循环条件是与 NULL 比较。

### 17.5.5 字符串类型

#### C 风格字符串

以 null（`\0`）结尾的字符数组，是 C 语言中最基本的字符串类型：

```c
char str[] = "Hello";   // 6字节：'H','e','l','l','o','\0'
char *p = "World";      // 指向只读段中的字符串常量
```

在反汇编中识别字符串：
- `lea rdi, [rip + .LC0]` 加载字符串地址，随后调用 `puts`/`printf`/`strlen`
- 连续的 ASCII 字节（可打印字符范围 0x20~0x7E）在 `.rodata` 或 `.data` 段
- IDA 自动将连续的可打印字节识别为字符串并标注

#### 宽字符串（Wide String）

使用 `wchar_t`（Linux 上 4字节/UTF-32，Windows 上 2字节/UTF-16）：

```c
wchar_t wstr[] = L"你好";  // Linux: 10字节（2个字符×4字节 + 4字节null）
                            // Windows: 6字节（2个字符×2字节 + 2字节null）
```

识别特征：`wprintf`/`wcslen` 等宽字符函数，或 `mov word ptr` 以2为步长递增访问字符。

#### C++ std::string 的内部结构

GCC libstdc++ 的 `std::string`（COW 实现，GCC < 5）和 SSO 实现（GCC ≥ 5）差异很大：

**GCC ≥ 5 的 SSO 实现**：

```cpp
struct basic_string {
    union {
        struct {                    // 长字符串（> 15字节）
            char*  _M_data;         // 偏移 0，堆上的数据指针
            size_t _M_size;         // 偏移 8，字符串长度
            size_t _M_capacity;     // 偏移 16，分配容量
        };
        struct {                    // 短字符串（≤ 15字节，SSO）
            unsigned char _M_size;  // 偏移 0，长度（实际存在最低字节）
            char _M_local_buf[15];  // 偏移 1~15，内联存储
        };
    };
};
// sizeof = 24字节（x64）
```

SSO 的判断：检查偏移 0 处的最低位——如果 `_M_size` 的值 ≤ 15 且数据直接存储在对象内部（不通过指针），则是 SSO 模式。在 IDA 中，如果看到对 string 对象本身地址（而非堆地址）的字符串操作，说明触发了 SSO。

**libc++（Clang）的 std::string**：

```cpp
struct basic_string {
    union {
        struct {                    // 长字符串
            size_t _M_cap;          // 偏移 0，容量（含标志位）
            size_t _M_size;         // 偏移 8，长度
            char*  _M_data;         // 偏移 16，数据指针
        };
        struct {                    // 短字符串
            unsigned char _M_size;  // 偏移 0
            char _M_buf[23];        // 偏移 1~23
        };
    };
};
// sizeof = 24字节（x64）
```

#### Windows 特有字符串

| 字符串类型 | 描述 | 终止方式 |
|-----------|------|---------|
| `LPSTR` / `char*` | ANSI 字符串 | null 结尾 |
| `LPWSTR` / `wchar_t*` | Unicode 字符串（UTF-16） | 双 null 结尾 |
| `BSTR` | COM 字符串，前4字节为长度 | null 结尾但通过长度访问 |
| `UNICODE_STRING` | NT 内核结构，包含长度和缓冲区指针 | 长度字段 |
| `ANSI_STRING` | NT 内核 ANSI 字符串 | 长度字段 |

```c
typedef struct _UNICODE_STRING {
    USHORT Length;        // 当前长度（字节）
    USHORT MaximumLength; // 缓冲区最大长度
    PWSTR  Buffer;        // 指向 UTF-16 字符串
} UNICODE_STRING;
// sizeof = 16字节（x64）
```

### 17.5.6 枚举与联合体

#### 枚举（Enum）

C 枚举本质是整数常量。编译后枚举值被替换为整数，类型信息完全丢失：

```c
enum Color { RED = 0, GREEN = 1, BLUE = 2 };
enum Color c = GREEN;
// 编译后等价于：int c = 1;
```

逆向中识别枚举的方法：
- 一组互斥的整数比较：`cmp eax, 0` / `je label` ... `cmp eax, 1` / `je label`
- `switch` 语句的跳转表（jump table）中连续的 case 值
- IDA 中可以创建 `enum` 类型并手动应用

#### 联合体（Union）

联合体的所有成员共享同一段内存，大小等于最大成员的大小：

```c
union Value {
    int i;          // 4字节
    float f;        // 4字节
    char str[8];    // 8字节
};
// sizeof = 8字节
```

联合体在逆向中的识别非常困难，因为同一地址可能以不同类型被访问。线索是：同一内存位置在不同代码路径中被用作不同类型（如某处作为整数参与算术，另一处被传递给浮点指令）。

### 17.5.7 标准容器的内存布局

#### std::vector

GCC libstdc++ 的 `std::vector` 内部结构：

```cpp
struct vector {
    T*     _M_start;      // 偏移 0，数据起始指针
    T*     _M_finish;     // 偏移 8，已使用区域的末尾
    T*     _M_end_of_storage; // 偏移 16，已分配空间的末尾
};
// sizeof = 24字节（x64），与元素类型无关
// size = _M_finish - _M_start
// capacity = _M_end_of_storage - _M_start
```

识别特征：三个紧密相关的指针，通过 `sub` 指令计算 `size` 或 `capacity`。

#### std::map / std::set（红黑树）

GCC libstdc++ 的 `std::map` 基于红黑树实现：

```cpp
struct _Rb_tree {
    _Rb_tree_node* _M_header;  // 偏移 0，哨兵节点指针
    size_t         _M_node_count; // 偏移 8，节点数量
};
```

红黑树节点结构：

```cpp
struct _Rb_tree_node {
    bool           _M_color;     // 偏移 0（实际占用更大对齐空间）
    _Rb_tree_node* _M_parent;    // 偏移 8
    _Rb_tree_node* _M_left;      // 偏移 16
    _Rb_tree_node* _M_right;     // 偏移 24
    // _M_value_type 在偏移 32 或之后
};
```

识别特征：递归的三指针结构（parent/left/right），遍历代码中包含与 `nil` 或哨兵节点的比较。

#### std::unordered_map（哈希表）

GCC libstdc++ 的 `std::unordered_map` 使用桶数组+链表的结构：

```cpp
struct unordered_map {
    _Hash_node**   _M_buckets;    // 桶数组指针
    size_t         _M_bucket_count;
    size_t         _M_element_count;
    float          _M_max_load_factor;
    // ...
};
```

识别特征：桶数组的索引计算（`hash % bucket_count`），桶中链表的遍历。

### 17.5.8 常见数据结构的逆向识别

#### 链表

```nasm
; 单链表遍历的典型模式
.loop:
    test    rax, rax            ; 检查指针是否为 NULL
    jz      .done
    ; ... 处理当前节点 ...
    mov     rax, [rax+0]        ; rax = node->next（偏移0暗示next是第一个成员）
    jmp     .loop
.done:
```

#### 二叉树

```nasm
; 二叉树递归遍历（前序）的典型模式
traverse:
    test    rdi, rdi            ; if (node == NULL)
    jz      .return
    ; ... 访问 node->data（偏移由具体结构决定）...
    mov     rdi, [rdi+8]        ; traverse(node->left)
    call    traverse
    mov     rdi, [rbx+16]       ; traverse(node->right)
    call    traverse
.return:
    ret
```

#### 哈希表

```nasm
; 哈希表查找的典型模式
    ; 计算 hash
    mov     eax, edi            ; key
    ; ... hash 函数计算 ...
    xor     edx, edx
    div     dword ptr [rsi+8]   ; hash % bucket_count
    ; 在桶中遍历
    mov     rax, [rsi]          ; buckets 数组
    mov     rax, [rax+rdx*8]    ; buckets[bucket_index]
    ; ... 链表遍历比较 key ...
```

# Ghidra Python 脚本：自动推断结构体

def infer_struct(base_addr, accesses):
    """根据内存访问模式创建结构体"""

### 17.5.10 类型系统与安全漏洞

类型混淆（Type Confusion）是一类重要的安全漏洞，根源在于类型系统信息丢失后对内存的错误解释：

#### Use-After-Free 与类型混淆

```cpp
class Base { virtual void f(); };
class Derived : public Base { int secret; void f() override; };

Base* obj = new Derived();
delete obj;                  // 析构，vptr 指向 Derived::vtable
// ... 内存被重分配为其他类型 ...
Base* obj2 = new UnrelatedClass();  // 可能重用同一块内存
obj->f();  // Use-After-Free：vptr 已被覆盖，调用不可预测的地址
```

#### 整数溢出导致的类型问题

```c
void process(char *buf, int len) {
    // len 为负数时（如 -1），len + HEADER_SIZE 可能变为小的正数
    char *payload = malloc(len + HEADER_SIZE);
    memcpy(payload, buf, len);  // 实际拷贝远超分配的大小 → 堆溢出
}
```

#### 虚表劫持

通过堆溢出或 UAF 覆盖对象的 vptr，控制虚函数调用目标：

```nasm
; 正常虚函数调用
mov     rax, [rcx]            ; rax = obj->vptr
call    [rax+16]              ; call vtable[2]（虚函数f）

; 攻击后：vptr 被篡改指向攻击者控制的假 vtable
mov     rax, [rcx]            ; rax = fake_vptr（指向攻击者内存）
call    [rax+16]              ; call gadget_addr → ROP/code execution
```

防御措施：CFI（Control-Flow Integrity）通过验证虚函数调用目标是否在合法 vtable 范围内来防止此类攻击。Clang 的 `-fsanitize=cfi-vcall` 和微软的 CFG（Control Flow Guard）是主流实现。

### 17.5.11 本节小结

类型系统与数据结构是逆向工程的知识基石。编译器将高级语言的丰富类型信息压缩为无类型的字节流，逆向分析者的任务就是反向还原这一过程。

关键要点：

- **基本类型**通过指令宽度、符号扩展方式和比较/除法指令的变体来识别
- **结构体**通过 `[base + offset]` 的访问模式和 `malloc` 大小来重建
- **C++ 类**通过 vtable 指针和构造函数初始化代码来识别继承层次
- **字符串**通过函数调用（`strlen`/`printf %s`）和内存中的可打印字节来发现
- **标准容器**的内部结构虽因实现而异，但都有可识别的模式（三指针 vector、红黑树、桶数组+链表）
- **类型混淆**漏洞提醒我们：类型信息丢失不仅是逆向的困难，也是安全风险的来源

掌握这些知识后，面对反汇编代码时，你看到的不再是孤立的寄存器和内存操作，而是有组织的数据结构和清晰的程序逻辑。
