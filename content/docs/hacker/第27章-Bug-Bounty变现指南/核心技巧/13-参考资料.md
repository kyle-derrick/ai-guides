# 参考资料：Bug Bounty 猎人必备知识库

本页汇总了 Bug Bounty 全流程中需要频繁查阅的核心参考资料——从漏洞分类标准到平台规则、从安全工具到法律合规、从技术文献到社区资源。建议收藏本页，作为日常狩猎的常备索引。

## 一、漏洞分类与标准体系

Bug Bounty 报告中引用标准化的漏洞编号（CWE/CVE）能大幅提升专业度和可信度，也让安全团队更快定位问题。

### 1.1 CWE（通用缺陷枚举）

CWE 是 MITRE 维护的软件缺陷分类标准，是 Bug Bounty 报告中引用率最高的标准体系。

| CWE 编号 | 漏洞类型 | 与 Bug Bounty 的关系 | 赏金参考 |
|----------|---------|---------------------|---------|
| CWE-79 | 跨站脚本（XSS） | 最常见的 Web 漏洞之一，几乎每个平台都有赏金 | 中低（$100-$2,000） |
| CWE-89 | SQL 注入 | 高危漏洞，可能导致数据泄露 | 高（$500-$10,000+） |
| CWE-918 | 服务器端请求伪造（SSRF） | 云环境/微服务中价值极高 | 高（$1,000-$15,000） |
| CWE-22 | 路径遍历 | 可能导致敏感文件泄露 | 中（$200-$3,000） |
| CWE-352 | 跨站请求伪造（CSRF） | 影响状态修改操作 | 中低（$100-$1,500） |
| CWE-287 | 身份验证不当 | 逻辑漏洞，影响账户安全 | 中高（$300-$8,000） |
| CWE-434 | 不受限制的文件上传 | 可能导致远程代码执行 | 高（$500-$10,000） |
| CWE-502 | 不安全的反序列化 | 可能导致 RCE | 极高（$2,000-$25,000） |
| CWE-601 | URL 重定向不当 | 钓鱼攻击入口 | 低中（$50-$1,000） |
| CWE-611 | XML 外部实体注入（XXE） | 可能导致 SSRF 或文件读取 | 中高（$500-$5,000） |
| CWE-776 | XML 炮弹（Billion Laughs） | 拒绝服务攻击 | 低（$50-$500） |
| CWE-917 | 表达式语言注入（ELI） | Java/Spring 应用中可 RCE | 高（$1,000-$10,000） |
| CWE-1321 | 原型链污染（Prototype Pollution） | Node.js 应用中可 RCE | 中高（$500-$8,000） |

**CWE 完整列表：** https://cwe.mitre.org/data/definitions.html

> **实战建议：** 在报告中同时引用 CWE 编号和 OWASP 分类，能让安全团队用两种视角理解你的发现。例如：「本漏洞属于 CWE-79（XSS），对应 OWASP Top 10 2021 A07:2021 - Identification and Authentication Failures 中的会话劫持攻击向量。」

### 1.4 CVSS 评分参考

CVSS（通用漏洞评分系统）是量化漏洞严重程度的标准。理解 CVSS 评分有助于在报告中准确定位漏洞影响。

| 评分区间 | 等级 | 赏金影响 |
|---------|------|---------|
| 0.1 - 3.9 | 低危 | 可能不收、赏金最低 |
| 4.0 - 6.9 | 中危 | 基础赏金范围 |
| 7.0 - 8.9 | 高危 | 赏金上浮 50%-200% |
| 9.0 - 10.0 | 严重 | 赏金翻倍，可能有额外奖励 |

**CVSS 3.1 计算器：** https://www.first.org/cvss/calculator/3.1

### 3.4 Nuclei 模板库

Nuclei 是当前 Bug Bounty 社区最受欢迎的自动化扫描工具之一，其核心价值在于社区维护的模板库。

| 模板库 | 维护者 | 用途 | 地址 |
|--------|-------|------|------|
| nuclei-templates | ProjectDiscovery | 官方模板，覆盖常见漏洞 | https://github.com/projectdiscovery/nuclei-templates |
| template-hub | 社区 | 社区贡献的自定义模板 | GitHub 搜索 nuclei-templates |

**快速上手：**
```bash
# 安装 nuclei
go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest

# 更新模板库
nuclei -update-templates

# 扫描目标
nuclei -l urls.txt -t nuclei-templates/ -severity critical,high

# 按漏洞类型扫描
nuclei -u https://target.com -t nuclei-templates/cves/
```

### 6.2 合规检查清单

在开始任何 Bug Bounty 活动之前，务必确认以下事项：

```text
□ 1. 目标在平台范围内（Scope）
     - 确认域名/IP/应用在授权范围内
     - 确认测试方法在允许列表中
     - 确认不在排除范围（Out of Scope）内

□ 2. 平台规则已读
     - 阅读并理解 Program Policy
     - 了解禁止的测试行为（如社工、DoS、物理测试）
     - 了解数据处理要求（如不得下载真实用户数据）

□ 3. 法律风险评估
     - 本国法律对未授权测试的定义
     - 确保测试行为不违反任何法律条款
     - 了解跨境测试的法律风险

□ 4. 数据保护
     - 不下载或存储真实用户数据
     - 发现敏感数据时立即报告，不扩散
     - 漏洞证明中使用脱敏数据
     - 遵守 GDPR/PIPL 等数据保护法规

□ 5. 报告伦理
     - 发现严重漏洞时负责任地披露
     - 不在漏洞修复前公开披露
     - 不向第三方出售漏洞信息（除非平台允许）
```

## 七、漏洞报告模板资源

### 7.1 HackerOne 报告模板参考

HackerOne 提供了标准化的报告模板，可作为编写报告的基础框架：

```markdown
## 概要（Summary）
[简述漏洞类型、影响范围和严重程度]

## 漏洞详情（Vulnerability Details）
[详细描述漏洞的技术原理和攻击路径]

## 复现步骤（Steps to Reproduce）
1. [精确的复现步骤]
2. ...
3. ...

## 预期结果（Impact）
[说明漏洞可造成的实际影响]

## 证据（Evidence）
[截图/PoC/请求响应包等证据材料]

## 修复建议（Remediation）
[可操作的修复建议]
```

### 7.2 CVSS 评分计算器

在报告中使用准确的 CVSS 评分能帮助安全团队正确评估优先级：

**CVSS 3.1 计算器：** https://www.first.org/cvss/calculator/3.1

**CVSS 向量字符串示例：**
```text
# 高危 XSS（需要用户交互）
```
