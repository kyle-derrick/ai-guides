## 五、自动化工具链构建

在渗透测试和安全竞赛中，效率就是生命线。一个熟练的安全工程师如果手动执行每一个命令，其产出可能只有自动化流水线的十分之一。自动化工具链构建不是"偷懒"，而是将重复性的信息收集、扫描、报告生成交给机器，让人类专注于需要创造性思维的漏洞发现和利用环节。本节将从理论基础出发，逐步构建一套完整的、可复用的自动化渗透测试工具链。

### 5.1 自动化的核心理念与架构设计

#### 5.1.1 为什么需要自动化

手动渗透测试面临三个核心瓶颈：

| 瓶颈类型 | 手动操作的问题 | 自动化解决方案 |
|---------|--------------|--------------|
| **时间成本** | 全端口扫描+目录枚举+子域名发现需要2-4小时手动等待和切换 | 串行/并行调度，一键启动全流程 |
| **可复现性** | 每次操作步骤可能不同，遗漏关键检查项 | 标准化流程，确保每次执行相同步骤 |
| **知识遗忘** | 忘记某个工具的参数或组合方式 | 工具链封装最佳实践，降低认知负担 |

在CTF竞赛场景下，时间通常为24-48小时。如果前2小时的信息收集阶段就能通过自动化完成，后续的漏洞分析和利用环节就能获得更多时间。在真实渗透测试项目中，自动化更是确保报告完整性和合规性的关键手段。

#### 5.1.2 工具链架构模型

一个成熟的自动化工具链遵循分层架构：

```text
┌─────────────────────────────────────────┐
│           报告生成层（Report Layer）       │
│    HTML/PDF报告、Jira工单、Slack通知      │
├─────────────────────────────────────────┤
│           分析聚合层（Analysis Layer）     │
│    结果去重、风险评分、交叉关联            │
├─────────────────────────────────────────┤
│           任务编排层（Orchestration Layer）│
│    任务调度、依赖管理、并行控制            │
├─────────────────────────────────────────┤
│           执行引擎层（Execution Layer）    │
│    Nmap/Nuclei/Gobuster/自定义PoC       │
├─────────────────────────────────────────┤
│           基础设施层（Infrastructure Layer）│
│    Docker容器、代理池、VPN链路、日志存储    │
└─────────────────────────────────────────┘
```

每一层都有独立的职责边界，上层依赖下层的输出，下层不感知上层的存在。这种松耦合设计意味着你可以随时替换某个扫描工具而不影响整体流程。

### 5.2 信息收集自动化

信息收集是渗透测试的第一步，也是自动化收益最高的环节。手动执行Nmap全端口扫描、子域名枚举、目录爆破、指纹识别等操作需要频繁切换工具和查看结果，自动化可以将这些步骤串联为一个无缝流程。

#### 5.2.1 综合侦察脚本

```bash
#!/bin/bash
# auto-recon.sh — 自动化信息收集框架
# 用法: ./auto-recon.sh <target_ip_or_domain> [output_dir]

set -euo pipefail

# ==================== 配置区 ====================
TARGET="${1:?用法: $0 <target> [output_dir]}"
OUTPUT_DIR="${2:-recon_$(date +%Y%m%d_%H%M%S)_$(echo $TARGET | tr '.' '_')}"
THREADS=20
TIMEOUT=300  # 单个工具超时时间（秒）
LOG_FILE="$OUTPUT_DIR/scan.log"

# 工具路径配置（支持自定义路径）
NMAP_BIN=$(command -v nmap 2>/dev/null || echo "/usr/bin/nmap")
GOBUSTER_BIN=$(command -v gobuster 2>/dev/null || echo "/usr/bin/gobuster")
SUBFINDER_BIN=$(command -v subfinder 2>/dev/null || echo "$HOME/go/bin/subfinder")
HTTPX_BIN=$(command -v httpx 2>/dev/null || echo "$HOME/go/bin/httpx")
NUCLEI_BIN=$(command -v nuclei 2>/dev/null || echo "$HOME/go/bin/nuclei")

# 字典路径
DIR_WORDLIST="/usr/share/wordlists/dirb/common.txt"
SUBDOMAIN_WORDLIST="/usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt"

# ==================== 工具函数 ====================
log() {
    local level="$1"; shift
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*" | tee -a "$LOG_FILE"
}

check_tool() {
    local tool_name="$1"
    local tool_path="$2"
    if [[ ! -x "$tool_path" ]]; then
        log "WARN" "工具 $tool_name 未找到或不可执行: $tool_path，跳过相关步骤"
        return 1
    fi
    return 0
}

run_with_timeout() {
    local name="$1"
    shift
    log "INFO" "启动: $name"
    if timeout "$TIMEOUT" "$@" 2>>"$LOG_FILE"; then
        log "INFO" "完成: $name"
        return 0
    else
        log "ERROR" "超时或失败: $name (timeout=${TIMEOUT}s)"
        return 1
    fi
}

# ==================== 初始化 ====================
mkdir -p "$OUTPUT_DIR"/{nmap,subdomains,http,dirs,vuln}
log "INFO" "目标: $TARGET | 输出目录: $OUTPUT_DIR"
log "INFO" "系统: $(uname -n) | 工具版本:"

# 检查关键工具
MISSING=0
for tool_pair in "nmap:$NMAP_BIN" "gobuster:$GOBUSTER_BIN" "subfinder:$SUBFINDER_BIN" "httpx:$HTTPX_BIN"; do
    IFS=':' read -r name path <<< "$tool_pair"
    check_tool "$name" "$path" || ((MISSING++))
done

if (( MISSING > 2 )); then
    log "ERROR" "缺少过多工具（${MISSING}个），请先安装依赖"
    exit 1
fi

# ==================== 阶段一：端口扫描 ====================
log "INFO" "===== 阶段一：端口扫描 ====="

# 快速端口发现（Top 1000端口，用于初步画像）
run_with_timeout "Nmap快速扫描" \
    "$NMAP_BIN" -sV -sC -T4 --top-ports 1000 \
    -oN "$OUTPUT_DIR/nmap/quick.txt" \
    -oX "$OUTPUT_DIR/nmap/quick.xml" \
    "$TARGET" || true

# 全端口深度扫描（仅当快速扫描发现开放端口时执行）
if [[ -f "$OUTPUT_DIR/nmap/quick.txt" ]] && grep -qE "^[0-9]+/tcp\s+open" "$OUTPUT_DIR/nmap/quick.txt"; then
    OPEN_PORTS=$(grep -oE "^[0-9]+" "$OUTPUT_DIR/nmap/quick.txt" | tr '\n' ',' | sed 's/,$//')
    log "INFO" "发现开放端口: $OPEN_PORTS"

    run_with_timeout "Nmap全端口扫描" \
        "$NMAP_BIN" -sV -sC -sA -T4 -p "$OPEN_PORTS" \
        -oN "$OUTPUT_DIR/nmap/full.txt" \
        -oX "$OUTPUT_DIR/nmap/full.xml" \
        "$TARGET" || true
else
    log "WARN" "未发现开放端口或快速扫描失败，跳过深度扫描"
fi

# ==================== 阶段二：子域名枚举 ====================
log "INFO" "===== 阶段二：子域名枚举 ====="

if check_tool "subfinder" "$SUBFINDER_BIN"; then
    # 仅对域名目标执行子域名枚举（排除纯IP）
    if echo "$TARGET" | grep -qE "^[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+$"; then
        log "INFO" "目标为IP地址，跳过子域名枚举"
    else
        run_with_timeout "Subfinder子域名枚举" \
            "$SUBFINDER_BIN" -d "$TARGET" -all -silent \
            -o "$OUTPUT_DIR/subdomains/subfinder.txt" || true

        # 对子域名进行HTTP存活探测
        if [[ -f "$OUTPUT_DIR/subdomains/subfinder.txt" ]] && check_tool "httpx" "$HTTPX_BIN"; then
            run_with_timeout "HTTPX存活探测" \
                "$HTTPX_BIN" -l "$OUTPUT_DIR/subdomains/subfinder.txt" \
                -silent -status-code -title -tech-detect -follow-redirects \
                -o "$OUTPUT_DIR/http/alive.txt" || true
        fi
    fi
fi

# ==================== 阶段三：目录枚举 ====================
log "INFO" "===== 阶段三：目录枚举 ====="

# 确定HTTP目标（优先使用存活探测结果，否则直接使用目标）
HTTP_TARGETS=()
if [[ -f "$OUTPUT_DIR/http/alive.txt" ]]; then
    while IFS= read -r line; do
        url=$(echo "$line" | awk '{print $1}')
        HTTP_TARGETS+=("$url")
    done < <(head -10 "$OUTPUT_DIR/http/alive.txt")  # 限制前10个，避免过多请求
else
    # 尝试HTTP和HTTPS
    HTTP_TARGETS+=("http://$TARGET" "https://$TARGET")
fi

if check_tool "gobuster" "$GOBUSTER_BIN"; then
    for url in "${HTTP_TARGETS[@]}"; do
        safe_name=$(echo "$url" | sed 's|[:/]|_|g')
        run_with_timeout "Gobuster目录枚举-$safe_name" \
            "$GOBUSTER_BIN" dir -u "$url" \
            -w "$DIR_WORDLIST" \
            -t "$THREADS" \
            --no-error \
            -o "$OUTPUT_DIR/dirs/${safe_name}.txt" || true
    done
fi

# ==================== 阶段四：漏洞扫描 ====================
log "INFO" "===== 阶段四：漏洞扫描 ====="

if check_tool "nuclei" "$NUCLEI_BIN"; then
    # 使用默认模板集进行漏洞扫描
    run_with_timeout "Nuclei漏洞扫描" \
        "$NUCLEI_BIN" -l "$OUTPUT_DIR/http/alive.txt" \
        -severity medium,high,critical \
        -silent -stats \
        -o "$OUTPUT_DIR/vuln/nuclei_results.txt" || true
fi

# ==================== 阶段五：结果汇总 ====================
log "INFO" "===== 阶段五：结果汇总 ====="

echo "==========================================" > "$OUTPUT_DIR/summary.txt"
echo "  自动化侦察报告 - $TARGET" >> "$OUTPUT_DIR/summary.txt"
echo "  扫描时间: $(date)" >> "$OUTPUT_DIR/summary.txt"
echo "==========================================" >> "$OUTPUT_DIR/summary.txt"
echo "" >> "$OUTPUT_DIR/summary.txt"

# 端口摘要
if [[ -f "$OUTPUT_DIR/nmap/quick.txt" ]]; then
    echo "【端口扫描结果】" >> "$OUTPUT_DIR/summary.txt"
    grep -E "^[0-9]+/tcp" "$OUTPUT_DIR/nmap/quick.txt" >> "$OUTPUT_DIR/summary.txt" 2>/dev/null || echo "  无结果" >> "$OUTPUT_DIR/summary.txt"
    echo "" >> "$OUTPUT_DIR/summary.txt"
fi

# 子域名摘要
if [[ -f "$OUTPUT_DIR/subdomains/subfinder.txt" ]]; then
    sub_count=$(wc -l < "$OUTPUT_DIR/subdomains/subfinder.txt")
    echo "【子域名发现】共 $sub_count 个" >> "$OUTPUT_DIR/summary.txt"
    head -20 "$OUTPUT_DIR/subdomains/subfinder.txt" >> "$OUTPUT_DIR/summary.txt"
    (( sub_count > 20 )) && echo "  ... (更多结果见 subdomains/subfinder.txt)" >> "$OUTPUT_DIR/summary.txt"
    echo "" >> "$OUTPUT_DIR/summary.txt"
fi

# 漏洞摘要
if [[ -f "$OUTPUT_DIR/vuln/nuclei_results.txt" ]]; then
    vuln_count=$(wc -l < "$OUTPUT_DIR/vuln/nuclei_results.txt")
    echo "【漏洞发现】共 $vuln_count 个" >> "$OUTPUT_DIR/summary.txt"
    head -30 "$OUTPUT_DIR/vuln/nuclei_results.txt" >> "$OUTPUT_DIR/summary.txt"
    echo "" >> "$OUTPUT_DIR/summary.txt"
fi

log "INFO" "全部完成。报告: $OUTPUT_DIR/summary.txt"
log "INFO" "完整结果目录: $OUTPUT_DIR/"
```

这个脚本的关键设计思想：

1. **容错优先**：每个工具调用都包裹在 `run_with_timeout` 中，单个工具失败不会中断整个流程
2. **阶段化执行**：侦察分为5个明确阶段，每个阶段的输出是下一阶段的输入
3. **自适应逻辑**：只在快速扫描发现端口后才执行全端口扫描，避免无谓的时间消耗
4. **结果聚合**：最后自动生成摘要报告，无需人工逐个检查输出文件

#### 5.2.2 被动信息收集自动化

在不允许直接扫描目标的场景（如授权受限的红队行动）中，被动信息收集尤为重要：

```bash
#!/bin/bash
# passive-recon.sh — 纯被动信息收集（不直接连接目标）
# 用法: ./passive-recon.sh <domain>

set -euo pipefail

DOMAIN="${1:?用法: $0 <domain>}"
OUTPUT_DIR="passive_recon_$(date +%Y%m%d)"
mkdir -p "$OUTPUT_DIR"

echo "[*] 被动侦察目标: $DOMAIN"

# ---- 1. WHOIS信息 ----
echo "[+] 查询WHOIS信息..."
whois "$DOMAIN" > "$OUTPUT_DIR/whois.txt" 2>/dev/null || echo "WHOIS查询失败"

# ---- 2. DNS历史记录查询 ----
echo "[+] 查询DNS历史..."
# 使用SecurityTrails等公开API（需要API key）
# curl -s "https://api.securitytrails.com/v1/domain/$DOMAIN/dns" \
#   -H "apikey: YOUR_KEY" > "$OUTPUT_DIR/dns_history.json"

# ---- 3. 子域名被动发现（利用 crt.sh 证书透明度日志） ----
echo "[+] 查询证书透明度日志..."
curl -s "https://crt.sh/?q=%25.$DOMAIN&output=json" \
    | jq -r '.[].name_value' 2>/dev/null \
    | sort -u > "$OUTPUT_DIR/crtsh_subdomains.txt" || echo "crt.sh查询失败"

# ---- 4. GitHub信息泄露搜索 ----
echo "[+] 搜索GitHub信息泄露..."
# 使用GitHub搜索API查找泄露的凭证和敏感信息
SEARCH_QUERIES=(
    "\"$DOMAIN\" password"
    "\"$DOMAIN\" api_key"
    "\"$DOMAIN\" secret"
    "\"$DOMAIN\" token"
)
for query in "${SEARCH_QUERIES[@]}"; do
    echo "  搜索: $query"
    # gh search code "$query" --json path,url --limit 10 >> "$OUTPUT_DIR/github_leaks.json" 2>/dev/null
done

# ---- 5. 子域名爆破（通过DNS解析，不直接连接目标服务器） ----
echo "[+] DNS子域名爆破..."
if command -v dnsx &>/dev/null; then
    # 使用dnsx进行DNS解析验证
    cat "$OUTPUT_DIR/crtsh_subdomains.txt" \
        | dnsx -silent -a -resp-only 2>/dev/null \
        > "$OUTPUT_DIR/resolved_subdomains.txt" || true
fi

# ---- 6. Shodan被动侦察 ----
echo "[+] Shodan信息查询..."
if [[ -n "${SHODAN_API_KEY:-}" ]]; then
    curl -s "https://api.shodan.io/dns/domain/$DOMAIN?key=$SHODAN_API_KEY" \
        > "$OUTPUT_DIR/shodan_dns.json" 2>/dev/null || echo "Shodan查询失败"
fi

echo "[*] 被动侦察完成，结果保存在 $OUTPUT_DIR/"
echo "[*] 文件清单:"
ls -la "$OUTPUT_DIR/"
```

被动侦察脚本的核心原则是：**只使用第三方服务的公开数据，不直接向目标发送任何请求**。这在法律受限的场景中尤为重要——即使是渗透测试，也可能因为网络环境限制而无法执行主动扫描。被动信息收集的结果可以作为主动侦察的有力补充，尤其是在缩小攻击面方面。

### 5.3 扫描与漏洞发现自动化

#### 5.3.1 多工具协同扫描

单一扫描工具的覆盖率有限，实际项目中需要多种工具协同工作：

```bash
#!/bin/bash
# multi-scan.sh — 多工具协同漏洞扫描
# 用法: ./multi-scan.sh <target_url>

set -euo pipefail

TARGET_URL="${1:?用法: $0 <target_url>}"
OUTPUT_DIR="scan_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$OUTPUT_DIR"/{nuclei,nikto,nmap,browser}
LOG="$OUTPUT_DIR/scan.log"

log() { echo "[$(date '+%H:%M:%S')] $*" | tee -a "$LOG"; }

# 并行启动多个扫描器
log "启动多工具并行扫描..."

# 1. Nuclei漏洞模板扫描
nuclei -u "$TARGET_URL" -severity medium,high,critical \
    -stats -silent \
    -o "$OUTPUT_DIR/nuclei/results.txt" 2>>"$LOG" &
PID_NUCLEI=$!

# 2. Nmap服务探测
nmap -sV -sC -T4 -p 80,443,8080,8443 \
    -oN "$OUTPUT_DIR/nmap/services.txt" \
    -oX "$OUTPUT_DIR/nmap/services.xml" \
    "$(echo "$TARGET_URL" | sed -E 's|https?://||;s|:.*||;s|/.*||')" 2>>"$LOG" &
PID_NMAP=$!

# 3. 目录深度枚举（使用ffuf替代gobuster，支持更多过滤）
if command -v ffuf &>/dev/null; then
    ffuf -u "$TARGET_URL/FUZZ" \
        -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt \
        -mc 200,201,301,302,307,401,403 \
        -fs 0 \
        -t 30 \
        -o "$OUTPUT_DIR/browser/directories.json" -of json 2>>"$LOG" &
    PID_FFUF=$!
fi

# 等待所有并行扫描完成
log "等待并行扫描完成..."
FAIL_COUNT=0
for pid_var in PID_NUCLEI PID_NMAP PID_FFUF; do
    pid=${!pid_var:-}
    if [[ -n "$pid" ]]; then
        if wait "$pid"; then
            log "✓ $pid_var 完成"
        else
            log "✗ $pid_var 失败 (exit=$?)"
            ((FAIL_COUNT++))
        fi
    fi
done

# 4. 结果交叉关联
log "执行结果交叉关联分析..."

# 将Nuclei发现的漏洞与Nmap开放端口进行关联
if [[ -f "$OUTPUT_DIR/nuclei/results.txt" && -f "$OUTPUT_DIR/nmap/services.txt" ]]; then
    echo "=== 交叉关联分析 ===" > "$OUTPUT_DIR/correlation.txt"
    echo "" >> "$OUTPUT_DIR/correlation.txt"

    # 提取Nuclei发现的漏洞端口
    nuclei_ports=$(grep -oP ':\K[0-9]+' "$OUTPUT_DIR/nuclei/results.txt" 2>/dev/null | sort -u || true)
    open_ports=$(grep -oP '^\K[0-9]+' "$OUTPUT_DIR/nmap/services.txt" 2>/dev/null | sort -u || true)

    echo "Nuclei漏洞涉及端口: ${nuclei_ports:-无}" >> "$OUTPUT_DIR/correlation.txt"
    echo "Nmap开放端口: ${open_ports:-无}" >> "$OUTPUT_DIR/correlation.txt"

    # 找出重叠端口（高优先级）
    if [[ -n "$nuclei_ports" && -n "$open_ports" ]]; then
        high_priority=$(comm -12 <(echo "$nuclei_ports") <(echo "$open_ports") || true)
        if [[ -n "$high_priority" ]]; then
            echo "" >> "$OUTPUT_DIR/correlation.txt"
            echo "⚠ 高优先级端口（同时被Nuclei标记且Nmap确认开放）:" >> "$OUTPUT_DIR/correlation.txt"
            echo "$high_priority" >> "$OUTPUT_DIR/correlation.txt"
        fi
    fi
fi

log "扫描完成。结果目录: $OUTPUT_DIR/"
log "总计失败: $FAIL_COUNT 个工具"
```

#### 5.3.2 基于Python的高级编排

当Shell脚本无法满足复杂的任务依赖和结果处理需求时，Python是更好的选择：

```python
#!/usr/bin/env python3
"""
auto_scanner.py — 基于Python的高级自动化扫描编排器
支持：任务依赖、并行执行、结果持久化、断点续扫
"""

import subprocess
import json
import os
import sys
import time
import sqlite3
from datetime import datetime
from pathlib import Path
from concurrent.futures import ThreadPoolExecutor, as_completed
from dataclasses import dataclass, field, asdict
from typing import Optional

# ==================== 数据模型 ====================

@dataclass
class ScanTask:
    """扫描任务定义"""
    name: str
    command: list[str]
    timeout: int = 300  # 秒
    depends_on: list[str] = field(default_factory=list)  # 依赖的任务名
    output_file: Optional[str] = None
    status: str = "pending"  # pending / running / completed / failed
    result: str = ""
    start_time: Optional[str] = None
    end_time: Optional[str] = None

@dataclass
class ScanResult:
    """扫描结果汇总"""
    target: str
    start_time: str
    end_time: str
    tasks_total: int
    tasks_completed: int
    tasks_failed: int
    findings: dict = field(default_factory=dict)

# ==================== 核心引擎 ====================

class AutoScanner:
    """自动化扫描引擎"""

    def __init__(self, target: str, output_dir: str = None):
        self.target = target
        self.output_dir = output_dir or f"scan_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
        self.tasks: dict[str, ScanTask] = {}
        self.max_workers = 4  # 最大并行任务数
        self.results = {}

        # 初始化输出目录
        Path(self.output_dir).mkdir(parents=True, exist_ok=True)

        # 初始化SQLite数据库用于持久化扫描状态
        self.db_path = os.path.join(self.output_dir, "scan_state.db")
        self._init_db()

    def _init_db(self):
        """初始化扫描状态数据库"""
        conn = sqlite3.connect(self.db_path)
        conn.execute("""
            CREATE TABLE IF NOT EXISTS scan_tasks (
                name TEXT PRIMARY KEY,
                command TEXT,
                status TEXT DEFAULT 'pending',
                result TEXT,
                start_time TEXT,
                end_time TEXT,
                output_file TEXT
            )
        """)
        conn.commit()
        conn.close()

    def add_task(self, task: ScanTask):
        """注册扫描任务"""
        self.tasks[task.name] = task

    def _resolve_dependencies(self, task: ScanTask) -> bool:
        """检查任务依赖是否全部完成"""
        for dep_name in task.depends_on:
            dep_task = self.tasks.get(dep_name)
            if not dep_task or dep_task.status != "completed":
                return False
        return True

    def _execute_task(self, task: ScanTask):
        """执行单个扫描任务"""
        task.status = "running"
        task.start_time = datetime.now().isoformat()

        print(f"[{task.start_time}] ▶ 启动: {task.name}")
        print(f"  命令: {' '.join(task.command)}")

        try:
            result = subprocess.run(
                task.command,
                capture_output=True,
                text=True,
                timeout=task.timeout
            )

            task.result = result.stdout
            task.status = "completed" if result.returncode == 0 else "failed"
            task.end_time = datetime.now().isoformat()

            # 保存输出到文件
            if task.output_file:
                output_path = os.path.join(self.output_dir, task.output_file)
                Path(output_path).parent.mkdir(parents=True, exist_ok=True)
                with open(output_path, 'w') as f:
                    f.write(result.stdout)
                    if result.stderr:
                        f.write(f"\n--- STDERR ---\n{result.stderr}")

            print(f"[{task.end_time}] {'✓' if task.status == 'completed' else '✗'} {task.name}")

            # 持久化到数据库
            self._save_task_state(task)

        except subprocess.TimeoutExpired:
            task.status = "failed"
            task.result = f"TIMEOUT after {task.timeout}s"
            task.end_time = datetime.now().isoformat()
            print(f"[{task.end_time}] ✗ {task.name} — 超时")

        except Exception as e:
            task.status = "failed"
            task.result = str(e)
            task.end_time = datetime.now().isoformat()
            print(f"[{task.end_time}] ✗ {task.name} — 异常: {e}")

    def _save_task_state(self, task: ScanTask):
        """将任务状态持久化到SQLite"""
        conn = sqlite3.connect(self.db_path)
        conn.execute("""
            INSERT OR REPLACE INTO scan_tasks
            (name, command, status, result, start_time, end_time, output_file)
            VALUES (?, ?, ?, ?, ?, ?, ?)
        """, (task.name, json.dumps(task.command), task.status,
              task.result[:10000], task.start_time, task.end_time, task.output_file))
        conn.commit()
        conn.close()

    def run(self):
        """执行所有任务（尊重依赖关系的并行调度）"""
        print(f"{'='*60}")
        print(f"  自动化扫描引擎")
        print(f"  目标: {self.target}")
        print(f"  输出: {self.output_dir}")
        print(f"  任务数: {len(self.tasks)}")
        print(f"{'='*60}")

        start_time = datetime.now()
        completed = set()

        with ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            pending_futures = {}

            while len(completed) < len(self.tasks):
                # 找出可以执行的任务（依赖已满足且未执行）
                ready = []
                for name, task in self.tasks.items():
                    if name not in completed and name not in pending_futures:
                        if self._resolve_dependencies(task):
                            ready.append(task)

                # 提交就绪的任务
                for task in ready:
                    future = executor.submit(self._execute_task, task)
                    pending_futures[future] = task.name

                # 等待任意一个任务完成
                if pending_futures:
                    done_futures = as_completed(pending_futures, timeout=1)
                    for future in done_futures:
                        task_name = pending_futures.pop(future)
                        completed.add(task_name)
                else:
                    # 防止死循环：如果没有任何就绪任务但还有未完成的，说明存在循环依赖
                    remaining = set(self.tasks.keys()) - completed
                    if remaining:
                        print(f"⚠ 可能存在循环依赖，无法执行的任务: {remaining}")
                        break
                    time.sleep(0.1)

        end_time = datetime.now()

        # 生成汇总报告
        self._generate_report(start_time, end_time)

    def _generate_report(self, start_time: datetime, end_time: datetime):
        """生成扫描报告"""
        completed = sum(1 for t in self.tasks.values() if t.status == "completed")
        failed = sum(1 for t in self.tasks.values() if t.status == "failed")

        report = {
            "target": self.target,
            "start_time": start_time.isoformat(),
            "end_time": end_time.isoformat(),
            "duration_seconds": (end_time - start_time).total_seconds(),
            "tasks": {
                "total": len(self.tasks),
                "completed": completed,
                "failed": failed
            },
            "task_details": {
                name: {
                    "status": task.status,
                    "duration": task.end_time and task.start_time and
                        (datetime.fromisoformat(task.end_time) -
                         datetime.fromisoformat(task.start_time)).total_seconds(),
                    "output_file": task.output_file
                }
                for name, task in self.tasks.items()
            }
        }

        report_path = os.path.join(self.output_dir, "report.json")
        with open(report_path, 'w') as f:
            json.dump(report, f, indent=2, ensure_ascii=False)

        print(f"\n{'='*60}")
        print(f"  扫描完成报告")
        print(f"  耗时: {report['duration_seconds']:.1f}秒")
        print(f"  成功: {completed}/{len(self.tasks)}")
        print(f"  失败: {failed}/{len(self.tasks)}")
        print(f"  报告: {report_path}")
        print(f"{'='*60}")

# ==================== 使用示例 ====================
```

```python
                if len(parts) >= 3:
                    severity = parts[0].strip("[]")
                    template_id = parts[1].strip("[]")
```

#### 5.5.1 自动化渗透测试报告生成器

手动编写渗透测试报告是最耗时的环节之一。一个自动化报告生成器可以从扫描结果中提取关键信息，生成结构化的报告框架：

```python
#!/usr/bin/env python3
"""
auto_report.py — 自动化渗透测试报告生成器
从扫描结果自动生成Markdown格式的渗透测试报告
"""

import json
import os
from datetime import datetime
from pathlib import Path
from dataclasses import dataclass

@dataclass
class Finding:
    """安全发现"""
    title: str
    severity: str  # critical / high / medium / low / info
    description: str
    affected_asset: str
    evidence: str
    recommendation: str
    cve: str = ""
    cvss: float = 0.0

class ReportGenerator:
    """自动化报告生成器"""

    SEVERITY_EMOJI = {
        "critical": "🔴",
        "high": "🟠",
        "medium": "🟡",
        "low": "🔵",
        "info": "⚪"
    }

    def __init__(self, scan_dir: str, target: str, client: str = "未指定"):
        self.scan_dir = Path(scan_dir)
        self.target = target
        self.client = client
        self.findings: list[Finding] = []
        self.scan_start = ""
        self.scan_end = ""

    def load_findings(self):
        """从扫描结果目录加载所有发现"""
        # 加载Nuclei结果
        nuclei_file = self.scan_dir / "nuclei" / "results.txt"
        if nuclei_file.exists():
            self._parse_nuclei(nuclei_file)

        # 加载自定义扫描结果
        custom_file = self.scan_dir / "custom_findings.json"
        if custom_file.exists():
            self._parse_custom(custom_file)

        print(f"共加载 {len(self.findings)} 个安全发现")

    def _parse_nuclei(self, filepath: Path):
        """解析Nuclei输出"""
        import re
        for line in filepath.read_text().splitlines():
            if not line.strip():
                continue
            match = re.search(r'\[(\w+)\]\s+\[([^\]]+)\]\s+(\S+)', line)
            if match:
                severity = match.group(1).lower()
                template = match.group(2)
                url = match.group(3)
                self.findings.append(Finding(
                    title=f"Nuclei: {template}",
                    severity=severity,
                    description=f"通过Nuclei模板 {template} 检测到的安全问题",
                    affected_asset=url,
                    evidence=line,
                    recommendation=f"建议针对 {template} 进行深入验证并修复"
                ))

    def _parse_custom(self, filepath: Path):
        """解析自定义JSON格式的发现"""
        data = json.loads(filepath.read_text())
        for item in data:
            self.findings.append(Finding(**item))

    def generate_markdown(self) -> str:
        """生成Markdown格式的报告"""
        now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")

        # 按严重性统计
        severity_count = {}
        for f in self.findings:
            severity_count[f.severity] = severity_count.get(f.severity, 0) + 1

        report = f"""# 渗透测试报告

## 基本信息

| 项目 | 内容 |
|------|------|
| 委托方 | {self.client} |
| 测试目标 | {self.target} |
| 测试时间 | {self.scan_start} ~ {self.scan_end} |
| 报告生成 | {now} |
| 报告版本 | 自动化生成 v1.0 |

## 执行摘要

本次渗透测试共发现 **{len(self.findings)}** 个安全问题：

| 严重等级 | 数量 | 占比 |
|---------|------|------|
| 🔴 严重 | {severity_count.get('critical', 0)} | {self._pct(severity_count.get('critical', 0))} |
| 🟠 高危 | {severity_count.get('high', 0)} | {self._pct(severity_count.get('high', 0))} |
| 🟡 中危 | {severity_count.get('medium', 0)} | {self._pct(severity_count.get('medium', 0))} |
| 🔵 低危 | {severity_count.get('low', 0)} | {self._pct(severity_count.get('low', 0))} |
| ⚪ 信息 | {severity_count.get('info', 0)} | {self._pct(severity_count.get('info', 0))} |

## 详细发现

"""
        # 按严重性排序输出
        severity_order = {"critical": 0, "high": 1, "medium": 2, "low": 3, "info": 4}
        sorted_findings = sorted(self.findings, key=lambda f: severity_order.get(f.severity, 5))

        for i, finding in enumerate(sorted_findings, 1):
            emoji = self.SEVERITY_EMOJI.get(finding.severity, "⚪")
            report += f"""### {i}. {emoji} {finding.title}

| 属性 | 值 |
|------|-----|
| 严重等级 | {finding.severity.upper()} |
| 影响资产 | `{finding.affected_asset}` |
| CVE编号 | {finding.cve or 'N/A'} |

**描述：** {finding.description}

**证据：**
```
{finding.evidence}
```python

**修复建议：** {finding.recommendation}

---

"""

        # 附录
        report += """## 附录

### 附录A：测试工具清单

| 工具名称 | 版本 | 用途 |
|---------|------|------|
| Nmap | latest | 端口扫描与服务探测 |
| Nuclei | latest | 漏洞模板扫描 |
| Gobuster | latest | 目录枚举 |
| Subfinder | latest | 子域名发现 |

### 附录B：免责声明

本报告由自动化工具辅助生成，所有发现均需人工复核确认。
扫描过程遵循授权范围，未对目标系统造成实质性损害。
"""

        return report

    def _pct(self, count: int) -> str:
        """计算百分比"""
        if not self.findings:
            return "0%"
        return f"{count / len(self.findings) * 100:.1f}%"

    def save(self, output_path: str = None):
        """保存报告"""
        report = self.generate_markdown()
        if output_path is None:
            output_path = f"report_{self.target}_{datetime.now().strftime('%Y%m%d')}.md"
        Path(output_path).write_text(report, encoding='utf-8')
        print(f"报告已生成: {output_path}")
        return output_path

# 使用示例
if __name__ == "__main__":
    import sys
    target = sys.argv[1] if len(sys.argv) > 1 else "example.com"
    gen = ReportGenerator("scan_results", target)
    gen.load_findings()
    gen.save()
```

### 5.7 常见误区与最佳实践

#### 5.7.1 典型误区

| 误区 | 正确做法 |
|------|---------|
| 只依赖单一扫描工具 | 多工具交叉验证，减少漏报 |
| 扫描全端口时不加速率限制 | 使用 `--max-rate` 或 `-T3` 避免触发IDS |
| 自动化脚本无错误处理 | 每个外部命令调用都检查返回值和超时 |
| 不保存中间结果 | 所有中间输出持久化到文件/数据库 |
| 把敏感凭证硬编码在脚本中 | 使用环境变量或 `.env` 文件 |
| 忽略代理和VPN配置 | 工具链集成代理切换，支持内网渗透场景 |
| 盲目追求自动化覆盖率 | 自动化信息收集，人工分析漏洞 |
| 忽视并发控制 | 限制并行扫描数量，避免目标服务崩溃 |

#### 5.7.2 生产环境最佳实践

1. **速率控制**：对目标的请求速率进行限制。Nmap使用 `-T3`（默认）或 `--max-rate 100`；Nuclei使用 `-rl 50`（每秒50个请求）；Gobuster使用 `-t 10`（10个线程）。

2. **日志分级**：使用 `DEBUG/INFO/WARN/ERROR` 四级日志，正常运行时只输出INFO以上级别，调试时开启DEBUG。

3. **断点续扫**：对于长时间运行的扫描，使用状态文件或数据库记录已完成的步骤。中断后重启时跳过已完成的步骤。

4. **结果去重**：不同工具可能发现同一个问题。在汇总报告时需要对重复发现进行去重（基于端口+漏洞类型+URL的组合键）。

5. **最小权限原则**：自动化脚本应该以最低必要权限运行。目录枚举不需要root权限，只有SYN扫描（Nmap `-sS`）才需要。

6. **网络隔离**：工具链应该在隔离的网络环境中运行，通过VPN或代理链连接目标。避免扫描流量直接暴露攻击机的真实IP。

7. **版本锁定**：工具版本应该固定，避免因工具更新导致行为变化。可以在Dockerfile中指定版本号而非使用 `@latest`。

### 5.8 进阶：构建个人渗透测试框架

当工具链积累到一定程度时，应该将其整合为一个统一的渗透测试框架：

```python
#!/usr/bin/env python3
"""
pentest_framework.py — 个人渗透测试框架入口
统一管理所有自动化工具，提供命令行接口
"""

import argparse
import sys
from pathlib import Path

def cmd_recon(args):
    """执行信息收集"""
    from auto_recon import AutoRecon
    recon = AutoRecon(args.target, args.output)
    recon.run_passive()
    if not args.passive_only:
        recon.run_active()

def cmd_scan(args):
    """执行漏洞扫描"""
    from auto_scanner import AutoScanner
    scanner = AutoScanner(args.target, args.output)
    # 根据参数注册任务
    scanner.add_default_tasks()
    if args.nuclei_only:
        scanner.add_nuclei_task()
    scanner.run()

def cmd_exploit(args):
    """执行漏洞利用"""
    from auto_exploit import AutoExploit
    exploit = AutoExploit(args.scan_dir)
    exploit.load_findings()
    if args.verify_only:
        exploit.execute_exploits(dry_run=True)
    else:
        exploit.execute_exploits(dry_run=False)

def cmd_report(args):
    """生成报告"""
    from auto_report import ReportGenerator
    gen = ReportGenerator(args.scan_dir, args.target, args.client)
    gen.load_findings()
    gen.save(args.output)

def main():
    parser = argparse.ArgumentParser(
        description="个人渗透测试自动化框架",
        formatter_class=argparse.RawDescriptionHelpFormatter,
        epilog="""
示例用法:
  %(prog)s recon example.com -o results/
  %(prog)s scan example.com --nuclei-only
  %(prog)s exploit --scan-dir results/ --verify-only
  %(prog)s report --scan-dir results/ --target example.com --client "客户A"
        """
    )
    subparsers = parser.add_subparsers(dest="command", help="可用命令")

    # recon 子命令
    p_recon = subparsers.add_parser("recon", help="信息收集")
    p_recon.add_argument("target", help="目标IP或域名")
    p_recon.add_argument("-o", "--output", default="recon_results", help="输出目录")
    p_recon.add_argument("--passive-only", action="store_true", help="仅执行被动收集")
    p_recon.set_defaults(func=cmd_recon)

    # scan 子命令
    p_scan = subparsers.add_parser("scan", help="漏洞扫描")
    p_scan.add_argument("target", help="目标IP或域名")
    p_scan.add_argument("-o", "--output", default="scan_results", help="输出目录")
    p_scan.add_argument("--nuclei-only", action="store_true", help="仅执行Nuclei扫描")
    p_scan.set_defaults(func=cmd_scan)

    # exploit 子命令
    p_exploit = subparsers.add_parser("exploit", help="漏洞利用")
    p_exploit.add_argument("--scan-dir", required=True, help="扫描结果目录")
    p_exploit.add_argument("--verify-only", action="store_true", help="仅验证不利用")
    p_exploit.set_defaults(func=cmd_exploit)

    # report 子命令
    p_report = subparsers.add_parser("report", help="生成报告")
    p_report.add_argument("--scan-dir", required=True, help="扫描结果目录")
    p_report.add_argument("--target", required=True, help="目标名称")
    p_report.add_argument("--client", default="未指定", help="委托方名称")
    p_report.add_argument("-o", "--output", help="报告输出路径")
    p_report.set_defaults(func=cmd_report)

    args = parser.parse_args()
    if not args.command:
        parser.print_help()
        sys.exit(1)

    args.func(args)

if __name__ == "__main__":
    main()
```

### 5.9 本节小结

自动化工具链构建不是一蹴而就的工程，而是一个持续迭代的过程。从最初的一键侦察脚本，到多工具协同扫描，再到完整的渗透测试框架，每一步都在提升效率和覆盖面。核心原则始终如一：**自动化处理重复性工作，人类专注于创造性思考**。

关键要点回顾：

1. **架构分层**：基础设施层 → 执行引擎层 → 任务编排层 → 分析聚合层 → 报告生成层
2. **容错优先**：每个外部工具调用都有超时和错误处理，单点故障不拖垮全局
3. **结果驱动**：所有扫描结果持久化存储，支持断点续扫和结果回溯
4. **工具互补**：不同工具发现不同类型的问题，交叉验证提高准确率
5. **环境一致**：使用Docker和配置文件管理工具链环境，确保可复现性
6. **持续迭代**：根据实战经验不断优化脚本、补充PoC、改进报告格式

记住：最好的自动化工具链是你自己用起来最顺手的那个。从简单的Shell脚本开始，逐步扩展功能，最终形成适合你个人风格的完整框架。
