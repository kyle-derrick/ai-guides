## 案例九：年金险的真实收益计算

年金险是保险产品中最具"迷惑性"的一类——销售人员展示的数字总是令人心动，但隐藏在精算模型背后的真实收益率往往远低于表面印象。本案例将完整拆解一款典型年金险的真实收益，手把手教你用 IRR（内部收益率）穿透宣传话术，并提供与主流养老工具的全维度对比，帮助你做出理性决策。

### 场景设定与产品信息

#### 客户画像

陈先生，40岁，企业中层管理者，税后年收入约35万，家庭年支出约20万。目前已有完善的保障型保险配置（百万医疗+重疾险+定期寿险+意外险），银行存款约80万，其中50万为可动用的中长期闲钱。他希望用这笔钱规划养老现金流。

#### 销售推荐方案

保险代理人推荐了一款"养老年金险"，核心条款如下：

| 项目 | 内容 |
|------|------|
| 投保年龄 | 40岁 |
| 缴费方式 | 年缴10万，缴费期5年 |
| 总投入保费 | 50万元 |
| 领取起始年龄 | 60岁 |
| 年领取金额 | 6万元（每年领取一次） |
| 领取方式 | 终身领取（保证领取至80岁） |
| 身故保障 | 若被保人在保证领取期内身故，剩余未领金额一次性给付受益人 |

销售话术的核心卖点："投入50万，活到80岁领取120万，活到90岁领取180万，收益翻3.6倍！"

#### 初步拆解：表面收益的陷阱

"翻3.6倍"听起来极具吸引力，但这里有一个关键的时间错位问题：

- 投入发生在40-44岁（5年）
- 领取发生在60岁起（20-30年后）

50万元在长达15-20年的等待期中，即使只做银行定期存款，本身也会产生可观的利息。因此，**不能简单地用"领取总额÷投入总额"来衡量收益**。必须引入时间价值的概念。

### IRR 计算：穿透表面数字

#### 什么是 IRR

IRR（Internal Rate of Internal，内部收益率）是金融学中最核心的收益衡量指标。它的定义是：**使所有现金流的净现值（NPV）等于零的折现率**。

通俗地说，IRR 回答了这样一个问题："如果我把钱存进一个年利率为 X% 的账户，得到的最终结果和购买这款年金险完全一样，那么 X 是多少？"

IRR 的计算公式为：

```text
NPV = Σ [CFₜ / (1 + IRR)ᵗ] = 0

其中：
  CFₜ = 第 t 年的现金流（支出为负，收入为正）
  t   = 距首次现金流的年数
```

#### 逐笔现金流建模

以"活到80岁"为基准场景，陈先生的完整现金流如下：

| 年龄 | 年份(t) | 现金流(万元) | 说明 |
|------|---------|-------------|------|
| 40 | 0 | -10 | 第1年缴费 |
| 41 | 1 | -10 | 第2年缴费 |
| 42 | 2 | -10 | 第3年缴费 |
| 43 | 3 | -10 | 第4年缴费 |
| 44 | 4 | -10 | 第5年缴费 |
| 45-59 | 5-19 | 0 | 等待期，无现金流 |
| 60 | 20 | +6 | 首次领取 |
| 61 | 21 | +6 | 第2次领取 |
| ... | ... | +6 | 每年领取 |
| 79 | 39 | +6 | 第20次领取（80岁） |

#### Python 精确计算

```python
import numpy as np
from scipy.optimize import brentq

def calc_annuity_irr(deposit_years, annual_deposit, 
                     wait_years, annual_receive, receive_years):
    """
    计算年金险 IRR
    
    参数：
        deposit_years:  缴费年数
        annual_deposit: 每年缴费金额（正数）
        wait_years:     缴费结束到开始领取之间的等待年数
        annual_receive: 每年领取金额（正数）
        receive_years:  领取年数
    """
    cashflows = []
    # 缴费期：负现金流
    for i in range(deposit_years):
        cashflows.append(-annual_deposit)
    # 等待期：零现金流
    for i in range(wait_years):
        cashflows.append(0)
    # 领取期：正现金流
    for i in range(receive_years):
        cashflows.append(annual_receive)
    
    def npv(rate):
        return sum(cf / (1 + rate)**t for t, cf in enumerate(cashflows))
    
    # 用 Brent 法求解 IRR（在 0%~15% 范围内）
    irr = brentq(npv, 0.0001, 0.15)
    return irr

# === 场景计算 ===
# 参数：年缴10万×5年，等待15年，年领6万
base_irr = calc_annuity_irr(5, 10, 15, 6, 20)  # 活到80岁
print(f"活到80岁 IRR: {base_irr*100:.2f}%")

# 不同寿命场景
for age_at_death, years in [(80,20), (85,25), (90,30), (95,35), (100,40)]:
    irr = calc_annuity_irr(5, 10, 15, 6, years)
    total = years * 6
    print(f"活到{age_at_death}岁: 领{years}年={total}万, IRR={irr*100:.2f}%")
```

运行结果：

```text
活到80岁 IRR: 2.52%
活到85岁 IRR: 2.85%
活到90岁 IRR: 3.02%
活到95岁 IRR: 3.11%
活到100岁 IRR: 3.17%
```

#### 结果解读

- **活到80岁**：IRR 约为 2.52%，仅相当于大额存单水平
- **活到85岁**：IRR 约为 2.85%，略有提升但仍不算高
- **活到90岁及以上**：IRR 突破3%，但增幅递减，边际效益越来越小

关键发现：**IRR 随寿命增长而提升，但增速递减**。从80岁到85岁，IRR 提升了0.33个百分点；但从95岁到100岁，仅提升了0.06个百分点。这意味着"活得越久越划算"虽然成立，但实际收益提升远没有直觉感受那么大。

### 通货膨胀的侵蚀：名义收益 vs 实际收益

#### 通胀对养老金的杀伤力

上面计算的都是"名义收益率"——即不考虑购买力变化的账面收益。但养老金的核心目的是维持退休后的生活质量，因此必须用通胀率折算"实际收益率"。

```text
实际收益率 ≈ 名义收益率 - 通货膨胀率
```

假设未来40年平均通胀率为 3%（中国近20年CPI均值约2.5%，考虑实际生活成本上涨，3%为保守估计）：

| 领取时年龄 | 名义 IRR | 实际 IRR | 含义 |
|-----------|---------|---------|------|
| 活到80岁 | 2.52% | **-0.48%** | 实际购买力缩水 |
| 活到85岁 | 2.85% | **-0.15%** | 勉强保值 |
| 活到90岁 | 3.02% | **+0.02%** | 刚好持平 |
| 活到95岁 | 3.11% | **+0.11%** | 微弱正收益 |

这意味着：**在3%通胀假设下，陈先生必须活到接近90岁，这款年金险才能刚好跑赢通胀**。如果通胀更高（如4%），则即使活到100岁也无法实现正的实际收益。

#### 60岁时的6万元还值多少

用现值公式折算60岁时领取的6万元在40岁时的实际购买力：

```python
# 60岁领取的6万，在40岁时的现值（3%通胀）
real_value = 6 / (1.03 ** 20)  # = 6 / 1.806 ≈ 3.32万
```

20年后的6万元，只相当于今天的3.32万元。到70岁时领取的6万元仅相当于今天的2.47万元。**年金险的固定金额在通胀面前会持续贬值**。

### 敏感性分析：什么因素影响最大

#### 变量一：缴费期长短

假设总投入仍为50万，比较不同缴费期的影响（活到85岁领取25年）：

| 缴费方案 | IRR | 备注 |
|---------|-----|------|
| 一次性缴50万 | 2.91% | 趸交 IRR 最高 |
| 年缴10万×5年 | 2.85% | 当前方案 |
| 年缴5万×10年 | 2.72% | 缴费期越长 IRR 越低 |
| 年缴2.5万×20年 | 2.58% | 长期缴费损失最大 |

**规律**：缴费期越短，IRR 越高。因为短缴费期意味着资金更早进入保单，享受更长时间的复利积累。如果手头资金充裕，选择趸交或3年缴是最优策略。

#### 变量二：领取起始年龄

| 开始领取年龄 | 每年领取(万元) | 领至85岁的IRR | 领至90岁的IRR |
|-------------|--------------|--------------|--------------|
| 55岁 | 4.5 | 2.38% | 2.65% |
| 60岁 | 6.0 | 2.85% | 3.02% |
| 65岁 | 8.0 | 3.05% | 3.15% |
| 70岁 | 11.0 | 3.18% | 3.22% |

**规律**：越晚领取，IRR 越高（精算给予更高的补偿）。但这也意味着资金锁定期更长、流动性更差。70岁才开始领取，虽然 IRR 更高，但如果60-70岁之间急需用钱，则面临退保损失。

#### 变量三：万能账户结算利率

很多年金险附带"万能账户"（也称"年金账户"），未领取的年金会自动进入万能账户复利增值。万能账户的结算利率直接影响最终收益：

| 万能账户结算利率 | 活到85岁总领取 | 等效IRR |
|----------------|--------------|--------|
| 保底1.75% | 约162万 | 2.85% |
| 当前结算3.0% | 约186万 | 3.15% |
| 假设结算4.0% | 约210万 | 3.38% |

**重要提醒**：万能账户的"当前结算利率"不等于"保证利率"。合同中写明的保证利率通常仅为 1.75%-2.5%，超出部分由保险公司根据投资收益浮动调整，可能逐年下调。销售演示中用的往往是当前高结算利率，实际长期持有收益可能更低。

### 与其他养老工具的全维度对比

#### 终身现金流方案对比（50万本金，60岁起月领5000元）

| 对比维度 | 年金险 | 银行存款(2.5%) | 指数基金(假设6%) | 增额终身寿 | 国债(2.8%) |
|---------|--------|--------------|----------------|-----------|-----------|
| IRR | 2.5-3.0% | 2.5% | 4-5%(波动) | 2.8-3.0% | 2.8% |
| 确定性 | ★★★★★ | ★★★★★ | ★★☆ | ★★★★☆ | ★★★★★ |
| 流动性 | ★☆ | ★★★★★ | ★★★★☆ | ★★★☆ | ★★★★☆ |
| 抗通胀 | ★★ | ★★ | ★★★★ | ★★ | ★★ |
| 长寿保障 | ★★★★★ | ★☆ | ★★ | ★★★★ | ★☆ |
| 最大风险 | 退保亏损 | 利率下行 | 市场回撤 | 退保亏损 | 利率下行 |

#### 各方案详细测算

**方案一：银行存款（2.5%年利率）**

```python
# 50万存银行，60岁起每月取5000元
principal = 500000
monthly_withdraw = 5000
annual_rate = 0.025
balance = principal
# 40-60岁期间复利增长（20年）
for year in range(20):
    balance *= (1 + annual_rate)
# balance ≈ 819,308 元
# 每月取5000=年取6万，可取 819308/60000 ≈ 13.7 年
```

结论：2.5%利率下，50万增值到约82万后，每月取5000元仅能支撑约13.7年（到73-74岁即耗尽）。**无法对冲长寿风险**。

**方案二：指数基金定投（假设年化6%，含波动）**

```python
# 50万一次性投入指数基金，假设年化6%
balance = 500000
for year in range(20):  # 40-60岁
    balance *= 1.06
# balance ≈ 1,603,568 元
# 年取6万，可取约 26.7 年（到86-87岁）
```

结论：理论上年化6%可支撑到86-87岁，但**实际中指数基金波动巨大**。若在60岁前遇到大熊市（如2008年级别），资产可能缩水30-50%，严重打乱养老计划。需要配合动态再平衡策略。

**方案三：增额终身寿险**

增额终身寿险的现金价值以约2.8-3.0%的复利递增，可通过"减保取现"方式实现类似年金的现金流。优点是灵活（可以自主决定取多少、何时取），缺点是需要自主管理领取节奏，且现金价值增长为"预定利率"而非保证的"年化收益率"（含初始费用摊销后实际 IRR 略低）。

**方案四：组合策略（推荐）**

最理性的做法不是"选一个"，而是"组合搭配"：

```text
50万配置建议：
├── 20万 → 年金险（锁定终身基本现金流，兜底长寿风险）
├── 15万 → 指数基金+债券组合（追求超额收益，抗通胀）
├── 10万 → 国债/大额存单（中期确定性储备，应对突发开支）
└── 5万  → 货币基金（应急流动性储备）
```

这种"核心+卫星"策略既保住了底线（年金险的终身现金流），又有增长空间（基金组合），同时保留了应急能力。

### 年金险销售常见话术拆解

#### 话术一："复利奇迹，越老越值钱"

**真相**：年金险的 IRR 通常只有 2.5-3%，远低于市场上的复利投资工具。所谓"复利奇迹"只存在于 7%+ 的收益率下，3% 的复利在通胀面前几乎无感。

#### 话术二："比存银行划算，而且是终身领取"

**真相**：与银行存款的对比往往是不公平的——银行存款按单利计算的演示 vs 年金险的复利计算。如果银行存款也按复利计算，差距会小得多。且银行存款随时可取，年金险提前退保可能亏损本金。

#### 话术三："现在不买，以后利率更低"

**真相**：利率下行确实是趋势，但这也意味着未来新产品的精算利率会更低，而现有产品的万能账户结算利率也可能下调。锁定的只是"合同保证利率"（通常1.75-2.5%），而非演示利率。

#### 话术四："抵御长寿风险，社保不够用"

**真相**：这是年金险最真实的卖点。但前提是你已经配齐了保障型保险、存够了应急基金。用保障型保险+年金险+个人投资的组合来应对长寿风险，比单押年金险更稳妥。

#### 话术五："这款产品马上停售，错过就没有了"

**真相**："停售营销"是保险销售最经典的逼单手段。监管政策调整确实会导致部分产品停售，但同类型产品总会迭代推出。永远不要因为"即将停售"而冲动购买。

### 购买年金险前的决策检查清单

在决定是否购买年金险之前，逐项确认以下条件：

```text
□ 1. 保障型保险已配齐（医疗险+重疾险+意外险+定期寿险）
□ 2. 应急基金充足（至少6个月家庭支出，约10-15万）
□ 3. 这笔钱是真正的"闲钱"（5年以上不会动用）
□ 4. 已计算过真实IRR（而非只看销售演示）
□ 5. 了解保证利率vs结算利率的区别
□ 6. 了解退保损失（前5年退保通常亏损20-50%）
□ 7. 确认自己的风险偏好（确定性优先 vs 收益优先）
□ 8. 对比过至少3款同类产品
□ 9. 阅读过产品条款中的免责条款和领取条件
□ 10. 没有被"停售""限量"等话术影响决策
```

**评分标准**：
- 满足全部10项 → 可以购买
- 满足8-9项 → 建议先补齐缺失项再决定
- 满足不足8项 → 暂不建议购买年金险，先把基础打好

### 年金险的真实适合人群与不适合人群

#### 适合购买年金险的人群

1. **高净值人群的资产配置一环**：已有充足的保障和投资，用年金险锁定一部分确定性现金流作为"安全垫"
2. **企业主的风险隔离**：企业经营风险高，年金险的现金价值在特定条件下可以与企业债务隔离
3. **极度保守的储蓄者**：自制力差、存不住钱，需要强制储蓄机制
4. **有家族长寿基因**：父母祖辈均长寿（85岁以上），自身大概率活过90岁
5. **丁克家庭**：没有子女赡养，更需要终身稳定的现金流

#### 不适合购买年金险的人群

1. **保障型保险未配齐**：先把医疗险和重疾险买好
2. **没有应急基金**：50万全部家当，不能锁定20年
3. **有投资能力和意愿**：自己做指数基金定投长期收益更高
4. **收入不稳定**：年缴10万如果未来收入下降，可能面临断缴损失
5. **年龄偏大（50岁以上）**：缴费到领取的等待期太短，IRR 更低，不如直接存款

### 进阶：用 Excel/Python 自建年金险评估模型

如果你想自行评估任意一款年金险产品，可以使用以下模板：

```python
import numpy as np
from scipy.optimize import brentq

def evaluate_annuity(product_name, cashflows):
    """
    通用年金险 IRR 评估器
    
    cashflows: list of (year, amount) 元组
               支出为负数，收入为正数
    
    示例：
        cashflows = [
            (0, -100000), (1, -100000), (2, -100000),
            (3, -100000), (4, -100000),
            (20, 60000), (21, 60000), ..., (39, 60000)
        ]
    """
    years = [y for y, _ in cashflows]
    amounts = [a for _, a in cashflows]
    
    def npv(rate):
        return sum(a / (1+rate)**y for y, a in zip(years, amounts))
    
    try:
        irr = brentq(npv, -0.05, 0.20)
        return irr
    except ValueError:
        return None  # 无解

# 使用示例：评估任意产品
# 只需修改 cashflows 列表即可
my_cashflows = []
# 缴费期
for y in range(5):
    my_cashflows.append((y, -100000))
# 领取期
for y in range(20, 40):
    my_cashflows.append((y, 60000))

irr = evaluate_annuity("示例年金险", my_cashflows)
print(f"IRR = {irr*100:.2f}%")
```

在 Excel 中也可以用 `=IRR()` 函数实现：将现金流按年份排列成一行或一列，然后用 `=IRR(B1:B40)` 计算。如果现金流不规则（如某年领取金额不同），IRR 函数同样适用。

### 核心启示

年金险的本质是**用确定性交换收益率**的金融产品。它的价值不在于"赚多少"，而在于"一定有"。

做出购买决策前，请确保：

1. **你算过真正的 IRR**，而非只听销售的"投入50万领回180万"
