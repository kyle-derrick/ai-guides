## 案例一：图像分类器的对抗性样本攻击

本案例以自动驾驶交通标志识别系统为靶标，完整复现从模型分析、对抗性样本生成、攻击效果评估到物理世界验证的全链路攻击流程。通过这个案例，读者将理解对抗性样本为什么能欺骗深度学习模型，以及攻击者如何在实际场景中利用这些弱点。

### 背景与威胁模型

#### 场景设定

某自动驾驶公司的感知模块使用 ResNet-50 对 GTSRB（German Traffic Sign Recognition Benchmark）数据集的 43 类交通标志进行分类。系统部署在车载嵌入式平台上，实时处理摄像头采集的图像帧，分类结果直接输入决策模块控制车辆行为。

安全研究人员的目标是评估该分类器在面对对抗性扰动时的鲁棒性——攻击者能否通过微小的、人眼几乎不可察觉的图像修改，让系统将"限速 120"误识别为"限速 30"，或者将"停止"标志误识别为"让行"。

#### 威胁模型定义

在开始攻击之前，必须明确威胁模型（Threat Model），它界定了攻击者的能力边界和攻击目标：

| 维度 | 本案例设定 | 说明 |
|------|-----------|------|
| 攻击者知识 | 白盒（White-box） | 攻击者拥有模型的完整权重和结构 |
| 攻击目标 | 非目标攻击（Untargeted） | 只要分类错误即算成功 |
| 扰动约束 | $L_\infty \leq \epsilon = 0.03$ | 每个像素的扰动不超过 0.03（归一化后） |
| 攻击者能力 | 物理世界 | 能在真实交通标志上施加扰动 |
| 攻击频率 | 每帧均可攻击 | 连续视频流中的实时攻击 |

不同的威胁模型对应不同的攻击策略。如果攻击者只能查询模型 API 而无法获取权重，则需要使用黑盒攻击方法（如迁移攻击或基于查询的攻击）。如果攻击者需要将分类结果误导为特定类别，则需要目标攻击（Targeted Attack）。本案例从最易实现的白盒非目标攻击入手，后续会扩展到更复杂的场景。

#### 为什么交通标志识别特别脆弱

交通标志识别系统面临对抗性攻击时有三个特殊弱点：

**1. 高安全性要求**：自动驾驶的决策链路中，一个分类错误可能导致严重事故。与图片分类"识别错了最多不好看"不同，交通标志的误分类直接关联物理安全。

**2. 环境可控性差**：交通标志暴露在开放环境中，攻击者可以物理接触并修改标志（贴纸、涂鸦、投影）。2017 年 Eykholt 等人在《Robust Physical-Adversarial Attacks on Deep Learning Visual Classification》中证明，贴在停车标志上的小贴纸就能让分类器产生错误输出。

**3. 模型架构的共性**：大多数视觉模型共享相似的架构（CNN 基础模块），在一个模型上有效的对抗性扰动往往可以迁移到其他模型上。

### 攻击全流程详解

整个攻击分为四个阶段：模型分析、对抗性样本生成、攻击效果评估、物理世界验证。每个阶段都包含完整的代码实现和原理说明。

#### 阶段一：模型分析与基线建立

在发起攻击之前，必须充分了解目标模型的行为特征。这一步的目标是建立基线（Baseline），即模型在正常输入下的性能表现，以便后续量化攻击效果。

**环境准备**：

```python

import torchvision.models as models
import torchvision.transforms as transforms
from torch.utils.data import DataLoader
import numpy as np

# 设备选择：优先使用 GPU 加速
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print(f'使用设备: {device}')
```

**加载目标模型**：

GTSRB 数据集包含 43 类德国交通标志，共超过 50,000 张训练图像和 12,000 多张测试图像。我们使用 ResNet-50 作为分类骨干网络，修改最后的全连接层以适配 43 类分类任务：

```python
# 加载预训练的 ResNet-50 并适配 GTSRB
model = models.resnet50(pretrained=False, num_classes=43)

# 加载已训练好的权重（假设模型已在一个 GTSRB 训练集上完成训练）
model.load_state_dict(torch.load('traffic_sign_model.pth', map_location=device))
model = model.to(device)
model.eval()  # 切换到评估模式，关闭 Dropout 和 BatchNorm 的训练行为

# 打印模型结构摘要
total_params = sum(p.numel() for p in model.parameters())
trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f'模型总参数量: {total_params:,}')
print(f'可训练参数量: {trainable_params:,}')
```

为什么 `model.eval()` 很重要？在评估模式下，BatchNorm 层使用训练阶段积累的全局均值和方差，而非当前批次的统计量；Dropout 层停止随机丢弃神经元。如果忘记切换到 eval 模式，前向传播的结果会引入随机性，导致对抗性样本的梯度方向不准确，攻击效果大打折扣。

**建立基线准确率**：

如果梯度全为零或为 `None`，可能的原因包括：模型参数被冻结（`requires_grad=False`）、使用了不可微的操作（如 `torch.round`）、或者梯度在某一层被截断。

#### 阶段二：对抗性样本生成

本阶段使用三种主流攻击方法生成对抗性样本，从快速的单步攻击到精细的优化攻击，逐步提升攻击强度。

**方法一：FGSM（Fast Gradient Sign Method）**

FGSM 由 Goodfellow 等人在 2015 年提出，是最经典的对抗性攻击方法。其核心思想极其简洁：沿着损失函数增长最快的方向（梯度的符号方向）添加扰动。

数学表达：

$$x_{adv} = x + \epsilon \cdot \text{sign}(\nabla_x L(f(x), y))$$

其中 $L$ 是交叉熵损失，$\epsilon$ 控制扰动大小，$\text{sign}$ 函数将梯度映射为 $\{-1, 0, +1\}$。

FGSM 的优势在于速度——只需一次前向传播和一次反向传播，就能生成对抗性样本。这使它适合大规模测试和实时攻击场景。其局限性在于扰动方向过于粗糙，在较大的 $\epsilon$ 下才能达到高攻击成功率。

```python
def fgsm_attack(model, images, labels, epsilon=0.03):
    """
    FGSM 对抗性攻击
    
    参数:
        model: 目标分类模型
        images: 原始输入图像批次，形状 (B, C, H, W)
        labels: 真实标签，形状 (B,)
        epsilon: 扰动预算，控制扰动的最大幅度
    
    返回:
        adv_images: 对抗性样本
        perturbation: 生成的扰动
    """
    # 复制输入以避免修改原始数据
    images = images.clone().detach().to(device)
    labels = labels.clone().detach().to(device)
    
    # 开启梯度计算
    images.requires_grad = True
    
    # 前向传播：计算模型对当前输入的预测
    outputs = model(images)
    
    # 计算损失：交叉熵衡量预测与真实标签的差距
    loss = nn.CrossEntropyLoss()(outputs, labels)
    
    # 反向传播：计算损失相对于输入图像的梯度
    model.zero_grad()
    loss.backward()
    
    # 收集梯度并计算扰动
    data_grad = images.grad.data
    perturbation = epsilon * data_grad.sign()
    
    # 生成对抗性样本
    adv_images = images + perturbation
    
    # 裁剪到有效像素范围 [0, 1]
    adv_images = torch.clamp(adv_images, 0, 1)
    
    return adv_images.detach(), perturbation.detach()
```

**方法二：PGD（Projected Gradient Descent）**

PGD 由 Madry 等人在 2018 年提出，是 FGSM 的多步迭代版本。可以理解为"反复进行小步 FGSM，每步之后将结果投影回 $\epsilon$ 球内"。PGD 被认为是一阶攻击中最强的方法，因为它近似求解了 $\epsilon$ 球内的最大损失问题。

数学表达（每一步）：

$$x^{(t+1)} = \Pi_{B_\epsilon(x)}\left(x^{(t)} + \alpha \cdot \text{sign}(\nabla_x L(f(x^{(t)}), y))\right)$$

其中 $\Pi_{B_\epsilon(x)}$ 表示投影操作，将结果裁剪回以原始样本为中心、半径为 $\epsilon$ 的 $L_p$ 球内；$\alpha$ 是每步的步长（通常远小于 $\epsilon$）。

```python
def pgd_attack(model, images, labels, epsilon=0.03, alpha=0.007, num_iter=40, random_start=True):
    """
    PGD 对抗性攻击（投影梯度下降法）
    
    参数:
        model: 目标分类模型
        images: 原始输入图像批次
        labels: 真实标签
        epsilon: 扰动预算（总扰动上限）
        alpha: 每步迭代的步长
        num_iter: 迭代次数
        random_start: 是否在 epsilon 球内随机初始化（增强攻击多样性）
    
    返回:
        adv_images: 对抗性样本
    """
    images = images.clone().detach().to(device)
    labels = labels.clone().detach().to(device)
    
    # 初始化对抗性样本
    adv_images = images.clone()
    
    if random_start:
        # 在 epsilon 球内随机初始化，避免陷入局部最优
        adv_images = adv_images + torch.empty_like(adv_images).uniform_(-epsilon, epsilon)
        adv_images = torch.clamp(adv_images, 0, 1)
    
    for i in range(num_iter):
        adv_images.requires_grad = True
        
        # 前向传播
        outputs = model(adv_images)
        loss = nn.CrossEntropyLoss()(outputs, labels)
        
        # 反向传播
        model.zero_grad()
        loss.backward()
        
        # 沿梯度方向更新
        adv_images = adv_images.detach() + alpha * adv_images.grad.sign()
        
        # 投影：将扰动裁剪到 [-epsilon, epsilon] 范围
        delta = torch.clamp(adv_images - images, -epsilon, epsilon)
        adv_images = torch.clamp(images + delta, 0, 1).detach()
    
    return adv_images
```

**random_start 的作用**：如果不随机初始化，PGD 从原始样本出发，可能会卡在一个局部极值点。随机初始化让搜索从不同起点开始，增加了找到更强对抗性样本的概率。Madry 等人证明，带随机启动的 PGD 在实践中几乎总是能找到对抗性样本（如果模型确实不鲁棒的话）。

**方法三：C&W（Carlini & Wagner）**

C&W 攻击由 Carlini 和 Wagner 在 2017 年提出，是基于优化的攻击方法。与 FGSM/PGD 的"加扰动"思路不同，C&W 直接求解一个优化问题：在使模型分类错误的前提下，最小化扰动大小。

C&W 的优势在于生成的扰动更小、更隐蔽，通常能找到 FGSM 和 PGD 找不到的对抗性样本。其代价是计算开销大，每张图像需要数百甚至上千次迭代。

```python
import torch.optim as optim

def cw_attack(model, images, labels, target=None, c=1.0, kappa=0, 
              num_iter=1000, lr=0.01, abort_early=True):
    """
    C&W 对抗性攻击（基于优化的方法）
    
    参数:
        model: 目标分类模型
        images: 原始输入图像
        labels: 真实标签
        target: 目标攻击时的目标类别（None 表示非目标攻击）
        c: 平衡扰动大小和攻击成功率的超参数
        kappa: 控制置信度间隔（越大攻击越"确信"）
        num_iter: 最大优化迭代次数
        lr: Adam 优化器学习率
        abort_early: 如果 500 步内没有改善则提前终止
    
    返回:
        best_adv: 找到的最优对抗性样本
    """
    images = images.to(device)
    labels = labels.to(device)
    
    # 在 tanh 空间中优化（自动满足像素值在 [0,1] 范围内）
    # 变换: x = (tanh(w) + 1) / 2，其中 w 是优化变量
    w = torch.zeros_like(images, requires_grad=True, device=device)
    optimizer = optim.Adam([w], lr=lr)
    
    best_adv = images.clone()
    best_l2 = torch.full((images.size(0),), float('inf'), device=device)
    prev_loss = torch.full((images.size(0),), float('inf'), device=device)
    
    for iteration in range(num_iter):
        # 将优化变量 w 转换为图像空间
        adv_image = torch.tanh(w + torch.atanh(torch.clamp(images * 2 - 1, -1 + 1e-6, 1 - 1e-6))) / 2 + 0.5
        
        # 计算 L2 距离
        l2_dist = torch.norm((adv_image - images).view(images.size(0), -1), dim=1, p=2)
        
        # 前向传播
        output = model(adv_image)
        
        # 计算攻击损失 f(x')
        if target is not None:
            # 目标攻击：让模型将输入误分类为目标类别
            target_logit = output[:, target]
            max_other_logit = output.clone()
            max_other_logit[:, target] = -float('inf')
            max_other_logit = max_other_logit.max(dim=1)[0]
            f_loss = torch.clamp(max_other_logit - target_logit + kappa, min=0)
        else:
            # 非目标攻击：让模型将输入误分类为任意非真实类别
            real_logit = output.gather(1, labels.unsqueeze(1)).squeeze(1)
            other_logit = output.clone()
            other_logit.scatter_(1, labels.unsqueeze(1), -float('inf'))
            max_other_logit = other_logit.max(dim=1)[0]
            f_loss = torch.clamp(real_logit - max_other_logit + kappa, min=0)
        
        # 总损失 = L2 距离 + c * 攻击损失
        # 当 f_loss = 0 时，攻击成功；此时只优化 L2 距离
        total_loss = l2_dist + c * f_loss
        
        optimizer.zero_grad()
        total_loss.sum().backward()
        optimizer.step()
        
        # 更新最优结果
        with torch.no_grad():
            success_mask = (f_loss == 0)
            better_mask = success_mask & (l2_dist < best_l2)
            best_l2[better_mask] = l2_dist[better_mask]
            best_adv[better_mask] = adv_image[better_mask]
            
            # 提前终止检查
            if abort_early and iteration % 100 == 0 and iteration > 0:
                if (total_loss >= prev_loss).all():
                    print(f'迭代 {iteration}: 损失未改善，提前终止')
                    break
                prev_loss = total_loss.clone()
    
    return best_adv.detach()
```

**超参数 c 的调优**：c 太小，攻击可能不成功（模型未被欺骗）；c 太大，扰动会很大（不隐蔽）。实践中常用二分搜索在 $[10^{-3}, 10^{3}]$ 范围内寻找最小的 c 使得攻击刚好成功。

**三种方法对比**：

| 特性 | FGSM | PGD | C&W |
|------|------|-----|-----|
| 计算速度 | 极快（1 步） | 中等（10-100 步） | 慢（数百至上千步） |
| 攻击强度 | 较弱 | 强 | 最强 |
| 扰动大小 | 较大 | 中等 | 最小 |
| 适用场景 | 大规模测试、实时攻击 | 标准鲁棒性评估 | 精细对抗性研究 |
| 超参数 | 仅 $\epsilon$ | $\epsilon$, $\alpha$, 迭代数 | $c$, $\kappa$, 迭代数, 学习率 |
| 年份 | 2015 | 2018 | 2017 |

#### 阶段三：攻击效果评估

生成对抗性样本后，需要从多个维度系统评估攻击效果。仅仅看"攻击成功率"是不够的——一个高成功率但扰动明显的攻击在物理世界中没有实用价值。

**综合评估框架**：

```python
def comprehensive_attack_evaluation(model, test_loader, epsilon=0.03):
    """
    系统评估三种攻击方法的效果
    """
    model.eval()
    
    results = {
        'fgsm': {'success': 0, 'total': 0, 'l2_distances': [], 'linf_distances': []},
        'pgd':  {'success': 0, 'total': 0, 'l2_distances': [], 'linf_distances': []},
        'cw':   {'success': 0, 'total': 0, 'l2_distances': [], 'linf_distances': []},
    }
    
    for batch_idx, (images, labels) in enumerate(test_loader):
        images, labels = images.to(device), labels.to(device)
        
        # 确保干净样本本身被正确分类（只测试正确分类的样本）
        with torch.no_grad():
            clean_pred = model(images).argmax(dim=1)
            correct_mask = (clean_pred == labels)
            if correct_mask.sum() == 0:
                continue
            images = images[correct_mask]
            labels = labels[correct_mask]
        
        # FGSM 攻击
        fgsm_adv, fgsm_pert = fgsm_attack(model, images, labels, epsilon=epsilon)
        with torch.no_grad():
            fgsm_pred = model(fgsm_adv).argmax(dim=1)
            fgsm_wrong = (fgsm_pred != labels)
        
        # PGD 攻击
        pgd_adv = pgd_attack(model, images, labels, epsilon=epsilon, alpha=epsilon/4, num_iter=40)
        with torch.no_grad():
            pgd_pred = model(pgd_adv).argmax(dim=1)
            pgd_wrong = (pgd_pred != labels)
        
        # C&W 攻击（为控制时间，只在前 100 个批次上运行）
        if batch_idx < 100:
            cw_adv = cw_attack(model, images, labels, c=1.0, num_iter=500)
            with torch.no_grad():
                cw_pred = model(cw_adv).argmax(dim=1)
                cw_wrong = (cw_pred != labels)
        
        # 统计结果
        batch_size = labels.size(0)
        
        results['fgsm']['success'] += fgsm_wrong.sum().item()
        results['fgsm']['total'] += batch_size
        
        results['pgd']['success'] += pgd_wrong.sum().item()
        results['pgd']['total'] += batch_size
        
        if batch_idx < 100:
            results['cw']['success'] += cw_wrong.sum().item()
            results['cw']['total'] += batch_size
        
        # 计算扰动距离
        for method, adv in [('fgsm', fgsm_adv), ('pgd', pgd_adv)]:
            delta = (adv - images).view(batch_size, -1)
            l2 = torch.norm(delta, p=2, dim=1).mean().item()
            linf = delta.abs().max(dim=1)[0].mean().item()
            results[method]['l2_distances'].append(l2)
            results[method]['linf_distances'].append(linf)
    
    # 输出评估报告
    print('=' * 60)
    print('对抗性攻击效果评估报告')
    print(f'扰动预算 epsilon = {epsilon}')
    print('=' * 60)
    
    for method_name, data in results.items():
        if data['total'] == 0:
            continue
        success_rate = 100 * data['success'] / data['total']
        avg_l2 = np.mean(data['l2_distances']) if data['l2_distances'] else 0
        avg_linf = np.mean(data['linf_distances']) if data['linf_distances'] else 0
        
        print(f'\n{method_name.upper()} 攻击:')
        print(f'  攻击成功率: {success_rate:.2f}% ({data["success"]}/{data["total"]})')
        print(f'  平均 L2 距离: {avg_l2:.4f}')
        print(f'  平均 L∞ 距离: {avg_linf:.4f}')
    
    return results
```

**关键评估指标解析**：

- **攻击成功率（Attack Success Rate, ASR）**：对抗性样本中被模型错误分类的比例。这是最直观的指标，但不能单独使用——一个成功的攻击如果需要极大的扰动，在实际中没有意义。
- **L2 距离**：扰动的欧几里得范数，衡量扰动的"总量"。L2 越小，扰动越难以被人类察觉。
- **L∞ 距离**：扰动的最大绝对值，衡量任何单个像素的最大偏移。L∞ 越小，每个像素的变化越小。
- **感知质量**：用 SSIM（结构相似性）或 PSNR（峰值信噪比）衡量对抗性样本与原始图像的视觉差异。

**不同 $\epsilon$ 下的攻击成功率曲线**：

```python
def epsilon_sweep(model, test_loader, method='pgd'):
    """绘制不同 epsilon 下的攻击成功率曲线"""
    epsilons = [0.005, 0.01, 0.015, 0.02, 0.025, 0.03, 0.04, 0.05, 0.07, 0.1]
    success_rates = []
    
    # 取一个固定批次进行测试
    images, labels = next(iter(test_loader))
    images, labels = images.to(device), labels.to(device)
    
    # 只保留正确分类的样本
    with torch.no_grad():
        clean_pred = model(images).argmax(dim=1)
        mask = clean_pred == labels
        images, labels = images[mask], labels[mask]
    
    for eps in epsilons:
        if method == 'fgsm':
            adv, _ = fgsm_attack(model, images, labels, epsilon=eps)
        else:
            adv = pgd_attack(model, images, labels, epsilon=eps, alpha=eps/4, num_iter=40)
        
        with torch.no_grad():
            pred = model(adv).argmax(dim=1)
            success = (pred != labels).float().mean().item()
            success_rates.append(success * 100)
        
        print(f'epsilon={eps:.3f}: 攻击成功率 {success*100:.1f}%')
    
    return epsilons, success_rates
```

典型的实验结果是：$\epsilon = 0.01$ 时攻击成功率很低（<10%），$\epsilon = 0.03$ 时中等（40-70%），$\epsilon = 0.1$ 时接近 100%。PGD 在相同 $\epsilon$ 下通常比 FGSM 高 10-20 个百分点。

#### 阶段四：物理世界验证

数字域的对抗性样本能否在物理世界中生效？这一步是将研究从实验室推向真实场景的关键跳跃，也是很多论文和实践中最容易被忽略的环节。

**物理世界攻击面临的挑战**：

1. **视角变换**：摄像头拍摄角度、距离不断变化，对抗性扰动在不同视角下的效果可能完全不同。
2. **光照变化**：白天/夜晚、阴天/晴天的光照差异会改变图像像素值，可能抵消扰动效果。
3. **打印/喷涂失真**：将数字扰动转化为物理媒介（打印贴纸、喷涂颜料）时，颜色和细节会失真。
4. **运动模糊**：车载摄像头的运动会导致图像模糊，模糊后的对抗性扰动可能失效。

**物理世界扰动应用函数**：

```python
def apply_perturbation_to_physical(image_path, perturbation, scale=0.5, 
                                     output_path='adversarial_sign.png'):
    """
    将数字域的对抗性扰动应用到物理世界的图像上
    
    参数:
        image_path: 原始交通标志的照片路径
        perturbation: 数字域生成的扰动张量
        scale: 扰动缩放因子（物理世界通常需要更大的扰动来补偿失真）
        output_path: 输出图像路径
    
    返回:
        PIL Image 对象
    """
    from PIL import Image, ImageEnhance
    import numpy as np
    
    # 加载原始图像
    image = Image.open(image_path).convert('RGB')
    image_np = np.array(image, dtype=np.float32)
    
    # 将扰动从张量格式转换为 numpy 数组
    if isinstance(perturbation, torch.Tensor):
        # 扰动是归一化的 [0,1] 范围，需要转换到 [0,255]
        pert_np = perturbation.detach().cpu().numpy()
        if pert_np.ndim == 4:
            pert_np = pert_np[0]  # 去掉批次维度
        pert_np = np.transpose(pert_np, (1, 2, 0))  # CHW -> HWC
        pert_np = pert_np * 255.0
    
    # 调整扰动大小以匹配目标图像
    from PIL import Image as PILImage
    pert_image = PILImage.fromarray(np.clip(pert_np, 0, 255).astype(np.uint8))
    pert_image = pert_image.resize(image.size, PILImage.BILINEAR)
    pert_np = np.array(pert_image, dtype=np.float32)
    
    # 应用缩放因子
    scaled_pert = pert_np * scale
    
    # 叠加扰动
    adversarial_np = np.clip(image_np + scaled_pert, 0, 255).astype(np.uint8)
    adversarial_image = Image.fromarray(adversarial_np)
    
    # 模拟打印效果（降低对比度和饱和度，模拟墨水扩散）
    enhancer = ImageEnhance.Contrast(adversarial_image)
    adversarial_image = enhancer.enhance(0.9)
    enhancer = ImageEnhance.Color(adversarial_image)
    adversarial_image = enhancer.enhance(0.95)
    
    adversarial_image.save(output_path)
    print(f'对抗性图像已保存到: {output_path}')
    print(f'扰动缩放因子: {scale}')
    print(f'图像尺寸: {adversarial_image.size}')
    
    return adversarial_image
```

### 防御方案详解

攻击不是目的，理解攻击是为了构建更好的防御。以下是四种主流防御方法的实现和分析。

#### 防御一：对抗性训练

对抗性训练（Adversarial Training）是最直接有效的防御方法。其核心思想是在训练过程中不断生成对抗性样本并将其加入训练集，让模型"见过"攻击并学会正确分类。

```python
def adversarial_training(model, train_loader, val_loader, epochs=50, 
                          epsilon=0.03, alpha=0.007, pgd_steps=7):
    """
    对抗性训练：使用 PGD 攻击生成的对抗性样本进行训练
    
    参数:
        epsilon: 训练时使用的扰动预算
        alpha: PGD 步长
        pgd_steps: PGD 迭代步数（训练时通常用较少步数以控制计算开销）
    """
    optimizer = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9, weight_decay=5e-4)
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)
    criterion = nn.CrossEntropyLoss()
    
    for epoch in range(epochs):
        model.train()
        total_loss = 0
        correct_clean = 0
        correct_adv = 0
        total = 0
        
        for batch_idx, (images, labels) in enumerate(train_loader):
            images, labels = images.to(device), labels.to(device)
            
            # 生成对抗性样本（使用 PGD）
            adv_images = pgd_attack(model, images, labels, 
                                     epsilon=epsilon, alpha=alpha, num_iter=pgd_steps)
            
            # 在对抗性样本上计算损失（也可混合干净样本和对抗性样本）
            outputs = model(adv_images)
            loss = criterion(outputs, labels)
            
            # 更新参数
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            # 统计
            total_loss += loss.item()
            total += labels.size(0)
            
            with torch.no_grad():
                clean_pred = model(images).argmax(dim=1)
                adv_pred = model(adv_images).argmax(dim=1)
                correct_clean += (clean_pred == labels).sum().item()
                correct_adv += (adv_pred == labels).sum().item()
            
            if batch_idx % 50 == 0:
                print(f'Epoch [{epoch+1}/{epochs}] Batch [{batch_idx}] '
                      f'Loss: {loss.item():.4f}')
        
        scheduler.step()
        
        clean_acc = 100 * correct_clean / total
        adv_acc = 100 * correct_adv / total
        print(f'Epoch [{epoch+1}/{epochs}] '
              f'干净准确率: {clean_acc:.2f}% | 对抗性鲁棒准确率: {adv_acc:.2f}%')
    
    return model
```

**对抗性训练的代价**：

- **计算开销**：每步训练需要多次前向/反向传播（PGD 的迭代次数），训练时间通常增加 3-10 倍。
- **干净准确率下降**：对抗性训练通常会导致干净样本的准确率下降 1-5 个百分点——这是一个鲁棒性-准确率的权衡。
- **不保证完全防御**：对抗性训练只能防御训练时使用的攻击方法和扰动预算。更强的攻击（更大的 $\epsilon$ 或不同的攻击方法）仍然可能成功。

#### 防御二：输入预处理

对输入图像进行预处理可以破坏对抗性扰动的精心结构，降低攻击效果。常用方法包括：

```python
class InputDefense:
    """输入预处理防御集合"""
    
    @staticmethod
    def jpeg_compression(image, quality=75):
        """JPEG 压缩：量化操作会抹除高频扰动信号"""
        from PIL import Image
        import io
        
        if isinstance(image, torch.Tensor):
            # 转换为 PIL Image
            image_np = (image.squeeze().cpu().numpy().transpose(1, 2, 0) * 255).astype(np.uint8)
            pil_image = Image.fromarray(image_np)
        else:
            pil_image = image
        
        # JPEG 压缩-解压缩
        buffer = io.BytesIO()
        pil_image.save(buffer, format='JPEG', quality=quality)
        buffer.seek(0)
        compressed = Image.open(buffer)
        
        # 转回张量
        result = torch.tensor(np.array(compressed), dtype=torch.float32).permute(2, 0, 1) / 255.0
        return result.unsqueeze(0) if image.dim() == 3 else result
    
    @staticmethod
    def spatial_smoothing(image, kernel_size=3):
        """空间平滑：用中值滤波去除小的高频扰动"""
        from torchvision.transforms import GaussianBlur
        
        blur = GaussianBlur(kernel_size=kernel_size, sigma=1.0)
        return blur(image)
    
    @staticmethod
    def bit_depth_reduction(image, bits=4):
        """位深度压缩：降低颜色精度，抹除精细扰动"""
        levels = 2 ** bits
        quantized = torch.round(image * (levels - 1)) / (levels - 1)
        return quantized
    
    @staticmethod
    def random_resizing(image, target_size=(224, 224), num_random=5):
        """随机缩放：多次随机缩放后取平均，扰动在缩放后会部分抵消"""
        from torchvision.transforms import Resize, InterpolationMode
        
        results = []
        for _ in range(num_random):
            # 随机缩放到稍大或稍小的尺寸
            scale = np.random.uniform(0.9, 1.1)
            temp_size = (int(target_size[0] * scale), int(target_size[1] * scale))
            resized = Resize(temp_size, interpolation=InterpolationMode.BILINEAR)(image)
            resized = Resize(target_size, interpolation=InterpolationMode.BILINEAR)(resized)
            results.append(resized)
        
        return torch.stack(results).mean(dim=0)
```

输入预处理的优势是计算开销小，不需要重新训练模型。但其防御效果通常不如对抗性训练，因为自适应攻击者可以在生成对抗性样本时将预处理步骤包含在攻击计算图中，从而绕过防御。

#### 防御三：对抗性样本检测

与其试图修复被扰动的输入，不如直接检测输入是否为对抗性样本，将其拦截在模型之外：

```python
class AdversarialDetector:
    """
    基于统计特征的对抗性样本检测器
    
    原理：对抗性样本的特征分布与干净样本存在统计差异。
    检测器通过统计特征（如主成分分析的特征值分布）来识别异常输入。
    """
    
    def __init__(self, model, clean_data_loader, threshold_percentile=95):
        self.model = model
        self.threshold = None
        self.clean_scores = []
        
        # 在干净数据上建立基线
        self._calibrate(clean_data_loader, threshold_percentile)
    
    def _compute_mahalanobis_score(self, image):
        """计算基于特征空间马氏距离的异常分数"""
        features = []
        
        # 提取中间层特征
        def hook_fn(module, input, output):
            features.append(output.detach())
        
        # 注册 hook 到倒数第二层
        handle = self.model.layer4.register_forward_hook(hook_fn)
        
        with torch.no_grad():
            self.model(image)
        
        handle.remove()
        
        # 计算特征统计量
        feat = features[0].mean(dim=[2, 3])  # 全局平均池化
        score = torch.norm(feat, p=2, dim=1).item()
        
        return score
    
    def _calibrate(self, data_loader, percentile):
        """在干净数据上校准检测阈值"""
        print('校准检测器...')
        for images, _ in data_loader:
            for i in range(images.size(0)):
                score = self._compute_mahalanobis_score(images[i:i+1].to(device))
                self.clean_scores.append(score)
        
        self.threshold = np.percentile(self.clean_scores, percentile)
        print(f'检测阈值 (第{percentile}百分位): {self.threshold:.4f}')
    
    def detect(self, image):
        """
        检测单张图像是否为对抗性样本
        
        返回: (is_adversarial: bool, score: float)
        """
        score = self._compute_mahalanobis_score(image)
        return score > self.threshold, score
```

检测方法的局限性在于：攻击者可以在生成对抗性样本时加入对检测器的规避约束（如对抗性训练中的"回避检测"损失），使得对抗性样本的统计特征与干净样本更加接近。这是一场持续的军备竞赛。

#### 防御四：认证防御

以上三种防御方法都是启发式的——它们在实践中有效，但无法提供数学上的安全保证。认证防御（Certified Defense）的目标是提供可证明的鲁棒性：对于给定的输入 $x$ 和扰动预算 $\epsilon$，证明模型在 $\epsilon$ 球内的所有扰动下都不会改变分类结果。

**随机平滑（Randomized Smoothing）**是目前最实用的认证防御方法。其核心思想是对输入添加高斯噪声后多次查询模型，用多数投票作为最终预测。

```python
import scipy.stats as stats

class RandomizedSmoothing:
    """
    随机平滑认证防御
    
    原理：对于基分类器 f，定义平滑分类器：
    g(x) = argmax_c P(f(x + δ) = c)，其中 δ ~ N(0, σ²I)
    
    通过采样估计 g(x) 的预测类别和置信度，
    然后用统计方法计算认证半径（在此半径内的 L2 扰动不会改变预测）。
    """
    
    def __init__(self, base_model, sigma=0.25, n_samples=1000, alpha=0.001):
        self.base_model = base_model
        self.sigma = sigma
        self.n_samples = n_samples
        self.alpha = alpha  # 认证失败的概率上界
    
    def certify(self, x):
        """
        对输入 x 进行认证
        
        返回:
            predicted_class: 预测类别
            radius: 认证鲁棒半径（L2 范数）
        """
        self.base_model.eval()
        
        # 计数每个类别的预测次数
        counts = torch.zeros(43, dtype=torch.long, device=device)
        
        for _ in range(self.n_samples):
            # 添加高斯噪声
            noise = torch.randn_like(x) * self.sigma
            noisy_x = x + noise
            
            with torch.no_grad():
                pred = self.base_model(noisy_x).argmax(dim=1)
                counts[pred] += 1
        
        # 取票数最多的类别
        top_class = counts.argmax().item()
        top_count = counts[top_class].item()
        
        # 计算第二名的票数
        counts[top_class] = 0
        second_count = counts.max().item()
        
        # 使用 Clopper-Pearson 置信区间计算认证半径
        # 如果 p_A > p_B（有统计显著性），则认证成功
        p_lower = stats.binom.ppf(1 - self.alpha, self.n_samples, top_count / self.n_samples)
        p_upper = stats.binom.ppf(self.alpha, self.n_samples, second_count / self.n_samples)
        
        if p_lower > p_upper:
            # 认证成功
            # 认证半径 = σ * Φ^{-1}(p_lower)
            radius = self.sigma * stats.norm.ppf(p_lower / self.n_samples)
        else:
            # 认证失败（无法确定预测的稳定性）
            radius = 0.0
        
        return top_class, radius
```

认证防御的优势在于提供了数学保证，而非经验性的安全。其代价是：认证半径通常较小（在 $L_2$ 范数下，$\epsilon = 0.25$ 的认证已经算不错），且计算开销较大（需要数千次模型查询）。

### 常见错误与实战经验

在实施对抗性攻击和防御时，以下是最常见的错误和对应的纠正方法：

#### 攻击侧常见错误

**错误一：忘记 `model.eval()`**

在训练模式下，BatchNorm 和 Dropout 会引入随机性，导致梯度方向不准确。攻击生成的扰动可能无法有效误导模型。

纠正：在每次攻击前确保调用 `model.eval()`，在对抗性训练循环中用 `model.train()` 训练、`model.eval()` 生成攻击样本。

**错误二：梯度累积未清理**

PyTorch 默认累积梯度。如果在多次攻击之间忘记调用 `model.zero_grad()`，梯度会叠加，导致扰动方向错误。

纠正：在每次反向传播前调用 `model.zero_grad()` 或 `optimizer.zero_grad()`。

**错误三：未裁剪像素范围**

对抗性样本添加扰动后可能超出 [0, 1] 的有效像素范围。如果不裁剪，模型接收到的是训练时从未见过的输入分布，攻击效果可能不真实。

纠正：每次生成对抗性样本后用 `torch.clamp(adv, 0, 1)` 裁剪。

**错误四：在错误分类的样本上测试攻击**

如果模型对某个样本本身就已经分类错误，攻击成功不代表什么。正确的做法是只在模型正确分类的样本上评估攻击。

纠正：在评估前过滤掉模型已经分错的样本。

#### 防御侧常见错误

**错误五：过拟合到特定攻击**

只用 FGSM 生成的对抗性样本做对抗性训练，模型可能对 FGSM 鲁棒但对 PGD 和 C&W 仍然脆弱。

纠正：使用多种攻击方法（PGD 为主）和多种 $\epsilon$ 进行对抗性训练。

**错误六：用迁移攻击验证防御**

在一个模型上生成的对抗性样本迁移效果不好，不代表该模型是安全的。迁移攻击是下界，白盒攻击才是上界。

纠正：始终用白盒攻击评估防御效果。

### 工具与框架

实施对抗性攻击和防御不需要从零实现所有算法。以下开源框架提供了经过充分测试的实现：

| 框架 | 维护方 | 特点 | 安装方式 |
|------|--------|------|---------|
| **CleverHans** | Google Brain / MILA | 学术界标准，算法实现权威 | `pip install cleverhans` |
| **Foolbox** | 团队开发 | API 简洁，支持 PyTorch/TensorFlow/JAX | `pip install foolbox` |
| **ART (Adversarial Robustness Toolbox)** | IBM | 功能最全面，包含攻击、防御、认证 | `pip install adversarial-robustness-toolbox` |
| **Torchattacks** | 韩国团队 | 专注 PyTorch，API 统一，实现清晰 | `pip install torchattacks` |
| **RobustBench** | 学术社区 | 标准化的鲁棒性基准测试排行榜 | `pip install robustbench` |

**使用 Torchattacks 的简洁示例**：

```python
import torchattacks

# 一行代码即可发起攻击
atk = torchattacks.PGD(model, eps=0.03, alpha=0.007, steps=40)
adv_images = atk(images, labels)

# 支持的攻击方法
# torchattacks.FGSM, PGD, CW, DeepFool, AutoAttack, BIM, ...
```

**AutoAttack**（由 Croce 和 Hein 提出）是目前公认的标准评估工具。它自动组合四种互补的攻击（APGD-CE、APGD-DLR、FAB、Square Attack），无需手动调参就能给出可靠的鲁棒性评估。如果只做一次评估，用 AutoAttack 就够了：

```python
from autoattack import AutoAttack

adversary = AutoAttack(model, norm='Linf', eps=0.03, version='standard')
adv_images = adversary.run_standard_evaluation(images, labels, bs=images.size(0))
```

### 进阶方向

本案例覆盖了对抗性攻击的基础流程。以下是值得深入探索的进阶方向：

**自适应攻击（Adaptive Attack）**：当防御方使用了多种防御组合时，攻击者需要针对具体防御设计自适应攻击。Tramer 等人在 2020 年的论文《On Adaptive Attacks to Adversarial Example Defenses》系统总结了绕过各种防御的方法。

**认证鲁棒性训练**：将随机平滑与对抗性训练结合，训练一个既经验鲁棒又可认证鲁棒的模型。Cohen 等人在 2019 年的工作首次在 ImageNet 上实现了可认证的鲁棒分类。

**对抗性样本在其他模态**：本案例聚焦图像分类，但对抗性样本存在于所有模态——文本（同义词替换、不可见字符注入）、音频（超声波注入）、点云（3D 物体点的扰动）都有对应的攻击方法。

**后门攻击（Backdoor Attack）**：与对抗性样本不同，后门攻击在训练阶段植入"触发器"——模型在正常输入下行为正常，但当输入包含特定模式（如角落的小标记）时输出攻击者指定的结果。这是供应链攻击在机器学习领域的体现。

### 本案例要点回顾

1. **白盒攻击流程**：加载模型 → 建立基线 → 生成对抗性样本 → 评估效果 → 物理世界验证
2. **三种攻击方法**：FGSM（快速）→ PGD（强）→ C&W（精细），根据场景选择
3. **评估不能只看成功率**：必须同时考虑扰动大小（L2/L∞）、感知质量、物理鲁棒性
4. **防御是军备竞赛**：没有一劳永逸的防御，对抗性训练 + 输入预处理 + 检测是目前最实用的组合
5. **物理世界验证是关键**：数字域成功的攻击在物理世界可能失效，必须实测验证
