## 七、实用别名和函数

Shell 别名（alias）和函数（function）是命令行效率的倍增器。对于安全从业者而言，精心设计的别名和函数不仅能大幅减少重复输入，更能将复杂的多步操作封装为一键完成的原子动作——在应急响应中争分夺秒，在渗透测试中保持隐蔽，在日常运维中减少人为失误。本节从原理机制出发，系统讲解别名与函数的设计哲学、分类体系和最佳实践，提供一套可直接落地的生产级配置方案。

### 7.1 别名与函数的本质

#### 7.1.1 Shell 命令解析顺序

当用户输入一条命令时，Bash 按以下优先级顺序解析：

```text
别名(alias) → 关键字(keyword) → 函数(function) → 内建命令(builtin) → 外部命令(hash/PATH)
```

这意味着别名的优先级最高，可以覆盖任何命令。理解这个顺序对安全至关重要——攻击者可以利用别名劫持（alias hijacking）植入恶意别名，将 `ls` 替换为会窃取密码的脚本。

```bash
# 验证命令解析顺序
type -a ls
# ls is aliased to 'ls --color=auto'    ← 别名优先
# ls is /usr/bin/ls                       ← 实际二进制

# 绕过别名执行原始命令
\ls                  # 反斜杠前缀
command ls           # command 内建
/usr/bin/ls          # 完整路径
'ls'                 # 引号包裹
```

#### 7.1.2 Alias 与 Function 的对比

| 特性 | Alias | Function |
|------|-------|----------|
| 语法 | `alias name='value'` | `name() { commands; }` |
| 参数传递 | 不支持（只能放在末尾） | 支持 `$1` `$2` `$@` `$#` 等 |
| 条件逻辑 | 不支持 | 支持 `if/else/case` |
| 循环 | 不支持 | 支持 `for/while/until` |
| 返回值 | 无 | 支持 `return` 和 `echo` 输出 |
| 局部变量 | 无 | 支持 `local` |
| 错误处理 | 无 | 支持 `trap`/`set -e` |
| 递归调用 | 不支持 | 支持 |
| 持久化 | 写入 `~/.bashrc` | 写入 `~/.bashrc` 或独立文件 |
| 加载方式 | `source ~/.bashrc` | `source` 或 `source ~/.bash_functions` |
| 性能 | 纯文本替换，零开销 | 需要 fork 子 shell，略慢 |
| 适用场景 | 简单命令缩写/默认参数 | 带逻辑的复合操作 |

**选择原则**：

- 只是给命令加默认参数或缩短名称 → 用 alias
- 需要参数传递、条件判断、错误处理 → 用 function
- 需要组合多个命令 → 用 function
- 安全敏感操作（需要参数校验）→ 用 function

```bash
# 适合用 alias 的场景
alias rm='rm -i'              # 加默认安全参数
alias grep='grep --color=auto' # 加默认显示参数
alias cls='clear'              # 简单缩写

# 适合用 function 的场景
# 需要参数：查找进程
fpid() { ps aux | grep -v grep | grep "$1"; }

# 需要逻辑：安全删除（带确认和回收站）
srm() {
    local target="$1"
    if [[ -z "$target" ]]; then
        echo "用法: srm <文件/目录>" >&2; return 1
    fi
    if [[ ! -e "$target" ]]; then
        echo "错误: '$target' 不存在" >&2; return 1
    fi
    mkdir -p ~/.trash
    mv "$target" ~/.trash/"$(basename "$target").$(date +%s)"
    echo "已移至 ~/.trash/"
}
```

#### 7.1.4 函数的高级特性

```bash
# 函数声明的两种方式（推荐第一种，POSIX 兼容）
myfunc() { echo "hello $1"; }
function myfunc { echo "hello $1"; }  # Bash 专属，不兼容 POSIX

# 局部变量（防止污染全局作用域）
demo() {
    local count=0           # 仅函数内可见
    local -i num=42         # 整数类型
    local -r CONST="固定值"  # 只读
    local -a arr=(a b c)    # 数组
    local -u upper="hello"  # 自动大写 → HELLO
    local -l lower="HELLO"  # 自动小写 → hello
}

# 返回值：两种方式
# 方式一：return 只能返回 0-255 的整数（通常 0=成功，非0=失败）
is_root() { [[ $(id -u) -eq 0 ]]; return $?; }

# 方式二：echo 输出，调用方用 $() 捕获
get_ip() { hostname -I | awk '{print $1}'; }
my_ip=$(get_ip)

# 陷阱（trap）用于清理
temp_work() {
    local tmpfile=$(mktemp)
    trap "rm -f '$tmpfile'" EXIT   # 函数退出时自动清理
    # ... 使用 tmpfile 做事 ...
}
```

### 7.4 高级函数模式

#### 7.4.1 带参数校验的安全函数

安全函数的核心是防御性编程：校验输入、处理错误、限制影响范围。

```bash
# 带完整参数校验的安全下载函数
safe-download() {
    # 参数校验
    if [[ $# -lt 1 ]]; then
        echo "用法: safe-download <URL> [输出目录]" >&2
        return 1
    fi

    local url="$1"
    local outdir="${2:-.}"

    # URL 格式校验
    if [[ ! "$url" =~ ^https?:// ]]; then
        echo "错误: 仅支持 HTTP/HTTPS URL" >&2
        return 1
    fi

    # 目录存在性校验
    if [[ ! -d "$outdir" ]]; then
        echo "错误: 目录 '$outdir' 不存在" >&2
        return 1
    fi

    # 下载并校验
    local filename=$(basename "$url")
    local outfile="${outdir}/${filename}"

    echo "正在下载: $url"
    if curl -fsSL --connect-timeout 10 --max-time 300 -o "$outfile" "$url"; then
        echo "下载成功: $outfile ($(du -h "$outfile" | cut -f1))"
        echo "SHA256: $(sha256sum "$outfile" | awk '{print $1}')"
    else
        echo "下载失败" >&2
        rm -f "$outfile"
        return 1
    fi
}

# 安全备份函数（带校验和回滚）
safe-backup() {
    local target="$1"
    local backup_dir="${2:-/tmp/backups}"

    if [[ -z "$target" || ! -e "$target" ]]; then
        echo "用法: safe-backup <文件或目录> [备份目录]" >&2
        return 1
    fi

    mkdir -p "$backup_dir"
    local basename=$(basename "$target")
    local timestamp=$(date +%Y%m%d_%H%M%S)
    local backup_file="${backup_dir}/${basename}.${timestamp}.tar.gz"

    tar czf "$backup_file" -C "$(dirname "$target")" "$basename" 2>/dev/null

    if [[ $? -eq 0 ]]; then
        echo "备份成功: $backup_file ($(du -h "$backup_file" | cut -f1))"
        echo "SHA256: $(sha256sum "$backup_file" | awk '{print $1}')"
    else
        echo "备份失败" >&2
        return 1
    fi
}
```

```bash
# 快速端口扫描（单个目标）
fast-scan() {

    done

# 网络连接监控（检测新建连接）
watch-connections() {
    local interval="${1:-5}"
    echo "每 ${interval} 秒检测新连接（Ctrl+C 退出）"
    local prev=$(ss -tnp state established | md5sum)
    while true; do
        sleep "$interval"
        local curr=$(ss -tnp state established | md5sum)
        if [[ "$prev" != "$curr" ]]; then
            echo "=== $(date) 检测到连接变化 ==="
            diff <(echo "$prev") <(echo "$curr")
            ss -tnp state established | grep -v "127.0.0.1\|::1"
            prev=$curr
        fi
    done
}
```

### 7.6 常见误区与排错

#### 7.6.1 别名陷阱

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 别名中管道不生效 | 别名只做文本替换，不解析管道 | 改用函数 |
| 别名内使用变量时取值错误 | 单引号延迟展开，双引号立即展开 | 根据需求选择引号 |
| sudo 别名不生效 | sudo 后命令不再查找别名 | `alias sudo='sudo '` （末尾加空格） |
| 脚本中别名无效 | 非交互式 shell 不加载别名 | 在脚本中用函数代替，或 `shopt -s expand_aliases` |
| 别名递归导致无限循环 | alias rm='rm -i' 后 `rm` 死循环 | 用 `\rm` 或 `command rm` 绕过 |

```bash
# sudo 别名陷阱详解
alias sudo='sudo '   # 末尾空格让 bash 继续解析后续别名
# 这样 sudo ll 才能展开为 sudo ls -alh

# 脚本中启用别名（非默认行为）
#!/bin/bash
shopt -s expand_aliases
source ~/.bash_aliases
ll   # 现在可以用了

# 引号陷阱
alias show1='echo $HOME'    # 单引号：每次执行时展开 $HOME → 正确
alias show2="echo $HOME"    # 双引号：定义时就展开 → 永远是定义时的值
# 区分：需要延迟展开用单引号，需要立即展开用双引号

# 别名中使用管道的正确方式
# 错误：alias mygrep='cat file | grep'    # 不工作
# 正确：用函数
mygrep() { cat file | grep "$@"; }
```

#### 7.6.2 函数常见错误

```bash
# 错误：忘记 local 导致变量污染全局
bad_func() {
    result=$(some_command)    # result 会污染全局作用域
}
good_func() {
    local result=$(some_command)    # 仅函数内可见
}

# 错误：未检查命令是否存在
bad_func() {
    docker ps    # docker 不存在时直接报错
}
good_func() {
    command -v docker >/dev/null 2>&1 || { echo "docker 未安装" >&2; return 1; }
    docker ps
}

# 错误：未处理空参数
bad_func() {
    rm -rf "$1/*"    # $1 为空时变成 rm -rf /*
}
good_func() {
    [[ -z "${1:-}" ]] && { echo "缺少参数" >&2; return 1; }
    rm -rf "$1"/*
}

# 错误：未使用 -- 终止选项解析
bad_func() { ls "$1"; }         # ls -l 会当作选项
good_func() { ls -- "$1"; }     # -- 后的参数都是文件名
```

### 7.8 小结

别名和函数是 Shell 效率工具链的基石。掌握它们的关键要点：

1. **选对工具**：简单缩写用 alias，带逻辑用 function，复杂场景用独立脚本
2. **安全优先**：`rm -I` 代替裸 `rm`，函数内校验参数，敏感命令不记录历史
3. **模块组织**：别名/函数/环境变量分文件管理，按功能分组，条件加载
4. **防御编程**：`local` 变量、参数校验、错误处理、`set -euo pipefail`
5. **持续迭代**：将反复使用的命令序列封装为函数，随工作流演进优化

> 实践建议：不要一次性部署所有别名。先从最常用的 5-10 个开始，用熟后再逐步扩展。每个别名/函数都应该有明确的使用场景，避免配置臃肿导致难以记忆和维护。
