## 练习一：手写类型推断引擎（预计60分钟）

### 目标

用 Python 实现一个简化版的 Hindley-Milner 类型推断引擎，能够自动推导简单表达式的类型。通过亲手编写合一算法和类型推导逻辑，深入理解类型系统的工作原理——这比阅读任何教材都更有效。

### 知识回顾

在开始之前，回顾本章理论基础中的关键概念：

- **类型变量**：表示未知类型，如 `α`、`β`
- **类型构造器**：从已有类型构造新类型，如函数类型 `→`、列表类型 `List`
- **合一（Unification）**：找到一个替换 θ 使得两个类型在 θ 下相同
- **泛化与实例化**：处理多态类型的核心机制

#### 第二步：实现合一算法（15分钟）

合一算法是类型推断的引擎。它的核心逻辑是：遍历两个类型结构，逐步建立类型变量到具体类型的绑定关系：

```python
class UnificationError(Exception):
    pass

def occurs_check(var: TypeVar, typ: Type) -> bool:
    """检查类型变量 var 是否出现在 typ 中（防止无穷类型）"""
    if isinstance(typ, TypeVar):
        if typ is var or typ.bound:
            return occurs_check(var, typ.bound) if typ.bound else (typ is var)
        return False
    elif isinstance(typ, FunType):
        return occurs_check(var, typ.param) or occurs_check(var, typ.ret)
    return False

def unify(t1: Type, t2: Type, subst: Dict[str, Type] = None) -> Dict[str, Type]:
    """
    合一两个类型，返回替换映射。
    
    算法核心：
    1. 解引用：沿 TypeVar.bound 链找到最终绑定
    2. 变量绑定：将类型变量绑定到另一个类型
    3. 结构匹配：递归合一类构造器的子类型
    """
    if subst is None:
        subst = {}

    # 解引用：沿 bound 链展开
    def deref(t: Type) -> Type:
        while isinstance(t, TypeVar) and t.bound:
            t = t.bound
        return t

    t1, t2 = deref(t1), deref(t2)

    # 相同类型：直接返回
    if type(t1) == type(t2):
        if isinstance(t1, TypeVar):
            if t1 is t2:
                return subst
        elif isinstance(t1, FunType):
            subst = unify(t1.param, t2.param, subst)
            subst = unify(t1.ret, t2.ret, subst)
            return subst
        else:
            return subst  # Int==Int, Bool==Bool

    # 类型变量绑定
    if isinstance(t1, TypeVar):
        if occurs_check(t1, t2):
            raise UnificationError(f"occurs check failed: {t1.name} in {t2}")
        t1.bound = t2
        subst[t1.name] = t2
        return subst

    if isinstance(t2, TypeVar):
        if occurs_check(t2, t1):
            raise UnificationError(f"occurs check failed: {t2.name} in {t1}")
        t2.bound = t1
        subst[t2.name] = t1
        return subst

    raise UnificationError(f"cannot unify: {t1} with {t2}")
```

#### 第三步：实现简单的类型推断器（20分钟）

实现一个处理简单表达式的类型推断器。支持的表达式语法：

expr := number              -- 整数字面量
      | true | false        -- 布尔字面量
      | variable            -- 变量引用
      | λx. body            -- Lambda 抽象（匿名函数）
      | expr1 expr2          -- 函数应用
      | let x = e1 in e2    -- Let 绑定

```python
# --- 表达式 AST ---

@dataclass
class Num:
    value: int

@dataclass
class Boolean:
    value: bool

@dataclass
class Var:
    name: str

@dataclass
class Lambda:
    param: str
    body: 'Expr'

@dataclass
class App:
    func: 'Expr'
    arg: 'Expr'

@dataclass
class Let:
    name: str
    value: 'Expr'
    body: 'Expr'

Expr = Num | Boolean | Var | Lambda | App | Let

# --- 类型推断器 ---

class TypeInferencer:
    def __init__(self):
        self.var_counter = 0

    def new_var(self) -> TypeVar:
        """生成全新的类型变量"""
        self.var_counter += 1
        return TypeVar(f"t{self.var_counter}")

    def infer(self, expr: Expr, env: Dict[str, Type]) -> Type:
        """
        核心推断函数。对应 Algorithm W 的简化版本。
        
        参数 env 是当前的类型环境（变量 → 类型的映射）。
        返回推断出的类型。
        """
        if isinstance(expr, Num):
            return IntType()

        if isinstance(expr, Boolean):
            return BoolType()

        if isinstance(expr, Var):
            if expr.name not in env:
                raise UnificationError(f"undefined variable: {expr.name}")
            return env[expr.name]

        if isinstance(expr, Lambda):
            param_type = self.new_var()
            body_env = {**env, expr.param: param_type}
            body_type = self.infer(expr.body, body_env)
            return FunType(param_type, body_type)

        if isinstance(expr, App):
            func_type = self.infer(expr.func, env)
            arg_type = self.infer(expr.arg, env)
            result_type = self.new_var()
            # 函数应用的关键：func_type 必须是 arg_type → result_type
            unify(func_type, FunType(arg_type, result_type))
            return result_type

        if isinstance(expr, Let):
            # let 的 value 部分在当前环境推断
            val_type = self.infer(expr.value, env)
            # 将绑定引入环境
            body_env = {**env, expr.name: val_type}
            return self.infer(expr.body, body_env)

        raise ValueError(f"unknown expression: {expr}")

    def infer_and_show(self, expr: Expr, env: Dict[str, Type] = None) -> str:
        """推断类型并返回可读字符串"""
        if env is None:
            env = {}
        self.var_counter = 0  # 重置变量计数器
        # 重置所有类型变量的 bound
        self._reset_bounds(expr)
        typ = self.infer(expr, env)
        return str(typ)

    def _reset_bounds(self, expr: Expr):
        """重置表达式中所有类型变量的绑定状态"""
        # 简化实现：实际中需要遍历 AST 收集所有 TypeVar
        pass
```

## 练习二：构造 SSA 形式（预计75分钟）

### 目标

从一个简单的中间表示（IR）出发，手动实现 SSA 构造的完整流程：构建控制流图（CFG）→ 计算支配关系 → 识别支配边界 → 插入 φ 函数 → 变量重命名。这是理解现代编译器（LLVM、V8、HotSpot）核心 IR 的最佳实践。

### 知识回顾

- **SSA（Static Single Assignment）**：每个变量只被赋值一次的 IR 形式
- **支配（Domination）**：从入口到节点 n 的所有路径都经过节点 d，则 d 支配 n
- **支配边界（Dominance Frontier）**：节点 n 的支配边界是这样的节点集合——n 支配其某些前驱，但不严格支配 n 本身
- **φ 函数**：在控制流汇合点合并同一变量的不同版本

### 步骤

#### 第一步：定义简单的 IR（10分钟）

创建文件 `ssa_builder.py`：

```python
from dataclasses import dataclass, field
from typing import List, Dict, Set, Optional

@dataclass
class IRInstruction:
    """简单的三地址码 IR 指令"""
    op: str          # 操作: 'assign', 'add', 'sub', 'mul', 'if_goto', 'goto', 'label', 'param', 'call'
    dest: str = ""   # 目标变量
    arg1: str = ""   # 操作数1
    arg2: str = ""   # 操作数2
    label: str = ""  # 标签（用于 goto/if_goto/label）

    def __repr__(self):
        if self.op == 'label':
            return f"{self.label}:"
        if self.op == 'goto':
            return f"goto {self.label}"
        if self.op == 'if_goto':
            return f"if {self.arg1} goto {self.label}"
        if self.op == 'assign':
            return f"{self.dest} = {self.arg1}"
        if self.op in ('add', 'sub', 'mul'):
            return f"{self.dest} = {self.arg1} {self.op} {self.arg2}"
        return f"{self.op} {self.dest} {self.arg1} {self.arg2}"

@dataclass
class BasicBlock:
    """基本块"""
    label: str
    instructions: List[IRInstruction] = field(default_factory=list)
    predecessors: List['BasicBlock'] = field(default_factory=list)
    successors: List['BasicBlock'] = field(default_factory=list)

    def __repr__(self):
        return f"Block({self.label})"

@dataclass
class CFG:
    """控制流图"""
    blocks: Dict[str, BasicBlock] = field(default_factory=dict)
    entry: Optional[BasicBlock] = None

def build_cfg(instructions: List[IRInstruction]) -> CFG:
    """
    从指令序列构建控制流图。
    
    算法：
    1. 找到所有 label 指令作为基本块的起始点
    2. 将指令序列切分为基本块
    3. 连接前驱/后继边
    """
    cfg = CFG()
    
    # 第一步：识别领导指令（label 和第一条指令）
    leaders = {0}  # 第一条指令总是领导指令
    label_map = {}  # label -> block_name
    
    for i, instr in enumerate(instructions):
        if instr.op == 'label':
            leaders.add(i)
            label_map[instr.label] = instr.label
        elif instr.op in ('goto', 'if_goto'):
            # 跳转目标是领导指令
            # 找到目标 label 的位置
            for j, other in enumerate(instructions):
                if other.op == 'label' and other.label == instr.label:
                    leaders.add(j)
                    break
            # 跳转后的下一条指令也是领导指令
            if i + 1 < len(instructions):
                leaders.add(i + 1)

    # 第二步：切分基本块
    sorted_leaders = sorted(leaders)
    block_ranges = []
    for i, start in enumerate(sorted_leaders):
        end = sorted_leaders[i + 1] if i + 1 < len(sorted_leaders) else len(instructions)
        block_ranges.append((start, end))

    block_name_counter = 0
    leader_to_block = {}  # leader_index -> block

    for start, end in block_ranges:
        block_label = f"B{block_name_counter}"
        block_name_counter += 1
        block = BasicBlock(label=block_label)
        
        # 添加属于该块的指令（跳过 label 指令）
        for i in range(start, end):
            instr = instructions[i]
            if instr.op != 'label':  # label 只作为块标识
                block.instructions.append(instr)
        
        if block.instructions:  # 空块不添加
            cfg.blocks[block_label] = block
            leader_to_block[start] = block
    
    # 第一步的入口块
    if 0 in leader_to_block:
        cfg.entry = leader_to_block[0]

    # 第三步：连接前驱/后继
    for start, end in block_ranges:
        if start not in leader_to_block:
            continue
        block = leader_to_block[start]
        if not block.instructions:
            continue
        
        last_instr = block.instructions[-1]
        
        if last_instr.op == 'goto':
            # 找到目标块
            target_block = _find_block_by_label(cfg, last_instr.label, instructions)
            if target_block:
                block.successors.append(target_block)
                target_block.predecessors.append(block)
        
        elif last_instr.op == 'if_goto':
            # 条件跳转有两个后继
            target_block = _find_block_by_label(cfg, last_instr.label, instructions)
            if target_block:
                block.successors.append(target_block)
                target_block.predecessors.append(block)
            # fall-through 后继
            fall_idx = end
            if fall_idx in leader_to_block:
                fall_block = leader_to_block[fall_idx]
                if fall_block not in block.successors:
                    block.successors.append(fall_block)
                    fall_block.predecessors.append(block)
        
        else:
            # 顺序执行，fall-through 到下一个块
            fall_idx = end
            if fall_idx in leader_to_block:
                fall_block = leader_to_block[fall_idx]
                block.successors.append(fall_block)
                fall_block.predecessors.append(block)

    return cfg

def _find_block_by_label(cfg: CFG, label: str, instructions: List[IRInstruction]) -> Optional[BasicBlock]:
    """根据 label 名找到对应的块"""
    for i, instr in enumerate(instructions):
        if instr.op == 'label' and instr.label == label:
            # 找到该 label 的起始位置对应的块
            for block in cfg.blocks.values():
                # 检查块是否包含该 label 对应的指令
                for b_instr in block.instructions:
                    if b_instr.label == label and b_instr.op in ('goto', 'if_goto'):
                        if any(succ.label == label for succ in block.successors):
                            pass
            break
    # 简化：直接用 label 查找
    for block in cfg.blocks.values():
        for instr in instructions:
            if instr.op == 'label' and instr.label == label:
                if block.label == label:
                    return block
    return None
```

#### 第二步：计算支配关系（15分钟）

```python
def compute_dominators(cfg: CFG) -> Dict[str, Set[str]]:
    """
    使用迭代算法计算每个块的支配者集合。
    
    核心数据流方程：
      dom(B) = {B} ∪ (∩ dom(P))  其中 P 是 B 的所有前驱
      dom(Entry) = {Entry}
    
    算法从所有块的 dominator = 全集开始，逐轮收窄直到不动点。
    """
    if not cfg.entry:
        return {}
    
    all_blocks = set(cfg.blocks.keys())
    entry_label = cfg.entry.label
    
    # 初始化：所有块的支配者为全集
    dom = {label: set(all_blocks) for label in all_blocks}
    dom[entry_label] = {entry_label}
    
    changed = True
    while changed:
        changed = False
        for label, block in cfg.blocks.items():
            if label == entry_label:
                continue
            
            # dom(B) = {B} ∪ (∩ dom(P))
            if block.predecessors:
                new_dom = set.intersection(*[dom[p.label] for p in block.predecessors])
            else:
                new_dom = set()
            new_dom.add(label)
            
            if new_dom != dom[label]:
                dom[label] = new_dom
                changed = True
    
    return dom

def compute_idom(cfg: CFG, dom: Dict[str, Set[str]]) -> Dict[str, str]:
    """
    计算直接支配者（immediate dominator）。
    
    idom(B) = 那个严格支配 B 且被所有其他严格支配 B 的节点所支配的节点。
    简化算法：从 dom(B) 中排除 B 自身，idom 就是这些节点中被其余节点支配的那一个。
    """
    idom = {}
    for label, dom_set in dom.items():
        strict_dom = dom_set - {label}
        if not strict_dom:
            idom[label] = None  # entry 节点
            continue
        # idom 是被 strict_dom 中所有其他节点所支配的节点
        for candidate in strict_dom:
            # candidate 被所有其他 strict_dom 中的节点支配？
            others = strict_dom - {candidate}
            if all(candidate in dom[other] for other in others):
                idom[label] = candidate
                break
    return idom
```

#### 第三步：计算支配边界并插入 φ 函数（25分钟）

```python
def compute_dominance_frontiers(cfg: CFG, idom: Dict[str, str]) -> Dict[str, Set[str]]:
    """
    计算支配边界（Dominance Frontier）。
    
    DF(B) = {N ∈ succ(B) | B 不严格支配 N}
           ∪ {N ∈ succ(X) | B 严格支配 X, X ≠ B, 且 B 不严格支配 N 的所有前驱}
    
    实际使用 Lengauer-Tarjan 或简单的迭代算法。
    这里使用基于 idom 的直接计算：
    """
    df = {label: set() for label in cfg.blocks}
    
    for label, block in cfg.blocks.items():
        if len(block.predecessors) < 2:
            # 只有多个前驱的块才可能在支配边界中
            continue
        
        # 控制流汇合点：检查每个前驱
        for pred in block.predecessors:
            runner = pred.label
            # 从前驱沿 idom 链向上走，直到遇到当前块
            while runner is not None and runner != idom.get(label):
                df[runner].add(label)
                runner = idom.get(runner)
    
    return df

def get_variable_uses(block: BasicBlock) -> Set[str]:
    """获取块中使用的变量（右值）"""
    uses = set()
    for instr in block.instructions:
        if instr.op in ('add', 'sub', 'mul'):
            if not instr.arg1[0].isdigit():  # 排除常量
                uses.add(instr.arg1)
            if not instr.arg2[0].isdigit():
                uses.add(instr.arg2)
        elif instr.op == 'assign':
            if not instr.arg1[0].isdigit():
                uses.add(instr.arg1)
        elif instr.op == 'if_goto':
            uses.add(instr.arg1)
    return uses

def get_variable_defs(block: BasicBlock) -> Set[str]:
    """获取块中定义的变量（左值）"""
    defs = set()
    for instr in block.instructions:
        if instr.op in ('assign', 'add', 'sub', 'mul') and instr.dest:
            defs.add(instr.dest)
    return defs

def insert_phi_functions(cfg: CFG, df: Dict[str, Set[str]]) -> Dict[str, Dict[str, List[str]]]:
    """
    使用经典算法插入 φ 函数。
    
    算法（Cytron et al., 1991）：
    1. 对每个变量 v，找到所有定义 v 的块 defs(v)
    2. 用 worklist 算法处理支配边界：
       - 对 defs(v) 中的每个块 B
       - 遍历 B 的支配边界 F
       - 如果 F 尚未有 v 的 φ 函数，插入 φ 函数，将 F 加入 defs(v)
       - 重复直到 worklist 为空
    """
    phi_insertions = {}  # block_label -> {var_name -> [phi_definitions]}
    
    # 收集每个块定义和使用的变量
    all_vars = set()
    block_defs = {}
    block_uses = {}
    for label, block in cfg.blocks.items():
        defs = get_variable_defs(block)
        uses = get_variable_uses(block)
        block_defs[label] = defs
        block_uses[label] = uses
        all_vars |= defs | uses
    
    # 对每个变量执行 φ 函数插入
    for var in all_vars:
        # 找到定义 var 的块集合
        defs_v = {label for label, defs in block_defs.items() if var in defs}
        has_phi = set()  # 已经插入 φ(var) 的块
        worklist = list(defs_v)
        
        while worklist:
            block_label = worklist.pop()
            for df_block in df.get(block_label, set()):
                if df_block not in has_phi:
                    has_phi.add(df_block)
                    # 插入 φ 函数
                    if df_block not in phi_insertions:
                        phi_insertions[df_block] = {}
                    phi_insertions[df_block][var] = []  # 空的来源列表，后续填充
                    # φ 函数本身也是一个定义
                    if df_block not in defs_v:
                        worklist.append(df_block)
    
    return phi_insertions
```

```python
def rename_variables(cfg: CFG, phi_insertions: Dict[str, Dict[str, List[str]]]) -> List[IRInstruction]:
    """
    SSA 构造的最后一步：变量重命名。
    
    使用深度优先遍历支配树，为每个变量维护一个版本栈。
    每次遇到变量定义时递增版本号。
    """
    # 收集所有指令（包括插入的 φ 函数）
    ssa_instructions = []
    
    for label, block in cfg.blocks.items():
        # 先输出块标签
        ssa_instructions.append(IRInstruction(op='label', label=label))

        if label in phi_insertions:

                    IRInstruction(op='phi', dest=f"{var}_φ", label=f"φ({', '.join(sources) if sources else '...'})")
                )
        
        # 输出原始指令（简化：不做实际重命名，标记出版本即可）
        for instr in block.instructions:
            ssa_instructions.append(instr)
    
    return ssa_instructions
```

#### 实验四：别名分析的影响（15分钟）

```c
#include <stdio.h>

// 场景 1：编译器不知道指针是否重叠
void sum1(int *a, int *b, int n, int *result) {
    for (int i = 0; i < n; i++) {
        result[i] = a[i] + b[i];
    }
}

// 场景 2：使用 restrict 关键字告诉编译器无重叠
void sum2(int *restrict a, int *restrict b, int n, int *restrict result) {
    for (int i = 0; i < n; i++) {
        result[i] = a[i] + b[i];
    }
}

// 场景 3：编译器能证明无重叠（栈上独立数组）
void sum3(int n, int *result) {
    int a[100], b[100];
    for (int i = 0; i < n; i++) {
        a[i] = i;
        b[i] = i * 2;
    }
    for (int i = 0; i < n; i++) {
        result[i] = a[i] + b[i];
    }
}
```

**观察要点**：

- **sum1 vs sum2**：没有 `restrict` 时，编译器必须保守地假设 `a`、`b`、`result` 可能重叠，因此不能将 `result[i] = a[i] + b[i]` 向量化（因为写入 `result[i]` 可能影响后续的 `a[i+1]` 或 `b[i+1]`）。加上 `restrict` 后，编译器可以放心向量化
- **sum3**：即使没有 `restrict`，编译器也能证明 `a` 和 `b` 是栈上独立数组，因此同样可以向量化
- **实践意义**：在性能敏感的数值计算代码中，使用 `__restrict__`（GCC/Clang）或 `_restrict`（MSVC）是免费的性能提升

## 练习四：实现简单常量传播（预计60分钟）

### 目标

实现一个基于 SSA 的稀疏条件常量传播（Sparse Conditional Constant Propagation, SCCP）算法。SCCP 是 LLVM 和 GCC 中广泛使用的经典优化，它结合了常量传播和不可达代码消除，比简单的迭代常量传播更精确。

### 知识回顾

- **常量传播**：如果变量 x 被赋值为常量 `5`，则后续所有使用 x 的地方都替换为 `5`
- **条件常量传播**：如果 `if (const_false)` 为假，则 `then` 分支不可达，其中的定值不参与分析
- **稀疏**：只分析实际相关的变量，而非全局遍历
- **格结构**：⊤（未确定）→ 常量值（确定）→ ⊥（非常量/冲突）

### 步骤

#### 第一步：定义 IR 和格结构（10分钟）

创建文件 `sccp.py`：

```python
from dataclasses import dataclass, field
from typing import Dict, List, Optional, Set, Tuple
from enum import Enum, auto

# --- 格（Lattice）---

class LatticeValue(Enum):
    """SCCP 格：⊤（未访问）→ 常量 → ⊥（非常量）"""
    TOP = auto()     # ⊤: 尚未确定（初始状态）
    CONST = auto()   # 常量值
    BOTTOM = auto()   # ⊥: 非常量（如变量合并为不同常量）

@dataclass
class Lattice:
    """单个变量在格中的状态"""
    state: LatticeValue = LatticeValue.TOP
    const_value: Optional[int] = None  # 仅在 state==CONST 时有效

    def join(self, other: 'Lattice') -> 'Lattice':
        """合并操作：两个格值的 join"""
        if self.state == LatticeValue.TOP:
            return Lattice(other.state, other.const_value)
        if other.state == LatticeValue.TOP:
            return Lattice(self.state, self.const_value)
        if self.state == LatticeValue.BOTTOM or other.state == LatticeValue.BOTTOM:
            return Lattice(LatticeValue.BOTTOM)
        # 两个都是 CONST：值相同则保持，不同则降为 BOTTOM
        if self.const_value == other.const_value:
            return Lattice(LatticeValue.CONST, self.const_value)
        return Lattice(LatticeValue.BOTTOM)

    def __repr__(self):
        if self.state == LatticeValue.TOP:
            return "⊤"
        if self.state == LatticeValue.BOTTOM:
            return "⊥"
        return str(self.const_value)

# --- IR 定义 ---

@dataclass
class SCCPInstruction:
    """简化 IR 指令"""
    op: str       # 'const', 'add', 'sub', 'mul', 'phi', 'branch', 'jmp'
    dest: str = ""
    args: List[str] = field(default_factory=list)
    true_label: str = ""
    false_label: str = ""
    target_label: str = ""

    def __repr__(self):
        if self.op == 'const':
            return f"{self.dest} = const {self.args[0]}"
        if self.op in ('add', 'sub', 'mul'):
            return f"{self.dest} = {self.args[0]} {self.op} {self.args[1]}"
        if self.op == 'phi':
            return f"{self.dest} = phi({', '.join(self.args)})"
        if self.op == 'branch':
            return f"branch {self.args[0]} -> {self.true_label}, {self.false_label}"
        if self.op == 'jmp':
            return f"jmp {self.target_label}"
        return f"{self.op} ..."

@dataclass
class SCCPBlock:
    label: str
    instructions: List[SCCPInstruction] = field(default_factory=list)
    predecessors: List['SCCPBlock'] = field(default_factory=list)
    successors: List['SCCPBlock'] = field(default_factory=list)

@dataclass
class SCCPCFG:
    blocks: Dict[str, SCCPBlock] = field(default_factory=dict)
    entry: Optional[SCCPBlock] = None
```

#### 第二步：实现 SCCP 算法（25分钟）

```python
class SCCPOptimizer:
    """
    稀疏条件常量传播（SCCP）算法实现。
    
    核心思想：
    1. 维护一个格（Lattice）记录每个变量的确定性
    2. 使用 worklist 驱动分析，只处理发生变化的变量
    3. 结合可达性分析：如果条件分支的条件是常量 false，
       则 then 分支不可达，其中的赋值不影响结果
    
    算法保证在有限步内收敛（格高度有限，每次操作严格单调下降）。
    """
    
    def __init__(self, cfg: SCCPCFG):
        self.cfg = cfg
        self.lattice: Dict[str, Lattice] = {}      # 变量 → 格值
        self.executed: Dict[str, bool] = {}          # 块是否可达
        self.executed_edges: Set[Tuple[str, str]] = set()  # 已执行的边
        self.worklist: List[str] = []                 # 待处理块
        self.edge_worklist: List[Tuple[str, str]] = []  # 待处理边
    
    def evaluate(self, instr: SCCPInstruction) -> Optional[int]:
        """
        在当前格状态下，尝试计算指令结果。
        如果能确定是常量，返回常量值；否则返回 None（表示非常量）。
        """
        if instr.op == 'const':
            return int(instr.args[0])
        
        if instr.op in ('add', 'sub', 'mul'):
            lhs = self.lattice.get(instr.args[0])
            rhs = self.lattice.get(instr.args[1])
            
            if not lhs or not rhs:
                return None
            
            if lhs.state != LatticeValue.CONST or rhs.state != LatticeValue.CONST:
                return None  # 至少一个操作数不是常量
            
            a, b = lhs.const_value, rhs.const_value
            if instr.op == 'add':
                return a + b
            elif instr.op == 'sub':
                return a - b
            elif instr.op == 'mul':
                return a * b
        
        if instr.op == 'phi':
            # phi 函数的值是所有已执行前驱提供的值的 join
            result = Lattice(LatticeValue.TOP)
            for arg in instr.args:
                val = self.lattice.get(arg)
                if val:
                    result = result.join(val)
            if result.state == LatticeValue.CONST:
                return result.const_value
            return None
        
        return None
    
    def run(self) -> Dict[str, Lattice]:
        """执行 SCCP 分析"""
        if not self.cfg.entry:
            return self.lattice
        
        # 初始化
        self.executed[self.cfg.entry.label] = True
        
        # 将入口块加入 worklist
        self.worklist.append(self.cfg.entry.label)
        
        while self.worklist or self.edge_worklist:
            # 处理边 worklist（可达性更新）
            while self.edge_worklist:
                src, dst = self.edge_worklist.pop(0)
                if (src, dst) not in self.executed_edges:
                    self.executed_edges.add((src, dst))
                    if dst not in self.executed or not self.executed[dst]:
                        self.executed[dst] = True
                        self.worklist.append(dst)
            
            # 处理块 worklist
            while self.worklist:
                block_label = self.worklist.pop(0)
                block = self.cfg.blocks.get(block_label)
                if not block:
                    continue
                
                for instr in block.instructions:
                    self._process_instruction(block, instr)
        
        return self.lattice
    
    def _process_instruction(self, block: SCCPBlock, instr: SCCPInstruction):
        """处理单条指令"""
        if instr.op in ('add', 'sub', 'mul', 'const'):
            result = self.evaluate(instr)
            if result is not None:
                new_lattice = Lattice(LatticeValue.CONST, result)
            else:
                new_lattice = Lattice(LatticeValue.BOTTOM)
            
            old = self.lattice.get(instr.dest, Lattice(LatticeValue.TOP))
            merged = old.join(new_lattice)
            
            if merged.state != old.state or (
                merged.state == LatticeValue.CONST and 
                old.state == LatticeValue.CONST and 
                merged.const_value != old.const_value
            ):
                self.lattice[instr.dest] = merged
                # 通知所有使用该变量的块
                self._notify_users(instr.dest)
        
        elif instr.op == 'phi':
            old = self.lattice.get(instr.dest, Lattice(LatticeValue.TOP))
            new_val = self.evaluate(instr)
            if new_val is not None:
                merged = old.join(Lattice(LatticeValue.CONST, new_val))
            else:
                merged = old.join(Lattice(LatticeValue.BOTTOM))
            
            if merged.state != old.state:
                self.lattice[instr.dest] = merged
                self._notify_users(instr.dest)
        
        elif instr.op == 'branch':
            # 条件分支：检查条件是否为常量
            cond_val = self.lattice.get(instr.args[0])
            
            if cond_val and cond_val.state == LatticeValue.CONST:
                # 条件已知：只添加可达的后继
                if cond_val.const_value != 0:
                    # 条件为真：只走 true 分支
                    self.edge_worklist.append((block.label, instr.true_label))
                else:
                    # 条件为假：只走 false 分支
                    self.edge_worklist.append((block.label, instr.false_label))
            else:
                # 条件未知：两个分支都可达
                self.edge_worklist.append((block.label, instr.true_label))
                self.edge_worklist.append((block.label, instr.false_label))
        
        elif instr.op == 'jmp':
            self.edge_worklist.append((block.label, instr.target_label))
    
    def _notify_users(self, var: str):
        """变量值变化后，将使用该变量的所有块加入 worklist"""
        for label, block in self.cfg.blocks.items():
            for instr in block.instructions:
                if var in instr.args or (instr.op == 'phi' and var in instr.args):
                    if label not in self.worklist:
                        self.worklist.append(label)
                    break
```

```python
def build_test_cfg():
```

## 练习五：PGO 对比实验（预计70分钟）

### 目标

通过实际操作完成配置文件导向优化（PGO）的完整流程：插桩编译 → 训练采集 → 优化重编译 → 性能量化对比。PGO 是目前投入产出比最高的优化手段之一，在 Chrome、MySQL、Redis 等大型项目中普遍使用，可带来 10%-30% 的性能提升。

### 知识回顾

- **PGO 流程**：插桩编译（instrument）→ 训练运行（train）→ 分析配置文件（profile）→ 用配置文件重编译（optimize）
- **为什么有效**：静态分析无法获得的信息（分支概率、热点函数、代码段可达性）可以通过运行时采集获得
- **适用场景**：分支密集的代码、虚函数调用多的代码、函数内联决策

#### 第一步：编写测试程序（15分钟）

创建文件 `pgo_benchmark.c`：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <time.h>

/*
 * 模拟一个简单的路由器/调度器：
 * 根据数据包类型执行不同的处理逻辑。
 * 
 * 实际场景：Web 服务器的请求分发、数据库查询路由、游戏逻辑分发等。
 * PGO 价值：编译器无法静态推断分支概率，但 PGO 可以告诉它哪些路径是热路径。
 */

typedef struct {
    int type;
    int priority;
    char data[64];
} Packet;

// 不同类型的处理函数
void handle_data(Packet *p) {
    // 数据包处理（简单计算）
    volatile int sum = 0;
    for (int i = 0; i < 100; i++) {
        sum += i * p->priority;
    }
}

void handle_control(Packet *p) {
    // 控制包处理（更重的计算）
    volatile int result = 1;
    for (int i = 0; i < 500; i++) {
        result = (result * 31 + p->type) % 1000000007;
    }
}

void handle_heartbeat(Packet *p) {
    // 心跳包：几乎不做处理
    volatile int x = p->type;
    (void)x;
}

// 模拟包类型分布：80% data, 15% control, 5% heartbeat
// 这个分布是 PGO 优化的关键——编译器静态分析不知道这个比例
void generate_packets(Packet *packets, int n, unsigned int seed) {
    srand(seed);
    for (int i = 0; i < n; i++) {
        packets[i].priority = rand() % 10;
        packets[i].type = rand() % 100;
        if (packets[i].type < 80)
            packets[i].type = 0;   // 80% data
        else if (packets[i].type < 95)
            packets[i].type = 1;   // 15% control
        else
            packets[i].type = 2;   // 5% heartbeat
    }
}

// 主调度逻辑
void dispatch(Packet *packets, int n) {
    for (int i = 0; i < n; i++) {
        switch (packets[i].type) {
            case 0: handle_data(&amp;packets[i]); break;
            case 1: handle_control(&amp;packets[i]); break;
            case 2: handle_heartbeat(&amp;packets[i]); break;
            default: break;
        }
    }
}

int main(int argc, char *argv[]) {
    int n = 1000000;
    unsigned int seed = 42;
    
    // 如果命令行有参数，用于训练模式
    if (argc > 1 &amp;&amp; strcmp(argv[1], "train") == 0) {
        seed = (unsigned int)time(NULL);
    }
    
    Packet *packets = malloc(n * sizeof(Packet));
    if (!packets) {
        fprintf(stderr, "malloc failed\n");
        return 1;
    }
    
    generate_packets(packets, n, seed);
    
    clock_t start = clock();
    dispatch(packets, n);
    clock_t end = clock();
    
    double elapsed = (double)(end - start) / CLOCKS_PER_SEC;
    printf("Processed %d packets in %.4f seconds\n", n, elapsed);
    
    free(packets);
    return 0;
}
```

```bash
#!/bin/bash
# benchmark.sh - 性能对比脚本

N=1000000
RUNS=10
SEED=42

echo "============================================"
echo "  PGO 性能对比实验"
echo "  每个版本运行 $RUNS 次，取平均值"
echo "============================================"
echo ""

for version in baseline optimized; do
    binary="pgo_benchmark_${version}"
    if [ ! -f "$binary" ]; then
        echo "⚠ $binary 不存在，跳过"
        continue
    fi
    
    total_time=0
    min_time=999
    max_time=0
    
    for i in $(seq 1 $RUNS); do
        output=$(./$binary 2>&amp;1)
        time_val=$(echo "$output" | grep -oP 'in \K[0-9.]+')
        total_time=$(echo "$total_time + $time_val" | bc)
        
        # 记录 min/max
        is_smaller=$(echo "$time_val < $min_time" | bc)
        is_bigger=$(echo "$time_val > $max_time" | bc)
        [ "$is_smaller" = "1" ] &amp;&amp; min_time=$time_val
        [ "$is_bigger" = "1" ] &amp;&amp; max_time=$time_val
    done
    
    avg_time=$(echo "scale=6; $total_time / $RUNS" | bc)
    echo "[$version] 平均: ${avg_time}s  最小: ${min_time}s  最大: ${max_time}s"
done

echo ""
echo "============================================"
echo "  分析：为什么 PGO 能加速？"
echo "============================================"
echo ""
echo "PGO 主要通过以下机制提升性能："
echo "1. 分支预测优化：编译器根据 profile 数据重排基本块，"
echo "   将热路径（80% 的 data 包）放在 fall-through 位置"
echo "2. 函数内联决策：高频调用的 handle_data() 被内联，"
echo "   低频的 handle_control() 和 handle_heartbeat() 不内联"
echo "3. 代码布局：热函数连续排列，提升指令缓存命中率"
echo "4. 去虚拟化：如果存在虚函数调用，PGO 可以推断实际类型"
```
