### 1.2 三种表格式核心操作对比（35分钟）

分别在三种表格式上执行相同的操作集，观察行为差异：

**Delta Lake 操作示例：**

```scala
// === Delta Lake：完整的ACID事务体验 ===

// 1. 创建表并写入数据
val ordersDF = spark.read.format("csv")
  .option("header", "true")
  .load("s3a://datalake/raw/orders.csv")

ordersDF.write.format("delta")
  .mode("overwrite")
  .save("s3a://datalake/silver/orders_delta")

// 2. 查看表的ACID事务日志（核心机制）
// Delta Log存储在表目录下的_delta_log/文件夹中
// 每次写入生成一个JSON事务日志文件

// 3. 执行Merge操作（Upsert）——Delta Lake的核心能力
import io.delta.tables.DeltaTable

val deltaTable = DeltaTable.forPath(spark, "s3a://datalake/silver/orders_delta")

val newOrders = spark.read.format("csv")
  .option("header", "true")
  .load("s3a://datalake/raw/orders_update.csv")

deltaTable.as("target")
  .merge(newOrders.as("source"), "target.order_id = source.order_id")
  .whenMatched.updateAll()
  .whenNotMatched.insertAll()
  .execute()

// 4. 时间旅行——查询历史版本的数据
// 查看历史版本列表
deltaTable.history().show()

// 查询v0版本的数据
spark.read.format("delta")
  .load("s3a://datalake/silver/orders_delta")
  .createOrReplaceTempView("orders_current")

spark.read.format("delta")
  .option("versionAsOf", 0)
  .load("s3a://datalake/silver/orders_delta")
  .createOrReplaceTempView("orders_v0")

// 对比两个版本的行数差异
spark.sql("SELECT 'v0' as version, COUNT(*) as cnt FROM orders_v0 UNION ALL SELECT 'current' as version, COUNT(*) as cnt FROM orders_current").show()

// 5. 小文件优化
spark.sql("OPTIMIZE delta.`s3a://datalake/silver/orders_delta`")
spark.sql("VACUUM delta.`s3a://datalake/silver/orders_delta` RETAIN 168 HOURS")
```

**Apache Iceberg 操作示例：**

```scala
// === Apache Iceberg：开放标准 + 隐藏分区 ===

// 1. 创建Iceberg表（含分区策略）
spark.sql("""
  CREATE TABLE spark_catalog.iceberg_db.orders_iceberg (
    order_id    BIGINT,
    customer_id BIGINT,
    order_date  DATE,
    amount      DECIMAL(10,2),
    status      STRING
  ) USING iceberg
  PARTITIONED BY (order_date)
  TBLPROPERTIES (
    'write.format.default' = 'parquet',
    'write.parquet.compression-codec' = 'zstd'
  )
""")

// 2. 写入数据
ordersDF.writeTo("spark_catalog.iceberg_db.orders_iceberg").append()

// 3. 查看Iceberg三层元数据（核心机制）
// Table Metadata → Manifest List → Manifest File
// 可以通过API查看元数据文件
spark.sql("SELECT * FROM spark_catalog.iceberg_db.orders_iceberg.files").show()

// 4. 分区演化——修改分区策略而不影响已有数据
spark.sql("""
  ALTER TABLE spark_catalog.iceberg_db.orders_iceberg
  DROP PARTITION BY order_date
""")
spark.sql("""
  ALTER TABLE spark_catalog.iceberg_db.orders_iceberg
  PARTITION BY (MONTH(order_date), status)
""")

// 5. 时间旅行——查询历史快照
spark.read.format("iceberg")
  .option("snapshot-id", "最初的快照ID")
  .load("spark_catalog.iceberg_db.orders_iceberg")
  .show()

// 6. 增量扫描——只查询自某个快照以来变更的数据
spark.read.format("iceberg")
  .option("start-snapshot-id", "起始快照ID")
  .option("snapshot-id", "结束快照ID")
  .load("spark_catalog.iceberg_db.orders_iceberg")
  .show()

// 7. 清理过期快照
spark.sql("""
  CALL spark_catalog.system.expire_snapshots(
    table => 'iceberg_db.orders_iceberg',
    older_than => TIMESTAMP '2024-01-01 00:00:00',
    retain_last => 5
  )
""")
```

**Apache Hudi 操作示例：**

```scala
// === Apache Hudi：增量处理 + COW/MOR对比 ===

// 1. 创建Hudi表（COW模式——Copy on Write）
spark.sql("""
  CREATE TABLE hudi_db.orders_hudi_cow (
    order_id    BIGINT,
    customer_id BIGINT,
    order_date  DATE,
    amount      DECIMAL(10,2),
    status      STRING
  ) USING hudi
  OPTIONS (
    'type' = 'COW',
    'hoodie.datasource.write.recordkey.field' = 'order_id',
    'hoodie.datasource.write.precombine.field' = 'order_date',
    'hoodie.table.name' = 'orders_hudi_cow'
  )
  PARTITIONED BY (order_date)
""")

// 2. 创建Hudi表（MOR模式——Merge on Read）
spark.sql("""
  CREATE TABLE hudi_db.orders_hudi_mor (
    order_id    BIGINT,
    customer_id BIGINT,
    order_date  DATE,
    amount      DECIMAL(10,2),
    status      STRING
  ) USING hudi
  OPTIONS (
    'type' = 'MERGE_ON_READ',
    'hoodie.datasource.write.recordkey.field' = 'order_id',
    'hoodie.datasource.write.precombine.field' = 'order_date',
    'hoodie.table.name' = 'orders_hudi_mor'
  )
  PARTITIONED BY (order_date)
""")

// 3. Upsert操作——Hudi的核心能力
ordersDF.write.format("hudi")
  .option("hoodie.datasource.write.operation", "upsert")
  .option("hoodie.datasource.write.recordkey.field", "order_id")
  .option("hoodie.datasource.write.precombine.field", "order_date")
  .mode("append")
  .save("s3a://datalake/silver/orders_hudi_cow")

// 4. 增量查询——只读取自上次以来变更的数据（Hudi原生支持）
spark.read.format("hudi")
  .option("hoodie.datasource.query.type", "incremental")
  .option("hoodie.datasource.read.begin.instanttime", "上次查询的时间戳")
  .load("s3a://datalake/silver/orders_hudi_cow")
  .show()

// 5. 对比COW和MOR的写入性能和文件布局
// COW：每次upsert重写整个文件，写入慢但查询快
// MOR：upsert写入日志文件，查询时合并，写入快但查询需合并
```

### 2.2 星型模型设计（30分钟）

**第一步：确定事实表粒度**

粒度选择决策树：

  业务过程是什么？ → "下单"
    ↓
  每一行代表什么？ → 选项A：一笔完整订单（order_id粒度）
                     选项B：一个订单明细项（order_id + product_id粒度）
    ↓
  分析需求需要哪种粒度？
    "按品类分析销售额" → 需要 product_id 粒度
    "按客户分析购买次数" → 需要 order_id 粒度
    ↓
  结论：选择最细粒度（订单明细项），因为：
    ① 粗粒度可从细粒度聚合得到，反之不行
    ② "按品类分析"需要product_id级别的数据
    ③ "按客户分析"可以通过COUNT(DISTINCT order_id)实现

**第二步：设计事实表**

```sql
-- DWD层：订单明细事实表（星型模型的核心）
CREATE TABLE dwd_order_detail (
    -- 代理键（Surrogate Key）
    order_detail_sk    BIGINT PRIMARY KEY,     -- 自增整数

    -- 自然键（业务键）
    order_id           VARCHAR(32),
    order_line_no      INT,

    -- 维度外键
    date_sk            INT,                    -- → dim_date
    customer_sk        BIGINT,                 -- → dim_customer
    product_sk         BIGINT,                 -- → dim_product
    store_sk           INT,                    -- → dim_store
    channel_sk         INT,                    -- → dim_channel

    -- 度量（Measures）
    quantity           INT,
    unit_price         DECIMAL(10,2),
    discount           DECIMAL(10,2),
    shipping_cost      DECIMAL(10,2),
    total_amount       DECIMAL(12,2),          -- 预计算：quantity × unit_price - discount
    net_amount         DECIMAL(12,2),          -- 预计算：total_amount - shipping_cost
    cost_amount        DECIMAL(12,2),          -- 预计算：quantity × cost_price

    -- 事务维度
    etl_load_time      TIMESTAMP,
    data_source        VARCHAR(50)
);
```

**第四步：实现SCD Type 2的ETL逻辑**

```sql
-- SCD Type 2 客户维度更新逻辑（以SQL伪代码展示核心思路）

-- 步骤1：找出源系统中发生变化的客户
WITH changed_customers AS (
    SELECT src.*
    FROM ods_customers src
    JOIN dim_customer tgt
      ON src.customer_id = tgt.customer_id
     AND tgt.is_current = TRUE
    WHERE src.customer_name != tgt.customer_name
       OR src.city != tgt.city
       OR src.province != tgt.province
       OR src.membership_level != tgt.membership_level
),

-- 步骤2：将旧记录标记为失效
close_old_records AS (
    UPDATE dim_customer
    SET valid_to = CURRENT_TIMESTAMP,
        is_current = FALSE
    WHERE customer_id IN (SELECT customer_id FROM changed_customers)
      AND is_current = TRUE
)

-- 步骤3：插入新版本记录
INSERT INTO dim_customer (
    customer_sk, customer_id, customer_name, city, province,
    membership_level, valid_from, valid_to, is_current
)
SELECT
    (SELECT COALESCE(MAX(customer_sk), 0) FROM dim_customer) + ROW_NUMBER() OVER (ORDER BY customer_id),
    customer_id, customer_name, city, province,
    membership_level,
    CURRENT_TIMESTAMP,          -- valid_from
    TIMESTAMP '9999-12-31',     -- valid_to（远未来表示当前记录）
    TRUE                        -- is_current
FROM changed_customers;
```

```sql
        order_id,
```

```sql
        quantity * unit_price - discount AS net_amount,
        quantity * cost_price AS cost_amount,

        channel,
        order_date,

        -- 数据质量

        -- 元数据

    FROM order_items
    WHERE data_quality_flag = 'valid'
)

{% if is_incremental() %}
    -- 增量模式：只处理新数据
    SELECT * FROM final
    WHERE dbt_loaded_at > (SELECT MAX(dbt_loaded_at) FROM {{ this }})
{% else %}
```

### 4.3 OLAP核心操作实践（20分钟）

```sql
-- === OLAP五种核心操作 ===

-- 1. 上卷（Roll-Up）：从日粒度聚合到月粒度
SELECT
    toStartOfMonth(order_date) AS month,
    category,
    SUM(total_amount) AS monthly_sales,
    COUNT(*) AS order_count
FROM olap_demo.sales
GROUP BY month, category
ORDER BY month, category;

-- 2. 下钻（Drill-Down）：从月粒度展开到日粒度
SELECT
    order_date,
    category,
    SUM(total_amount) AS daily_sales
FROM olap_demo.sales
WHERE toStartOfMonth(order_date) = '2024-01-01'  -- 先定位到某个月
GROUP BY order_date, category
ORDER BY order_date, category;

-- 3. 切片（Slice）：选择某个维度的特定值
SELECT
    order_date,
    SUM(total_amount) AS sales
FROM olap_demo.sales
WHERE category = 'Electronics'   -- 只看电子产品
GROUP BY order_date
ORDER BY order_date;

-- 4. 切块（Dice）：选择多个维度的范围
SELECT
    category,
    region,
    SUM(total_amount) AS sales
FROM olap_demo.sales
WHERE category IN ('Electronics', 'Clothing')
  AND region IN ('East', 'West')
  AND order_date >= '2024-01-01'
  AND order_date <= '2024-06-30'
GROUP BY category, region
ORDER BY category, region;

-- 5. 旋转（Pivot）：行列转换
SELECT *
FROM (
    SELECT category, toStartOfMonth(order_date) AS month, total_amount
    FROM olap_demo.sales
)
PIVOT (SUM(total_amount) FOR month IN (
    '2024-01-01', '2024-02-01', '2024-03-01',
    '2024-04-01', '2024-05-01', '2024-06-01'
))
ORDER BY category;
```

```sql
-- === 优化技巧1：利用ORDER BY键加速查询 ===

-- 对于频繁查询的聚合结果，创建物化视图自动维护
CREATE MATERIALIZED VIEW olap_demo.sales_monthly_mv
ENGINE = SummingMergeTree()
ORDER BY (month, category, region)

    toStartOfMonth(order_date) AS month,

    region,
    SUM(total_amount) AS total_sales,
    SUM(quantity) AS total_quantity,
    COUNT(*) AS order_count
FROM olap_demo.sales

-- 查询物化视图（秒级响应，因为数据已经预聚合）
SELECT month, category, SUM(total_sales) AS sales
FROM olap_demo.sales_monthly_mv
WHERE month >= '2024-01-01'
GROUP BY month, category
ORDER BY month, sales DESC;

-- 投影是ClickHouse 21.6+的特性，类似物化视图但更灵活
ALTER TABLE olap_demo.sales ADD PROJECTION projection_by_region (
    SELECT *
    ORDER BY (region, order_date)

-- === 优化技巧5：数据压缩优化 ===
-- 查看表的压缩情况
SELECT
    table,
    formatReadableSize(sum(data_compressed_bytes)) AS compressed,
    formatReadableSize(sum(data_uncompressed_bytes)) AS uncompressed,
    round(sum(data_uncompressed_bytes) / sum(data_compressed_bytes), 2) AS ratio
```

### 5.1 质量规则设计（15分钟）

首先为每个数据湖层级设计具体的质量规则：

```yaml
# data_quality_rules.yml
# 数据质量规则定义

bronze_zone:
  description: "原始区——检查数据完整性和基本格式"
  rules:
    - name: row_count_check
      type: completeness
      description: "检查每个批次的数据量是否在合理范围"
      sql: |
        SELECT
          CASE
            WHEN COUNT(*) < 1000 THEN 'FAIL: 数据量过少（疑似摄入失败）'
            WHEN COUNT(*) > 10000000 THEN 'WARN: 数据量异常偏大'
            ELSE 'PASS'
          END AS check_result,
          COUNT(*) AS actual_count
      threshold:
        min_rows: 1000
        max_rows: 10000000

    - name: null_rate_check
      type: completeness
      description: "检查关键字段的空值率"
      columns:
        order_id: { max_null_rate: 0.0 }      # 主键不能为空
        customer_id: { max_null_rate: 0.02 }   # 外键空值率不超过2%
        order_date: { max_null_rate: 0.0 }

    - name: format_check
      type: accuracy
      description: "检查字段格式是否符合预期"
      rules:
        - column: order_date
          pattern: '^\d{4}-\d{2}-\d{2}$'
          max_invalid_rate: 0.0
        - column: status
          allowed_values: ['pending', 'completed', 'cancelled', 'refunded']
          max_invalid_rate: 0.0

silver_zone:
  description: "清洗区——检查数据一致性和业务规则"
  rules:
    - name: deduplication_check
      type: uniqueness
      description: "检查去重是否成功"
      sql: |
        SELECT
          COUNT(*) AS total_rows,
          COUNT(DISTINCT order_id) AS distinct_orders,
          COUNT(*) - COUNT(DISTINCT order_id) AS duplicate_count,
          CASE
            WHEN COUNT(*) - COUNT(DISTINCT order_id) > 0 THEN 'FAIL'
            ELSE 'PASS'
          END AS check_result

    - name: referential_integrity
      type: consistency
      description: "检查外键引用完整性"
      sql: |
        SELECT
          COUNT(*) AS orphan_count,
          CASE
            WHEN COUNT(*) > 0 THEN 'FAIL'
            ELSE 'PASS'
          END AS check_result
        FROM silver_orders o
        LEFT JOIN silver_customers c ON o.customer_id = c.customer_id
        WHERE c.customer_id IS NULL

    - name: range_check
      type: accuracy
      description: "检查数值字段的合理范围"
      rules:
        - column: quantity
          min: 1
          max: 10000
        - column: unit_price
          min: 0.01
          max: 999999.99
        - column: discount
          min: 0
          max_ratio: 0.5    # 折扣不超过单价的50%

gold_zone:
  description: "精炼区——检查业务指标的一致性和时效性"
  rules:
    - name: metric_consistency
      type: consistency
      description: "检查汇总指标与明细数据的一致性"
      sql: |
        -- 对比Gold层汇总数据与Silver层明细数据
        SELECT
          g.monthly_total,
          s.computed_total,
          ABS(g.monthly_total - s.computed_total) AS diff,
          CASE
            WHEN ABS(g.monthly_total - s.computed_total) > 0.01 THEN 'FAIL'
            ELSE 'PASS'
          END AS check_result
        FROM gold_monthly_sales g
        JOIN (
          SELECT DATE_TRUNC('month', order_date) AS month,
                 SUM(total_amount) AS computed_total
          FROM silver_orders
          GROUP BY 1
        ) s ON g.month = s.month

    - name: freshness_check
      type: timeliness
      description: "检查数据是否按时更新"
      sql: |
        SELECT
          MAX(order_date) AS latest_date,
          CURRENT_DATE - MAX(order_date) AS days_delay,
          CASE
            WHEN CURRENT_DATE - MAX(order_date) > 2 THEN 'FAIL: 数据延迟超过2天'
            WHEN CURRENT_DATE - MAX(order_date) > 1 THEN 'WARN: 数据延迟1-2天'
            ELSE 'PASS'
```

### 5.2 质量检查脚本实现（25分钟）

```python
#!/usr/bin/env python3
"""
data_quality_monitor.py
端到端数据质量监控脚本

使用方式：
  python data_quality_monitor.py --config data_quality_rules.yml
  python data_quality_monitor.py --zone bronze   # 只检查某个层级
  python data_quality_monitor.py --report html   # 生成HTML报告
"""

import yaml
import json
import sqlite3
from datetime import datetime, timedelta
from dataclasses import dataclass, field
from typing import Optional
from pathlib import Path


@dataclass
class QualityCheckResult:
    """单条质量检查结果"""
    check_name: str
    zone: str
    status: str           # PASS / WARN / FAIL
    actual_value: Optional[float] = None
    expected_value: Optional[str] = None
    message: str = ""
    executed_at: str = field(default_factory=lambda: datetime.now().isoformat())


class DataQualityMonitor:
    def __init__(self, db_path: str, rules_path: str):
        self.conn = sqlite3.connect(db_path)
        self.conn.row_factory = sqlite3.Row
        self.rules = self._load_rules(rules_path)
        self.results = []

    def _load_rules(self, path: str) -> dict:
        with open(path) as f:
            return yaml.safe_load(f)

    def run_all_checks(self, zone: str = None):
        """执行所有质量检查"""
        zones_to_check = [zone] if zone else ['bronze_zone', 'silver_zone', 'gold_zone']

        for z in zones_to_check:
            if z not in self.rules:
                continue
            zone_rules = self.rules[z]
            print(f"\n{'='*50}")
            print(f"  检查区域: {zone_rules['description']}")
            print(f"{'='*50}")

            for rule in zone_rules.get('rules', []):
                result = self._execute_rule(z, rule)
                self.results.append(result)
                status_icon = {'PASS': '✅', 'WARN': '⚠️', 'FAIL': '❌'}[result.status]
                print(f"  {status_icon} {result.check_name}: {result.status}")
                if result.message:
                    print(f"     {result.message}")

        return self.results

    def _execute_rule(self, zone: str, rule: dict) -> QualityCheckResult:
        """执行单条质量规则"""
        check_name = rule['name']
        check_type = rule.get('type', 'custom')

        if check_type == 'completeness' and 'threshold' in rule:
            return self._check_completeness(zone, rule)
        elif check_type == 'uniqueness':
            return self._check_uniqueness(zone, rule)
        elif check_type == 'consistency':
            return self._check_consistency(zone, rule)
        elif check_type == 'accuracy' and 'columns' in rule:
            return self._check_null_rates(zone, rule)
        elif check_type == 'accuracy' and 'rules' in rule:
            return self._check_value_ranges(zone, rule)
        else:
            return self._execute_sql_rule(zone, rule)

    def _check_completeness(self, zone: str, rule: dict) -> QualityCheckResult:
        """检查数据完整性（行数范围）"""
        table = f"{zone.replace('_zone', '')}_orders"
        cursor = self.conn.execute(f"SELECT COUNT(*) FROM {table}")
        count = cursor.fetchone()[0]
        threshold = rule['threshold']

        if count < threshold['min_rows']:
            return QualityCheckResult(
                check_name=rule['name'], zone=zone, status='FAIL',
                actual_value=count,
                message=f"数据量{count}低于最小阈值{threshold['min_rows']}"
            )
        elif count > threshold['max_rows']:
            return QualityCheckResult(
                check_name=rule['name'], zone=zone, status='WARN',
                actual_value=count,
                message=f"数据量{count}超过建议上限{threshold['max_rows']}"
            )
        return QualityCheckResult(
            check_name=rule['name'], zone=zone, status='PASS',
            actual_value=count
        )

    def _check_uniqueness(self, zone: str, rule: dict) -> QualityCheckResult:
        """检查数据唯一性（去重检查）"""
        # 执行SQL规则
        return self._execute_sql_rule(zone, rule)

    def _check_consistency(self, zone: str, rule: dict) -> QualityCheckResult:
        """检查数据一致性"""
        return self._execute_sql_rule(zone, rule)

    def _check_null_rates(self, zone: str, rule: dict) -> QualityCheckResult:
        """检查关键字段空值率"""
        table = f"{zone.replace('_zone', '')}_orders"
        results = []
        for col_config in rule['columns']:
            col_name = col_config
            max_null_rate = rule['columns'][col_config]['max_null_rate']
            cursor = self.conn.execute(f"""
                SELECT
                    COUNT(*) AS total,
                    SUM(CASE WHEN {col_name} IS NULL THEN 1 ELSE 0 END) AS null_count
                FROM {table}
            """)
            row = cursor.fetchone()
            if row[0] == 0:
                continue
            null_rate = row[1] / row[0]
            if null_rate > max_null_rate:
                results.append(f"{col_name}: 空值率{null_rate:.2%}超过阈值{max_null_rate:.0%}")

        if results:
            return QualityCheckResult(
                check_name=rule['name'], zone=zone, status='FAIL',
                message='; '.join(results)
            )
        return QualityCheckResult(
            check_name=rule['name'], zone=zone, status='PASS',
            message='所有字段空值率在阈值内'
        )

    def _check_value_ranges(self, zone: str, rule: dict) -> QualityCheckResult:
        """检查数值字段的合理范围"""
        table = f"{zone.replace('_zone', '')}_orders"
        violations = []
        for r in rule['rules']:
            col = r['column']
            min_val = r.get('min')
            max_val = r.get('max')
            where_clause = []
            if min_val is not None:
                where_clause.append(f"{col} < {min_val}")
            if max_val is not None:
                where_clause.append(f"{col} > {max_val}")
            if where_clause:
                cursor = self.conn.execute(f"""
                    SELECT COUNT(*) FROM {table}
                    WHERE {' OR '.join(where_clause)}
                """)
                bad_count = cursor.fetchone()[0]
                if bad_count > 0:
                    violations.append(f"{col}: {bad_count}条记录超出范围")

        if violations:
            return QualityCheckResult(
                check_name=rule['name'], zone=zone, status='FAIL',
                message='; '.join(violations)
            )
        return QualityCheckResult(
            check_name=rule['name'], zone=zone, status='PASS'
        )

    def _execute_sql_rule(self, zone: str, rule: dict) -> QualityCheckResult:
        """执行通用SQL规则"""
        if 'sql' in rule:
            try:
                cursor = self.conn.execute(rule['sql'])
                row = cursor.fetchone()
                if row and 'check_result' in dict(row.keys()):
                    result_str = row['check_result']
                    status = 'PASS' if result_str == 'PASS' else 'WARN' if 'WARN' in result_str else 'FAIL'
                    return QualityCheckResult(
                        check_name=rule['name'], zone=zone, status=status,
                        message=result_str
                    )
            except Exception as e:
                return QualityCheckResult(
                    check_name=rule['name'], zone=zone, status='FAIL',
                    message=f"SQL执行失败: {e}"
                )
        return QualityCheckResult(check_name=rule['name'], zone=zone, status='PASS')

    def generate_report(self) -> dict:
        """生成质量报告摘要"""
        total = len(self.results)
        passed = sum(1 for r in self.results if r.status == 'PASS')
        warned = sum(1 for r in self.results if r.status == 'WARN')
        failed = sum(1 for r in self.results if r.status == 'FAIL')

        quality_score = (passed / total * 100) if total > 0 else 0

        return {
            'timestamp': datetime.now().isoformat(),
            'total_checks': total,
            'passed': passed,
            'warned': warned,
            'failed': failed,
            'quality_score': f"{quality_score:.1f}%",
            'results': [
                {'name': r.check_name, 'zone': r.zone, 'status': r.status, 'message': r.message}
                for r in self.results
            ]
        }

    def save_to_history(self):
        """将检查结果保存到历史记录表（支持趋势分析）"""
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS quality_check_history (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                check_name TEXT,
                zone TEXT,
                status TEXT,
                message TEXT,
                checked_at TIMESTAMP
            )
        """)
        for r in self.results:
            self.conn.execute(
                "INSERT INTO quality_check_history (check_name, zone, status, message, checked_at) VALUES (?, ?, ?, ?, ?)",
                (r.check_name, r.zone, r.status, r.message, r.executed_at)
            )
        self.conn.commit()

    def get_quality_trend(self, days: int = 30) -> list:
        """获取质量趋势数据"""
        cursor = self.conn.execute("""
            SELECT
                DATE(checked_at) AS check_date,
                status,
                COUNT(*) AS count
            FROM quality_check_history
            WHERE checked_at >= datetime('now', ? || ' days')
            GROUP BY check_date, status
            ORDER BY check_date
        """, (f'-{days}',))
        return cursor.fetchall()


# === 主程序 ===
if __name__ == '__main__':
    import argparse

    parser = argparse.ArgumentParser(description='数据质量监控')
```

### 5.3 数据血缘追踪练习（15分钟）

理解数据血缘的追踪方式——从源表到目标表的完整链路：

```sql
-- === 数据血缘的SQL解析方式 ===
-- 通过解析SQL语句提取表级别的血缘关系

-- 示例：分析以下dbt模型的血缘
-- models/marts/sales/fct_orders.sql 的内容:
--
-- SELECT
--     o.order_id,
--     c.customer_name,
--     p.category,
--     SUM(o.total_amount) AS total_amount
-- FROM {{ ref('stg_orders') }} o
-- JOIN {{ ref('stg_customers') }} c ON o.customer_id = c.customer_id
-- JOIN {{ ref('stg_products') }} p ON o.product_id = p.product_id
-- GROUP BY o.order_id, c.customer_name, p.category

-- 血缘关系可以解析为图结构：
-- stg_orders ─────┐
--                  ├──→ fct_orders
-- stg_customers ──┤
--                  │
-- stg_products ───┘

-- 而 stg_orders 的血缘是：
-- raw_data.orders ──→ stg_orders

-- 完整血缘链路：
-- raw_data.orders ──→ stg_orders ─────┐
--                                     ├──→ fct_orders
-- raw_data.customers ──→ stg_customers ┘
--                                     │
-- raw_data.products ──→ stg_products ─┘
```

在实际项目中，可以使用以下方式采集血缘：

```python
#!/usr/bin/env python3
"""
data_lineage_tracker.py
简化版数据血缘追踪器
通过解析SQL中的表引用关系，自动构建血缘图
"""

import re
from dataclasses import dataclass
from typing import Dict, List, Set, Tuple


@dataclass
class LineageNode:
    """血缘节点"""
    name: str
    node_type: str      # 'source' / 'staging' / 'intermediate' / 'mart'
    upstream: List[str] = None
    downstream: List[str] = None

    def __post_init__(self):
        if self.upstream is None:
            self.upstream = []
        if self.downstream is None:
            self.downstream = []


class DataLineageTracker:
    def __init__(self):
        self.nodes: Dict[str, LineageNode] = {}
        self.edges: List[Tuple[str, str]] = []

    def register_model(self, model_name: str, sql_content: str):
        """从SQL内容中提取表引用，注册血缘关系"""
        # 提取 FROM / JOIN 后面的表名
        table_pattern = r'(?:FROM|JOIN)\s+(?:{{\s*ref\([\'"](.+?)[\'"]\s*\)\s*}}|(\w+\.\w+))'
        referenced_tables = re.findall(table_pattern, sql_content, re.IGNORECASE)
        referenced_tables = [t[0] or t[1] for t in referenced_tables]

        # 注册节点
        if model_name not in self.nodes:
            self.nodes[model_name] = LineageNode(name=model_name, node_type='unknown')

        for ref in referenced_tables:
            if ref not in self.nodes:
                self.nodes[ref] = LineageNode(name=ref, node_type='unknown')

            # 添加边
            self.edges.append((ref, model_name))
            self.nodes[ref].downstream.append(model_name)
            self.nodes[model_name].upstream.append(ref)

    def visualize(self) -> str:
        """生成文本格式的血缘图"""
        output = []
        output.append("=== 数据血缘图 ===\n")

        # 按层级分组
        sources = [n for n in self.nodes.values() if not n.upstream]
        layers = [sources]

        visited = set(sources)
        current_layer = sources
        while current_layer:
            next_layer = []
            for node in current_layer:
                for ds_name in node.downstream:
                    if ds_name not in visited:
                        next_layer.append(self.nodes[ds_name])
                        visited.add(ds_name)
            if next_layer:
                layers.append(next_layer)
            current_layer = next_layer

        for i, layer in enumerate(layers):
            layer_names = [n.name for n in layer]
            output.append(f"Layer {i}: {' → '.join(layer_names)}")

        return '\n'.join(output)


# 使用示例
tracker = DataLineageTracker()

# 注册模型的SQL内容
models_sql = {
    'stg_orders': "SELECT * FROM {{ ref('raw_orders') }}",
    'stg_customers': "SELECT * FROM {{ ref('raw_customers') }}",
    'int_order_joined': """
        SELECT * FROM {{ ref('stg_orders') }} o
        JOIN {{ ref('stg_customers') }} c ON o.customer_id = c.customer_id
    """,
    'fct_orders': """
        SELECT * FROM {{ ref('int_order_joined') }} i
        JOIN {{ ref('stg_products') }} p ON i.product_id = p.product_id
    """,
}

for model_name, sql in models_sql.items():
    tracker.register_model(model_name, sql)

print(tracker.visualize())
```
