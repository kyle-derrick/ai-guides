## 20.4 差分隐私核心技巧

差分隐私（Differential Privacy, DP）是当前隐私保护机器学习中最具数学严谨性的技术框架。本节从攻击者视角出发，深入讲解差分隐私在模型训练中的落地技巧——包括噪声校准、梯度裁剪策略、隐私预算追踪、主流框架实战，以及对抗隐私攻击的具体防御方案。

### 20.4.1 为什么需要差分隐私：从攻击场景说起

在没有差分隐私保护的情况下，机器学习模型会"记住"训练数据的细节。攻击者可以通过以下方式窃取隐私：

| 攻击类型 | 攻击方式 | 危害程度 |
|---------|---------|---------|
| 成员推理攻击（Membership Inference） | 判断某条数据是否参与了模型训练 | 高——可确认敏感记录存在性 |
| 模型反转攻击（Model Inversion） | 从模型权重重建训练样本 | 极高——可还原人脸、医疗记录 |
| 梯度泄露攻击（Gradient Leakage） | 从梯度更新反推训练数据 | 极高——联邦学习场景尤为严重 |
| 属性推理攻击（Attribute Inference） | 推断训练数据的隐藏属性 | 中——可推断收入、健康状况等 |

差分隐私的核心保证：**无论攻击者拥有多少背景知识，都无法确定某条特定数据是否参与了模型训练**。这不是"安全性"的模糊承诺，而是数学定理级别的保障。

### 20.4.2 差分隐私的数学基础与直觉

差分隐私的正式定义（已在 20.5 理论基础中介绍）可以转化为工程直觉：

**核心参数解读：**

| 参数 | 含义 | 典型取值 | 工程意义 |
|------|------|---------|---------|
| ε（epsilon） | 隐私损失上界 | 1~10 | 越小越安全，但噪声越大，模型精度越低 |
| δ（delta） | 失败概率 | 10⁻⁵~10⁻³ | 通常设为 1/n（n为数据集大小） |
| σ（sigma） | 噪声标准差 | 由ε和δ决定 | 核心工程参数，直接影响模型质量 |

**直觉理解 ε 的含义：**
- ε = 0：完美隐私，输出与输入完全无关（无用）
- ε = 1：强隐私保护，实用中被认为是"安全"的标准
- ε = 8：中等保护，学术界常见实验值
- ε → ∞：无隐私保护，等价于不加噪声

**高斯机制的噪声公式：**

当使用高斯噪声机制时，噪声标准差为：

$$\sigma = \frac{\Delta_2 \cdot \sqrt{2 \ln(1.25/\delta)}}{\epsilon}$$

其中 Δ₂ 是 L2 敏感度——即单条数据变化导致的查询结果最大 L2 距离。在 SGD 场景中，Δ₂ 等价于梯度裁剪阈值 C。

### 20.4.3 差分隐私 SGD（DP-SGD）完整实现

DP-SGD 是差分隐私在深度学习中最核心的算法，由 Abadi 等人（2016）提出。其关键区别在于标准 SGD 的三处修改：

1. **逐样本梯度计算**：必须计算每个样本的独立梯度（而非 batch 平均梯度）
2. **梯度裁剪**：将每个样本的梯度范数限制在阈值 C 以内
3. **噪声注入**：在聚合梯度上添加校准的高斯噪声

```python
import torch
import torch.nn as nn
import numpy as np
from torch.utils.data import DataLoader, TensorDataset


class DPSGDFull:
    """
    差分隐私SGD完整实现，包含隐私预算追踪（RDP会计）
    """
    
    def __init__(self, model, epsilon, delta, max_grad_norm, lr=0.01):
        """
        参数:
            model: PyTorch模型
            epsilon: 隐私预算ε
            delta: 失败概率δ
            max_grad_norm: 梯度裁剪阈值C（即L2敏感度Δ₂）
            lr: 学习率
        """
        self.model = model
        self.epsilon = epsilon
        self.delta = delta
        self.max_grad_norm = max_grad_norm
        self.lr = lr
        self.privacy_spent = 0.0  # 已消耗的隐私预算
        self.steps = 0
        
        # 使用momentum优化器（DP-SGD推荐）
        self.optimizer = torch.optim.SGD(model.parameters(), lr=lr, momentum=0.9)
    
    def _compute_per_sample_gradients(self, data, target, loss_fn):
        """
        逐样本计算梯度——DP-SGD的关键步骤
        
        标准SGD使用batch平均梯度，但DP-SGD需要：
        1. 每个样本独立计算梯度
        2. 裁剪每个样本的梯度
        3. 聚合后再加噪声
        
        为什么逐样本？因为噪声校准依赖于单个样本的最大贡献。
        如果用batch平均梯度，无法精确控制单条数据的影响范围。
        """
        # 方法1：逐样本遍历（简单但慢）
        per_sample_grads = []
        for i in range(data.shape[0]):
            self.optimizer.zero_grad()
            output = self.model(data[i:i+1])
            loss = loss_fn(output, target[i:i+1])
            loss.backward()
            
            # 收集每个参数的梯度
            sample_grad = []
            for param in self.model.parameters():
                if param.grad is not None:
                    sample_grad.append(param.grad.clone())
                else:
                    sample_grad.append(None)
            per_sample_grads.append(sample_grad)
        
        return per_sample_grads
    
    def _clip_gradients(self, per_sample_grads):
        """
        梯度裁剪——将每个样本的梯度范数限制在C以内
        
        裁剪策略:
        - L2裁剪（推荐）：g_clip = g * min(1, C/||g||₂)
        - L1裁剪：g_clip = g * min(1, C/||g||₁)
        - 逐层裁剪：每层独立裁剪（更精细但更复杂）
        
        裁剪阈值C的选择：
        - C太大：隐私保护弱（噪声相对于梯度太小）
        - C太小：梯度信息被截断，模型欠拟合
        - 实践中C通常在0.1~10之间，需要调参
        """
        clipped_grads = []
        
        for sample_grad in per_sample_grads:
            # 计算该样本所有参数梯度的L2范数
            total_norm = 0.0
            for g in sample_grad:
                if g is not None:
                    total_norm += g.norm(2).item() ** 2
            total_norm = total_norm ** 0.5
            
            # 应用裁剪
            clip_factor = min(1.0, self.max_grad_norm / (total_norm + 1e-8))
            clipped = [g * clip_factor if g is not None else None for g in sample_grad]
            clipped_grads.append(clipped)
        
        return clipped_grads
    
    def _aggregate_and_add_noise(self, clipped_grads, batch_size):
        """
        聚合裁剪后的梯度并添加高斯噪声
        
        噪声校准公式:
        σ = C * √(2 ln(1.25/δ)) / ε
        
        注意：这里是每步的噪声校准。
        累积隐私损失通过RDP会计精确追踪，不会简单地ε累加。
        """
        # 聚合梯度
        aggregated = []
        num_params = len(clipped_grads[0])
        
        for param_idx in range(num_params):
            param_sum = None
            for sample_grad in clipped_grads:
                if sample_grad[param_idx] is not None:
                    if param_sum is None:
                        param_sum = sample_grad[param_idx].clone()
                    else:
                        param_sum += sample_grad[param_idx]
            
            if param_sum is not None:
                aggregated.append(param_sum / batch_size)
            else:
                aggregated.append(None)
        
        # 添加高斯噪声
        noise_std = (self.max_grad_norm * 
                     np.sqrt(2 * np.log(1.25 / self.delta)) / self.epsilon)
        
        noisy_grads = []
        for g in aggregated:
            if g is not None:
                noise = torch.normal(0, noise_std, size=g.shape)
                noisy_grads.append(g + noise / batch_size)
            else:
                noisy_grads.append(None)
        
        return noisy_grads
    
    def train_step(self, data, target, loss_fn=nn.CrossEntropyLoss()):
        """单步DP-SGD训练"""
        self.model.train()
        batch_size = data.shape[0]
        
        # Step 1: 逐样本计算梯度
        per_sample_grads = self._compute_per_sample_gradients(data, target, loss_fn)
        
        # Step 2: 梯度裁剪
        clipped_grads = self._clip_gradients(per_sample_grads)
        
        # Step 3: 聚合并添加噪声
        noisy_grads = self._aggregate_and_add_noise(clipped_grads, batch_size)
        
        # Step 4: 手动更新参数
        self.optimizer.zero_grad()
        param_idx = 0
        for param in self.model.parameters():
            if param.grad is not None and param_idx < len(noisy_grads):
                if noisy_grads[param_idx] is not None:
                    param.grad = noisy_grads[param_idx]
                param_idx += 1
        
        self.optimizer.step()
        self.steps += 1
        
        return self.model


# 使用示例
def dp_sgd_training_example():
    """DP-SGD训练完整流程"""
    # 构建简单模型
    model = nn.Sequential(
        nn.Linear(784, 256),
        nn.ReLU(),
        nn.Linear(256, 128),
        nn.ReLU(),
        nn.Linear(128, 10)
    )
    
    # 初始化DP-SGD
    dp_trainer = DPSGDFull(
        model=model,
        epsilon=8.0,           # 隐私预算
        delta=1e-5,            # 失败概率
        max_grad_norm=1.0,     # 梯度裁剪阈值
        lr=0.01
    )
    
    # 模拟训练数据
    X = torch.randn(1000, 784)
    y = torch.randint(0, 10, (1000,))
    dataset = TensorDataset(X, y)
    loader = DataLoader(dataset, batch_size=64, shuffle=True)
    
    # 训练循环
    for epoch in range(10):
        for batch_data, batch_target in loader:
            dp_trainer.train_step(batch_data, batch_target)
        
        print(f"Epoch {epoch+1}, 隐私消耗: ε={dp_trainer.privacy_spent:.2f}")
    
    return model
```

### 20.4.4 使用 Opacus 实现高效 DP-SGD

逐样本梯度的手动实现效率很低。Opacus（Meta 开源）通过梯度钩子（Gradient Hooks）实现了高效计算，速度提升 5-10 倍，是 PyTorch 生态中 DP-SGD 的首选库。

```python
from opacus import PrivacyEngine
from opacus.validators import ModuleValidator
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset


def dp_sgd_with_opacus():
    """
    使用Opacus的DP-SGD实现
    
    优势:
    - 自动处理逐样本梯度计算（通过钩子）
    - 内置RDP隐私会计
    - 自动检测兼容性问题
    - 支持虚拟batch（解决内存问题）
    """
    # 1. 定义模型（注意：某些层不兼容DP，需要替换）
    model = nn.Sequential(
        nn.Linear(784, 256),
        nn.ReLU(),
        nn.Linear(256, 128),
        nn.ReLU(),
        nn.Linear(128, 10)
    )
    
    # 2. 修复不兼容的层
    # BatchNorm在DP训练中有问题（统计信息泄露），需要替换为GroupNorm
    # 如果模型使用了BatchNorm：
    # errors = ModuleValidator.validate(model, strict=False)
    # model = ModuleValidator.fix(model)
    
    # 3. 创建数据加载器
    X = torch.randn(5000, 784)
    y = torch.randint(0, 10, (5000,))
    dataset = TensorDataset(X, y)
    train_loader = DataLoader(dataset, batch_size=64, shuffle=True)
    
    # 4. 创建优化器
    optimizer = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
    loss_fn = nn.CrossEntropyLoss()
    
    # 5. 关键步骤：将隐私引擎绑定到模型和优化器
    privacy_engine = PrivacyEngine()
    
    model, optimizer, train_loader = privacy_engine.make_private_with_epsilon(
        module=model,
        optimizer=optimizer,
        data_loader=train_loader,
        epochs=10,
        target_epsilon=8.0,        # 目标隐私预算
        target_delta=1e-5,         # 失败概率
        max_grad_norm=1.0,         # 梯度裁剪阈值
    )
    
    # 6. 训练循环
    model.train()
    for epoch in range(10):
        for batch_data, batch_target in train_loader:
            optimizer.zero_grad()
            output = model(batch_data)
            loss = loss_fn(output, batch_target)
            loss.backward()
            optimizer.step()
        
        # 查询已消耗的隐私预算
        epsilon = privacy_engine.get_epsilon(delta=1e-5)
        print(f"Epoch {epoch+1}: ε = {epsilon:.2f}")
    
    return model


def handle_opacus_compatibility():
    """
    Opacus兼容性问题排查
    
    常见不兼容层及修复方案：
    """
    # 问题1：BatchNorm层
    # 原因：BatchNorm的均值/方差统计依赖整个batch，泄露信息
    # 修复：替换为GroupNorm或LayerNorm
    # nn.BatchNorm2d(64) → nn.GroupNorm(8, 64)
    
    # 问题2：Embedding层
    # 原因：Embedding的梯度是稀疏的，裁剪行为特殊
    # 修复：使用Opacus提供的兼容Embedding
    
    # 问题3：注意力机制中的dropout
    # 原因：标准dropout在DP下有信息泄露风险
    # 修复：使用确定性dropout或调整dropout位置
    
    # 批量检查和修复
    model = nn.Sequential(
        nn.Linear(784, 256),
        nn.BatchNorm1d(256),   # 不兼容
        nn.ReLU(),
        nn.Linear(256, 10)
    )
    
    # 自动检查
    errors = ModuleValidator.validate(model, strict=False)
    print(f"兼容性问题: {len(errors)} 个")
    for error in errors:
        print(f"  - {error}")
    
    # 自动修复
    model = ModuleValidator.fix(model)
    print("修复后模型:", model)
    
    return model
```

### 20.4.5 使用 TensorFlow Privacy 实现 DP 训练

对于 TensorFlow 生态，`tensorflow-privacy` 提供了等价的 DP-SGD 实现。

```python
import tensorflow as tf
import tensorflow_privacy as tfp
from tensorflow_privacy.privacy.optimizers.dp_optimizer_keras import DPKerasSGDOptimizer
import numpy as np


def dp_training_with_tf_privacy(epsilon, delta, num_train_samples, batch_size):
    """
    TensorFlow Privacy 差分隐私训练
    
    关键参数说明：
    - l2_norm_clip: 梯度裁剪阈值（等价于Opacus的max_grad_norm）
    - noise_multiplier: 噪声乘数（噪声标准差 = noise_multiplier * l2_norm_clip）
    - num_microbatches: 微批次数量（影响逐样本梯度计算的精度）
    
    num_microbatches的设置：
    - 设为batch_size：每个样本一个微批次（最精确，最慢）
    - 设为1：整个batch一个微批次（最快，但裁剪粒度粗）
    - 推荐：设为batch_size或batch_size的因子
    """
    # 1. 计算满足隐私预算所需的噪声乘数
    # 这个函数内部使用RDP会计，比手动计算更精确
    noise_multiplier = tfp.compute_noise_multiplier(
        target_epsilon=epsilon,
        target_delta=delta,
        epochs=10,
        batch_size=batch_size,
        num_train_samples=num_train_samples
    )
    print(f"计算得到噪声乘数: {noise_multiplier:.4f}")
    
    # 2. 创建DP优化器
    optimizer = DPKerasSGDOptimizer(
        l2_norm_clip=1.0,               # 梯度裁剪阈值
        noise_multiplier=noise_multiplier, # 噪声乘数
        num_microbatches=batch_size,       # 每样本一个微批次
        learning_rate=0.01,
        momentum=0.9
    )
    
    # 3. 定义模型
    model = tf.keras.Sequential([
        tf.keras.layers.InputLayer(input_shape=(784,)),
        tf.keras.layers.Dense(256, activation='relu'),
        tf.keras.layers.Dense(128, activation='relu'),
        tf.keras.layers.Dense(10, activation='softmax')
    ])
    
    # 4. 编译模型
    # 注意：使用SparseCategoricalCrossentropy时需要设置reduction=NONE
    # 因为DP-SGD需要逐样本损失
    loss = tf.keras.losses.SparseCategoricalCrossentropy(
        from_logits=False,
        reduction=tf.keras.losses.Reduction.NONE
    )
    
    model.compile(
        optimizer=optimizer,
        loss=loss,
        metrics=['accuracy']
    )
    
    return model


def tf_privacy_training_loop():
    """
    TensorFlow Privacy 完整训练流程
    """
    # 准备数据
    (X_train, y_train), (X_test, y_test) = tf.keras.datasets.mnist.load_data()
    X_train = X_train.reshape(-1, 784).astype('float32') / 255.0
    X_test = X_test.reshape(-1, 784).astype('float32') / 255.0
    
    # 参数
    batch_size = 256
    epsilon = 8.0
    delta = 1e-5
    epochs = 10
    
    # 创建模型
    model = dp_training_with_tf_privacy(
        epsilon=epsilon,
        delta=delta,
        num_train_samples=len(X_train),
        batch_size=batch_size
    )
    
    # 训练
    model.fit(
        X_train, y_train,
        epochs=epochs,
        batch_size=batch_size,
        validation_data=(X_test, y_test)
    )
    
    # 验证隐私保证
    # 使用get_computation_graph查询实际隐私消耗
    print(f"训练完成，目标ε={epsilon}")
    
    return model


def compute_rdp_accountant(noise_multiplier, batch_size, dataset_size, epochs, delta):
    """
    使用RDP会计精确计算隐私损失
    
    RDP（Rényi Differential Privacy）会计比简单组合定理紧得多：
    - 简单组合：n次查询 → 总ε = n * ε_per_query（过于保守）
    - 高级组合：总ε ≈ √(2n * ln(1/δ)) * ε_per_query
    - RDP会计：最紧的界限，是目前工业界标准
    
    这里手动实现RDP会计以展示原理
    """
    from scipy.special import logsumexp
    import math
    
    # RDP参数α的候选值
    alphas = [1 + x / 10.0 for x in range(1, 100)] + list(range(12, 64))
    
    # 每步的RDP损失
    # 对于高斯机制：RDP(α) = α / (2σ²)
    q = batch_size / dataset_size  # 采样概率
    steps = int(epochs * dataset_size / batch_size)
    
    def rdp_gaussian(alpha, sigma):
        """高斯机制的RDP损失"""
        return alpha / (2 * sigma ** 2)
    
    def rdp_sampled_gaussian(alpha, sigma, q):
        """
        下采样高斯机制的RDP损失（更紧的界限）
        使用Poisson采样的RDP组合
        """
        if q == 0:
            return 0
        if q == 1:
            return rdp_gaussian(alpha, sigma)
        
        # 使用Mironov的公式
        rdp = rdp_gaussian(alpha, sigma)
        # 对于下采样，使用更紧的bound
        return min(rdp, q ** 2 * alpha * (alpha - 1) / (2 * sigma ** 2))
    
    # 计算累积RDP损失
    sigma = noise_multiplier * 1.0  # noise_multiplier对应l2_norm_clip=1
    rdp_total = []
    
    for alpha in alphas:
        rdp_per_step = rdp_sampled_gaussian(alpha, sigma, q)
        rdp_total.append(steps * rdp_per_step)
    
    # 将RDP转换为(ε, δ)-DP
    # ε(α) = RDP(α) + log(1/δ)/(α-1)
    epsilons = []
    for i, alpha in enumerate(alphas):
        if alpha > 1:
            eps = rdp_total[i] + math.log(1.0 / delta) / (alpha - 1)
            epsilons.append(eps)
    
    best_epsilon = min(epsilons)
    best_alpha = alphas[epsilons.index(best_epsilon)]
    
    print(f"RDP会计结果:")
    print(f"  最优ε = {best_epsilon:.4f} (α = {best_alpha:.1f})")
    print(f"  噪声乘数 = {noise_multiplier:.4f}")
    print(f"  总训练步数 = {steps}")
    
    return best_epsilon
```

### 20.4.6 隐私预算追踪与组合定理

差分隐私的组合定理是工程落地的核心——每次查询或每一步训练都会消耗隐私预算，必须精确追踪累计损失。

```python
import math


class PrivacyAccountant:
    """
    隐私预算追踪器
    
    支持三种会计方法：
    1. 基本组合定理（最简单但最松）
    2. 高级组合定理（中等紧度）
    3. RDP会计（最紧，推荐使用）
    """
    
    def __init__(self, epsilon_per_step, delta_per_step, mechanism='gaussian'):
        """
        参数:
            epsilon_per_step: 每步的ε消耗
            delta_per_step: 每步的δ消耗
            mechanism: 噪声机制类型
        """
        self.epsilon_per_step = epsilon_per_step
        self.delta_per_step = delta_per_step
        self.mechanism = mechanism
        self.steps = 0
    
    def basic_composition(self, k):
        """
        基本组合定理
        k次(ε,δ)-DP查询 → (kε, kδ)-DP
        简单但过于保守，实际中不推荐使用
        """
        total_epsilon = k * self.epsilon_per_step
        total_delta = k * self.delta_per_step
        return total_epsilon, total_delta
    
    def advanced_composition(self, k, delta_prime=1e-5):
        """
        高级组合定理（Dwork等人，2010）
        k次(ε,δ)-DP查询 → (ε', kδ + δ')-DP
        其中 ε' = ε * √(2k * ln(1/δ')) + k * ε * (e^ε - 1)
        
        比基本组合紧√k倍
        """
        eps = self.epsilon_per_step
        total_epsilon = (eps * math.sqrt(2 * k * math.log(1 / delta_prime)) + 
                        k * eps * (math.exp(eps) - 1))
        total_delta = k * self.delta_per_step + delta_prime
        return total_epsilon, total_delta
    
    def moment_accountant(self, noise_multiplier, batch_size, 
                          dataset_size, epochs, delta):
        """
        Moment Accountant（Abadi等人，2016）
        使用Rényi散度精确追踪隐私损失
        
        这是当前工业界标准方法，比高级组合紧得多
        通常结果比高级组合好5-10倍
        """
        q = batch_size / dataset_size
        sigma = noise_multiplier
        steps = int(epochs * dataset_size / batch_size)
        
        # 使用RDP的α-moment
        best_epsilon = float('inf')
        
        for alpha in range(2, 64):
            # RDP损失
            rdp = alpha / (2 * sigma ** 2)
            # 下采样修正
            rdp_sampled = q * alpha * (alpha - 1) / (2 * sigma ** 2)
            rdp_per_step = min(rdp, rdp_sampled)
            
            # 累积
            rdp_total = steps * rdp_per_step
            
            # 转换为(ε,δ)-DP
            epsilon = rdp_total + math.log(1.0 / delta) / (alpha - 1)
            best_epsilon = min(best_epsilon, epsilon)
        
        return best_epsilon
    
    def track_training(self, noise_multiplier, batch_size, 
                       dataset_size, target_epsilon, delta):
        """
        训练过程中的隐私预算追踪
        
        返回：在给定隐私预算下可以训练的最大epoch数
        """
        q = batch_size / dataset_size
        sigma = noise_multiplier
        max_epochs = 0
        
        for epochs in range(1, 1000):
            epsilon = self.moment_accountant(
                noise_multiplier, batch_size, dataset_size, epochs, delta
            )
            if epsilon <= target_epsilon:
                max_epochs = epochs
            else:
                break
        
        print(f"隐私预算分析:")
        print(f"  目标ε = {target_epsilon}")
        print(f"  噪声乘数 = {noise_multiplier}")
        print(f"  batch_size = {batch_size}")
        print(f"  数据集大小 = {dataset_size}")
        print(f"  可训练最大epoch数 = {max_epochs}")
        
        return max_epochs


# 使用示例
```

### 20.4.7 梯度裁剪策略与调参技巧

梯度裁剪阈值 C 是 DP-SGD 中最关键的超参数，直接影响隐私-效用权衡。

**裁剪阈值选择方法：**

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| 固定阈值 | 手动设定C值 | 快速实验 |
| 自适应裁剪 | 根据梯度分布动态调整C | 生产环境 |
| 百分位裁剪 | 使C覆盖p%的梯度 | 推荐默认方案 |
| 噪声感知裁剪 | 根据目标ε反推最优C | 精确隐私控制 |

```python
import torch
import numpy as np


class AdaptiveGradientClipping:
    """
    自适应梯度裁剪——根据梯度分布动态调整裁剪阈值
    
    为什么需要自适应？
    - 固定阈值在训练初期可能太松（梯度大），后期可能太紧（梯度小）
    - 不同层的梯度范数差异很大，统一裁剪不合理
    """
    
    def __init__(self, target_quantile=0.5, warmup_steps=100):
        """
        参数:
            target_quantile: 目标裁剪百分位（0.5表示裁剪掉50%的大梯度）
            warmup_steps: 预热步数，用于估计梯度分布
        """
        self.target_quantile = target_quantile
        self.warmup_steps = warmup_steps
        self.gradient_norms = []
        self.current_threshold = 1.0
        self.step_count = 0
    
    def compute_gradient_norm(self, model):
        """计算当前batch所有样本梯度的L2范数"""
        total_norm = 0.0
        for param in model.parameters():
            if param.grad is not None:
                total_norm += param.grad.norm(2).item() ** 2
        return total_norm ** 0.5
    
    def update_threshold(self):
        """根据历史梯度范数更新裁剪阈值"""
        if len(self.gradient_norms) > 10:
            self.current_threshold = np.percentile(
                self.gradient_norms, self.target_quantile * 100
            )
    
    def clip_and_report(self, model):
        """
        裁剪并返回统计信息
        
        返回:
            clip_ratio: 被裁剪的梯度比例
            current_threshold: 当前裁剪阈值
        """
        norm = self.compute_gradient_norm(model)
        self.gradient_norms.append(norm)
        self.step_count += 1
        
        if self.step_count <= self.warmup_steps:
            # 预热阶段，只收集统计信息
            return 0.0, self.current_threshold
        
        # 更新阈值
        self.update_threshold()
        
        # 裁剪
        clip_ratio = min(1.0, self.current_threshold / (norm + 1e-8))
        for param in model.parameters():
            if param.grad is not None:
                param.grad.mul_(clip_ratio)
        
        return clip_ratio, self.current_threshold


def percentile_clipping(per_sample_grads, percentile=50):
    """
    百分位裁剪——使C覆盖p%的梯度
    
    优点：自动适应梯度分布
    缺点：需要预先计算整个batch的梯度范数
    """
    # 计算所有样本的梯度范数
    norms = []
    for sample_grad in per_sample_grads:
        total_norm = 0.0
        for g in sample_grad:
            if g is not None:
                total_norm += g.norm(2).item() ** 2
        norms.append(total_norm ** 0.5)
    
    # 使用百分位数作为阈值
    threshold = np.percentile(norms, percentile)
    
    # 裁剪
    clipped_grads = []
    for i, sample_grad in enumerate(per_sample_grads):
        clip_factor = min(1.0, threshold / (norms[i] + 1e-8))
        clipped = [g * clip_factor if g is not None else None for g in sample_grad]
        clipped_grads.append(clipped)
    
    return clipped_grads, threshold
```

```text
2. 选择覆盖80%梯度的阈值作为起点
3. 逐步降低阈值，观察模型精度下降幅度
4. 找到精度下降<2%的最大裁剪比例
```

```python
from torch.utils.data import DataLoader, TensorDataset, Subset

class MembershipInferenceAttack:
    """
    成员推理攻击
    
    攻击原理：
    1. 训练一个"影子模型"来学习目标模型的行为模式
    2. 影子模型在已知数据上训练（成员）和未训练数据上测试（非成员）
    3. 训练攻击分类器，根据模型输出判断样本是否为成员
    
    如果目标模型使用了差分隐私，攻击成功率应该接近50%（随机猜测）
    """
    
    def __init__(self, target_model, shadow_model, attack_model):
        self.target_model = target_model
        self.shadow_model = shadow_model
        self.attack_model = attack_model
    
    def get_model_outputs(self, model, data):
        """获取模型的softmax输出（置信度向量）"""
        model.eval()
        with torch.no_grad():
            outputs = model(data)
            probs = torch.softmax(outputs, dim=1)
        return probs

        """

        non_member_outputs = self.get_model_outputs(self.shadow_model, shadow_test)

        non_member_outputs = self.get_model_outputs(self.target_model, target_non_member_data)
        
        with torch.no_grad():
            member_preds = self.attack_model(member_outputs).squeeze()
            non_member_preds = self.attack_model(non_member_outputs).squeeze()
        
        # 计算准确率
        member_correct = (member_preds > 0.5).float().mean().item()
        non_member_correct = (non_member_preds < 0.5).float().mean().item()
        accuracy = (member_correct + non_member_correct) / 2
        
        print(f"攻击结果:")
        print(f"  成员识别率: {member_correct:.2%}")
        print(f"  非成员识别率: {non_member_correct:.2%}")
        print(f"  总体准确率: {accuracy:.2%}")
        print(f"  随机基线: 50.00%")
        print(f"  隐私泄露: {max(0, accuracy - 0.5):.2%}")
        
        return accuracy
```

### 20.4.9 联邦学习中的差分隐私

联邦学习是差分隐私最重要的应用场景之一——每个参与方在本地训练时添加噪声，保护数据隐私。

```python
import torch
import torch.nn as nn
import numpy as np
from collections import OrderedDict


class DPFederatedLearning:
    """
    差分隐私联邦学习
    
    两种DP模式：
    1. 本地差分隐私（Local DP）：参与方在本地添加噪声
       - 优点：参与方完全控制自己的隐私
       - 缺点：噪声累积，模型精度下降严重
    
    2. 中心差分隐私（Central DP）：聚合服务器添加噪声
       - 优点：噪声更小，模型精度更高
       - 缺点：需要信任聚合服务器
    
    实践中通常采用混合方案：
    - 本地添加小噪声（保护传输过程）
    - 中心再添加噪声（保护聚合结果）
    """
    
    def __init__(self, global_model, num_clients, epsilon, delta, 
                 dp_mode='central'):
        self.global_model = global_model
        self.num_clients = num_clients
        self.epsilon = epsilon
        self.delta = delta
        self.dp_mode = dp_mode
    
    def client_update(self, client_model, client_data, epochs=1, 
                      local_noise_scale=0.0):
        """
        客户端本地训练
        
        参数:
            client_model: 客户端模型副本
            client_data: 客户端本地数据
            epochs: 本地训练轮数
            local_noise_scale: 本地噪声规模（Local DP时>0）
        """
        optimizer = torch.optim.SGD(client_model.parameters(), lr=0.01)
        loss_fn = nn.CrossEntropyLoss()
        
        for epoch in range(epochs):
            for data, target in client_data:
                optimizer.zero_grad()
                output = client_model(data)
                loss = loss_fn(output, target)
                loss.backward()
                
                # 本地差分隐私：在梯度上添加噪声
                if local_noise_scale > 0:
                    for param in client_model.parameters():
                        if param.grad is not None:
                            noise = torch.normal(
                                0, local_noise_scale, 
                                size=param.grad.shape
                            )
                            param.grad += noise
                
                optimizer.step()
        
        return client_model.state_dict()
    
    def aggregate_with_dp(self, client_updates, central_noise_scale=0.0):
        """
        聚合客户端更新并添加中心DP噪声
        
        聚合方式：FedAvg（加权平均）
        """
        # 计算平均更新
        avg_update = OrderedDict()
        
        for key in client_updates[0].keys():
            avg_update[key] = torch.zeros_like(client_updates[0][key])
            
            for update in client_updates:
                avg_update[key] += update[key]
            
            avg_update[key] /= len(client_updates)
        
        # 中心差分隐私：在聚合结果上添加噪声
        if central_noise_scale > 0:
            for key in avg_update:
                noise = torch.normal(
                    0, central_noise_scale, 
                    size=avg_update[key].shape
                )
                avg_update[key] += noise
        
        return avg_update
    
    def federated_training_round(self, client_datasets, epochs=1):
        """
        一轮联邦训练
        
        完整流程：
        1. 分发全局模型给各客户端
        2. 各客户端本地训练（可选本地DP）
        3. 收集客户端更新
        4. 聚合并添加中心DP噪声
        5. 更新全局模型
        """
        # 计算噪声规模
        if self.dp_mode == 'local':
            # 本地DP：每个客户端分担总隐私预算
            local_epsilon = self.epsilon / np.sqrt(2 * np.log(1.25/self.delta))
            local_noise_scale = 1.0 / local_epsilon
            central_noise_scale = 0.0
        elif self.dp_mode == 'central':
            local_noise_scale = 0.0
            # 中心DP：所有隐私预算由服务器承担
            central_noise_scale = (1.0 * np.sqrt(2 * np.log(1.25/self.delta)) / 
                                   self.epsilon)
        else:  # 混合模式
            local_noise_scale = 0.1
            central_noise_scale = (1.0 * np.sqrt(2 * np.log(1.25/self.delta)) / 
                                   (self.epsilon * 0.5))
        
        # 收集客户端更新
        client_updates = []
        for client_data in client_datasets:
            # 创建客户端模型副本
            client_model = type(self.global_model)()
            client_model.load_state_dict(self.global_model.state_dict())
            
            # 本地训练
            update = self.client_update(
                client_model, client_data, 
                epochs=epochs, 
                local_noise_scale=local_noise_scale
            )
            client_updates.append(update)
        
        # 聚合并添加噪声
        aggregated = self.aggregate_with_dp(
            client_updates, 
            central_noise_scale=central_noise_scale
        )
        
        # 更新全局模型
        self.global_model.load_state_dict(aggregated)
        
        return self.global_model
```

|------|---------|---------|
| ε过大失去保护意义 | ε > 10时隐私保证微乎其微 | 控制ε ≤ 8，最好 ≤ 3 |
| 忽略δ的影响 | δ设置过大（如0.01）削弱保证 | δ ≤ 1/n，n为数据集大小 |
| 忽略组合效应 | 多轮训练累积隐私损失 | 使用RDP会计精确追踪 |
| 裁剪阈值过小 | 梯度被严重截断，模型欠拟合 | 使用自适应裁剪或百分位裁剪 |
| 忽略数据预处理 | 归一化不当导致梯度范数异常 | 训练前标准化数据 |
| 使用BatchNorm | BN统计量泄露信息 | 替换为GroupNorm/LayerNorm |
| 模型过大 | 参数多→噪声累积→精度崩塌 | 使用更小的模型或知识蒸馏 |

**生产环境部署清单：**

```text
□ 隐私预算明确定义（ε, δ, 组合定理）
□ 梯度裁剪阈值经过调参验证
□ 噪声校准使用RDP会计（非简单组合）
□ 模型兼容性检查（无BatchNorm等不兼容层）
□ 隐私损失追踪贯穿整个训练流程
□ 攻击实验验证DP有效性
□ 隐私保证文档化（给用户/监管机构）
□ 模型发布前的最终隐私审计
```

### 20.4.11 本节小结

差分隐私是当前隐私保护机器学习中数学基础最扎实的技术方案。核心要点：

- **DP-SGD三要素**：逐样本梯度、梯度裁剪、噪声注入——缺一不可
- **隐私预算追踪**：使用RDP会计（而非简单组合），可获得5-10倍的紧度提升
- **裁剪阈值是关键**：C太大则隐私弱，C太小则模型废，需要系统调参
- **Opacus是首选工具**：比手动实现快5-10倍，内置兼容性检查和隐私会计
- **验证必须做攻击实验**：成员推理攻击成功率应接近50%才能确认DP生效
- **联邦学习+DP是黄金组合**：本地DP保护传输，中心DP保护聚合，分层防护
